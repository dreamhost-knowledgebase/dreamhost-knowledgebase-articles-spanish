Generalidades

Para acceder a tu base de datos MySQL, necesitas un nombre de host único (nombre de dominio) o una dirección IP para tu servidor MySQL. Este nombre de host puede ser cualquier nombre que desees, siempre que lo haya registrado, pero comúnmente es un subdominio de tu dominio principal y se llama "mysql.example.TLD".

Por ejemplo, supongamos que tu dominio es example.com: puede crear el subdominio mysql.example.com, que se utilizas para conectarte a tu servidor de base de datos.

Este arreglo es conveniente para usar con las instalaciones MySQL de DreamHost, ya que puedes crear y administrar todas tus bases de datos desde la página Bases de Datos MySQL.

¿Qué nombre de host se debe usar si DNS aún no apunta a DreamHost?

Durante una migración de servidor o servidor web, el DNS para example.tld puede no apuntar a los nameservers de DreamHost, por lo que no podrás usar mysql.example.tld para el nombre de host de MySQL.

Si tienes otros dominios alojados en DreamHost, puedes usar tu nombre de host en tu lugar. Esto se debe a que tu servidor MySQL (que aloja todas tus bases de datos) es accesible desde cualquier subdominio que le asignes dentro de tu cuenta DreamHost.

Por ejemplo, si tienes otro dominio como dreamhostexample.com y tus nameservers apuntan a DreamHost, puedes crear un nombre de host mysql titulado mysql.dreamhostexample.com. Luego puedes usar este nombre de host para conectarte a CUALQUIER base de datos en tu cuenta DreamHost.

¿Qué sucede si no tengo dominios apuntados a DreamHost?

Si no tienes ningún dominio alojado en DreamHost, aún puedes conectarte a tu base de datos. Para hacer esto, debe agregar un subdominio dreamhosters.com gratis. Aparecería como 'example.dreamhosters.com'. Luego puedes crear un nombre de host mysql como 'mysql.example.dreamhosters.com'.
