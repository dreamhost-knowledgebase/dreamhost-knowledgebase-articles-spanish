Generalidades

DreamHost no puede garantizar que los datos se retendrán en cuentas inactivas (suspendidas/cerradas/deshabilitadas) durante más de dos semanas después del cambio de estado de la cuenta. Revisa los siguientes artículos del centro de ayuda para obtener más información sobre cómo puedes hacer una copia de seguridad de tu cuenta o realizar un pago:

  • Respaldar una cuenta en el panel
  • Hacer un pago

Tu panel DreamHost ofrece una opción de restauración de dominio para restaurar tu dominio. Esto solo restaura archivos en tu servidor web, no lo que están en tu servidor de bases de datos. Algunos sitios web no usan una base de datos, por lo que este artículo sería todo lo que necesitas para restaurar tu sitio.

Como se indica en la página Restaurar (descrita a continuación), DreamHost no garantiza las copias de seguridad de tus dominios. Es muy recomendado que siempre mantengas copias de seguridad locales del contenido de tu sitio en caso de que una copia de seguridad DreamHost no esté disponible.

Los archivos se restauran desde el servidor web en el que está alojado tu sitio actualmente. El servicio de restauración no restaura datos de ningún otro servidor web desde el que hayas migrado previamente.

Si tu sitio ejecuta WordPress o cualquier otro software que use una base de datos, visita el siguiente artículo para obtener instrucciones sobre cómo restaurar también tu base de datos.

  • ¿Cómo restaurar mi base de datos en el panel?

Restaurar tu sitio web

Para restaurar tu sitio web:

  1. Dirígete a la página Administrar Dominios en el panel, el cual muestra cada dominio en tu cuenta.
  2. Haz clic en el botón Restaurar en el extremo derecho.
    Aparece la página Restore:
  3. Elige entre las siguientes secciones:
    • Fechas de respaldo disponibles: Hay tres opciones disponibles con diferentes fechas de los respaldos de tu dominio:
      • 1 hora a 1 día.
      • 1 día a 2 semanas.
      • 2 semanas o posiblemente más.
    • Opciones de restauración: Las dos opciones te permiten restaurar las copias de seguridad seleccionadas a tu sitio actual en vivo o restaurar las copias de seguridad seleccionadas como una carpeta separada. 
      Copy the backup files to: /home/username/example.com_TIMESTAMP
      La primera opción mantiene vivo el sitio actual. Tus archivos de copia de seguridad se recuperan y almacenan en un nuevo directorio. Este directorio lleva el nombre de tu sitio, seguido de una marca de tiempo. Usa esta opción si deseas ver tus archivos respaldados antes de cambiar tu sitio a la versión anterior.
      Make the backup copy live and back up all the current live files to
/home/username/example.com_TIMESTAMP
      La segunda opción toma la copia de seguridad existente e inmediatamente la convierte en tu sitio web en vivo. Los archivos actualmente en el sitio se mueven a un directorio con el nombre de tu sitio, seguido de una marca de tiempo. Usa esta opción si deseas que tu sitio vuelva inmediatamente a la versión anterior.
  4. Haz clic en el botón ¡Restaurar example.com ahora! para completar este proceso. (Reemplaza el nombre de dominio que figura en el botón con tu propio nombre de dominio que se está restaurando).

Una vez que haces clic en el botón Restaurar, volverás a la página Administrar Dominios donde verás un mensaje verde en la parte superior de la página que indica que la restauración se está procesando y se completará según el tamaño de la copia de seguridad. Se te notificará por correo electrónico una vez que se complete.
