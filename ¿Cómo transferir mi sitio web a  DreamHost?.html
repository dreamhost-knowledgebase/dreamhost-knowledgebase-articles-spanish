Generalidades

Este artículo describe cómo transferir manualmente tu sitio web y correo de tu antigua compañía de alojamiento a DreamHost. Este proceso implica los siguientes pasos:

  • Paso 1 — Respaldar tu información en la companía de alojamiento anterior
  • Paso 2 — Cargar la información a DreamHost
  • Paso 3 — Ver tu sitio en DreamHost antes de actualizar los DNS
  • Paso 4 — Apuntar tus registros DNS a DreamHost

Servicio de migración

Si tienes problemas para migrar tu sitio web desde otro alojamiento, o simplemente no tienes tiempo, DreamHost actualmente ofrece un servicio que se encarga de eso por tí. Puedes encontrar más información en el artículo de los servicios de migración del sitio web DreamHost.

Transferencia de registro

Este artículo explica cómo transferir la información de tu sitio web a DreamHost. Esto es diferente a tu registro de dominio.

Tu NO necesitas transferir el registro de dominio a DreamHost durante este proceso. Siempre podrás transferirlo en cualquier momento en el futuro. Revisa el siguiente artículo cuando también estés listo para transferir el registro:

    • Transferir el registro de un dominio a DreamHost

Crear una cuenta en DreamHost

Antes de continuar, asegúrate de haber creado una cuenta en DreamHost registrándote en un plan de alojamiento. Revisa el siguiente artículo para más detalles:

  • Cómo crear una cuenta en DreamHost

Durante el proceso de registro, selecciona la opción Ya tengo un dominio. Esto te permite agregar tu dominio actual a tu nueva cuenta de DreamHost.

Agregar tu dominio a DreamHost en este punto NO afecta tu sitio web existente de ninguna forma. Este permanecerá funcionado normalmente en tu alojamiento actual hasta que apuntes los registros DNS en el Paso #4.

 
 

Paso 1 — Respaldar tu información en la companía de alojamiento anterior

Este paso respalda tus datos de la compañía de alojamiento antigua a tu computador personal.

Necesitarás respaldar lo siguiente:

  • Archivos del sitio web
  • Archivo de la base de datos (solo tu si tu sitio web usa una base de datos)
  • Correo electrónico

Asegúrate de que tu base de datos es exportada a un archivo con la extensión .sql.

Contacta a tu antigua companía de alojamiento para ver sus instrucciones de como respaldar este contenido. Ellos deben tener una manera simple de exportar la información. Si no, necesitarás hacer una copia de seguridad manual de todo.

 

Paso 2 — Cargar la información a DreamHost

Este paso carga o sube la información desde tu computador personal a tu nueva cuenta de DreamHost.

Archivos de sitio web

Puedes usar un cliente de FTP para iniciar sesión en tu nuevo servidor de DreamHost y cargar los archivos de tu sitio web. Revisa el siguiente artículo para más instrucciones de dónde subir los archivos en el servidor.

  • ¿Dónde debo poner mis archivos una vez conectado vía FTP?

Archivo de la base de datos

Este paso solo es necesario si tu sitio web usa una base de datos.

Tu base de datos debe ser respaldada en un solo archivo terminando con la extensión .sql. Revisa el siguiente artículo para ver instrucciones de cómo importar este archivo de base de datos en tu cuenta de DreamHost.

  • ¿Cómo migrar mi base de datos a DreamHost?

Correo electrónico

Revisa el siguiente archivo para ver instrucciones de cómo importar los correos que descargaste de tu antigua compañía de alojamiento en tu nuevo servidor de correo de DreamHost.

  • Importar el correo otro alojamiento a DreamHost
 

Paso 3 — Ver tu sitio en DreamHost (antes de actualizar los DNS)

Este paso te permite ver como tu sitio aparecerá en DreamHost antes de apuntar tus DNS. Esto no afecta tu sitio de ninguna manera. Solo te ofrece la habilidad de confirmar que tu sitio esté configurado correctamente en DreamHost antes de apuntar sus DNS.

Aunque hayas subido el contenido de tu sitio web a DreamHost, todavía está alojado en tu antigua compañía de alojamiento. Esto significa que cuando lo ves en un navegador, aparece como en tu host anterior, no en DreamHost. Solo se cargará desde tu servidor DreamHost después de que apuntes sus registros DNS a DreamHost.

Antes de apuntar los registros DNS, debes probar el sitio para confirmar que se ve y funciona según lo previsto en tu nuevo servidor DreamHost.

Actualizar el archivo host de tu computador

La mejor manera de ver tu sitio tal como aparece en tu servidor de DreamHost es actualizar el archivo hosts en tu computador personal. Este es un archivo que ya existe en tu computador. Su trabajo es asignar un nombre de dominio a una dirección IP. De esta manera, puedes obligar al computador de tu hogar a cargar el sitio web desde tu servidor de DreamHost, aunque todavía esté alojado en tu antigua compañía de alojamiento.

Revisa el siguiente artículo para obtener instrucciones sobre cómo actualizar el archivo host en tu computador para que apunte a DreamHost:

  • Ver tu sitio DreamHost usando un archivo hosts

Una vez que hayas confirmado que tu sitio funciona correctamente, asegúrate de eliminar las líneas que acabas de agregar durante la prueba.

 

Paso 4 — Apuntar tus registros DNS a DreamHost

Este paso apunta los registros DNS de tu sitio web a DreamHost. Este es el paso final para alojar tu sitio web en tu nuevo servidor de DreamHost.

En este punto, tu sitio web está correctamente configurado en tu servidor DreamHost, pero está cargando desde el servidor en tu antigua compañía de alojamiento.

Para cargar tu sitio desde tu servidor DreamHost, debes apuntar ahora sus registros DNS a DreamHost. Tienes dos opciones cuando apuntes tu DNS.

  • Apuntar los nameservers (recomendado)
  • Apuntar los registros A

¿Qué registros debería apuntar?

No importa qué opción elijas. Los registros y los nameservers funcionarán de la misma manera para cargar tu sitio web. La única diferencia es que apuntar tus nameservers es mucho más simple.

Nameservers (recomendado)

Apuntar tus nameservers es más fácil porque automáticamente apunta todos tus otros registros a DreamHost.

Los siguientes artículos explican más sobre los nameservers, cómo ver hacia dónde apuntan y cómo apuntarlos a DreamHost.

  • Generalidades de los Nameservers
  • Verificar hacia dónde apuntan tus nameservers
  • Cambiar los nameservers en tu alojamiento actual

Registros A

Apuntar los Registros A solo se recomienda si necesitas mantener activos servicios específicos en tu antigua compañía de alojamiento. Si deseas apuntar todos tus registros DNS para este dominio, se recomienda apuntar tus nameservers en su lugar.

Apuntar tus registros A es más difícil porque hay varios registros A para cada tipo de servicio que utiliza. Por ejemplo, deberás actualizar los registros de tu:

  • Sitio web
  • Base de datos
  • Correo
  • Registros personalizados que requieras

Revisa el siguiente artículo para obtener instrucciones sobre cómo ver las direcciones IP en DreamHost a las que deberás apuntar tus registros A.

  • ¿A cuáles registros DNS debo apuntar mi sitio web?

Luego deberás revisar con tu antigua compañia de alojamiento para obtener instrucciones sobre cómo actualizar estos registros.

Esperar para que los registros DNS actualicen

Después de apuntar tus registros DNS a DreamHost, necesitarán tiempo para propagarse en línea.

  • Generalidades de la propagación de DNS

Los Registros A pueden tardar 6 horas en actualizarse, pero los nameservers pueden tardar hasta 72 horas en actualizarse por completo. Hasta que todos los registros apunten a DreamHost, tu sitio aún se cargará desde tu compañía de alojamiento anterior.

No es posible acelerar este proceso ya que los servidores de todo el mundo actualizan los registros en su propio horario.

Una vez que los registros DNS se hayan actualizado en línea, tu sitio web se cargará tal como está configurado en tu servidor de DreamHost.
