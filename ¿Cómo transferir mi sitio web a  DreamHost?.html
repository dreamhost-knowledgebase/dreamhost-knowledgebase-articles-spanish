Generalidades

Este artículo describe cómo transferir manualmente tu sitio web y correos electrónicos desde tu antiguo proveedor de alojamiento a DreamHost. Este proceso implica los siguientes pasos:

  • Paso 1 — Respaldar los datos desde tu antiguo proveedor de alojamiento
  • Paso 2 — Subir la información a DreamHost
  • Paso 3 — Ver tu sitio en DreamHost antes de actualizar el DNS
  • Paso 4 — Apuntar el DNS a DreamHost

Requisitos previos

Antes de continuar, asegúrate de haber creado una cuenta en DreamHost al registrarte para un plan de alojamiento. Durante el proceso de registro, selecciona la opción Ya tengo un dominio. Esto te permitirá agregar tu nombre de dominio actual a tu nueva cuenta en DreamHost.

Preguntas frecuentes

¿Agregar mi dominio a DreamHost afectará mi sitio web en vivo?

No. Agregar tu dominio a DreamHost en este momento NO afecta tu sitio web actual de ninguna manera. Seguirá funcionando normalmente en tu alojador actual hasta que apuntes tus registros DNS en el Paso #4.

¿Existen otras opciones de transferencia?

Las siguientes opciones alternativas están disponibles para transferir tu sitio a DreamHost.

Servicio de migración — Si tienes problemas para migrar tu sitio web o simplemente no tienes tiempo, DreamHost ofrece un servicio que se encarga de esto por ti. Puedes encontrar más información en el artículo Servicios de migración de sitios web.

Opciones de migración de WordPress — Revisa el artículo Generalidades de las migraciones de WordPress para obtener una lista de opciones alternativas de WordPress.

¿Necesito transferir mi dominio?

No. No es necesario que transfieras tu registro a DreamHost durante este proceso. Este artículo solo explica cómo transferir la información de tu sitio web a DreamHost, lo cual es independiente del registro de tu dominio.

Siempre puedes transferirlo en cualquier momento en el futuro siguiendo los pasos de este artículo.

Cómo transferir tu sitio en línea

Los siguientes pasos transfieren el contenido de tu sitio web a DreamHost.

 
 

Respaldar los datos desde tu antiguo proveedor de alojamiento

Este paso hace un respaldo de la información del sitio web de tu antigua compañía de alojamiento en tu computador de hogar.

Necesitarás respaldar lo siguiente:

  • Archivos del sitio web
  • Archivo de base de datos (Solo si tu sitio utiliza una base de datos. Asegúrate de que esté exportado con una extensión .sql.)
  • Correo electrónico

Contacta a tu antigua compañía de alojamiento para obtener sus instrucciones sobre cómo respaldar este contenido. Es posible que tengan una forma sencilla de exportar los datos. Si no es así, deberás respaldar todo manualmente.

 

Subir la información a DreamHost

Este paso carga los datos desde tu computador de hogar a tu nueva cuenta de DreamHost.

Archivos de sitio web — Usar un cliente FTP para iniciar sesión en tu servidor de DreamHost y subir tus archivos del sitio web. Revisa este artículo para obtener instrucciones sobre dónde subir los archivos en el servidor.

Archivos de base de datos — Revisa este artículo para obtener instrucciones sobre cómo importar tu base de datos.

Correo electrónico — Revisa este artículo para obtener instrucciones sobre cómo importar el correo a tu nueva cuenta de DreamHost.

 

Ver tu sitio en DreamHost (antes de apuntar el DNS)

Este paso te permite ver cómo aparecerá tu sitio en DreamHost antes de apuntar tu DNS.

  • Esto no afecta a tu sitio en vivo en absoluto.
  • Simplemente te permite confirmar que tu sitio está configurado correctamente en DreamHost antes de apuntar tu DNS.

En este punto, has subido el contenido de tu sitio web a DreamHost, pero aún está alojado en tu antiguo proveedor de alojamiento. Esto significa que, cuando lo veas en un navegador, se cargará desde tu antiguo alojamiento. Solo se cargará desde tu servidor de DreamHost después de que apuntas los registros DNS a DreamHost.

Sin embargo, antes de apuntar los registros DNS, deberías probar para confirmar que tu sitio se ve y funciona como se espera en tu nuevo servidor de DreamHost.

Editar el archivo host de tu computador

Para ver tu sitio tal como aparece en tu servidor de DreamHost, deberás editar el archivo de hosts de tu computador. Este archivo se usa para asignar un nombre de dominio a una dirección IP, por lo que al editarlo, puedes forzar a tu computador a cargar el sitio web desde tu servidor de DreamHost (aunque aún esté alojado en tu antiguo proveedor de alojamiento). Revisa este artículo para obtener instrucciones sobre cómo editar el archivo hosts para que apunte a DreamHost.

Después de confirmar que tu sitio funciona correctamente, asegúrate de eliminar las líneas que acabas de agregar durante la prueba.

 

Apuntar el DNS DreamHost

Este paso apunta los registros DNS de tu sitio web a DreamHost.

Tienes dos opciones al apuntar DNS.

  • Apuntar los nameservers (recomendado)
  • Apuntar los registros A

¿Que opción debería escoger?

Ambas opciones funcionan para cargar tu sitio web en DreamHost. Sin embargo, según los requisitos de alojamiento de tu sitio web, es posible que debas utilizar una opción en lugar de la otra.

Revisa el artículo Visión general del DNS de DreamHost para obtener una descripción general rápida de cómo se utiliza el DNS para controlar dónde se alojan los servicios de tu dominio.

Nameservers (recomendado)

Se recomienda apuntar tus nameservers porque automáticamente apunta TODOS tus registros DNS a DreamHost. Revisa los siguientes artículos para obtener más información sobre los nameservers, cómo ver dónde apuntan y apuntarlos a DreamHost.

  • Generalidades de los nameservers
  • Verificar hacia donde apuntan tus nameservers
  • Cambiar los nameservers en tu alojamiento actual

Registros A

Solo es necesario apuntar los registros A si necesitas mantener activos servicios específicos (como el correo) en tu antigua compañía de alojamiento. Esto es más difícil porque debes confirmar qué servicios apuntar a DreamHost y cuáles continuar alojando en tu compañía actual.

Revisa este artículo para obtener instrucciones sobre cómo ver las direcciones IP en DreamHost a las que necesitarás apuntar tus Registros A. Luego, deberás consultar con tu antigua compañía de alojamiento para obtener instrucciones sobre cómo actualizar estos registros.

Esperar que actualicen los DNS

Después de apuntar tus registros DNS a DreamHost, necesitarán tiempo para propagarse en línea.

  • Generalidades de la propagación del DNS

Los registros A pueden tardar 6 horas en actualizarse, pero los nameservers pueden tardar hasta 72 horas en actualizarse por completo.

No es posible acelerar este proceso ya que los servidores de todo el mundo actualizan los registros DNS según su propio horario.

Una vez que los registros DNS se hayan actualizado en línea, tu sitio web se cargará tal como está configurado en tu servidor DreamHost.
