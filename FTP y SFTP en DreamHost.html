Generalidades

Cuando necesitas cargar la información de un sitio web de tu computador local a tu servidor web para que aparezca online, usarías un programa como FileZilla o Cyberduck. Estos programas se conectan al servidor web usando uno de los siguientes protocolos para conectarte.

  • FTP
  • SFTP

Este artículo explica la diferencia entre estos protocolos y cual debes usar cuando te conectes a tu servidor.

Diferencias entre FTP y SFTP

FTP

FTP (File Transfer Protocol) es un protocolo de red antiguo que transfiere archivos entre tu computador y el servidor web. El problema con FTP es que los datos no se cifran durante la transferencia, lo que los hace inseguros.

Debido a que los datos no están encriptados, una persona excepcionalmente astuta podría espiar lo que estás enviando. Por esta razón, nunca se debe utilizar FTP.

SFTP

SFTP (Secure File Transfer Protocol) es el protocolo de red moderno que transfiere archivos entre tu computador y el servidor web. Realiza las mismas funciones básicas que el FTP y, al mismo tiempo, cifra tus datos durante la transferencia.

Dado que los datos están encriptados, este protocolo es seguro de usar.

FTP y SFTP en DreamHost

Todos los usuarios web de DreamHost están configurados para usar SFTP y el puerto 22.

Cuando uses un programa como FileZilla o Cyberduck para conectarte a tu servidor web, DreamHost recomienda usar siempre SFTP y el puerto 22 para garantizar que sus datos permanezcan seguros y protegidos.
