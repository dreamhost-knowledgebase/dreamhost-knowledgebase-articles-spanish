Generalidades

Un registro de subdominio DNS estándar tiene un nombre correspondiente como blog.example.com. Sin embargo, a veces puede ser necesario habilitar el DNS Wildcard que no tiene un solo registro asignado. En lugar de aparecer como blog.example.com (o cualquier otro subdominio), aparece como *.example.com. El carácter * se usa para hacer coincidir nombres de subdominios no existentes.

En pocas palabras, el DNS de Wildcard se utiliza para proporcionar registros DNS a subdominios que no existen.

El DNS de Wildcard es independiente de los certificados SSL Wildcard. Actualmente no es posible utilizar un certificado SSL Wildcard en DreamHost. Visita el siguiente artículo para más detalles.

  • Generalidades de los certificados SSL: Certificados Wildcard

¿Cómo puedo usar un DNS Wildcard en DreamHost?

Puedes implementar el DNS Wildcard para dominios que se ejecutan solo en un Servidor Privado Virtual (VPS) o un  Servidor Dedicado de DreamHost. Esto se debe al uso extremadamente alto de recursos (y al posible abuso) que generalmente viene con la instalación de software que se basa en los DNS Wildcard para funcionar (es decir, WordPress Multi-User, o WPMU, y otros).

DreamHost puede, en casos especiales, configurar esto para un cliente. Este es un procedimiento manual para el personal de soporte, así que asegúrate de que sea algo que realmente necesites y que el dominio en cuestión se esté ejecutando actualmente en un Servidor VPS o Dedicado antes de solicitarlo. Asegúrate también de conocer las implicaciones técnicas de tener una entrada DNS de Wildcard configurado en tu dominio.

Opción no administrada mediante DreamCompute

Las configuraciones anteriores solo se aplican a VPS y Servidores Dedicados que son administrados por DreamHost. También tienes la opción de crear un servidor en la nube usando DreamCompute, el servicio de computación en la nube de DreamHost.

Cuando creas una instancia de DreamCompute, tienes control total sobre el servidor, lo que significa que puedes instalar lo que quieras en él.

Después de apuntar tu sitio web a tu instancia DreamCompute, puedes agregar una entrada DNS de Wildcard en tu panel DreamHost. Cualquier configuración adicional debe hacerse iniciando sesión en tu instancia de DreamCompute.

Sin embargo, ten en cuenta que el soporte técnico no puede ayudar de ninguna manera con las personalizaciones realizadas en un servidor en la nube. Si eliges usar esta opción, debes estar listo para instalar todo el software y solucionar cualquier problema que pueda surgir.
