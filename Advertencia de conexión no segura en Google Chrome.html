Generalidades

Si tu sitio web no tiene un certificado SSL configurado, los visitantes de tu sitio verán una advertencia de "inseguro" en tu navegador. Puedes verlo haciendo clic en el icono de advertencia en la barra de URL a la izquierda de tu dominio.

Cuando haces clic en el icono de advertencia, aparece un mensaje que te informa que tu conexión al sitio no es segura.

Para no perder visitantes y clientes potenciales, debes asegurarte de que tu sitio web esté usando un certificado SSL.

Cuando un certificado SSL es habilitado, todo el tráfico dentro del dominio se encripta. Esto asegura que toda la comunicación de datos personales (como información de tarjetas de crédito o contraseñas) esté protegida.

Incluso si tu sitio no maneja datos confidenciales (como información de tarjetas de crédito o contraseñas), los navegadores web lo marcarán como "no seguro". Los visitantes del mismo pueden estar preocupados por la advertencia y optar por evitarlo.

Resolver la advertencia — Propietarios del sitio web

Agregar un certificado SSL a tu sitio

Para asegurar que el tráfico de tu sitio web no se vea afectado, se recomienda encarecidamente que agregues un certificado SSL a tu sitio web. DreamHost recomienda agregar un 'Let's Encrypt' gratuito o un certificado 'Sectigo' pago.

Un certificado autofirmado no se recomienda pues seguirá mostrando una advertencia de seguridad en cualquier buscador.

  • Agregar un certificado gratuito Let's Encrypt

Actualizar enlaces para usar HTTPS

Si tu sitio web contiene enlaces HTTP, se lanzará una advertencia de contenido mixto. Aquí es cuando se agrega un certificado SSL, pero los enlaces dentro del sitio no se actualizaron. Revisa el siguiente artículo para obtener más detalles.

  • Resolver advertencias de sitios inseguros y contenido mixto

Agregar encabezados de seguridad

Los encabezados HTTP son informaciones que se transmites entre un navegador web y un sitio web cuando se visita. Hay diferentes tipos de encabezados que puede agregar según el servicio que deseas implementar. Por ejemplo, podrías agregar encabezados de seguridad para obligar a tu sitio a cargar solo contenido seguro. Revisa el siguiente artículo para obtener una lista de encabezados de seguridad.

Resolver la advertencia — Visitantes del sitio web

Limpiar el caché y cookies de tu buscador

Es posible que el caché y cookies viejas en tu buscador estén causando que el sitio web cargue información que no está actualizada. Limpiarlos puede resolver esta advertencia.

  • Chrome: Caché y cookies
  • Edge: Cookies y Caché
  • Firefox: Cookies y Caché
  • Safari: Cookies y Caché

Reparar la hora y la zona horaria

Los certificados SSL tienen fechas de expiración. Si la fecha y la hora de tu computador están configuradas incorrectamente, los certificados SSL de un sitio web podrían parecer no válidos y, por lo tanto, mostrar una advertencia de seguridad.

  • Windows — Establecer la hora y la zona horaria
  • Mac — Configurar la fecha y la hora de tu computador
