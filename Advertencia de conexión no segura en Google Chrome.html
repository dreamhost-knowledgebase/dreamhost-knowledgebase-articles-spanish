Generalidades

A partir de julio de 2018, el navegador web Google Chrome muestra una advertencia 'insegura' a los visitantes de cualquier sitio que no tenga un certificado SSL configurado. Puede ver esto haciendo clic en el icono de advertencia en la barra de URL a la izquierda de tu dominio. Parece así:

You should not enter any sensitive information on this site (for example, passwords or credit cards), because it could be stolen by attackers.

No debes ingresar ninguna información confidencial en este sitio (por ejemplo, contraseñas o tarjetas de crédito), porque los atacantes podrían robarla.

Esto significa que definitivamente debes asegurarte de que tu sitio web ahora está usando un certificado SSL.

Este artículo te brinda una descripción general de por qué sucede esto y los pasos recomendados sobre cómo actualizar tu sitio web.

¿Por qué está pasando esto?

Google ha estado haciendo un esfuerzo en los últimos años para alentar a los propietarios de sitios web a proteger su sitio y los datos del cliente mediante el uso de un certificado SSL.

Cuando se habilita un certificado SSL, todo el tráfico dentro del dominio se cifra. Esto garantiza que toda la comunicación de datos confidenciales (como la información de la tarjeta de crédito o las contraseñas) esté protegida.

¿Otros navegadores mostrarán la misma advertencia?

En este momento, solo Google Chrome está configurado para mostrar una advertencia insegura. Sin embargo, es probable que en un futuro muy cercano, otros navegadores sigan este ejemplo y adopten la misma política.

¿Realmente importa si mi sitio no maneja información confidencial?

Si. A partir de febrero de 2020, el navegador Chrome representa el 82% por ciento del tráfico total del navegador.

  • https://www.w3schools.com/browsers/default.asp

Esto significa que la mayoría de los visitantes de tu sitio probablemente estén usando Chrome. Incluso si tu sitio no maneja datos confidenciales (como información de tarjeta de crédito o contraseñas), tu sitio seguirá siendo marcado como 'inseguro' por Google. Los visitantes de tu sitio pueden estar preocupados por la advertencia y elegir evitar tu sitio.

¿Cómo puedo arreglar mi sitio web?

Para garantizar que el tráfico de tu sitio web no se vea afectado, se recomienda que agregue un certificado SSL a su sitio web. DreamHost recomienda agregar un 'Let's Encrypt' gratuito o de un certificado pago 'Sectigo'.

No se recomienda un certificado autofirmado, ya que aún mostrará una advertencia de seguridad en cualquier navegador.

  • Comprar un certificado SSL firmado profesionalmente

¿Qué sucede si Chrome todavía dice que el sitio no es seguro después de agregar el certificado SSL?

Es posible que Chrome siga mostrando el sitio como inseguro después de que haya agregado el certificado SSL. Esto sucede debido a enlaces inseguros en tu sitio. Consulte el siguiente artículo para obtener detalles sobre cómo resolver esta advertencia.
