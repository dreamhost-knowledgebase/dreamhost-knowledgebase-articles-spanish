Generalidades

Agregar un certificado SSL a su sitio de WordPress es un excelente primer paso para proteger sus datos. Sin embargo, hay otros pasos adicionales que debe seguir para garantizar que todos los links dentro de su sitio apunten a la versión segura de su nombre de dominio (Esto significa la versión 'HTTPS').

Tenga en cuenta que estas instrucciones no se aplican a los sitios de DreamPress. Si su sitio de WordPress está en DreamPress, siga estas instrucciones:

  • Forzando HTTPS con DreamPress

Paso 1 — Agregar un certificado SSL al dominio

Asegúrese de que su sitio esté configurado correctamente para HTTPS. Debe tener un certificado SSL aplicado a su sitio y posiblemente una dirección IP única, dependiendo de sus requisitos para el sitio. Consulte el artículo de IP única para obtener más información.

Asegúrese de que su certificado SSL esté activo antes de continuar.

Paso 2 — Actualizar los URL de la base de datos

Tu base de datos de WordPress tiene varias URL codificadas que apuntan a la versión 'http' de tu dominio. Tienes dos opciones para actualizar las URL de tu base de datos:

  • Usa un complemento (plugin)
  • Usa la línea de comando

Consulta el siguiente artículo para obtener más instrucciones:

  • Cambiar todas las URL en tu base de datos de WordPress

Paso 3 — Verificar que el URL de dirección de WordPress y del sitio estén actualizados

  1. Inicie sesión en WordPress utilizando el nuevo URL de HTTPS seguro.
    • https://example.com/wp-login.php
  2. Vaya a la página de Configuración > General.
  3. En la página General, debería ver que su URL de dirección de WordPress y del sitio ahora apunten a 'https'.

Limpia el caché de tu complemento (plugin)

En este punto, también debes eliminar tu caché si usas WP Super Cache.

  • Eliminar el caché

Si revisas tu sitio ahora, todas las URL dentro de tu sitio de WordPress usarán el nuevo nombre de dominio.

Paso 4 — Actualiza tu archivo wp-config.php

  1. Inicie una sesión en su servidor de web a través de SSH o SFTP.
  2. Encuentre el archivo llamado 'wp-config.php' en el directorio de su sitio de WordPress y luego ábralo en un editor de texto.
  3. ​Agregue las dos líneas siguientes arriba de la línea que dice /* That's all, stop editing! Happy blogging. */: 
    define('FORCE_SSL', true);
define('FORCE_SSL_ADMIN',true);
 

Paso 5 — Fuerza la URL para redireccionar a HTTPS

Después de haber agregado un ceritficado SSL a tu dominio, DreamHost redirige automáticamente la URL que usan los visitantes para ver tu sitio de HTTP a HTTPS. La 'S' asegura que tu conexión esté encriptada. Por ejemplo:

  • https://example.com

No hay nada de tu parte que debas hacer para obligar a tu sitio a utilizar la URL segura. Sin embargo, en algunos casos especiales, puede ser necesario que crees tu propio archivo de configuración personalizado para forzar la redirección de HTTP a HTTPS.

Agregar código personalizado solo es necesario si tu sitio web requiere un código específico para forzar el redireccionamiento. Si es así, primero deberás desactivar la redirección automática en tu panel.

  • Deshabilitando la redirección automática a HTTPS

Una vez deshabilitado, puedes proceder a añadir tu archivo de redirección personalizada.

La mayoría de los sitios de WordPress NO requieren que agregues código al archivo .htaccess para redirigirlo. Sin embargo, si tu sitio específico de WordPress requiere un código personalizado por alguna razón, puedes forzar a tu sitio a redirigir de HTTP a HTTPS de dos maneras:

  • Actualizando las reglas del archivo .htaccess de tu sitio
  • Usar un plugin HTTPS de WordPress

Paso 6 — Prevenir advertencias de contenido mixto usando un archivo .htaccess

Es una buena idea agregar una línea de código a tu archivo .htaccess para evitar posibles advertencias de contenido mixto. Consulta el siguiente artículo para obtener más detalles.

  • Resolviendo advertencias de sito inseguro y contenido mixto

Una vez que haya completado todos los pasos anteriores, cualquier solicitud a su sitio de web se redirigirá a la versión de HTTPS.
