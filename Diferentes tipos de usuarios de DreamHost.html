Generalidades

DreamHost ofrece la capacidad de crear muchos tipos diferentes de usuarios, el cual es necesario para diferentes tareas dentro del panel, así como para interactuar con tu sitio. Por ejemplo:

  • Restablecer contraseñas
  • Usuarios Web (FTP, SFTP, Usuarios Shell)
  • Usuarios de correo
  • Usuarios MySQL
  • Usuarios de estadísticas
  • Usuario Administrador de Servidores Dedicados
  • Usuarios de Aplicación
  • Usuarios Remixer
  • Usuarios del Panel de control de cuenta
  • Usuarios WebID

Este artículo enumera los diferentes tipos y cómo se usan.

Restablecer contraseñas

Visita el artículo de Contraseñas para obtener instrucciones detalladas sobre cómo restablecer la contraseña para diferentes tipos de usuarios.

  • Restablecer contraseñas

Usuarios web

Un usuario del sitio web es el nombre de usuario asignado a tu dominio en el servidor. Este es el nombre de usuario que utiliza para iniciar sesión en tu servidor utilizando un cliente FTP para cargar/descargar archivos. Mira el siguiente artículo para obtener instrucciones sobre cómo agregar un usuario del sitio web al servidor.

  • Agregar un usuario a un servidor

Usuarios de Correo

Los usuarios de correo solo se usan para leer y enviar correos, y no pueden iniciar sesión en el servidor web a través de FTP. Estos usuarios tampoco tienen acceso de Shell a los servidores web o de correo. En DreamHost, todos los buzones se nombran actualmente en el formato m######## aunque también se puede acceder a ellos utilizando toda la dirección de correo principal asociada. Cada buzón tiene una dirección de correo principal asociada, pero también puedes tener otras direcciones de correo ilimitadas que también se entregan. Puede configurar esto desde la página Administrar Correo.

Si se elimina un dominio del panel, se eliminan todos los correos asociados con ese dominio. Asegúrate de hacer una copia de seguridad de tu correo antes de eliminar un dominio. Mira el siguiente artículo para obtener más información.

  • ¿Cómo hacer una copia de seguridad de mi correo?

Usuarios MySQL

Los usuarios de MySQL se utilizan para conectarse a tu servidor de base de datos.

MySQL tiene su propio sistema de permisos complejo, y cada vez que necesites acceder a una base de datos MySQL, debes iniciar sesión con un nombre de usuario y una contraseña que reconozcas. Cada vez que crees una nueva base de datos MySQL con DreamHost, deberás especificar un nombre de usuario único que desees crear para permitir el acceso a él. Los usuarios de MySQL se crean en la página Bases de Datos MySQL.

Cuando creas un usuario, solo tienes acceso a MySQL y debe ser único en todas las demás cuentas de MySQL en DreamHost. MySQL también verifica desde dónde se está conectando y solo permite el acceso desde direcciones IP o hosts autorizados.

De forma predeterminada, esto está configurado para que pueda acceder desde cualquier servidor DreamHost; pero si deseas acceder a su base de datos MySQL directamente desde la computador de tu casa o desde otro servidor, puedes agregar más hosts permitidos (direcciones IP o nombres de host) desde el área MySQL del panel.

Usuarios de estadísticas

Los usuarios de estadísticas son usuarios que puedes crear con el único propósito de ver las estadísticas del sitio web.

  • Puedes ver estas estadísticas a través de una URL como https://example.com/stats/.
  • Se mantienen en un archivo .htpasswd que administra desde la página Estadísticas del Sitio.
  • Para más información sobre esto, visita Acceder estadísticas.

Usuarios Administradores de Servidores Dedicados

Un usuario administrador del Servidor dedicado es un usuario especial que solo está disponible en estos planes de alojamiento.

  • Estos usuarios pueden proporcionar acceso sudo a áreas de la configuración de configuración del Servidor Dedicado que normalmente administra DreamHost a través del panel DreamHost. Puedes agregar un usuario dedicado en la página Administrador de Usuarios.
  • Un usuario administrador puede ser muy útil para instalar scripts personalizados o realizar cambios avanzados de configuración de PHP o Apache.

Ten en cuenta que configurar un usuario administrador puede ser arriesgado. Como un usuario administrador tiene acceso a todas las áreas, puede alterar cualquier configuración del sistema. Debes saber lo que estás haciendo antes de configurar un usuario administrador.

Usuarios de Aplicaciones

Muchas aplicaciones web (sistemas CMS, sistemas de blogs, etc.) tienen la capacidad de crear sus propios usuarios para administrar y proporcionar acceso limitado a las características, archivos y contenido de una aplicación. Estos no son administrados por DreamHost, sino por las propias aplicaciones.

Remixer

Remixer es el creador de sitios de un solo clic "personalizado" de DreamHost. El propietario de la cuenta es el 'usuario' que controla cada sitio Remixer, y en este momento no es posible agregar usuarios adicionales a un sitio Remixer. Visita el artículo Comenzar con con Remixer para más detalles.

Usuarios del Panel de control de cuenta

Los usuarios del panel de control de la cuenta se utilizan para iniciar sesión en el panel de control de la cuenta de DreamHost. Este puede ser el propietario principal de la cuenta u otro usuario que el propietario haya proporcionado con credenciales para iniciar sesión en el panel.

Agregar un usuario de cuenta

El propietario de la cuenta puede crear usuarios adicionales con acceso limitado a la cuenta. Revisa el siguiente artículo para obtener más información:

  • Generalidades de Privilegios de Cuenta

Renombrar un usuario de cuenta

Para cambiar el nombre del usuario de la cuenta, dirígete a la página Administrar Cuenta. En la parte superior, haz clic en el nombre de tu cuenta.

El campo se vuelve editable para que puedas cambiar el nombre. Haz clic en el Guardar cuando termines.

Usuarios WebID

Conocido cariñosamente como tu webID, este usuario es el único que no le da acceso a ningún servidor DreamHost físico. En cambio, las webID solo se usan para iniciar sesión en el Panel de Control de DreamHost y administrar tu cuenta. Los webID tienen la intención de representar lógicamente a un ser humano real, por lo que una webID puede poseer varias cuentas de facturación, con múltiples dominios, usuarios (FTP/ Shell/Correo), bases de datos MySQL, etc. También puedes compartir el acceso a cualquiera de las funciones de tu cuenta agregando Privilegios de Cuenta.

Debido a esto, no hay razón para tener más de un solo WebID. Los WebID también son permanentes y no se pueden cambiar de nombre ni cambiar.
