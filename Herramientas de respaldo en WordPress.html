Generalidades

Hay muchos plugins populares para usar para hacer una copia de seguridad en WordPress. Puedes almacenar tus copias de seguridad en Amazon S3, DropBox o incluso en DreamObjects. Siempre que sea posible, DreamHost recomienda que limites la cantidad de copias de seguridad a una cantidad razonable, ya que las copias de seguridad infinitas pueden causar problemas de rendimiento en tu sitio.

Plugins sugeridos

  • Copia de seguridad de BoldGrid – (Gratis con cualquier plan de WordPress DreamHost).
  • Plugin de copias de seguridad de DreamObjects – Diseñado específicamente para trabajar con DreamObjects.
  • UpdraftPlus – Admite DreamObjects de forma nativa.
  • BlogVault
  • BackupBuddy
  • VaultPress

Para obtener más información sobre el servicio DreamObjects, visita el siguiente artículo:

  • Generalidades de DreamObjects

Almacenar copias de seguridad personales o copias de seguridad del sitio en los servidores host va contra la política de DreamHost (Esto no aplica para VPS o Servidores Dedicados.) Debes eliminar cualquier contenido de respaldo del sitio y almacenarlo localmente. Puedes leer más sobre la política ilimitada aquí:

  • https://www.dreamhost.com/es/legal/politica-ilimitada/

Hacer una copia de seguridad manual de tu sitio WordPress

Puedes hacer una copia de seguridad manual de tu sitio WordPress usando el proceso a continuación.

¿Dónde correr el comando?

Debes iniciar sesión en tu servidor web vía SSH para correr los siguientes comandos. Revisa el siguiente artículo para obtener instrucciones completas sobre cómo iniciar sesión en tu servidor en una terminal:

  • Generalidades de SSH

Hacer la copia de la base de datos de tu sitio usando wp-cli

Inicia sesión en una terminal. Asegúrate de que estás en el directorio del sitio web. Luego, corre el siguiente comando:

[server]$ wp db export ~/backup`date +"%d-%m-%Y"-%T`.sql

Hacer la copia de los archivos de tu sitio usando SSH

Navega de nuevo al directorio home del usuario. Luego, corre el siguiente comando:

[server]$ zip –r exampledomain.com.zip exampledomain.com

Asegúrate de cambiar el comando para que coincida con el nombre del directorio de tu sitio web.

Descargar la copia de tu sitio

Si los archivos .zip y .sql de la copia de seguridad tienen un tamaño inferior a 1 GB, puedes descargarlos mediante WebFTP. Revisa el artículo vinculado a continuación para obtener más detalles:

  • Usar el administrador de archivos del sitio web en el panel

De lo contrario, deberás descargar cualquier archivo de la copia de más de 1 GB mediante FTP. Revisa el artículo a continuación para obtener más detalles:

  • Generalidades sobre FTP y sus credenciales
