Generalidades

Cuando subes datos a DreamObjects, estos están sin cifrar predeterminadamente. Aunque los datos son seguros dentro de DreamObjects, existen algunas opciones disponibles para protegerlos aún más mediante el cifrado.

Este artículo explica diferentes formas en las que puedes asegurarte de que tus datos estén encriptados dentro de DreamObjects.

Encriptación del lado del servidor

El cifrado del lado del servidor es cuando tus datos se cifran mientras se escriben en el servidor. Cuando desees acceder a los datos, se descifrarán del servidor. DreamHost actualmente no admite el cifrado del lado del servidor con DreamObjects.

Cifrado del lado del cliente

El cifrado del lado del cliente es cuando tu cifras los datos ANTES de cargarlos en el servidor. Esto significa que los datos que cargas ya están encriptados mientras se escriben en el servidor. La siguiente aplicación admite el cifrado del lado del cliente:

  • Encriptación de datos con S3cmd

SSE-C

SSE-C está disponible, sin embargo, esta es una opción solo para usuarios avanzados. Esto se debe a que requiere que crees tus propias claves de cifrado para cargar/descargar contenido y cifrar datos.
