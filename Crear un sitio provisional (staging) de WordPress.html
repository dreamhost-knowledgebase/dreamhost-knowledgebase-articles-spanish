Generalidades

Este artículo explica cómo crear un sitio de prueba de WordPress.

Requisitos previos

Estas instrucciones suponen que has instalado WordPress manualmente o has utilizado el Instalador de WordPress de DreamHost. También suponen que tu sitio de WordPress está en un alojamiento Shared, VPS, o Dedicado.

Para el resto de esta guía, se supone que tu sitio web es example.com y que el sitio de prueba es example.dreamhosters.com.

Antecedentes

Un sitio web de prueba es un sitio que puedes usar para probar los cambios que te gustaría ver en tu sitio web activo. Por ejemplo, puedes probar nuevos temas, plugins, y actualizaciones para ver si algo afecta negativamente la funcionalidad de tu sitio. Esto te permite trabajar en el mismo sin afectar el sitio web activo.

Diferencia entre un sitio de prueba de WordPress o DreamPress

¿Aún no tienes DreamPress?

Para actualizar tu sitio de WordPress a DreamPress, visita el siguiente artículo:

DreamPress es el alojamiento WordPress administrado de DreamHost. Es un servicio premium con varias características adicionales sobre el alojamiento estándar de WordPress.

DreamPress ya incluye la capacidad de preparar su sitio web, por lo que las instrucciones de este artículo no son necesarias. Consulte los siguientes artículos para obtener más información sobre cómo usar la preparación con DreamPress:

  • Cómo hacer un sitio provisional en DreamPress
  • Copiar tu sitio de DreamPress a Staging para probar los cambios
  • Publicar tus cambios de Staging en tu sitio de DreamPress en vivo

Con tu instalación de WordPress de DreamHost, tendrás que crear manualmente el sitio de prueba.

Crear un sitio de prueba de WordPress

Usa los siguientes tres pasos para crear un sitio web de prueba y luego trasladarlo a tu sitio en vivo.

 

Crear un subdominio dreamhosters.com

Para preparar un sitio de WordPress, deberás crear manualmente un duplicado del mismo creando un nuevo subdominio dreamhosters.com. Puedes nombrar este nuevo subdominio dreamhosters.com como desees.

Puede que el DNS tarde unas horas en actualizarse, pero una vez que lo haga, deberías ver una página Próximamente. Sigue estas instrucciones para aprender a copiar el contenido de example.com a example.dreamhosters.com mientras actualizas las URL en la base de datos. Esto es muy importante ya que cada base de datos de WordPress tiene URL codificadas que deben coincidir con el nombre de tu sitio web.

Instala y prueba

En este punto, el sitio example.dreamhosters.com debería ser una copia exacta de example.com. Ahora puedes instalar y probar lo que quieras.

Cuando el sitio example.dreamhosters.com esté configurado exactamente como lo deseas, será momento de mover los datos nuevamente a tu sitio web activo en example.com.

Mueve el sitio preparado nuevamente al sitio en vivo

Para mover tu sitio preparado de nuevo a tu sitio activo, repite el paso que realizaste anteriormente para copiar el sitio. Estas instrucciones te muestran cómo importar los datos de nuevo a tu sitio web activo y cambiar las URL en la base de datos.

Una vez completado, tu sitio web en vivo se actualiza con el contenido de tu sitio de prueba.
