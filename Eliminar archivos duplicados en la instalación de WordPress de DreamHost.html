Generalidades

Hay varias formas de instalar WordPress. Por ejemplo:

  • Usando el instalador WordPress de DreamHost en el panel.
  • Descargando los archivos directamente de WordPress.org y subirlos a tu sitio.
  • Usando wp-cli vía SSH para instalarlo directamente en tu servidor.

Este artículo explica cómo el instalador de WordPress de DreamHost duplicará los archivos instalados manualmente y cómo eliminarlos del servidor.

Instalaciones manuales de WordPress

Si subiste un sitio WordPress anteriormente a tu servidor de forma manual, no se mostrará en el panel, pero los archivos de WordPress existirán en el directorio del mismo. Si luego usas el instalador de WordPress de DreamHost en el panel, los nuevos archivos de WordPress se instalarán normalmente y los archivos anteriores se les agregará una marca de tiempo.

Cómo se renombran los archivos antiguos

Por ejemplo, los archivos de WordPress se ven de esta manera:

  • index.php
  • wp-login.php
  • wp-settings.php

Si estos archivos ya existen y corres la instalación de WordPress de DreamHost, se agregará una marca de tiempo a los archivos recién instalados. Por ejemplo:

  • index.php.1555975213
  • wp-login.php.1555975213
  • wp-settings.php.1555975213

Los siguientes pasos explican cómo eliminar estos archivos duplicados.

Eliminar los archivos de instalación duplicados

  1. Inicia sesión en tu servidor a través de SSH.
  2. Navega al directorio de tu sitio web. Cambia example.com a tu sitio. 
    [server]$ cd ~/example.com
  3. Corre el siguiente comando para eliminar estos archivos duplicados. Solo cambia el número de marca de tiempo. 
    [server]$ find . -name "*.1555975213" -type f -delete
  4. Corre ls -la y verás que se han eliminado todos los archivos duplicados.
