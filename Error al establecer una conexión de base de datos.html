Generalidades

Lo siguiente te ayudará a identificar y solucionar algunos problemas comunes de la base de datos cuando veas el mensaje Error al establecer una conexión a la base de datos.

Cómo corregir el 'error al establecer una conexión a la base de datos'

Este error indica que los archivos de tu sitio web en el servidor web no pueden conectarse a la base de datos del servidor de la misma. En las siguientes secciones se describen los motivos más comunes de este error y cómo resolverlos.

Corregir las credenciales de base de datos en tu archivo de configuración

Cada sitio web que se conecta a una base de datos tiene un archivo de configuración que almacena las credenciales esenciales para la conexión:

  • Nombre de usuario de la base de datos.
  • Contraseña de usuario de la base de datos.
  • Hostname de la base de datos.
  • Nombre de la base de datos.

Si la información de inicio de sesión de la base de datos cambia en el panel sin actualizar el archivo de configuración del sitio, el sitio no se conectará a la base de datos. A continuación, se muestran los archivos de configuración estándar para el software más conocido:

Software Nombre del archivo de configuración Ubicación del archivo de configuración
Concrete CMS database.php application/config/
Joomla configuration.php En el directorio principal del sitio
MediaWiki Localsettings.php En el directorio principal del sitio
phpBB config.php En el directorio principal del sitio
WordPress wp-config.php En el directorio principal del sitio
Zen Cart configure.php includes/

Encontrar las credenciales de tu base de datos

Revisa this article para obtener instrucciones sobre cómo ubicar las credenciales de tu base de datos:

Nunca uses localhost para tu hostname. Solo debes usar el hostname configurado en tu panel DreamHost.

Corregir el hostname

El hostname es la URL que utilizas para acceder a tu base de datos. Revisa este artículo para obtener más información sobre cómo encontrar esta URL en tu panel.

Para probar el hostname, intenta acceder a este a través de un navegador web. Si no se carga nada, es probable que tenga un hostname incorrecto. Los hostname en el mismo servidor son intercambiables; si funciona otro hostname, puedes utilizar ese. Si no tienes una alternativa, crea un nuevo hostname y espera a que se propague. Para obtener más información, revisa el artículo phpMyAdmin.

Si se carga el hostname pero se solicita un nombre de usuario y una contraseña, revisa que el usuario y la contraseña en tu archivo de configuración sean correctos. Revisa este artículo para obtener detalles sobre cómo obtener las credenciales correctas en tu panel.

Asegúrate de que el servidor de la base de datos esté disponible

Es posible que su servidor MySQL no responda debido a problemas de red. Si ves un error al intentar conectarte a phpMyAdmin, contacta a soporte para más asistencia.
