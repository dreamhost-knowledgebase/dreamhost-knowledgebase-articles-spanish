Generalidades

Es posible que veas el siguiente error al visitar su sitio:

Error establishing a database connection / Error al establecer una conexión de base de datos

Este error significa que los archivos de tu sitio web (en el servidor web) no pueden conectarse a tu base de datos (en el servidor de la base de datos). Este artículo enumera algunas razones comunes por las que este error podría aparecer en tu sitio, incluyendo:

  • Credenciales incorrectas de la base de datos en tu archivo config
  • El hostname no está funcionando
  • La base de datos del servidor no está disponible

Credenciales de base de datos incorrectas en tu archivo de configuración

Cada sitio web que se conecta a una base de datos tiene un archivo de configuración. Este archivo de configuración almacena las siguientes credenciales de base de datos que te permiten conectarte.

  • Nombre de usuario de la base de datos.
  • Contraseña de usuario de la base de datos.
  • Nombre de host de la base de datos.
  • Nombre de la base de datos.

Si la información de inicio de sesión de la base de datos se modificó en el panel sin actualizar la información de conexión en el archivo de configuración del sitio, tu sitio no puede conectarse a la base de datos. Los archivos de configuración estándar para software popular se muestran a continuación:

Software Nombre del archivo de configuración Ubicación del archivo de configuración
Concrete CMS database.php application/config/
Joomla configuration.php En el directorio principal del sitio
MediaWiki Localsettings.php En el directorio principal del sitio
phpBB config.php En el directorio principal del sitio
WordPress wp-config.php En el directorio principal del sitio
Zen Cart configure.php includes/

¿Dónde puedo encontrar las credenciales de mi base de datos?

Consulta el siguiente artículo para obtener instrucciones sobre cómo ubicar las credenciales de tu base de datos:

  • Encontrar las credenciales de inicio de sesión de tu base de datos

Nunca uses localhost para tu nombre de host. Solo debes usar el nombre de host configurado en tu panel DreamHost.

El hostname no está funcionando

El nombre de host es la URL que usa para acceder a tu base de datos. Consulta el siguiente artículo para obtener instrucciones sobre cómo ubicar esta URL en tu panel.

  • Encontrar tu nombre de host

Primero, prueba el nombre de host intentando acceder a él con un navegador web. Si no se carga nada, es probable que tengas un nombre de host incorrecto y deberás reemplazarlo por uno que funcione. Afortunadamente, los nombres de host en el mismo servidor son intercambiables, por lo que si tienes otro nombre de host que funcione en el mismo servidor, puedes usar el que funcione. Si no tienes otros nombres de host para ese servidor, deberás crear uno y esperar a que se propague. Mira el artículo phpMyAdmin para más detalles.

Si se carga el nombre de host, se le solicitará un nombre de usuario y contraseña. Si se rechaza el nombre de usuario y la contraseña del archivo de configuración, es posible que debas actualizarlos con la información correcta. Consulta el siguiente artículo para obtener detalles sobre cómo obtener las credenciales correctas en tu panel.

  • Encontrar las credenciales de inicio de sesión de tu base de datos

El servidor de la base de datos no está disponible

Es posible que su servidor MySQL no responda debido a problemas de red. Si ves un error al intentar conectarte a phpMyAdmin, contacta a soporte para más asistencia.
