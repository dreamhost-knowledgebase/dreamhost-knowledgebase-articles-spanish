Generalidades

Es posible que DreamHost necesite realizar cambios en el servidor de vez en cuando para optimizar el entorno de alojamiento y hacerlo más robusto y confiable.

Este artículo te brinda información sobre qué cambia exactamente cuando un servidor se mueve y cómo resolver cualquier problema.

¿Qué cambia cuando un servidor se mueve?

Nombre del servidor

Cuando se cambia tu servidor, tu nombre también cambia. Cada servidor compartido en DreamHost tiene un nombre específico.

Por favor vé el artículo Encontrar el nombre de host de tu servidor FTP para obtener información sobre cómo ver el nombre de tu servidor web

A continuación hay algunas cosas que debes verificar antes de la mudanza para asegurarte de que las cosas salgan lo más suavemente posible:

  • Asegúrate de que cuando te conectes a tu dominio a través de FTP/SFTP/SSH, uses tu dominio como nombre de host.
  • Si tu dominio es example.com, ingresaría exactamente eso como tu nombre de host.
  • No añadas https:// a la URL.

Configuración DNS

Aquí hay algunos pasos para configurar los ajustes de DNS cuando se mueve un servidor:

  1. Revisa el correo que se te envía una vez que se ha completado la migración del servidor – tu nuevo servidor aparece en la línea Asunto.
  2. Si no estás utilizando los nameservers de DreamHost, actualiza tu dirección IP una vez que migres el servidor.
    Si tu IP continúa apuntando a la antigua IP del servidor, tu sitio no funcionará.
  3. Ve el artículo DNS para obtener información sobre cómo encontrar la dirección IP correcta de tu sitio.
  4. Si es necesario, actualiza la IP correcta de tu sitio en la compañía que aloja tus nameservers.

Ten en cuenta que las IP únicas también cambiarán, ya que DreamHost no ofrece direcciones IP estaticas.

Solucionando problemas después de la mudanza

Una vez que se completa la migración, es posible que notes errores en tu sitio o que no responda como lo hizo una vez.

Las siguientes secciones te ayudan a solucionar las causas más comunes de este.

Mi sitio no responde

Este es probablemente un problema de DNS. Como el servidor cambió, la IP del sitio también cambió. En tu host DNS actual, debes apuntar su DNS a tu nueva IP DreamHost. Vea la sección de configuración de DNS anterior para corregir esto.

Si tu sitio apunta correctamente a tu nueva IP de DreamHost, es posible que veas un error 'Site Not Found'. Ve el artículo Página web no encontrada para arreglar esto.

Si tu sitio aún no responde, ponte en contacto con el soporte de DreamHost para obtener ayuda.

Mi formulario de correo o cron job  no funciona

Es posible usar tu nombre de servidor como parte de tu dirección de correo en un formulario de correo personalizado o cron job. Por ejemplo, podrías usar:

username@servername.dreamhost.com

En este ejemplo, 'username' es tu usuario FTP y 'servername' es el nombre de tu servidor web. Si usas este formato, asegúrate de actualizar el nombre del servidor que estás usando para la dirección de correo.

Mis archivos cargados no se muestran en línea

Esto es probable porque estás utilizando el antiguo nombre del servidor como tu nombre de host FTP. Si estás utilizando algo como SERVER.dreamhost.com como tu nombre de host, deberás cambiarlo a tu nuevo nombre de servidor (que se te proporciona en la línea Asunto del correo que confirma que se ha movido con éxito). Consulta el artículo Encontrando el nombre de host de tu servidor FTP para obtener más información sobre cómo ver el nombre de tu servidor web.

Una idea aún mejor es usar tu nombre de dominio. El nombre de dominio apuntará automáticamente al servidor correcto sin importar dónde se mueva.

Mi llave de servidor es incorrecta

Como te mudarás a un nuevo servidor, las claves SSH también cambiarán. Si tienes la clave del servidor anterior guardada en tu computador local, deberás eliminarla. Cuando inicies sesión en el nuevo servidor después de eso, te pedirá que guardes la nueva clave.

Usar Linux (OpenSSH)

La versión OpenSSH de SSH proporciona una utilidad de línea de comandos para eliminar claves de host específicas sin molestar a las demás. Esto es especialmente útil si usas SSH para conectarse a más de un host y no deseas editar manualmente el archivo known_hosts.

En el indicador de shell, escribe:

[server]$ ssh-keygen -R www.example.com
  • Esto creará una copia de seguridad del archivo ~/.ssh/known_hosts y eliminará la clave anterior de www.example.com de los actuales known_hosts.
  • Repite según sea necesario para cada dominio que se movió.
