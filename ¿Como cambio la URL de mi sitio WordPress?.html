Generalidades

Cambiar la URL del sitio WordPress es solo una parte de una serie de pasos que necesitas tomar para asegurarte que tu sitio WordPress funcione con la nueva URL. Revisa el siguiente artículo para ver una lista con pasos para migrar tu sitio WordPress a tu nuevo dominio.

  • Cómo mover WordPress a un dominio diferente

Tu puedes mover el nombre de dominios tu sitio WordPress en el momento que quieras, pero hay unas configuraciones importantes que deben hacerse para cambiarlo.

Asegúrate de seguir cuidadosamente los siguientes pasos para hacerlo correctamente.

Muchas razones para cambiar la URL

Lo que verás son instrucciones de cómo mover una instalación de WordPress de una URL a otra. Hay diferentes casos para mover una instalación de WordPress, como muestra abajo.

Caso #1 — Mover de un subdirectorio

El primer caso es cuando un sitio WordPress se mueve de un subdirectorio a su dominio padre. Por ejemplo:

  • Mover de https://example.com/blog
    a
  • https://example.com

Si ya has movido tu dominio a una nueva URL, no vas a poner iniciar sesión pues WordPress está configurado para usar la vieja URL.

Caso #2 — Agregar o remover www

Otra razón para actualizar tus URL es si necesitas agregar o remover www de tu dominio. Por ejemplo, si estás usando Cloudflare con DreamHost, necesitas agregar www a tu dominio. Entonces si antes era example.com, deberás cambiarlo a www.example.com.

Después de ajustar la URL, puedes ver error de redireccionamiento del buscador cuando veas tu sitio- Puede decir 'Muchas redirecciones' o 'Too many redirects' o 'Esta página no está redireccionando correctamente' o 'The page isn't redirecting properly'.

Caso #3 — Agregar 'https' a una URL

Si estás usando un certificado SSL, la primera parte de la URL cambia de 'http' o 'https'. Si ya has hecho este cambio, no vas a poder iniciar sesion pues WordPres está configurado para usar la vieja URL.

Caso #4 — Mover a un nuevo dominio

El último caso es cuando un sitio WordPress se mueve de un dominio totalmente alojado a uno diferente. Por ejemplo:

  • Mover de http://example.com
    a
  • http://dreamhost.com

Si esta opción aplica a tu sitio, mira el siguiente artículo para ver instrucciones de cómo mover una instalación de WordPress a un dominio diferente.

  • Cómo mover WordPress a un dominio diferente en DreamHost

Diferentes maneras de mover una URL de WordPress

Las siguientes dos opciones explica cómo cambiar tu HOME y tu URL de tu sitio para ganar acceso inmediato a tu sitio WordPress y el dashboard.

Después de iniciar sesión, deberás usar un plugin para actualizar todas las URL en tu base de datos con la nueva URL. Vé el siguiente artículo para ver las instrucciones completas.

  • Cambiar todas las URL en tu base de datos de WordPress

Opción 1 – Cambiar el HOME y el SITE URL usando phpMyAdmin

Si tu ya has hecho cualquiera de los cambios de arriba, no vas a poder iniciar sesión en tu dashboard. Puedes en vez de eso usar phpMyAdmin para actualizar el HOME y el SITE URL.

Visita el siguiente artículo para más información de cómo iniciar sesión en phpMyAdmin:

  • Encontrar tus credenciales de inicio de sesión de phpMyAdmin
  1. Abre phpMyAdmin.
  2. Da click en la parte izquierda de la base de datos. Muchas tablas aparecerán debajo. Da click en la tabla wp_options.
  3. Localiza las filas de siteur y home. Da click en el ícono de lápiz en cada uno para editarlos.
    Cuando bajes por estas opciones, el nombre 'browse' o 'buscar' aparecerá.
  4. Da click en el ícono de lápiz en cada uno para editarlos.
  5. Altera la URL escribiendo una nueva
  6. Da click en el botón GO para guardar las configuraciones. Haz esto para las opciones 'siteurl' y 'home'.

Deberías poder acceder ahora a tu sitio a través de example.com/wp-login.php.

Aún cuando esto te da acceso a tu sitio, Todas las URL antiguas en tu base de datos siguen usando la vieja URL. Ahora podrás actualizar todas las URL en tu base de datos a la nueva URL. Vé el siguiente artículo para ver las instrucciones completas:

  • Cambiar todas las URL de tu base de datos de WordPress

Opción 2 – Cambiar las URL en tu archivo wp-config.php

Tu puedes cambiar también las URL en tu archivo wp-config.php. Esto es un poco más fácil que editar tu base de datos y te dará acceso inmediato a tu sitio.

El siguiente ejemplo asume tu Home y Site URL son example.com.

  1. Inicia sesión en tu sitio usando FTP o SSH.
  2. En tu directorio WordPress, edita el archivo llamado wp-config.php.
    • Crear y editar archivos vía FTP
    • Crear y editar archivos usando SSH
  3. En ese archivo, agrega las siguientes líneas de abajo /* That's all, stop editing! Happy blogging. */. 
    define('WP_HOME','https://example.com');
define('WP_SITEURL','https://example.com');
  4. Si tu visitas tu sitio ahora, verás que ahora puedes iniciar sesión.

Aún cuando esto te da acceso inmediato a tu sitio, la base de datos no está configurada todavía para usar correctamente HOME y SITE URL. Ahora podrás usar el plugin para actualizar todos las URL en tu base de datos a la nueva URL. Da click en el siguiente artículo para ver las instrucciones completas:

  • Cambiar todas las URL en tu base de datos de WordPress

Verificar el nuevo HOME y SITE URL

Si usaste las opciones #1 de arriba, las URL HOME y SITE URL fueron actualizadas en la base de datos. Puedes confirmar que tu sitio está usando ahora el URL correcto en tu dashboard.

    1. Inicia sesión en tu dashboard de WordPress.
    2. Navega a Settings > General.
    3. Aquí podrás ver cómo están configuradas las URL de HOME y SITE URL de tu sitio WordPress.

Si usaste la opción #2, la base de datos no actualizó tu HOME y SITE URL. En Configuraciones > General verás que las entradas están en gris y no accesibles con los valores que ingresaste en tu archivo wp-config.php.

Deberás ahora usar un plugin para actualizar todas las URL en tu base de datos a la nueva URL. Vé el siguiente artículo para ver las instrucciones completas:

  • Cambiar todas las URL en tu base de datos de WordPress

Una vez actualizados, elimina las líneas de tu archivo wp-config.php. Vé a la página de Configuraciones > General y verás que esos campos ya no están en gris.
