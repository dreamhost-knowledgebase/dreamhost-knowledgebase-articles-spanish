Generalidades

Hay muchas maneras de evitar el uso excesivo del disco en tu plan de alojamiento y garantizar que tu servidor y sitio web continúen funcionando de manera rápida y eficiente. Este artículo describe por qué el almacenamiento en bloque y las opciones de terceros pueden ser una solución más rentable y estable que almacenar copias de seguridad en un servidor web.

Almacenamiento en bloque

El almacenamiento en bloque es un servicio de almacenamiento basado en la nube que está diseñado específicamente para almacenar grandes cantidades de datos. Los siguientes son ejemplos de servicios de almacenamiento en bloque que podrías usar:

DreamObjects

DreamObjects es un servicio de almacenamiento de DreamHost basado en la nube que puede ser usado para lmacenar cualquier cosa; desde datos estáticos para tu sitio web hasta copias de serguridad del mismo. Tu puedes acceder a DreamObjects en el panel DreamHost, mediante programación a través de las API estándar de S3 o mediante cualquier aplicación compatible con la conectividad de S3. Soluciones de respaldo de sitios web

Al crear y almacenar copias de seguridad, el objetivo siempre es garantizar que sigan siendo redundantes. Si se almacenan localmente (en el mismo servidor que tu sitio), se vuelven ineficaces por varias razones:

  • DreamHost hace todo lo posible para evitar la pérdida de datos en todos los servidores. Sin embargo, en el improbable caso de que algo le sucediera a tu servidor, las copias de seguridad almacenadas en el mismo lugar que tu sitio se verían afectadas.
  • Si tu sitio web alguna vez se ve comprometido, las copias de seguridad almacenadas localmente también podrían estar expuestas a vulnerabilidades de seguridad que afectan al mismo.

Los siguientes son ejemplos de soluciones de respaldo que puedes usar.

Respaldos de WordPress

El plugin UpdraftPlus de WordPress incluye opciones para copiar tus datos de respaldo en un servidor de terceros, lo que garantiza que tus copias de seguridad no se almacenen localmente. Alternativas de almacenamiento de medios de terceros

Las alternativas al almacenamiento en bloque incluyen proveedores que ya usan tecnologías de almacenamiento masivo similares. Los sitios web como YouTube, Vimeo, y SoundCloud están diseñados en torno a la necesidad específica de alojar archivos de video y audio, lo que brinda una variedad de formas de incrustar o vincular el contenido que ha subido a esas plataformas.

Optimización de imágenes y limpieza de sitios web

Un sitio web optimizado es rápido mientras mantienes el uso del disco bajo. Aquí hay algunos consejos para ayudarte a optimizar tu sitio web:

  • Elimina las imágenes duplicadas de tu sitio (si necesitas usar la misma imagen más de una vez, vuelve a usar la misma imagen en lugar de usar varias copias).
  • Usa una herramienta de optimización de imágenes para optimizarlas antes de cargarlas en el servidor.
  • Si estás usando plugins de WordPress, como el plugin EWWW Image Optimizer puede optimizar tus imágenes sobre la marcha.

Optimización del archivo de registros

DreamHost mantiene los registros de tu sitio web en horarios establecidos. Sin embargo, es posible que las aplicaciones que has instalado en tu sitio no estén configuradas de una manera que garantice que se mantengan en un tamaño utilizable. La forma más sencilla de hacerlo es a través de un cron job.

Usa el siguiente sintaxis cuando crees tu cron job.

Asegúrate de cambiar username al usuario de tu sitio web y /app/log_file a la ruta del archivo de registro de tu aplicación.

/usr/sbin/logrotate /home/username/app/log_file
