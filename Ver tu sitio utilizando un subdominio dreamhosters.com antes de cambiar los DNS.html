Generalidades

Este artículo explica cómo ver tu sitio antes de apuntar tus DNS a DreamHost utilizando un subdominio dreamhosters.com. Para explorar otra opción sobre cómo ver tu sitio en DreamHost antes de apuntar los DNS, visita el siguiente artículo:

  • Cómo ver tu sitio en DreamHost usando tu archivo hosts

Si estás transfiriendo tu sitio a DreamHost, eventualmente necesitarás apuntar los DNS tu sitio en DreamHost. Sin embargo, también es posible probar tu sitio en DreamHost antes de cambiar los DNS. Puedes hacerlo utilizando un subdominio dreamhosters.com.

Hay dos formas de usar un subdominio dreamhosters.com para ver tu sitio en DreamHost antes de apuntar los DNS a DreamHost.

Opción # 1 (funciona con cualquier sitio web)

Por favor contacta a soporte si quieres habilitar SSL en tu subdominio dreamhosters.com.

Subir tu sitio directamente al subdominio dreamhosters.com.

Ten en cuenta que si tu sitio utiliza enlaces absolutos (como un sitio de WordPress), debes actualizar manualmente todas las URL en los archivos y la base de datos del sitio al nuevo nombre de dominio temporal. Si no lo haces, los enlaces en el sitio apuntarán a la URL del sitio existente que aún no está alojado en DreamHost.

Opción # 2 (solo funciona con sitios sencillos que no usan un certificado SSL)

Sube tu sitio en DreamHost, luego usa el subdominio dreamhosters.com para Reflejar tu sitio real.

Ten en cuenta que esta opción no se puede usar si tu sitio usa un certificado SSL. Esto se debe a que la opción de espejo en el panel DreamHost no funcionará con el certificado SSL de otro sitio.

Opción # 1 — Subir tu sitio directamente al subdominio dreamhosters.com

La mejor opción es subir tu sitio a un subdominio dreamhosters.com. Este es un dominio gratuito que puedes agregar a tu cuenta para que actúe como un sitio temporal. Los siguientes artículos explican cómo crear un subdominio de dreamhosters.com y luego subir los archivos y la base de datos de tu sitio web.

  • Generalidades sobre FTP y credenciales
  • ¿Cómo migrar mi base de datos a DreamHost?

¿Qué pasa si mi sitio usa WordPress?

Si su sitio usa WordPress, o es un sitio que contiene enlaces absolutos (URL completas a tu sitio), debes actualizar manualmente esos enlaces para que el sitio funcione en el subdominio dreamhosters.com.

Por ejemplo, supongamos que tu sitio es example.com y has subido el contenido a example.dreamhosters.com. Si tu sitio contiene enlaces absolutos (que apuntan a example.com), cuando hagas clic en enlaces en example.dreamhosters.com, los enlaces te llevarán al sitio actual en vivo de example.com.

Para solucionar esto, debes actualizar todas las URL en tu base de datos que apuntan a tu sitio web en vivo (example.com) a example.dreamhosters.com. El siguiente artículo explica cómo actualizar las URL en tu base de datos para apuntar al nuevo subdominio dreamhosters.com:

  • Cambiar todas las URL en tu base de datos de WordPress

Una vez que hayas actualizado todas las URL del sitio, deberías ver que este carga con example.dreamhosters.com.

Migrar de nuevo a la URL en vivo

Cuando estés listo para que el sitio sea publicado, debes actualizar una vez más las URL de tu sitio. Simplemente invierte los pasos anteriores para cambiar todas las URL del subdominio dreamhosters.com (example.dreamhosters.com) a la URL de tu sitio en vivo. Solo después de que se hayan actualizado las URL es que debes apuntar tus DNS.

Opción #2 — Ver tu sitio a través de un 'Espejo' de dreamhosters.com

Esta opción solo se puede usar si lo siguiente se cumple:

  • Tu sitio NO está utilizando un certificado SSL
  • Tu sitio NO contiene enlaces absolutos (URL completas a su sitio)

Para una visualización web rápida, agrega un subdominio de dreamhosters.com en tu página Manage Domains. Luego podrás usar esto para reflejar tu sitio web real. Visita el siguiente artículo para obtener más instrucciones:

  • Generalidades del dominio como espejo

Esperar a que se actualicen los DNS

Este espejo recién creado puede demorar hasta 6 horas en actualizarse, pero una vez que lo haga, podrás ver tu dominio en DreamHost (example.com) usando el espejo en example.dreamhosters.com.

Si confirmas que el sitio se ve y funciona como debería, estás preparado para apuntar tus DNS a DreamHost. Cuando estés listo para apuntar los DNS, visita las siguientes páginas de artículos que proporcionan instrucciones completas:

  • Generalidades de los DNS
  • Generalidades de los Servidores de Nombres

No intentes visitar tu sitio en DreamHost por su dirección IP, ya que la dirección IP es para el servidor, no para el sitio web. Si visitas la IP, verás una página con el mensaje Sitio No Encontrado del que puedes leer más aquí:

  • Sitio no encontrado
