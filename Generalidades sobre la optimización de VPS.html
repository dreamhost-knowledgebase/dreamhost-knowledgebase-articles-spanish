Generalidades

Las siguientes sugerencias te ayudarán a identificar problemas comunes de memoria y a optimizar tu VPS de DreamHost. Optimizar tu VPS ayuda a garantizar un menor uso de memoria mientras acelera tus sitios web.

Cómo bloquear bots abusivos

A common problem for websites is caused by search engine indexing bots repeatedly hitting the website. The following can help you troubleshoot this issue.

Crear un archivo robots.txt: Para minimizar el problema de los bots abusivos, deberías configurar un archivo robots.txt. Este archivo ayuda a controlar qué bots pueden buscar y con qué frecuencia.

Ver el archivo access.log: Encontrar bots abusivos puede ser más un arte que una ciencia, pero puedes obtener algunos datos sobre esto analizando el archivo access.log de tus sitios web. Revisa este artículo para obtener más información sobre cómo revisar tu archivo access.log y ver de dónde proviene la mayor parte del tráfico.

Prevenir enlaces directos de imágenes o hotlinking

El hotlinking ocurre cuando otro sitio web utiliza una imagen de tu sitio web. Esto sucede cuando se usa la URL completa de una imagen de tu sitio en su propio sitio. Esto puede causar problemas importantes ya que incluso un sitio con relativamente poco tráfico puede causar un rendimiento deficiente del servidor.

Revisa el artículo Prevenir hotlinking de imágenes para más información sobre cómo prevenir que esto suceda.

Cómo almacenar caché en tu sitio

El almacenamiento en caché ayuda a almacenar archivos de sitios web, lo que hace que tus visitantes accedan a ellos más rápidamente. Asegúrate de que el software de tu sitio web use alguna funcionalidad de almacenamiento en caché para evitar problemas de memoria y mejorar el rendimiento general.

Trabajar con procesos de PHP: Los procesos de PHP pueden consumir una cantidad significativa de memoria. Si tu sitio está recibiendo mucho tráfico, habilitar el almacenamiento en caché ayudará a reducir la cantidad de procesos de PHP generados. El almacenamiento en caché renderiza un script PHP y luego "almacena en caché" esa salida HTML como un archivo. La próxima vez que se acceda a esa página, el servidor mostrará la página HTML estática en lugar de procesar nuevamente el script PHP.

Usar Cloudflare: También puedes habilitar un servicio de terceros como Cloudflare, que ofrece opciones adicionales de almacenamiento en caché para tus sitios web si esperas un gran volumen de visitantes.

Usar OPcache con sitios PHPs: OPcache es un motor de almacenamiento en caché integrado en PHP que, cuando está habilitado, aumenta drásticamente el rendimiento de los sitios web basados en PHP. Revisa este artículo para obtener más información sobre cómo usar OPcache con tu sitio web.

Optimizaciones específicas de WordPress

Revisa la categoría de Optimización de WordPress para obtener artículos detallados si estás ejecutando múltiples instancias de WordPress en tu VPS. Asegúrate de hacer lo siguiente con cada instalación:

  • Ejecuta siempre la versión más reciente de WordPress.
  • Instala sólo plugins esenciales.
  • Mantén actualizados tus plugins.
  • Elimina plugins que no utilices en lugar de simplemente desactivarlos.
  • Confirma que el tema que está activo no esté generando ningún error en tu archivo error.log (ubicado en el mismo directorio que access.log).
  • Revisa la página Optimizing Page Load Time (optimización del tiempo de carga de la página) y asegúrate de tener un plugin de almacenamiento en caché de WordPress instalado y configurado. Por ejemplo, W3TC o WP Super Cache.
  • Comprime tus archivos de imagen, lo cual mejora el tiempo de carga y reduce el uso general de memoria.

Configurar las Tablas de IPs

No es posible personalizar las tablas IP en un VPS de DreamHost. Esto se debe a que las tablas IP están configuradas en una sola máquina host a la que no hay acceso.

Si necesitas ejecutar tus propias tablas de IPs, la única opción sería actualizar a un Servidor Dedicado o DreamCompute e instalar este software manualmente.
