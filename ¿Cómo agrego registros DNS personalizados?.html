Generalidades

Tu puedes agregar registros DNS personalizados para un dominio configurado como 'Dominio Alojado', así como un dominio configurado solo para DNS. Todo depende de qué registros estés creando y para qué los necesites.

En términos generales, si deseas que tu sitio esté alojado en los servidores de DreamHost, debes mantener tu sitio como 'Dominio Alojado' (Fully Hosted). Sin embargo, si deseas que tu sitio apunte, mediante la creación de registros A o CNAME, a un servicio de alojamiento externo, el primer paso es eliminar el servicio de alojamiento que esté utilizando tu dominio en DreamHost.

Los siguientes pasos explican cómo configurar un dominio para Solo DNS (si es necesario) e igualmente cómo crear los siguientes tipos de registros DNS personalizados.

  • Registro A
  • Registro CNAME
  • Servidores de Nombre
  • Registro SRV
  • Registro TXT

Configurar tu dominio a Solo DNS

Visita el siguiente artículo para obtener instrucciones sobre cómo configurar tu dominio como 'Solo DNS'.

  • Configurando tu dominio a 'Solo DNS'

Esto es necesario si tu sitio está alojado actualmente en DreamHost y deseas apuntar tu DNS a otra compañía de alojamiento utilizando registros A o CNAME.

Registro A 

Los registros A son direcciones IP. Se pueden agregar para apuntar tu sitio web a una compañía de alojamiento diferente, o diferentes subdominios de tu sitio a servicios externos.

Apuntar un subdominio a una dirección IP específica

El siguiente ejemplo muestra cómo apuntar un subdominio de example.com a una dirección IP específica. Por ejemplo, si tu sitio es example.com y este utiliza los servidores de nombres, de DreamHost, puedes crear un sitio llamado blog.example.com o store.example.com. Luego de esto, puedes configurar este subdominio para que apunte a una empresa de alojamiento externa. Esto es útil si deseas que un subdominio apunte a un alojamiento externo.

El siguiente ejemplo muestra cómo crear un subdominio llamado subdomain.example.com y cómo apuntarlo a una IP. Asegúrate de cambiar la dirección IP a la IP que te brinda el servicio de alojamiento externo.

  1. Haz clic en el enlace 'DNS' ubicado bajo tu dominio en la página Manage Domains.
  2. Ingresa lo siguiente en tres campos para crear el registro A:
    • Name: Para el dominio principal, deja en blanco el campo 'Nombre'. Para 'www' u otros subdominios, puedes ingresar el nombre del subdominio en este campo.
    Este campo suele ser cualquiera de los siguientes: www, blog, dev, store, etc.

    No agregues el símbolo @ en este campo. Este se agrega automáticamente.

    • Type: El registro 'Tipo' por defecto es 'A'.
    • Value: Introduce la dirección IP en el campo 'Valor'.
    Esta es la dirección IP a la que quieres que apunte este dominio/subdominio.
  3. Haz clic en botón Add Record Now!.

La propagación del nuevo registro A comienza y puede demorar hasta 6 horas antes de que los nuevos registros se actualicen en línea.

Registros CNAME

Al igual que con los registros A personalizados, puedes apuntar tu alojamiento a otro diferente a DreamHost con un registro CNAME. Este registro se crea para señalar un dominio a otro en lugar de apuntar un dominio a una dirección IP como con el registro A personalizado.

Ten en cuenta que el registro raíz A (por ejemplo, example.com) no puede ser un registro CNAME y siempre debe ser un registro A.

  1. Haz clic en el enlace DNS debajo del nombre de dominio en la página Manage Domains.
  2. Introduce lo siguiente:
    • Name: Este campo puede ser cualquier cosa. Pero muy a menudo es sólo www. Al agregar www, vas a apuntar el subdominio www a un servicio de alojamiento diferente a DreamHost.
    En el ejemplo anterior, el registro CNAME se denomina "calendar" y apunta a "ghs.google.com".
    • Type: En el menú desplegable, selecciona 'CNAME'.
    • Value: Ingresa la ubicación a la que deseas que apunte este registro.

      Los registros CNAME no pueden comenzar con un guión bajo.

  3. Haz clic en botón Add Record Now!.

Después de hacer clic en el botón Add Record Now!, comienza la propagación del nuevo registro. Esto puede llevar hasta 6 horas antes de que los nuevos registros se actualicen en línea.

Es posible que veas el siguiente error al agregar un CNAME:

You already have a record for this name. You can't have a CNAME and any other record on the same name.

Esto generalmente significa que estás intentando agregar un CNAME para 'www', pero como ese registro ya existe en tu configuración de DNS, no puedes agregar un segundo. Si necesitas agregar un CNAME para el subdominio www, primero debes configurar tu dominio a Solo DNS.

  • Configuración de tu dominio a Solo DNS

Si estás intentando agregar un registro CNAME para otra cosa, revisa tus registros DNS para ver si el registro que estás agregando ya existe.

Registro SRV

Los registros SRV (servicio) son otro conjunto de registros que se pueden aplicar a tu dominio. Ciertos programas y aplicaciones pueden requerir la configuración de estos tipos de registros para activar estos servicios.

Tu registro SRV estándar tendrá este tipo de formato:

_service._protocol.name. TTL class SRV priority weight port target.

Cuando estés ingresando un registro SRV en tu panel, ignora los campos TTL, clase y SRV. Un ejemplo completo está abajo.

Los siguientes dos registros SRV de Microsoft son algunos de los registros SRV más comunes que están configurados para un dominio. El ejemplo se puede ver aquí:

  • Dos registros SRV de Microsoft
Type Name Target Protocol Service Priority Weight Port TTL
SRV @ sipdir.online.lync.com _tls _sip 100 1 443 1 hour
SRV @ sipfed.online.lync.com _tcp _sipfederationtls 100 1 5061 1 hour

A continuación se muestra un ejemplo de cómo configurar uno de los registros anteriores.

  1. En tu panel, ve a Manage Domains.
  2. Haz clic en el enlace 'DNS' debajo de tu nombre de dominio:

    • Name: _sipfederationtls._tcp
    Este es el 'Servicio' seguido por el 'Protocolo'.
    En la captura de pantalla anterior, solo se está creando uno de los registros SRV. Repite estos pasos para el segundo registro.
    • Type: Seleccione SRV en el menú desplegable.
    • Value: Debes ingresar los valores en el orden (prioridad) (peso) (puerto) (destino)
    Usando el ejemplo del alojamiento de correo de MS anterior, el valor del primer registro es:
    100 1 5061 sipfed.online.lync.com.
    Los siguientes valores se utilizan en el ejemplo anterior:
    prioridad = 100
    peso = 1
    puerto = 5061
    destino = sipfed.online.lync.com
  3. Haz clic en el botón Add Record Now!.

Después de hacer clic en el botón Add Record Now!, la propagación del nuevo registro A comienza y puede demorar hasta 6 horas antes de que los nuevos registros se actualicen en línea.

Los dos registros van a verse de la siguiente manera una vez creados:

_sipfederationtls._tcp	SRV	100 1 5061 sipfed.online.lync.com.

_sip._tls	        SRV	100 1 443 sipdir.online.lync.com.

Se aplica un período final al final del registro una vez que se crea. Esto no causará ningún problema.

Registro TXT

Ver el registro TXT en Wikipedia.

  1. Ve a la página Manage Domains.
  2. Haz clic en el enlace 'DNS' debajo del nombre de dominio.
  3. Introduce lo siguiente:
    • Name: Este campo puede ser cualquier cosa. Muy a menudo se deja en blanco.
    • Type: En el menú desplegable, selecciona 'TXT'.
    • Value: Ingrese la ubicación a la que deseas que apunte este registro.
    El ejemplo anterior es un registro TXT típico de Google para verificación de sitios.
  4. Haga clic en el botón Add Record Now! para guardar los cambios.

Después de hacer clic en el botón Add Record Now!, comienza la propagación del nuevo registro. Esto puede llevar hasta 6 horas antes de que los nuevos registros se actualicen en línea.

Detalles adicionales

  • Al igual que con todos los cambios en el DNS, estos registros necesitan tiempo para actualizarse en línea. Es posible que las cosas empiecen a tardar algunas horas en comenzar a funcionar. Puedes vaciar el DNS para ver si esto acelera el proceso de propagación, pero no hay garantías de que esto ayude. Puedes leer las instrucciones para descargar el DNS aquí:
    • Vaciar DNS
  • Los cambios de DNS pueden tardar hasta 6 horas en resolverse. Si tus servidores de nombre apuntan a DreamHost y los servicios configurados para el registro creado no están activos dentro de este tiempo, puedes ponerte en contacto con soporte para obtener ayuda en Contact Support.

El soporte de DreamHost puede indicarte dónde puede crear registros personalizados y verificar si el registro se aplica correctamente a tu dominio. Sin embargo, el soporte no puede ayudar a configurar estos registros personalizados o confirmar si el registro creado es correcto o no.
