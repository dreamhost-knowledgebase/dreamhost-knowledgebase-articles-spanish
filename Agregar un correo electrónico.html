Una de las mejores cosas de tener tu propio dominio es que puedes tener cualquier dirección personal que te guste para ese dominio, que se ve más profesional que una dirección de correo genérica, como @gmail.com.

Agregar una dirección de correo solo requiere unos pocos pasos simples que esta guía explica. Sin embargo, si necesitas más detalles sobre otros aspectos de una dirección de correo, revisa el artículo Configuración de cliente de correo.

Requisitos previos

Debes tener un plan de alojamiento activo, o un plan pago adicional de correo para usar el correo.

Todos los planes de alojamiento, excepto Shared Starter, ofrecen correo sin costo adicional.

Si bien los planes de alojamiento Shared Starter y las cuentas de registro de dominio únicamente no incluyen correo electrónico, puedes agregar un plan de correo pago a tu cuenta que incluya hasta 5 direcciones. Revisa el siguiente artículo para obtener más información:

  • Plan pago adicional de correo

Correo completamente alojado

En el pasado, era posible crear un correo completamente alojado que también se reenviaba a otra bandeja de entrada. Esta herramienta ya no es soportada, y las nuevas direcciones de correo creadas no tendrán esta opción. Las direcciones de correo existentes que usan esta función no se verán afectadas hasta que se editen, momento en el que deberás seleccionar entre reenviar solo o completamente alojado.

Una dirección de correo completamente alojada es un correo que tiene un buzón activo para que puedas iniciar sesión para enviar y recibir mensajes. Los usuarios de correo completamente alojados pueden acceder a su propio buzón y usar la interfaz de Webmail y la interfaz de Mailboxes si no deseas acceder a a través de la interfaz de Webmail.

Agregar un correo completamente alojado

  1. Navega a la página Administrar Correo.
  2. Haz clic en el botón Crear Correo.
  3. Del menú desplegable 'Seleccionar un dominio' que queda bajo Correo Electrónico selecciona tu dominio.
  4. Ingresa el nombre de tu correo electrónico en el campo de texto que aparece.
  5. Selecciona Correo Completamente Alojado debajo de Tipo de Buzón y haz clic en el botón Continuar.
  6. Ingresa un nombre para el buzón (este puede ser tu nombre o algo que describa el propósito del buzón).
  7. Haz clic en el botón Generar contraseña segura para generar una contraseña (o ingresa una tuya si prefieres).
  8. Da clic en el botón Crear Correo en la parte inferior de la página para guardar la nueva información.
    Aparece un mensaje exitoso.

Ahora estás listo para configurar tu Cliente de correo y comenzar a enviar y recibir correos.

Para encontrar rápidamente tu configuración de correo en el panel, abre la página Administrar Correo y busca el enlace Ayuda en la parte superior derecha. Haz clic en el texto para abrir las instrucciones de Configuración de Cliente de Correo de tu correo electrónico:

Entrante
imap.dreamhost.com
pop.dreamhost.com
Saliente
smtp.dreamhost.com

¿Cuánto tiempo tarda una nueva dirección de correo en funcionar?

De forma predeterminada, tu dominio no está configurado con registros MX (alojamiento de correo) con DreamHost cuando el dominio está configurado para ser alojado. Los registros MX y todos los demás registros de alojamiento de correo se aplican al dominio una vez que se crea la primera dirección de correo.

Si este es el primer usuario de correo creado para el dominio, el dominio necesitará algo de tiempo para que se propague el DNS del registro de correo, lo que puede tardar hasta 6 horas para cada registro. Debido a esto, no se podrá acceder al usuario hasta que el DNS haya terminado de actualizarse.

Si tu dominio ya está configurado con los registros para el alojamiento de correo, el usuario recién creado debe estar completamente activo dentro de los 15 minutos posteriores a su creación.

Posibles errores

Los pasos anteriores suponen que tu DNS está alojado con DreamHost. Si tienes Registros MX personalizados para un dominio, no podrás crear nuevas direcciones. Esto se debe a que si estás usando Gmail (u otro proveedor de correo alternativo), debes crear nuevas direcciones de correo en ese proveedor.

Otro problema a tener en cuenta es que, a veces, cuando ya tienes una gran cantidad de direcciones creadas, recibes un mensaje de error que dice:

“Tu cuenta no puede crear direcciones de correo electrónico adicionales en este momento.
Esta suele ser una situación temporal y debe resolverse en un plazo de 24 horas.
Comunícate con Soporte si tienes alguna pregunta.”

En caso de que recibas este mensaje, por favor contacta a soporte

Eliminar una dirección de correo

Para eliminar un correo en el panel:

  1. Navega a la página Administrar Correo.
  2. Haz clic en el botón Administrar a la derecha de tu correo.
  3. Da clic en el botón rojo Eliminar, luego da clic una vez más para confirmar su eliminación.

Registros DNS de tu nueva dirección de correo

Los siguientes registros se configuran cuando se crea el primer usuario:

  • _domainkey (dkim)
  • _autodiscover (srv)
  • mail. (A)
  • mail. (MX)
  • mail. (MX)
  • mailboxes. (A)
  • www.mailboxes. (A)
  • webmail. (A)
  • www.webmail. (A)

Hay dos registros MX para el alojamiento de correo del dominio junto con los registros de los subdominios para buzones y correo web. Cada registro puede resolverse en diferentes momentos, por lo que un servicio puede estar activo mientras que otro aún no se ha actualizado.

Durante este tiempo, los mensajes enviados a tu usuario serán rechazados con una notificación de devolución que contiene el siguiente error:

Recipient Address Rejected

Hasta que se resuelva el DNS, los mensajes enviados a tu usuario continuarán siendo rechazados. Sin embargo, tu usuario debería poder enviar mensajes incluso cuando el DNS aún no se haya resuelto. Esto a menudo causa confusión sobre por qué un usuario puede enviar, pero no puede recibir correos. Dale al DNS un poco más de tiempo para que resuelva antes de que el nuevo usuario pueda recibir el correo entrante.

Direcciones de correo estándar de Internet

Los estándares de Internet especifican que tu creas las siguientes direcciones de correo específicas, que todos los dominios deben tener y usar para los siguientes propósitos:

Dirección Propósito
postmaster@example.com Contacto para cualquier problema de correo, debe existir para cualquier dominio que tenga servicio de correo
webmaster@example.com Contacto para cualquier problema del sitio web, debe existir para cualquier dominio que tenga un sitio web o servicio http
hostmaster@example.com Contacto para cualquier problema de DNS, debe existir para todos los dominios ya que todos usan DNS para todo
abuse@example.com Contacto para cualquier "comportamiento público inapropiado"
