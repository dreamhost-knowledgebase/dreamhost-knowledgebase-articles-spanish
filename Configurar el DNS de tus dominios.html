Generalidades

DNS es la tecnología que permite que tu dominio apunte a una dirección IP específica y permite que el servidor que lo escucha aloje tu dominio. Hay varios tipos diferentes de registros, pero los más utilizados son:

  • A - Se utiliza para vincular un dominio o subdominio a una dirección IPv4.
  • AAAA - Se utiliza para vincular un dominio o subdominio a una dirección IPv6.
  • CNAME - Se utiliza para vincular un dominio o subdominio a otro dominio o subdominio.

Para la mayoría de los propósitos, solo necesitarás usar registros A y AAAA.

Ubicar de tu dirección IP pública

A cada instancia de DreamCompute en el clúster US-East 2 se le asigna una dirección IPv4 e IPv6 pública.

Puedes ver estas direcciones de dos formas:

  • En la página Instancias del panel de DreamCompute
  • Usando el cliente de línea de comando de OpenStack

Si tienes varias instancias, tendrás varias direcciones IPv4 e IPv6 públicas. Deberás rastrear qué direcciones alojan qué sitios web u otros servicios.

Agregar direcciones IP a tu dominio en el panel DreamHost

Para agregar una nueva dirección IP:

  1. Configura tu dominio como 'Solo DNS'. Mira el artículo Configurar tu dominio como 'Solo DNS' para más información.
    • Asegúrate de que tu dominio esté ya alojado. Mira el artículo Agregar un Sitio Web y Hosting para más información sobre como agregar un alojamiento a tu dominio.
  2. Crea un nuevo registro A para el dominio o subdominio. Mira el artículo Agregar registros DNS personalizados para más información.
  3. Crea registros A adicionales según sea necesario para los dominios que deseas alojar en tu instancia. También puedes crear un registro A para la versión sin www de la URL de tu sitio. Todo lo que necesitas hacer es dejar el campo 'Host' en blanco.

Esperar la propagación de DNS

Una vez que se guardan las entradas de DNS, el DNS debe propagarse, lo que puede llevar varias horas. El TTL (tiempo de vida) predeterminado para los nameservers DreamHost es de 5 minutos, por lo que cualquier cambio en tus registros DNS debe actualizarse en casi todas partes del mundo dentro de ese tiempo.
