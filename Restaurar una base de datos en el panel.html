Generalidades

A continuación se explica cómo restaurar una base de datos en el panel.

DreamHost's daily backups

Todas las bases de datos MySQL de DreamHost se respaldan diariamente. Las copias de seguridad comienzan a medianoche y se completan después de que se haya realizado una de cada base de datos. En general, hay cinco días de respaldos disponibles.

Se RECOMIENDA ENCARECIDAMENTE que crees tus propios respaldos externos de todas tus bases de datos para complementar las que a corto plazo crea DreamHost. DreamHost no garantiza que las copias de seguridad estén disponibles.

Cuentas suspendidas/cerradas/deshabilitadas

DreamHost no puede garantizar que los datos se retendrán en cuentas inactivas (suspendidas/cerradas/deshabilitadas) durante más de dos semanas después del cambio de estado de la cuenta. Consulta los siguientes artículos para obtener más información sobre cómo puedes hacer una copia de seguridad de tu cuenta o realizar un pago:

Restaurar una base de datos

Para restaurar una base de datos:

  1. Navega a la página Bases de Datos MySQL.
  2. Haz clic en el botón Restaurar BD que se encuentra a la derecha de tu base de datos.
  3. Escoge lo que te gustaría restaurar:
    • ¿Restaurar qué copia de seguridad? — Elige cualquier copia de seguridad disponible del menú desplegable.
    • Tablas para restaurar — Si deseas restaurar toda la base de datos, deja este espacio con un asterisco, como *
    • Al restaurar — Tienes 4 opciones de cómo se restauran tus tablas.
    Opción Selección Descripción
    1 Mis tablas restauradas deberían llamarse tablename__Timestamp y las tablas que están ahí ahora deberían estar intactas. Las opciones 1 y 2 restauran las tablas, pero se modifican los nombres. De esta manera, los archivos restaurados NO reemplazan los archivos actuales. Deberás cambiar manualmente los nombres de las tablas o ajustar el archivo de configuración de tu sitio para ver la base de datos restaurada.
    2 Mis tablas restauradas deberían llamarse Timestamp__tablename y las tablas que están ahí ahora deberían estar intactas.
    3 Mis tablas restauradas deberían llamarse tablename y las tablas que están ahí ahora deberían cambiar su nombre a tablename__Timestamp. Las opciones 3 y 4 reemplazan inmediatamente las tablas de la base de datos. Las tablas actuales se renombran. Lo más frecuente es que desees una de las dos últimas opciones.
    4 Mis tablas restauradas deberían llamarse tablename y las tablas que están ahí ahora deberían cambiar su nombre a Timestamp__tablename.

    Sitios de WordPress

    Al restaurar una base de datos del sitio de WordPress, se recomienda encarecidamente que selecciones la Opción 4. Esto te permite revertir fácilmente el sitio a la versión anterior de tu base de datos si hay algún problema con los datos restaurados.

    Puedes volver rápidamente a la versión anterior actualizando el valor del prefijo en tu archivo wp-config.php. Inicia sesión en tu sitio a través de FTP o SSH y abre el archivo wp-config.php (está en el directorio de tu sitio). Verás la siguiente línea:

    $table_prefix = 'wp_';

    Reemplaza el valor con la nueva marca de tiempo de la copia de seguridad que creaste. Por ejemplo:

    $table_prefix = '20240329_141233_wp_';
  4. Haz clic en el botón ¡Restaurar mi copia de seguridad ahora!

Aparecerá un mensaje de éxito notificándote que tu base de datos se restaurará dentro de 10 minutos.

Administrar el uso de disco de la base de datos de MySQL

DreamHost alienta a sus clientes a desarrollar las bases de datos lo más pequeñas y eficientes posibles.
