Generalidades

Al eliminar tu servidor privado, tienes la opción de eliminar permanentemente todos los usuarios y datos, o migrar todo a un servidor diferente. Asegúrate de seguir los pasos a continuación cuidadosamente, de lo contrario podrías perder datos en tu servidor.

Si deseas migrar tus datos a un servidor diferente

Este suele ser el caso al eliminar un servidor privado. Esto también puede considerarse como la migración de los datos de tu servidor privado a un plan de alojamiento diferente.

Si deseas mantener tu sitio web activo y migrarlo a un plan de alojamiento diferente, debes tener activo un plan de Alojamiento Shared o Dedicado al momento de eliminar el servidor privado. Solo entonces tendrás la oportunidad de migrar los datos a este nuevo servidor cuando lo canceles.

Si no hay otro servicio de alojamiento, el sitio web se cerrará y tu cuenta solo estará disponible para registros de dominio.

Asegúrate de tener otro plan de alojamiento activo si deseas mantener tu sitio en funcionamiento.

  1. Primero, asegúrate de haber comprado un segundo plan de alojamiento. Este podría ser un compartido, Dedicado, o incluso otro plan de VPS.
  2. Navega a la página VPS.

    Si el uso de la RAM del VPS es muy alto, puede que no veas el botón Eliminar. Se necesita que el uso disminuya antes de que puedas eliminar un VPS. Revisa el siguiente artículo para más información sobre reducir el uso de memoria de RAM de un VPS:

    • Generalidades sobre optimización de VPS
  3. A la derecha del VPS debajo de la columna Acciones, haz clic en el botón rojo Eliminar.
  4. Se muestra una página que te ofrece la opción de migrar todos los datos a un servidor diferente. En la captura de pantalla anterior, todo se está migrando a un servidor compartido.
  5. Haz clic en Mover todo en ps### a un <newserver> y luego destruir ps###!

Cuando termine, DreamHost le enviará al propietario de la cuenta un correo para confirmar que el VPS ha sido eliminado.

Si ya no deseas alojar sitios en tu servidor privado

Si tienes un VPS MySQL con 3GB o más de datos y no tiene un Servidor Dedicado, no puedes usar el botón Eliminar en el panel para eliminar tu VPS. Primero debes disminuir el tamaño a menos de 3GB para cada base de datos MySQL VPS para eliminar el VPS en el panel.

Si ya no necesitas que tu servidor aloje tus sitios web, puedes eliminar el servidor privado en la página VPS. Primero asegúrate de haber hecho una copia de seguridad de todos los datos necesarios, ya que no serán recuperables después de eliminar el servidor.
