Generalidades

En lugar de utilizar contraseñas, DreamCompute usa claves SSH para acceder a tus instancias. Esto se hace para aumentar la seguridad y simplifica la administración de usuarios para OpenStack, ya que la clave pública se inyecta en la instancia durante la creación. Esto significa que no necesitas modificar archivos o contraseñas en el sistema operativo al iniciar.

Hay varias formas de crear una Llave de Emparejamiento. Elige una de las siguientes opciones.

  • Crea una nueva Llave de Emparejamiento en el panel de DreamCompute
  • Crea una nueva Llave de Emparejamiento en Windows
  • Crea una nueva Llave de Emparejamiento en Mac o Linux
  • Creación e importación de una Llave de Emparejamiento mediante la CLI de OpenStack

A cada instancia solo se le puede asignar una única Llave de Emparejamiento. Cuando lanzas originalmente una instancia, puedes seleccionar el par de claves que te gustaría usar con la nueva instancia.

No es posible dentro del panel DreamCompute cambiar la Llave de Emparejamiento asignada a una instancia después de que la instancia se ha creado sin sobrescribir/borrar datos. Asegúrate de asignar la Llave de Emparejamiento correcta cuando crees originalmente la instancia.

Componentes de la clave SSH

Hay dos partes en una clave SSH:

  • Una llave privada
  • Una llave pública

La clave privada es conocida solo por tí y debe protegerse. En sistemas basados en UNIX como Mac y Linux, la clave privada generalmente se almacena en:

/home/username/.ssh/id_rsa

La clave pública se puede compartir con cualquier servidor al que te gustaría permitir el acceso SSH. En un sistema basado en UNIX, la clave pública generalmente se almacena en:

/home/username/.ssh/id_rsa.pub

Las claves SSH generadas en sistemas Windows mediante el generador de claves PuTTY (puttygen.exe) no se almacenan en una ubicación uniforme, sino donde el usuario decide guardarlas.
