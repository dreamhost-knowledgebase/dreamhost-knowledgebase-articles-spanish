Generalidades

Este artículo explica qué configuraciones cambian cuando se mueve un servidor y cómo resolver cualquier problema posterior.

DreamHost puede mover tu servidor en ocasiones para optimizar el entorno de alojamiento y asegurar que permanezca confiable y seguro. Recibirás avisos por correo electrónico antes y después de este tipo de movimiento para mantenerte informado sobre el proceso.

¿Qué cambia cuando se mueve un servidor?

Nombre del servidor

Cuando se mueve tu servidor, también cambia su nombre. Cada servidor compartido en DreamHost tiene un hostname único que puedes ver en el panel.

Si estás utilizando el nombre del servidor para conectarte en tu cliente FTP, debes actualizarlo al nuevo nombre. De lo contrario, aún puedes usar el nombre de tu sitio web para conectarte.

Configuraciones de DNS

Solo necesitas ajustar tu configuración de DNS si no estás utilizando los nameservers de DreamHost. En este caso, debes actualizar las direcciones IP de tu sitio web en la compañía que aloja tus nameservers.

Si tu IP sigue apuntando a la IP del servidor antiguo, tu sitio no funcionará.

Revisa el artículo ¿A qué registros DNS debo apuntar mi sitio? para obtener información sobre cómo encontrar las direcciones IP correctas de tu sitio.

Las IPs únicas también cambiarán, ya que DreamHost no ofrece direcciones IP estáticas.

Solucionar problemas después del movimiento

Después de completar la migración, puedes notar errores en tu sitio o que no responde como lo hacía antes. Las siguientes secciones te ayudarán a solucionar las causas más comunes de esto.

El sitio no responde

Aquí hay algunos errores del sitio que puedes ver y cómo solucionarlos.

El sitio no carga

Esto es muy probablemente un problema de DNS. Revisa la sección de configuraciones de DNS anterior para corregir esto.

Error de sitio no encontrado

Si tu sitio está apuntado correctamente a tu nueva IP de DreamHost, es posible que veas un error de Sitio No Encontrado. Revisa el artículo Sitio no encontrado para solucionarlo.

El formulario de correo o el cron job no funciona

Es posible usar el nombre de tu servidor como parte de tu dirección de correo en un formulario de correo personalizado o en un cron job donde username es tu usuario del sitio web y el nombre del servidor es el nombre de tu servidor web. Por ejemplo:

username@servername.dreamhost.com

Si estás usando este formato, asegúrate de actualizar el nombre de tu servidor en la dirección de correo electrónico.

Los archivos subidos no aparecen

Esto probablemente se deba a que estás utilizando el nombre del servidor antiguo en tu cliente FTP, por lo que los archivos se subieron al servidor antiguo. Revisa la sección anterior nombre del servidor para más detalles.

Llave del servidor incorrecta

Las llaves SSH también cambiarán cuando se mueva el servidor. Si tu cliente SSH o FTP guardó previamente la llave del servidor antiguo, necesitarás eliminarla. Revisa el artículo de Actualizar las llaves del host para obtener instrucciones sobre cómo resolver este error.
