Generalidades

Un sitio web provisional es un sitio de prueba que puedes usar para probar los cambios que te gustaría ver en tu sitio web en vivo. Por ejemplo, podrías probar nuevos temas, plugins y actualizaciones para ver si algo afecta negativamente la funcionalidad de tu sitio. Esto te permite trabajar en el sitio provisional sin afectar el sitio web en vivo.

Diferencia entre un sitio provisional de WordPress o Dreampress

¿Todavía no tienes DreamPress?

Puedes actualizar un sitio de WordPress existente a DreamPress usando el enlace 'Upgrade to DreamPress' en la página Manage Domains, o ve a la página Managed WordPress para comprar.

DreamPress es el alojamiento administrado de WordPress de DreamHost. Es un servicio premium con varias características adicionales sobre el alojamiento estándar de WordPress.

DreamPress ya incluye la capacidad de provisionar tu sitio web, por lo que las instrucciones en este artículo no son necesarias. Consulta los siguientes artículos para obtener más información sobre el uso de sitio provisional con DreamPress:

  • Cómo usar tu sitio provisional en DreamPress
  • Copiar tu sitio DreamPress a tu sitio provisional para probar los cambios
  • Publicar tus cambios de tu sitio provisional a tu sitio DreamPress en vivo

Con tu instalación estándar de WordPress de un click, deberás crear manualmente el sitio de provisional.

Crear el sitio de WordPress provisional

Estas instrucciones asumen que has instalado WordPress manualmente o utilizando el Instalador de un clic. Esto también supone que tu sitio de WordPress está en alojamiento compartido, VPS o dedicado.

Para el resto de esta guía, se supone que tu sitio web es 'example.com' y el sitio provisional es 'example.dreamhosters.com'.

Para provisionar un sitio de WordPress, deberás crear manualmente un duplicado del mismo. Sigue los pasos a continuación para crear una copia de tu sitio.

  1. Crea un nuevo dominio dreamhosters.com. Puedes nombrarlo como quieras. Esta guía usa el nombre 'example.dreamhosters.com'. El DNS puede tardar algunas horas en actualizarse, pero una vez que lo haga, deberías ver la página 'Coming Soon'
  2. Copia el contenido de 'example.com' a 'example.dreamhosters.com'. Puedes usar cualquiera de los siguientes dos artículos para lograr esto:
    • Uso del plugin de migración de WordPress (DreamHost Automated Migration)
    • Cómo mover WordPress a un dominio diferente en DreamHost

    Si seguiste los pasos en esos artículos, tu dominio no solo se copia al nuevo sitio, sino que las URL en la base de datos también se actualizan.

    Esto es muy importante ya que cada base de datos de WordPress tiene URL codificadas que deben coincidir con el nombre de tu sitio web.

  3. En este punto, tu sitio example.dreamhosters.com debería ser una copia exacta de example.com. Ahora puedes instalar y probar lo que quieras.
  4. Cuando hayas configurado tu sitio example.dreamhosters.com exactamente como le gustaría, es hora de migrar esos datos a tu sitio web en vivo en example.com.

Migrar el sitio provisional nuevamente al sitio en vivo

Para migrar tu sitio provisional a tu sitio en vivo, puedes repetir el paso que realizaste anteriormente para copiar el sitio:

  • Uso del plugin de migración de WordPress (DreamHost Automated Migration)
  • Cómo mover WordPress a un dominio diferente en DreamHost

Ambos artículos muestran cómo volver a importar los datos a tu sitio web en vivo y cambiar las URL en la base de datos.

Una vez completado, tu sitio web en vivo se actualizará con el contenido de tu sitio provisional.
