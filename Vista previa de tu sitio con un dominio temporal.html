Generalidades

Este artículo explica cómo ver tu sitio antes de apuntar su DNS a DreamHost utilizando un subdominio de dreamhosters.com o un dominio temporal de dream.website. Para ver otra opción sobre cómo ver tu sitio en DreamHost antes de apuntar al DNS, consulta el siguiente artículo:

  • Cómo ver tu sitio DreamHost usando su archivo de hosts

Si estás transfiriendo tu sitio a DreamHost, eventualmente deberás apuntar tu DNS a tu sitio DreamHost. Sin embargo, también es posible probar tu sitio en DreamHost primero antes de cambiar tu DNS. Puedes hacerlo utilizando un subdominio de dreamhosters.com o un dominio temporal de dream.website.

Hay varias opciones, que se describen en detalle a continuación:

Opción #1 (funciona con cualquier sitio web)

Subir tu sitio directamente al subdominio dreamhosters.com.

Por favor contacta a soporte si quieres habilitar SSL on your dreamhosters.com subdomain.

Ten en cuenta que si tu sitio utiliza enlaces absolutos (como un sitio de WordPress), debes actualizar manualmente todas las URL en los archivos y la base de datos del sitio al nuevo nombre de dominio temporal. Si no lo haces, los enlaces en el sitio apuntarán a la URL del sitio existente que aún no está alojada en DreamHost.

Oción #2 (solo funciona con sitios simples que no usan un certificado SSL)

Sube tu sitio a DreamHost, luego usa el subdominio dreamhosters.com para 'Reflejar' tu sitio real.

Ten en cuenta que esta opción no se puede utilizar si tu sitio utiliza un certificado SSL. Esto se debe a que la opción de espejo en el panel DreamHost no funcionará con el certificado SSL de otro sitio.

Opción #3 (usando un dominio temporal que ya está instalado en tu plan de alojamiento DreamHost)

Puede utilizar el subdominio temporal dream.website que se incluye automáticamente con tu sitio web alojado en DreamHost.

Opción #1 — Subir tu sitio directamente al subdominio dreamhosters.com

La mejor opción es subir tu sitio a un​ subdominio dreamhosters.com. Este es un dominio gratuito que puedes agregar a tu cuenta para que actúe como un sitio temporal. ¿Qué pasa si mi sitio utiliza WordPress?

Si tu sitio ejecuta WordPress, o es un sitio que contiene enlaces absolutos (URL completas a su sitio), debes actualizar manualmente esos enlaces para que el sitio funcione en el subdominio dreamhosters.com.

Por ejemplo, digamos que tu sitio es example.com y has subido el contenido a example.dreamhosters.com. Si tu sitio contiene enlaces absolutos (que apuntan a example.com), cuando hagas clic en los enlaces de example.dreamhosters.com, los enlaces lo llevarán al sitio activo actual de example.com.

Para solucionar esto, debes actualizar todas las URL en tu base de datos que apuntan a tu sitio web en vivo (example.com), a example.dreamhosters.com. El siguiente artículo explica cómo actualizar las URL en su base de datos para que apunten al nuevo subdominio dreamhosters.com:

  • Cambiar todas las URL en tu base de datos de WordPress

Una vez que hayas actualizado todas las URL del sitio, deberías ver cómo se resuelve en example.dreamhosters.com.

Migrando de nuevo a la URL activa

Cuando estés listo para que el sitio entre en funcionamiento, debes actualizar una vez más las URL de tu sitio. Simplemente invierta los pasos anteriores para cambiar todas las URL del subdominio dreamhosters.com (ejemplo.dreamhosters.com) a la URL de tu sitio en vivo. Solo después de que se hayan actualizado las URL, debes apuntar tu DNS.

Opción #2 — Ver tu sitio a través de un 'Mirror' dreamhosters.com

Esta opción solo se puede utilizar si se cumple lo siguiente:

  • Tu sitio NO utiliza un certificado SSL
  • Tu sitio NOcontiene enlaces absolutos (URL completas a tu sitio)

Para una visualización rápida de la web, agrega un subdominio de dreamhosters.com en tu página Administrar Dominios.Luego puede usar esto para reflejar tu sitio web real. Consulta el siguiente artículo para obtener instrucciones:

Esperando que se actualice el DNS

Este espejo recién creado puede tardar hasta 6 horas en actualizarse, pero una vez que lo haga, podrás ver tu dominio en DreamHost (example.com) utilizando el espejo en example.dreamhosters.com.

Si confirmas que el sitio se ve y funciona como debería, estás preparado para apuntar tu DNS a DreamHost. Cuando estés listo para apuntar su DNS, visita las siguientes páginas del artículo que brindan instrucciones completas:

Opción #3 — Uso del dominio temporal dream.website para ver tu sitio

Cada sitio web alojado en DreamHost incluye un subdominio dream.website gratuito, que puedes usar como dominio temporal para probar tu sitio. Esto es particularmente útil en los casos en los que deseas ver el sitio antes de apuntar el DNS a DreamHost. Es muy parecido a tener un dominio temporal "siempre disponible" que puedes utilizar para este propósito.

Ten en cuenta lo siguiente cuando trabaje con un subdominio de dream.website:

  • La protección SSL (en forma de cualquier tipo de certificado instalado) NO está disponible
  • Si ya no necesitas utilizar su subdominio dream.website y deseas eliminarlo, deberás contactar a soporte para desactivarlo.
