Generalidades

Analog es un programa de análisis de estadísticas de sitios web de código abierto y está instalado en todos los servidores DreamHost de forma predeterminada, sin embargo, debes habilitarlo manualmente en tu panel para poder usarlo.

Puedes habilitar stats en la página Site Statistics al crear un usuario de stats.

Las estadísticas del sitio no están disponibles en los planes de DreamPress (excepto los planes antiguos de DreamPress Legacy). Asegúrate de habilitar Jetpack si deseas capturar estadísticas del sitio en tu sitio DreamPress.

Habilitando stats

Visita el siguiente artículo para obtener detalles completos sobre cómo habilitar y ver stats:

Algunas aplicaciones web (sistemas de administración de contenido, software de blogs, etc.) pueden implementar reglas de reescritura de URL de Apache que pueden impedirte acceder a esta página a menos que modifiques esas reglas. Las instrucciones generales para realizar dichas modificaciones (e instrucciones específicas para ciertas aplicaciones web que se encuentran comúnmente, como WordPress), se pueden encontrar en el artículo hacer stats accesible con htaccess.

Resumen de funciones de estadísticas

La siguiente es la primera sección que aparece cuando un usuario inicia sesión en las estadísticas de su sitio:

Desde la sección ‘Go To’ en la parte superior de la página, los usuarios pueden navegar a secciones y resúmenes separados. El área restante de la sección ‘General Summary’ de la manera siguiente:

Successful requests
La cantidad de veces que alguien logró acceder a cualquier archivo en tu sitio.
Average successful requests per day
La cantidad promedio de solicitudes si el tráfico se mantuvo constante.
Successful requests for pages
La cantidad de veces que alguien logró acceder a un archivo de "page" en tu sitio (archivos que terminan en .html, .htm, .php, .cgi, y demás).
Average successful requests for pages per day
El número promedio si las visitas al sitio fueran constantes.
Failed requests
La cantidad de veces que alguien intentó acceder a un archivo desde tu sitio, pero no tuvo éxito por cualquier motivo.
Redirected requests
La cantidad de veces que un visitante intentó acceder a un archivo en particular, pero el servidor lo redirigió a otro diferente. Hay dos razones principales por las que esto sucede:
  • El visitante ha solicitado incorrectamente un nombre de directorio sin un slash seguido. Por ejemplo, una solicitud de www.example.com/yourdirectory (el nombre incorrecto) devuelve www.example.com/yourdirectory/ (el nombre correcto).
  • Los anuncios de banner "Click-thru" también pueden hacer que se redirija una solicitud.
Distinct files requested
Cuántos archivos (páginas, imágenes, programas, etc.) se solicitaron a tu sitio.
Distinct hosts served
Básicamente, cuántas personas diferentes visitaron tu sitio. Sin embargo, el número puede ser un poco engañoso, ya que los clientes de algunos ISP grandes pasan por un servidor proxy. Como resultado, una gran cantidad de visitantes puede representarse como una pequeña cantidad de hosts distintos.
Data transferred
La cantidad total de datos transferidos.
Average data transferred per day
Cuál sería la cantidad promedio de datos transferidos con tráfico constante.

Reportes basados en tiempo

Como puedes ver en la siguiente captura de pantalla, los tipos de informes se pueden dividir en resúmenes semanales, diarios y horarios:

Aquí hay una descripción de las cuatro secciones:

Weekly Report
Un resumen de las últimas semanas por la fecha de inicio semanal. También enumera la semana más ocupada en la parte inferior.
Daily Report
Similar al Weekly Report, pero desglosa el sitio más ocupado por día.
Daily Summary
Resumen general de las estadísticas de un solo día.
Hourly Summary
Desglosa las solicitudes y las páginas por hora del día anterior.

Reportes basados en visitantes

Debido al tamaño de la página de estadísticas, esta sección está ligeramente dividida para revisar las distintas secciones individualmente:

Organization Report
Enumera las organizaciones, instituciones e ISP bajo los cuales se registraron las computadoras que accedieron a la información.
Host Reports
Muestra todas las computadoras individuales que accedieron a la información en su sitio web.
User Reports
Listado de todos los usuarios que han visitado tu sitio durante el período de tiempo especificado. Si tu sitio requiere que los usuarios se autentiquen (iniciando sesión), aparecerá su nombre de usuario. De lo contrario, se muestra la cookie del usuario.
Browser Report/Summary
Desglosa las solicitudes del sitio y las visitas exitosas a la página de diferentes navegadores.
Operating System Report
Una lista de los sistemas operativos utilizados por las personas que visitan tu sitio (por ejemplo, Windows, Linux, Apple, etc.). Estos están ordenados por número de solicitudes.
Status Code Report
Enumera cada código de estado HTTP que registra su sitio.
File Type Report
La cantidad de veces que se solicitó a tu sitio cada tipo de archivo (por ejemplo, imagen, página, etc.).
Directory Report
Enumera donde se descargaron los archivos de directorios.
Failure Report
Enumera los nombres de archivo que provocaron un error.
Request Report
Enumera todos los archivos/páginas que se descargaron de tu sitio web durante el período de tiempo especificado.
