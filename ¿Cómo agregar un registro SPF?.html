Generalidades

Este artículo trata sobre el registro SPF predeterminado de DreamHost y cómo puedes ajustarlo..

Si no estás usando los nameservers de DreamHost, debes configurar los registros SPF con el proveedor DNS de correo actual.

¿Necesito agregar el registro SPF manualmente?

Si estás usando Google Workpace, no puedes actualizar el registro SPF. Este campo permanece ineditable.

DreamHost agrega automáticamente los registros SPF de DreamHost a tu sitio. Solo necesitas agregar manualmente un registro SPF si también estás enviando un correo a través de un tercero. Por ejemplo, también puedes enviar correo a través de Google o Mailchimp. Si estás enviando correo a través de servicios como esos y también estás enviando correo desde DreamHost, deberás agregar un registro SPF personalizado que contenga ambos servidores de correo.

Revisa el siguiente artículo para más información:

  • Generalidades sobre el SPF

Agregar un registro SPF

Solo puedes tener un único registro SPF. Si necesitas enviar desde múltiples servidores, asegúrate de combinarlos todos dentro del mismo registro SPF.

El siguiente ejemplo supone que estás enviando un correo desde Google y DreamHost, donde se agregan el registro personalizado de Google Y el SPF de DreamHost.

v=spf1 mx include:netblocks.dreamhost.com include:relay.mailchannels.net include:_spf.google.com -all
  1. Navega a la página Administrar Sitios Web.
  2. Abre la página Administrar de tu sitio web.
    • Si estás en la vista de Cuadrícula, mueve el cursor sobre tu sitio web y da clic al botón Administrar.
    • Si estás en la vista de Lista, da clic al botón Administrar a la derecha de tu sitio web.
  3. Da clic en la pestaña DNS.
  4. Haz clic en el botón Agregar Registro.
  5. Mueve el cursor sobre la sección Registro TXT y da clic al enlace AÑADIR.
  6. Ingresa lo siguiente:
    • Host: Este campo puede ser cualquier cosa. Muy a menudo se queda en blanco.
    • Valor de TXT*: Ingresa el valor de registro SPF de este registro para apuntarlo.
  7. Haz clic en el botón Agregar Registro para guardar.

Espera alrededor de 6 horas para que el registro DNS se propague.

  • Ten en cuenta que el registro SPF trata los subdominios (como sub.example.com) por separado, por lo que es posible que quieras especificar registros SPF tanto para example.com como para cualquier host como tuhost.example.com. Consulta las preguntas frecuentes sobre SPF en subdominios para obtener más información.
  • Wildcard para registros TXT no son compatibles con DreamHost. Esto generalmente no se aconseja, como se indica en el siguiente artículo: RFC 4408 §3.1.5.

SPF Avanzado

Un registro SPF avanzado incluye más que solo los servidores de correo predeterminados: también tiene información sobre todos los demás servidores que envían correo para el dominio. El siguiente es un ejemplo de un registro SPF avanzado:

v=spf1 ip4:321.321.321.321 include:_spf.google.com include:shaw.ca mx ~all
Indicador Descripción
v=spf1
Identifica este registro DNS como un registro SPF versión 1.
ip4:321.321.321.321
Dirección IP de un servidor específico, por ejemplo tu servidor web para secuencias de comandos que envían correo directamente desde ese servidor. Puedes obtener la IP de tu servidor web desde la página Administrar Dominios. Mira el artículo de DNS para más detalles.
include:_spf.google.com
Incluye todos los registros SPF de Google, que en este ejemplo, es donde se aloja el servicio de correo del dominio.
include:shaw.ca
Incluye todos los registros SPF para Shaw Cable en Canadá, un proveedor de servicios de internet. En este ejemplo, el correo del dominio a veces se envía a través del servidor SMTP del proveedor de servicios de internet.
mx
Incluye todos los servidores MX que usa el dominio, enumerados en los registros MX DNS del dominio.
-all
Dice que todos los demás servidores no están autorizados y que solo se 'pasará' el correo enviado desde los servidores enumerados.

?all (signo de interrogación), -all (guión), o ~all (virgulilla)

El símbolo antes de "all" indica cuán estricto es el modo en el que el registro SPF que se está aplicando.

  • ?, el signo de interrogación hace que todo el registro esté inactivo, como si el dominio no tuviera ningún registro SPF.
  • -, guión, hace que el registro sea estricto, y cualquier correo de los servidores que no figuran en la lista se marcará como "fallido" y puede marcarse como spam o rechazado por completo.
  • ~, virgulilla, está entre las otras dos opciones en rigor. Cualquier correo de servidores que no figuran en la lista se marcará "softfail" (significa que la dirección IP puede o no estar autorizada para enviar desde el dominio). Si bien está destinado a las pruebas, se recomienda su uso para evitar problemas de entrega como se indica en este artículo.
