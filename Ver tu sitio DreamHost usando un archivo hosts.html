Generalidades

Un archivo hosts es un archivo de texto en el computador de tu hogar. Se utiliza para asignar un nombre de dominio a un servidor específico. Si lo editas, puedes cambiar el lugar donde tu computador busca un sitio web.

Por ejemplo, cuando buscas una URL en tu navegador, tu computador envía una consulta a Internet para encontrar dónde se encuentra tu sitio. tu sitio está ubicado donde apuntan los DNS. Si editas el archivo hosts, puedes cambiar dónde tu computador busca un sitio web.

De esta manera, puedes ver un sitio en tu servidor de DreamHost cuando el DNS apunta a otra compañía.

Este artículo explica cómo ver tu sitio antes de apuntar tus DNS a DreamHost editando el archivo hosts en tu computador de escritorio o portátil (no un dispositivo móvil).

Para probar en un dispositivo móvil, o para otra opción sobre cómo ver tu sitio en DreamHost antes de apuntar DNS, visita el siguiente artículo:

  • Previsualizar tu sitio con un subdominio dreamhosters.com antes de cambiar los DNS

Ubicar tus registros A

Cada sitio en DreamHost tiene dos registros A para la URL. Uno para tu dominio con y sin www. Por ejemplo:

  • 208.113.137.245 example.com
  • 208.113.137.245 www.example.com

Visita el artículo de DNS para obtener detalles sobre cómo ver estos registros. Toma nota de esto, ya que deberás agregar la IP a tu archivo hosts en el siguiente paso.

Usando Windows 10 & 11

En la mayoría de los sistemas Windows, puedes encontrar el archivo hosts en el siguiente directorio:

C:\Windows\System32\Drivers\etc

El Bloc de notas es un programa básico de edición de texto que ya está instalado en Windows. Para abrir el archivo hosts con el Bloc de notas, sigue los pasos a continuación:

  1. Abre el menú Inicio.
  2. En el cuadro de búsqueda, escribe Bloc de notas.
  3. Haz clic derecho en la opción Bloc de notas y elije Ejecutar como administrador.

    Es posible que recibas un mensaje emergente llamado Control de acceso de usuarioque pregunta: ¿Desea permitir que esta aplicación realice cambios en tu PC? Haz clic en Sí. Esto abre un archivo de Bloc de notas en blanco.

  4. En la parte superior izquierda de este archivo en blanco, haz clic en la opción de menú Archivo > Abrir....
  5. En la ventana del 'Explorador', navega hasta Este PC. Luego al directorio C:\Windows\System32\drivers\etc.
  6. Elige el archivo hosts y haz clic en Abrir. Es posible que debas cambiar los Todos los documentos (* .txt) en la esquina inferior derecha a Todos los archivos en la lista desplegable.
  7. En la parte inferior del archivo, agrega la dirección IP de tu sitio DreamHost seguido del dominio.
    208.113.137.245 example.com 
    208.113.137.245 www.example.com
  8. Guarda el archivo.
  9. Asegúrate de vaciar los DNS de tu computador para garantizar la actualización de la nueva configuración.
    • Limpiar los DNS
  10. Reinicia tu navegador e intenta acceder a tu dominio.

Ahora puedes ver tu sitio como aparece en tu servidor DreamHost. Una vez que apuntes tus DNS a DreamHost, puedes eliminar estas líneas.

Usar Apple Mac OS

  1. Inicia Terminal presionando el comando (⌘) + barra espaciadora que abre el cuadro de búsqueda.
  2. Escribe la palabra Terminal.
  3. Presiona la tecla enter o haz clic en el ícono Terminal debajo de TOP HIT.
  4. Visita el artículo Creación y edición de un archivo a través de SSH para obtener instrucciones sobre cómo editar este archivo. Este ejemplo usa el editor nano.
  5. Escribe lo siguiente en tu terminal:
    [server]$ sudo nano /private/etc/hosts
  6. Ingresa la contraseña del administrador. Nota: Las contraseñas escritas no se muestran en la línea de comandos.
  7. Una vez que el archivo de hosts esté abierto, usa las teclas de flecha para navegar hacia la parte inferior y realizar cualquier modificación.
  8. Cuando termines, guarda y cierra el archivo para volver a la terminal.
  9. Cierra la Terminal cuando hayas terminado.

Asegúrate de limpiar tus DNS para así asegurar de que se actualice las nuevas configuraciones. Una vez que apuntes tus DNS a DreamHost, puedes eliminar estas.

Usar Linux

En la mayoría de los sistemas basados en Linux:

  • El archivo de hosts se puede ubicar en el directorio /etc/.
  • El archivo de hosts generalmente se llama hosts.
  • La ruta completa es /etc/hosts.

Los pasos para editar y guardar el archivo son los mismos que las instrucciones de 'Mac OS' presentadas anteriormente.

Instrucciones para el movimiento de WordPress

Esta sección explica cómo ver tu sitio de WordPress (en DreamHost) si fue migrado por el Equipo de Servicios Profesionales. Esta sección solo se aplica a las migraciones que no son de DreamPress.

Primero, lee las secciones anteriores para ubicar y editar el archivo hosts de tu computador.

Normalmente, actualizar un archivo hosts solo requiere que agregues dos líneas (como se mencionó anteriormente). Con este proceso de migración, también debes agregar una tercera línea de texto para el servidor MySQL. Por ejemplo, si tu sitio era example.com, agregaría estas líneas al final de tu archivo hosts:

111.111.111.111 example.com
111.111.111.111 www.example.com.com
222.222.222.222 mysql.example.com

Reemplaza las direcciones IP de example anteriores con las direcciones IP reales de tu sitio.

Puedes usar las siguientes instrucciones para encontrar las direcciones IP en la página registros DNS:

  1. Navega a la página Administrar Sitios Web.
  2. Abre la página Configuraciones DNS:
    • Si estás en la Vista de Malla, da clic en el botón de los '3 puntos verticales' debajo de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
    • Si estás en la Vista de Lista, da clic en el botón de los '3 puntos verticales' a la derecha de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
      Aparecen los registros DNS del dominio.

Si necesitas más ayuda para ubicar las direcciones IP de tu sitio, revisa el artículo DNS para obtener instrucciones más detalladas.

Confirmar que el archivo 'hosts' funciona

Puedes confirmar que está cargando el sitio migrado desde los servidores de DreamHost si puedes cargar la siguiente página (reemplazando example.com con tu dominio):

  • http://example.com/quickstart.html

Ver tu sitio público a través de un proxy

Una vez que tu archivo hosts esté funcionando, tu computador te mostrará cómo se ve tu sitio en DreamHost. Si también deseas ver tu sitio actual (público), debes usar un sitio web proxy.

Un ejemplo de un sitio proxy es kproxy.com. Visita este enlace en tu navegador, ingresa tu nombre de dominio en el cuadro, luego haz clic en Enter. Utiliza el DNS público de tu sitio (no el archivo hosts local) para ver el sitio en tu empresa de alojamiento actual.

Comparar los sitios

Ver tu sitio usando un archivo hosts y un sitio proxy te permite ver ambas versiones de tu sitio para una comparación lado a lado. Puedes tener una pestaña del navegador abierta en el sitio migrado (visto usando tu archivo hosts) y otra pestaña del navegador abierta en el sitio proxy (visto usando DNS) para ver tu sitio público actual.

Realizar más cambios en tu sitio

Durante tu inspección del sitio, puedes sentir la necesidad de cambiar la configuración de algo que estaba fuera del alcance del servicio de migración. Abstente de cambiar cualquier cosa que pueda alterar el aspecto o el comportamiento del sitio, ya que el Equipo de Servicios Profesionales debe poder comparar el sitio migrado (en DreamHost) con el sitio en vivo (en tu alojamiento actual).

Si debes actualizar algo, asegúrate de documentar cualquier cambio para que el Equipo de Servicios Profesionales pueda separar lo que pudo haber sido afectado por la migración de las modificaciones posteriores por tu parte.
