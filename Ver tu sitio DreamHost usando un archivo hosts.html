Generalidades

Este artículo explica cómo ver tu sitio web antes de apuntar tu DNS a DreamHost editando el archivo hosts en tu computador computador.

Para probar en un dispositivo móvil, o para otra opción sobre cómo ver tu sitio en DreamHost antes de apuntar DNS, visita el siguiente artículo:

  • Previsualizar tu sitio con un subdominio dreamhosters.com antes de cambiar los DNS

¿Qué es un archivo hosts?

Un archivo hosts es un archivo de texto utilizado por tu computador que cambia dónde este busca un sitio web específico. Por ejemplo:

  • cuando visitas un sitio web, tu computador sigue los registros DNS públicos de Internet.
  • cuando editas el archivo hosts, controlas los registros DNS que tu computador utiliza.

Esto te permite ver un sitio en tu servidor de DreamHost cuando los registros DNS públicos están apuntando a otra compañía.

Registros A

El archivo hosts utiliza registros A (direcciones IP) para localizar tu sitio en DreamHost. Cada sitio en DreamHost tiene dos registros A para la URL: uno para tu dominio con www y otro sin www. Por ejemplo:

  • 111.222.333.444 example.com
  • 111.222.333.444 www.example.com

Revisa el siguiente artículo para localizar estos dos registros para tu sitio web:

  • ¿A qué registros DNS debo apuntar mi sitio?

Actualizar tu archivo hosts

Las siguientes secciones describen cómo localizar y actualizar tu archivo hosts en diferentes sistemas operativos.

Windows 10

En la mayoría de los sistemas Windows, puedes encontrar el archivo hosts en el siguiente directorio:

C:\Windows\System32\Drivers\etc

El Bloc de notas es un programa básico de edición de texto que ya está instalado en Windows. Para abrir el archivo hosts con el Bloc de notas, sigue los pasos a continuación:

  1. Abre el menú Inicio.
  2. En el cuadro de búsqueda, escribe Bloc de notas.
  3. Haz clic derecho en la opción Bloc de notas y elije Ejecutar como administrador.

    Es posible que recibas un mensaje emergente llamado Control de acceso de usuarioque pregunta: ¿Desea permitir que esta aplicación realice cambios en tu PC? Haz clic en Sí. Esto abre un archivo de Bloc de notas en blanco.

  4. Haz clic en la opción de menú Archivo > Abrir... en la parte superior izquierda de este archivo en blanco.
  5. Navega hasta Este PC en la ventana del 'Explorador'. Luego al directorio C:\Windows\System32\drivers\etc.
  6. Elige el archivo hosts y haz clic en Abrir. Es posible que debas cambiar los Todos los documentos (* .txt) en la esquina inferior derecha a Todos los archivos en la lista desplegable.
  7. Agrega las IP de tu sitio DreamHost seguidas del dominio en la parte inferior del archivo. 
    xxx.xxx.xxx.xxx example.com 
    xxx.xxx.xxx.xxx www.example.com
  8. Guarda el archivo.
  9. Limpia el DNS de tu computador para asegurar de que las nuevas actualizaciones se lleven a cabo.
  10. Reinicia tu navegador y visita tu dominio.

Ahora puedes ver tu sitio como aparece en tu servidor DreamHost. Una vez que apuntes tus DNS a DreamHost, puedes eliminar estas líneas.

Mac OS

  1. Inicia Terminal presionando el comando (⌘) + barra espaciadora que abre el cuadro de búsqueda.
  2. Escribe la palabra Terminal.
  3. Presiona la tecla enter o haz clic en el ícono Terminal debajo de TOP HIT.
  4. Visita el artículo Creación y edición de un archivo a través de SSH para obtener instrucciones sobre cómo editar este archivo. Este ejemplo usa el editor nano.
  5. Escribe lo siguiente en tu terminal:
    [server]$ sudo nano /private/etc/hosts
  6. Ingresa la contraseña del administrador.
  7. Usa las teclas de flecha para navegar hacia la parte inferior Una vez que el archivo de hosts esté abierto para hacer cualquier modificación.
  8. guarda y cierra el archivo cuando termines para volver a la terminal.
  9. Cierra la Terminal cuando hayas terminado.
  10. Limpia tu DNS para asegurarte de que las nuevas configuraciones se actualicen.

Ahora puedes ver tu sitio tal como aparece en tu servidor de DreamHost. Una vez que apuntes tus DNS a DreamHost, puedes eliminar estas líneas..

Linux

En la mayoría de los sistemas basados en Linux:

  • El archivo de hosts se puede ubicar en el directorio /etc/.
  • El archivo de hosts generalmente se llama hosts.
  • La ruta completa es /etc/hosts.

Los pasos para editar y guardar el archivo son los mismos que las instrucciones de 'Mac OS' presentadas anteriormente.

Migraciones de los Servicios Profesionales de WordPress

Esta sección explica cómo ver tu sitio de WordPress (en DreamHost) si fue migrado por el Equipo de Servicios Profesionales. Esta sección solo se aplica a las migraciones que no son de DreamPress.

Localiza y edita el archivo hosts de tu computador como se describió anteriormente. Con este proceso de migración, debes agregar una tercera línea de texto para el servidor MySQL. Por ejemplo, si tu sitio fuera example.com, agregarías estas líneas al final de tu archivo hosts:

111.111.111.111 example.com
111.111.111.111 www.example.com.com
222.222.222.222 mysql.example.com

Realizando más cambios en tu sitio

Por favor, abstente de realizar cambios que puedan alterar el aspecto o comportamiento del sitio, ya que el equipo de Servicios Profesionales debe poder comparar el sitio migrado (en DreamHost) con el sitio en vivo (en tu alojamiento actual).

Ver tu sitio

Esta sección explica cómo comparar tu sitio de DreamHost con tu sitio web en vivo.

Confirmar que el archivo 'hosts' funciona

Puedes confirmar que está cargando el sitio migrado desde los servidores de DreamHost si puedes cargar la siguiente página (reemplazando example.com con tu dominio):

  • http://example.com/quickstart.html

Ver tu sitio público a través de un proxy

Una vez que tu archivo hosts funcione, tu computador te mostrará cómo se ve tu sitio en DreamHost. Si también quieres ver tu sitio actual (público), debes usar un sitio web proxy.

Un ejemplo de un sitio proxy es kproxy.com. El sitio proxy utilizará el DNS público de tu sitio (no tu archivo local hosts) para ver el sitio en tu compañía de alojamiento actual. Ver tu sitio usando un archivo hosts y un sitio proxy te permite ver ambas versiones de tu sitio para una comparación lado a lado.
