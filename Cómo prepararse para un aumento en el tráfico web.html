Generalidades

A continuación se presentan sugerencias para optimizar un sitio web en un plan de Alojamiento Compartido si esperas un aumento significativo en el tráfico web.

También podrías enviar un mensaje al soporte de DreamHost cuando esperes un aumento en el tráfico web para que puedan monitorear tu sitio web durante ese período.

Regulación/Deshabilitación de tu sitio

Si el aumento de tráfico causa problemas en el servidor, puede ser necesario que el soporte:

  • "regule" tu sitio (restringir el número de visitantes simultáneos)
  • o posiblemente deshabilitarlo.

Esto se hace por equidad hacia los otros clientes en tu servidor para asegurar que no se vean negativamente afectados por tu sitio web.

Ten en cuenta que estás en un plan de Alojamiento Compartido, donde los recursos se comparten con otros clientes. Por lo tanto, DreamHost no puede permitir que un solo sitio web ponga en peligro el funcionamiento del servidor para los demás.

Aunque este artículo ofrece sugerencias de optimización para ayudar a que tu sitio se mantenga en funcionamiento, en última instancia es tu responsabilidad asegurar que tu sitio no afecte negativamente a otros clientes.

¿Qué sucede si mi sitio es regulado?

El soporte de DreamHost contactará al propietario de la cuenta si es necesario regular el tráfico de tu sitio.

Si hay un problema con tu sitio que ya has resuelto, asegúrate de notificar al soporte con los detalles de lo que hiciste para solucionar el problema. El soporte podría entonces eliminar la regulación existente.

Optimizar tu sitio

Actualizar tu software

Asegúrate de que tus aplicaciones estén actualizadas. Por ejemplo, si tienes un sitio de WordPress, asegúrate de que lo siguiente esté actualizado:

  • Versión de WordPress
  • Versiones de los plugins
  • Versión del tema

Si estás utilizando una instalación de WordPress de DreamHost, tu sitio se actualiza automáticamente.

Habilitar el caché

Si tu sitio proporciona contenido dinámico o utiliza una base de datos, es posible que desees habilitar alguna forma de "caché". Deberás revisar la documentación de ayuda de tu aplicación en particular para obtener más detalles. Aquí hay algunos artículos generales para comenzar:

  • Opciones de caché para WordPress
  • Crear una cuenta de Cloudflare gratuita
 

Optimizar tu base de datos

Revisa el artículo Optimizar una base de datos para obtener consejos sobre cómo optimizar la base de datos de tu sitio web.

 

Deshabilitar software en desuso

Desactiva los plugins innecesarios o que consuman muchos recursos que tu sitio pueda estar utilizando. Para sitios de WordPress, elimina el software antiguo y desactiva cualquier tema, plugin, o instalación adicional de WordPress que no estés usando.

Optimizar los archivos de multimedia

Intenta optimizar tu multimedia reemplazando temporalmente gráficos, archivos, películas, etc., de gran tamaño con versiones de menor resolución (más pequeñas).

Convertir contenido dinámico a contenido estático

Aunque puede ser difícil, convertir páginas dinámicas en páginas estáticas podría ayudar significativamente. Algunas herramientas de caché, como wp-super-cache, incluyen la capacidad de hacer esto utilizando el plugin.

Mejorar tu plan de alojamiento

VPS

Si no es posible optimizar más tu sitio, puede ser necesario actualizar tu plan de alojamiento Shared a un VPS. Este es un plan de alojamiento separado que te permite asignar una mayor cantidad de RAM para ejecutar tu sitio, lo que podría ayudar a evitar que tu sitio se vea abrumado por el tráfico entrante. Revisa los siguientes artículos para obtener más información:

  • VPS
  • Solución de problemas de VPS
  • Optimización de VPS

DreamPress

Si tu sitio está ejecutando WordPress, considera actualizar a un plan DreamPress, que proporciona mayor robustez del servidor, caché y memoria a tu sitio de WordPress. Revisa los siguientes artículos para obtener más información sobre DreamPress:

  • Generalidades de DreamPress
  • Mejorar un sitio de WordPress a DreamPress

Asegurar el correo

Prevenir spam

Si tu sitio permite a los visitantes dejar comentarios, asegúrate de estar usando algún tipo de plugin para prevenir comentarios de spam. Si estás usando WordPress, Akismet es una buena opción.

Usar formularios de contacto seguros

Asegúrate de que cualquier formulario de contacto esté configurado para usar SMTP. La dirección FROM también debe estar configurada correctamente. Revisa los siguientes artículos para obtener más detalles:

  • Cómo hacer spoofing o suplantar tu CMS o aplicación web
  • Solución de problemas de los formularios de contacto de WordPress

Habilitar estadísticas

Habilitar las estadísticas del sitio te permite ver y monitorear los cambios en el tráfico de tu sitio.
