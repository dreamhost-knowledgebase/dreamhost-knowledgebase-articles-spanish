Generalidades

Para cambiar el nombre de tu base de datos, debes crear una nueva base de datos e importarla. Esto implica los siguientes pasos:

  1. Generar una copia de seguridad de tu base de datos antigua.
  2. Crear una nueva base de datos con el nuevo nombre.
  3. Importar tu antiguo archivo de base de datos a la nueva base de datos.
  4. Ajustar el archivo de configuración de tu sitio web.

Este artículo te guía a través de estos pasos.

Paso 1 — Generar una copia de seguridad de tu base de datos antigua

Primero, debes hacer una copia de seguridad de tu base de datos antigua. Hay algunas maneras en que puedes hacer esto:

  • Generar una copia de seguridad de toda tu cuenta en el panel.
  • Exportar tu base de datos usando phpMyAdmin.
  • Exportar tu base de datos usando SSH.

Consulta el siguiente artículo para obtener instrucciones sobre cómo proceder con cualquiera de estas opciones:

  • Generalidades de copias de seguridad de bases de datos

Esto hará una copia de seguridad de tu base de datos en un archivo que termina con .sql.

Paso 2 — Crear una nueva base de datos en el panel

A continuación, deberás crear una nueva base de datos con el nuevo nombre. Consulta el siguiente artículo para obtener instrucciones sobre cómo crear esta base de datos en tu panel:

  • Crear una base de datos MySQL

Paso 3 — Importar tu base de datos

En este punto, has realizado una copia de seguridad de tu base de datos anterior en un archivo que termina en .sql y se ha creado una nueva base de datos. Ahora puedes importar tu archivo .sql a la nueva base de datos. Esto se puede hacer usando phpMyAdmin o SSH. Elige una de las siguientes opciones.

Importar con phpMyAdmin

Si tu archivo de copia de seguridad SQL tiene un tamaño inferior a 64 MB, se puede importar a tu nueva base de datos a través de phpMyAdmin. Ve el siguiente artículo para importar una base de datos respaldada a través de phpMyAdmin:

  • Cómo importar una base de datos a través de phpMyAdmin

Importar usando SSH

Si tiene 64MB o más (o si encuentras un error tiempo de espera al importar), la única otra opción es importarlo desde la línea de comandos a través de SSH.

  • Importar una base de datos a través de SSH

Paso 4 — Ajustar el archivo de configuración de tu sitio web

El último paso es actualizar el archivo de configuración de tu sitio web. Cada sitio web que se conecta a una base de datos tiene un archivo de configuración. Este archivo de configuración almacena las siguientes credenciales de base de datos que le permiten conectarse.

  • Nombre de usuario de base de datos.
  • Contraseña de usuario de base de datos.
  • Hostname de base de datos.
  • Nombre de base de datos.

Si alguno de ellos es incorrecto, tu sitio no puede conectarse a la base de datos. Los archivos de configuración estándar para software popular se muestran a continuación:

Software         Nombre archivo Config       Ubicación archivo Config                

Concrete5        database.php                        application/config/                             

Joomla            configuration.php                 En el directorio principal del sitio      

MediaWiki       Localsettings.php                  En el directorio principal del sitio    

phpBB             config.php                             En el directorio principal del sitio      

WordPress      wp-config.php                       En el directorio principal del sitio       

Zen Cart            configure.php                        includes/                                              

Creaste todas estas credenciales al crear la nueva base de datos en el paso #2 anterior. Después de actualizar tu archivo de configuración con estas nuevas credenciales, tu sitio se conectará a la nueva base de datos.
