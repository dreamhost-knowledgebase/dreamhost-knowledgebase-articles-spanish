Generalidades

Un dominio "redireccionado" actúa como un puntero a otro dominio. Al acceder a una página web en el dominio redirigido, la URL del navegador se cambia al dominio al que se está redirigiendo. Al configurar esto a través del panel de DreamHost, se crea automáticamente una redirección 301.

Este artículo explica cómo redirigir un dominio o subdominio con DreamHost.

Los ejemplos en este artículo utilizan el protocolo HTTPS. Para que esto funcione, el dominio al que está redirigiendo debe tener un certificado SSL configurado. Si no, solo use HTTP en su lugar.

¿Cómo funciona la redirección?

Cualquier página que visite en su sitio actual (la que se redirige) se reenvía a la página en el sitio redirigido. Por ejemplo, digamos que está redirigiendo example.com a dreamhost.com. Si visita example.com/contact.html, la URL cambia a dreamhost.com/contact.html.

¿Funciona la redirección para las versiones www y no www del sitio?

Sí. Si configura una redirección en example.com, también funcionará para www.example.com.

¿La redirección cambia los registros MX?

No. La redirección solo cambia donde está alojado el sitio web. Tus registros MX (correo) no se cambiarán de ninguna manera.

¿El redireccionamiento funciona con https (certificado SSL)?

A continuación se describen algunas formas diferentes de configurar esto.

Opción 1 — Redirigir a la versión segura de un sitio diferente

Solo puedes redirigir a otro sitio que ya esté configurado con un certificado SSL. Por ejemplo:

http://example.com a https://dreamhostexample.com

No puedes utilizar la URL https del dominio que se redirige. Esto se debe a que el dominio desde el que estás redirigiendo no está alojado, por lo que no tiene un certificado SSL configurado.

Solo puede redirigir desde la URL http del primer dominio a la URL https del segundo dominio.

Opciónn 2 — Redirigir a la versión segura del mismo sitio

No es posible utilizar la opción Redirigir del panel (que se describe a continuación) para redirigir a HTTPS, ya que esto elimina el alojamiento del dominio. Por ejemplo:

http://example.com a https://example.com

Esto tampoco es necesario, ya que después de haber agregado un certificado SSL a tu dominio, DreamHost redirige automáticamente la URL que usan los visitantes para ver tu sitio de HTTP a HTTPS. La 'S' asegura que tu conexión esté encriptada. Por ejemplo:

  • https://example.com

No hay nada de tu parte que deba hacer para obligar a tu sitio a utilizar la URL segura. Sin embargo, en algunos casos especiales, puede ser necesario que crees tu propio archivo de configuración personalizado para forzar el redireccionamiento de HTTP a HTTPS.

Agregar código personalizado solo es necesario si tu sitio web requiere un código específico para forzar el redireccionamiento. Si es así, primero deberás desactivar la redirección automática en tu panel.

  • Deshabilitar la redirección automatica a HTTPS

Una vez deshabilitada, puedes proceder a añadir tu archivo personalizado de redirección.

Redireccionando un nuevo dominio o subdominio

  1. Navegar a la página Manage Domains.
    Se abrirá la página Manage Domains.
  2. Haga clic en el botón Agregar alojamiento a un dominio / subdominio.
  3. Desplácese hacia abajo hasta la sección 'Redirigir'.
  4. Ingrese la información en la siguiente sección:
    • Dominio para redirigir – ingrese el nuevo dominio o subdominio que desea redireccionar a otro lugar.
    • Redirigir a URL – ingresa la URL a dónde deseas redirigir este nuevo dominio o subdominio.
  5. Haga clic en el botón Redirigir este dominio para completar la configuración.

Cambiar un dominio o subdominio existente para redirigir

  1. Navegar a la página Manage Domains.
    Se abrirá la página Manage Domains.
  2. Haga clic en el botón Edit situado a la derecha del dominio / subdominio en la columna 'Web Hosting'.
  3. Desplácese hacia abajo hasta la sección 'Redirect'.
    En el siguiente ejemplo, el campo ‘Domain to Redirect’ ya se rellena con el dominio que está cambiando a una redirección:
  4. En el campo 'Redirect to URL' ingrese la URL donde desea que se redirija este dominio.
  5. Haga clic en el botón Redirect this domain para completar la configuración.

Creando un redireccionamiento a un subdirectorio

Si necesita crear una redirección a un subdirectorio de otro sitio, asegúrese de agregar una barra diagonal hacia adelante a la URL redirigida. Si no lo hace, la URL no se redireccionará correctamente.

Por ejemplo, si su sitio es 'example.com' y desea redirigir a un subdirectorio de otro sitio llamado 'dreamhostexample.com/blog', asegúrese de que la redirección que cree agregue la barra inclinada hacia delante como esta:

  • dreamhostexample.com/blog/

Si no agrega la barra diagonal, los archivos a los que se acceda dentro de ese subdirectorio no se cargarán, ya que la URL será incorrecta.

No puede redirigir a un subdirectorio del mismo sitio web.

Apagar una redirección

Para desactivar un Redireccionamiento, debe cambiar la configuración de alojamiento a cualquier otra opción, como 'Alojamiento completo' (Fully Hosted) o 'Solo DNS' (DNS Only). Vea los siguientes artículos para obtener instrucciones:

¿Cuánto tiempo hasta que la redirección esté funcionando?

Redirigir su sitio es lo mismo que cualquier otro cambio de DNS. Esto significa que el DNS debe propagarse primero antes de que pueda ver los cambios en línea. En general, esto no debería tomar más de 6 horas. Puede realizar un seguimiento de cuándo se actualizan los nuevos registros DNS en diferentes ubicaciones en este sitio.
