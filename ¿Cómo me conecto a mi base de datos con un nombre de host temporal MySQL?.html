Generalidades

Puedes acceder a tu base de datos mientras el DNS de tu sitio principal no está dirigido a DreamHost. Esto requiere tres pasos:

  1. Crea un nuevo subdominio dreamhosters.com (por ejemplo: example.dreamhosters.com).
  2. Usando ese subdominio, crea un nombre de host MySQL.
  3. Usa este nombre de host MySQL para acceder a tu base de datos a través de phpMyAdmin.

Paso 1 — Agregar un subdominio dreamhosters.com

Agrega un subdominio dreamhosters.com (por ejemplo: example.dreamhosters.com) que refleja tu dominio principal en la página Administrar Dominios.

Para obtener instrucciones más detalladas sobre este proceso, consulta el siguiente artículo:

  • Ver tu sitio utilizando un subdominio dreamhosters.com antes de cambiar los DNS

Paso 2 — Agregar un nombre de host MySQL

Para agregar un nombre de host para tu nuevo subdominio:

  1. Navega a la página MySQL Databases.
    Se abre la página MySQL Databases:
  2. Da click en el botón Add New Hostname (agregar nuevo nombre de host).
    Se abre la página "Agregar un nombre de host de MySQL":

    En el ejemplo anterior:

    • El dominio primario es "example.dreamhosters.com".
    • El nombre de host de MySQL es "mysql.example.dreamhosters.com".
  3. Ingresa el prefijo de nombre de host de tu elección (comúnmente "mysql"), y luego selecciona el nuevo subdominio usando el menú desplegable.
  4. Da click en el botón Create this MySQL hostname now! para terminar.

Paso 3 — Acceder a tu base de datos con phpMyAdmin

Una vez que los cambios de DNS se propagan (lo que puede demorar entre 4 y 24 horas), puedes acceder a la utilidad de administración MySQL phpMyAdmin utilizando el nombre de host mysql que acaba de crear. En el ejemplo anterior, sería mysql.example.dreamhosters.com.

Si intentas acceder a phpMyAdmin antes de que se actualicen los registros DNS del nombre de host MySQL, la interfaz phpMyAdmin de DreamHost no podrá reconocer tu información de inicio de sesión.

Para obtener más información sobre phpMyAdmin y cómo iniciar sesión, consulta el siguiente artículo:

  • Generalidades sobre phpMyAdmin
