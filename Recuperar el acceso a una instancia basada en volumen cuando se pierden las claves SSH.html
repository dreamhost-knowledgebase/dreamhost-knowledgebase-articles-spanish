Introducción

Las instancias de DreamCompute se crean con una llave SSH específica que se utiliza para obtener acceso inicial. Esta guía cubre algunos pasos básicos para intentar recuperar los datos y obtener acceso nuevamente si lo deseas.

Cosas para tener en cuenta

  • La dirección IP cambiará

    Para volver a iniciar sesión en una instancia, es necesario volver a crearla. La nueva instancia recibe nuevas direcciones IPv4 e IPv6.

  • Conoce el tipo de tu instancia

    En general, se puede averiguar si una instancia utilizó un disco de arranque efímero o de volumen consultando la página 'Instances' en el panel de DreamCompute. Allí puedes ver si el nombre de una imagen personalizada o un sistema operativo aparece en la columna Image Name. Si aparece uno, la instancia es efímera. Si no lo es, usó un disco de arranque por volumen.

  • Instancia iniciada por volumen de respaldo

    Cuando se crea una instancia, hay una opción titulada "Delete Volume on Instance Delete". Desafortunadamente, es difícil verificar en qué se configuró esa opción después de la creación. Si se seleccionó "Yes" los datos del volumen se eliminarán de forma permanente cuando se elimine la instancia.

    Para hacer una copia de seguridad, ve a la página 'Compute > Instances'. A la derecha de la instancia, haz clic en el botón Create Snapshot. Este snapshot usa una cuota de almacenamiento de volumen, por lo que es posible que debas agregar almacenamiento adicional en el panel DreamHost para obtener más espacio. Una vez que se completa todo este proceso, se puede eliminar el snapshot y reducir la cuota.
 

Recuperar el acceso a la instancia basada en volumen

  1. Navega a la página Instances del panel DreamCompute.
  2. Si no sabes si "Delete Volume on Instance Delete" se estableció en "Yes" para tu instancia cuando se creó, asegúrate de que se haya creado una instantánea. Un  snapshot utiliza tanto espacio en disco como el volumen, por lo que si el volumen es de 80 GB, confirma que hay otros 80 GB libres. Puedes asegurarte de que haya suficiente volumen de almacenamiento en la página Generalidades.
  3. Toma nota del tamaño de la instancia, el nombre de la instancia, el UUID y el nombre del volumen, y cualquier otra característica que deba conservarse en la nueva instancia.
  4. Elimina la instancia (esto dejará un volumen disponible).
  5. Haz clic en el botón Launch Instance para crear una nueva instancia. En la pestaña Source, establece Select Boot Source en Volume y haz clic en el símbolo de la flecha hacia arriba junto al volumen recién disponible de la instancia anterior.
  6. En la pestaña Key Pair, seleciona la Llave par para ser inyectada por el usuario predeterminado. Si no hay claves válidas disponibles, puedes usar los botones Create Key Pair o Import Key Pair para asociar un nuevo par de claves con tu proyecto (y esta instancia).
  7. Cuando hayas terminado de personalizar la instancia, haz clic en el botón Launch en la parte inferior derecha.
  8. Una vez que la nueva instancia se ha reiniciado, intenta iniciar sesión con el nombre de usuario predeterminado y la nueva dirección IP.
  9. Si todo parece correcto y se ha restaurado el acceso, debes eliminar el snapshot de volumen realizado en el paso #2 para recuperar la cuota de almacenamiento de volumen.
