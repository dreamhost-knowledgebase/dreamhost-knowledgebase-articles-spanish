Generalidades

Traceroute es una herramienta de diagnóstico de red que se utiliza para verificar la red entre dos ubicaciones. Por ejemplo, para determinar la causa de la lentitud o los tiempos de espera, un Traceroute debe ejecutarse desde el punto de inicio de una conexión de red a tu servidor DreamHost.

¿Cuándo debo usar Traceroute?

A continuación, se incluyen algunos ejemplos de cuándo se puede necesitar un traceroute para solucionar un problema que estés experimentando.

Lentitud con la página web

Si notas que tu sitio está cargando lentamente, ejecuta un traceroute desde tu computadora a tu sitio web para ver si hay problemas de red entre tu ubicación y el servidor de DreamHost.

Tiempo de espera del cliente de correo electrónico

Si tienes problemas relacionados con tu conexión de correo, ejecuta un traceroute para determinar la calidad de la conexión al servidor de correo.

  1. Visita la página Configuración del cliente de correo electrónico para encontrar el nombre de tu servidor de correo. Puede ser imap.dreamhost.com o pop.dreamhost.com, según tu preferencia de conexión.
  2. Ejecuta un traceroute.
    El siguiente ejemplo muestra un traceroute ejecutándose en Windows:
    C:\Users\DreamHost> tracert imap.dreamhost.com

Cómo ejecutar un traceroute

Usando Windows

  1. Abre el Símbolo del Sistema.
  2. Escribe la palabra 'tracert' seguido de un espacio y luego el nombre del sitio/servidor al que deseas conectarte. Por ejemplo, si deseas conectarte a example.com, deberás ejecutar esto:
    C:\Users\DreamHost> tracert example.com
  3. Presiona la tecla "Enter" para ejecutar el comando.
    • También puedes probar tu servidor de correo DreamHost ejecutando un traceroute al subdominio de correo. Por ejemplo:
    C:\Users\DreamHost> tracert imap.dreamhost.com

Usando linux

Para la mayoría de las distribuciones de Linux, primero deberás instalar el paquete traceroute.

Para instalar:

  1. Abre tu terminal.
  2. Ejecuta lo siguiente para realizar la instalación en Ubuntu:
    [server]$ sudo apt-get install traceroute
  3. Una vez instalado, puedes ejecutar el comando de la siguiente manera:
    [server]$ traceroute example.com
    • Algunas variantes de Linux requieren que también especifiques el protocolo después de -I. Por ejemplo:
    [server]$ traceroute -I ICMP example.com

Mac OS X

Para ejecutar traceroute:

  1. Visita el artículo de SSH para Mac OS X para obtener instrucciones sobre cómo abrir tu terminal.
  2. Ejecuta el siguiente comando:
    [server]$ traceroute example.com

Otra opción es usar el programa gráfico Network Utility (puedes usar Finder y buscar aplicaciones para ubicar y abrir el programa):

  1. Haz clic en el botón Traceroute en la barra superior.
  2. Escribe el nombre del servidor.
  3. Haz clic en el botón Trace.

Copiar y pegar los resultados

Si deseas compartir tus resultados con un representante de soporte, deberás pegarlos en tu ticket.

Windows

  1. En el símbolo del sistema de Windows, haz clic con el botón derecho y elije 'Marcar'.
  2. Arrastra el cursor sobre el área completa que deseas copiar.
  3. Haz clic derecho de nuevo.

Luego puedes pegar en cualquier editor de texto o en tu ticket para soporte.

Otra opción es tener resultados de traceroute escritos directamente en un archivo de texto.

Para crear un archivo de texto de sus resultados de traceroute, ejecuta el siguiente comando:

C:\Users\DreamHost> tracert example.com > C:\tracert.txt

Esto crea un archivo llamado tracert.txt dentro de tu carpeta C: \. Puedes seleccionar cualquier carpeta en la que desees guardar el archivo.

Mac & Linux

Puedes ejecutar el siguiente comando para escribir los resultados de traceroute en un archivo, y puedes especificar la ubicación que desees.

En el siguiente ejemplo, los resultados se escriben en un archivo llamado trace.txt debajo del usuario llamado 'username':

[server]$ traceroute -I example.com > /home/username/trace.txt
