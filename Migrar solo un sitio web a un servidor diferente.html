Generalidades

Este artículo explica cómo mover un dominio entre los servidores web de DreamHost. Estas instrucciones solo mueven los archivos del sitio web, los archivos de la base de datos permanecerán en el mismo servidor MySQL.

Si solo deseas cambiar el nombre de usuario asignado a un dominio pero no migrarlo a un servidor diferente, mira el siguiente artículo:

  • Mover un dominio a un usuario nuevo

Mover un solo dominio a un servidor diferente

Edita el dominio en tu panel para seleccionar un nuevo nombre de usuario en un servidor diferente:

  1. Navega a la página Dominios Alojados.
  2. Haz clic en el botón Editar a la derecha del dominio debajo de la columna Alojamiento Web.
  3. Desplázate hacia abajo hasta la sección Usuarios, archivos y rutas.

    Las capturas de pantalla a continuación muestran cómo se vería al pasar a un servidor compartido o privado.


    Migrar a un servidor compartido

    Migrar a un VPS

  4. A la derecha de ‘Ejecutar este dominio bajo el usuario', ingresa lo siguiente:
    • Haz clic en el menú desplegable y selecciona Crear un nuevo usuario (recomendado).
    • Ingresa el nuevo nombre de usuario.
    • Asegúrate que el servidor correcto está seleccionado.
    • Deja la casilla Mover archivos al nuevo usuario? seleccionada.
  5. Baja y da clic al botón Cambiar ajustes para guardar los cambios.
    Se abrirá una página de confirmación que indica que los cambios se guardaron correctamente.
    • En 15 minutos, el nuevo usuario está listo para usar.
    • Según la cantidad de datos que se muevan, puede llevar varias horas completar la migración al nuevo usuario.
    • Para verificar el estado de la migración de un solo dominio, consulta la página Dominios Alojados. A la derecha de tu dominio, puedes ver un icono de reloj y un texto que indica que los datos se están moviendo al nuevo usuario. Una vez completado, el icono y el texto ya no aparecerán.

Propagación de DNS después de migrar

No hay tiempo de inactividad durante el movimiento ya que los datos se copian mientras el servidor anterior continúa sirviendo al sitio web. Sin embargo, no debes trabajar en tu sitio durante este período de 4 a 6 horas. Esto se debe a que es posible que tus cambios se realicen en el sitio anterior, no en el nuevo. Debes esperar hasta que el DNS se haya actualizado completamente para continuar trabajando en tu sitio web.

Cuando migras tu dominio a un servidor diferente, la IP de tu sitio cambia. Si tu dominio usa los nameservers de DreamHost, la IP del dominio se actualiza automáticamente.

Si tu dominio no está usando los nameservers de DreamHost, deberás actualizar tus registros DNS manualmente en la compañía donde están alojados tus nameservers.

Puedes ver todos los registros DNS configurados para cualquier sitio que aloje en DreamHost en tu panel. Ten en cuenta que después de cualquier cambio de DNS, los registros deben propagarse en línea.
