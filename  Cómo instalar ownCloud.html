Instalarlo manualmente

  1. Visita la página de descarga de ownCloud.
  2. Busca la sección para descargar un archivo .Tar o .Zip.

    Este ejemplo usa el archivo .Tar.

  3. La siguiente página detalla todos los pasos para instalar. Haz clic derecho en el botón Download Tar y elige Copy link Address.
  4. Inicia sesión en tu servidor a través de SSH.
  5. Navega hasta el directorio en el que deseas instalar ownCloud
  6. Corre un comando wget para descargar el archivo .tar.bz2. Escribe wget, seguido por el enlace que copiaste anteriormente. 
    [server]$ wget https://download.owncloud.org/community/owncloud-10.0.8.tar.bz2
  7. Bajo el botón Download Tar, haz clic derecho en 'sha256' y elige Copy link Address. Luego corre wget para descargar este archivo. 
    [server]$ wget https://download.owncloud.org/community/owncloud-10.0.8.tar.bz2.sha256
  8. Luego, verifica la firma sha256 con el archivo .tar.bz2 para asegurarte de que coincida. 
    [server]$ sha256sum -c owncloud-10.0.8.tar.bz2.sha256 < owncloud-10.0.8.tar.bz2
owncloud-10.0.8.tar.bz2: OK
  9. Si dice 'OK', descomprime el archivo: 
    [server]$ tar xvjf owncloud-10.0.8.tar.bz2
    Esto crea un nuevo subdirectorio llamado /owncloud en el mismo directorio en el que corriste el comando.
  10. En tu navegador, navega hasta la carpeta /owncloud. Lo más probable es que la URL sea algo como example.com/owncloud.
  11. Ingresa un nombre de usuario y contraseña, y luego haz clic en el botón Finish setup.

Conectarse con DreamObjects

Ahora que has instalado ownCloud, puedes comenzar a usarlo para administrar tus datos. Si ya estás utilizando una cuenta de DreamObjects para almacenar tus datos, puedes vincularla a tu instalación de ownCloud. Ve el siguiente artículo para obtener instrucciones:

Eliminar ownCoud

Para eliminar la instalación de ownCloud de tu sitio web, simplemente elimina el directorio de archivos de ownCloud.

Para los clientes de escritorio, desinstala el programa como cualquier otro programa en tu sistema operativo.

Solución de problemas

Después de la instalación, es posible que veas la siguiente advertencia en la página Configuración > General:

Transactional file locking should be configured to use memory-based locking, not the default slow database-based locking

Esta es solo una recomendación para mejorar el rendimiento al instalar Redis y usar Memcache como se indica aquí:

  • https://doc.owncloud.org/server/10.0/admin_manual/configuration/server/caching_configuration.html#configuring-transactional-file-locking-label

Dado que esta instalación requiere acceso a sudo, solo es posible en un servidor dedicado o DreamCompute, ya que estos planes permiten el acceso root sudo al servidor:
