Generalidades

Para solucionar problemas, es posible que a veces tengas que revisar los registros de errores de tu sitio y acceder a los registros para obtener más información sobre por qué puede ocurrir un problema. Este artículo explica cómo ver estos registros usando SFTP.

Si deseas ver y examinar estos archivos a través de SSH, visita los siguientes artículos en su lugar:

  • Ver y examinar el access.log vía SSH
  • Ver y examinar el registro de errores vía SSH
  • Los registros se rotan cada noche para que los archivos access.log y error.log solo contengan los registros de hoy, mientras que error.log.0 y access.log.0 contienen registros del día anterior.
  • Los archivos de registro se eliminan poco después de esto, por lo que se recomienda verificarlos lo antes posible.
  • Por GDPR, DreamHost almacena los registros HTTP de forma predeterminada durante 3 días.

Ver los registros a través de SFTP

 

No es posible ver tus registros usando el administrador de archivos en el panel.

  • Ver tus registros de acceso y error via SFTP

Debes usar un cliente FTP completamente funcional como Filezilla o Cyberduck para ver los registros.

Como se mencionó anteriormente, los registros están dentro del directorio /logs/example.com/http(s) bajo tu nombre de usuario en el servidor.

Para ver los registros:

  1. Asegúrate de que tu usuario sea un usuario de SFTP o Shell. Visita el artículo Cómo crear un usuario con acceso Shell (SSH) para más instrucciones.
  2. Inicia sesión en tu servidor usando tu cliente. Visita el artículo FTP para más detalles.
    Una vez que hayas iniciado sesión, verás varios archivos; en este ejemplo, estamos en el directorio de ejemplo del usuario anotado por el campo Sitio remoto: que muestra /home/Usuarioejemplo:
  3. Haz Clic en el directorio /logs.
    A continuación, verás una lista de sitios asociados con este usuario:
  4. Haz clic en el sitio apropiado de este directorio siguiente.
  5. Haz clic en el directorio http o https según los registros que desees ver.

    Es posible que el directorio http o https tenga una serie de números aleatorios o que veas varios directorios. Solo mira la fecha a la derecha para elegir el directorio más reciente.

    Verás el error.log y access.log a partir de hoy.

    Una vez en el directorio /http, notarás que tu sitio remoto se ve diferente en la parte superior, lo que muestra que los directorios seleccionados han cambiado, esto está bien.

  6. Haz clic derecho en el archivo de registro y abrelo con tu programa de edición de texto.
