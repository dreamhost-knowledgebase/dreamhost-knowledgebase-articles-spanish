Generalidades

Este artículo explica cómo crear registros DNS personalizados en tu dominio.

Este tipo de registros solo deben agregarse si tu dominio requiere un servicio específico para funcionar, como alojar tu correo en otra compañía o configurar registros de verificación de sitios de Google.

Requisitos previos

Estas instrucciones solo funcionarán si tus nameservers están apuntando a DreamHost. Antes de continuar, asegúrate de haber revisado el artículo Visión general de DNS y tener una comprensión de cómo funciona el DNS.

Configurar tu dominio

Puedes agregar registros DNS personalizados a un dominio que esté 'Completamente alojado' o configurado solo para DNS. Todo depende de qué registros estés creando y por qué los necesitas.

  • Dominios Completamente Alojados: Un dominio Completamente Alojado significa que has agregado alojamiento al dominio para que esté configurado en un servidor web y pueda ser visitado en Internet. En términos generales, si deseas que tu sitio esté alojado en los servidores de DreamHost, mantén tu sitio Completamente Alojado. 
  • Solo DNS: Sin embargo, si deseas que tu sitio apunte fuera de DreamHost creando registros A o CNAME, el primer paso es eliminar el servicio de alojamiento ya asignado a tu dominio en DreamHost. Revisa este artículo para obtener instrucciones sobre cómo configurar tu dominio como 'Solo DNS'.

Acceder a la página de Configuración de Dominio

Necesitarás acceder a la página de configuración de tu dominio para agregar los registros personalizados en este artículo. Para acceder a esta página:

  1. Navega a la página Administrar Sitios Web.
  2. Abre la página Configuraciones DNS:
    • Si estás en la Vista de Malla, da clic en el botón de los '3 puntos verticales' debajo de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
    • Si estás en la Vista de Lista, da clic en el botón de los '3 puntos verticales' a la derecha de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
      Aparecen los registros DNS del dominio.

Propagación de DNS

Las actualizaciones de DNS pueden tardar varias horas en actualizarse completamente en línea. Este proceso se conoce como 'Propagación de DNS' y el período de tiempo puede variar dependiendo del tipo de registro actualizado. Revisa el artículo sobre propagación de DNS para obtener más detalles.

 

Agregar registros de DNS personalizados

Las siguientes secciones explican cómo actualizar registros DNS específicos en tu panel de DreamHost.

No necesitas agregar el símbolo @ cuando creas registros DNS personalizados. Este símbolo se agrega automáticamente cuando el campo Nombre se deja en blanco.

Registro A

Los registros A son direcciones IP. Pueden ser agregados para apuntar tu sitio web a una compañía de alojamiento diferente o subdominios de tu sitio web a servicios externos.

  1. Haz clic en en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro A y haz clic en el enlace AGREGAR.
    • Host: Este es el dominio raíz o un subdominio (blog, tienda, etc.). No necesitas agregar el nombre del dominio en el campo Host.
      • Registro A raíz: Si deseas crear un registro A para el dominio raíz y apuntar fuera de DreamHost, primero debes configurar el dominio como Solo DNS. Luego puedes dejar este campo en blanco.
      • Registro A de subdominio: Para un subdominio (blog, dev, tienda, etc.), ingresa el nombre del subdominio en este campo.
    • Apunta a: Ingresa la dirección IP a la que apuntará el subdominio.
  3. Haz clic en el botón Agregar Registro para guardar.

Registro AAAA

Los registros AAAA son direcciones IPv6. Pueden ser agregados para apuntar tu sitio web a una compañía de alojamiento diferente o subdominios de tu sitio web a servicios externos.

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro AAAA y haz clic en el enlace AGREGAR.
    • Host: Este es el dominio raíz o un subdominio. No necesitas agregar el nombre del dominio en el campo Host.
      • Registro AAAA raíz: Si deseas crear un registro AAAA para el dominio raíz y apuntar fuera de DreamHost, primero debes configurar el dominio como Solo DNS. Luego puedes dejar este campo en blanco.
      • Registro AAAA de subdominio: Para un subdominio (blog, dev, tienda, etc.), ingresa el nombre del subdominio en este campo.
    • Apunta a: Ingresa la dirección IPv6 a la que apuntará el subdominio. Por ejemplo: 
      2607:f298:5:104f::765:5be
  • Haz clic en el botón Agregar Registro para guardar.

Registro ALIAS

Un registro ALIAS (también conocido como registro ANAME) es un registro DNS virtual similar a un CNAME, pero con la capacidad de agregarlo al nombre de dominio raíz (lo cual no es posible con un CNAME). También puede ser agregado a un registro de subdominio que ya exista (donde un CNAME debe ser único).

Para aclarar más la diferencia entre los registros ALIAS y CNAME:

Registro Dominio raíz (example.com) Subdominio (blog.example.com)
ALIAS Es POSIBLE agregar un registro ALIAS al dominio raíz incluso si ya tiene asignado un registro A. 

Los registros ALIAS también pueden ser agregados a un subdominio activo. Por ejemplo, si www.example.com o calendar.example.com ya tienen registros A o CNAME configurados, aún puedes agregar un registro ALIAS a esos mismos nombres.
CNAME No es posible agregar un registro CNAME al dominio raíz. El registro CNAME se utiliza para apuntar un subdominio como www.example.com o calendar.example.com a otro servicio usando un nombre de host (no una IP). Los registros CNAME solo pueden ser agregados a un subdominio que no esté utilizando actualmente registros A.

Agregar un registro ALIAS

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro ALIAS y haz clic en el enlace AGREGAR.
    • Host: Este es el dominio raíz o un subdominio (blog, tienda, etc.). No necesitas agregar el nombre del dominio en el campo Host.
      • Registro raíz: Si deseas crear un registro ALIAS para el dominio raíz, deja este campo en blanco.
      • Registro de subdominio: Para un subdominio (blog, dev, tienda, etc.), ingresa el nombre del subdominio en este campo.
    • Apunta a: Ingresa la ubicación del nombre de host a la que te gustaría que apunte este registro.
  3. Haz clic en el botón Agregar Registro para guardar.

Registro CNAME

Los registros CNAME se pueden utilizar para redirigir tu alojamiento fuera de DreamHost. Este registro se crea para apuntar un dominio a otro, en lugar de apuntar un dominio a una dirección IP (como el registro A personalizado).

Por favor ten en cuenta que un registro CNAME solo se puede crear en un subdominio. Esto significa que el registro raíz (example.com) no puede ser un registro CNAME.

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro CNAME y haz clic en el enlace AGREGAR.
  3. Ingresa lo siguiente:
    • Host: Este campo puede ser cualquier cosa, pero a menudo es simplemente www. Agregar www redirige el subdominio www fuera de DreamHost. No necesitas agregar el nombre del dominio en el campo Host.
    • Apunta a: Ingresa la ubicación a la que te gustaría que apunte este registro.
  4. Haz clic en el botón Agregar Registro para guardar.

Es posible que veas el siguiente error al agregar un CNAME:

Ya tienes un registro para este nombre. No puedes tener un CNAME y cualquier otro registro con el mismo nombre.

Esto generalmente significa que estás intentando agregar un CNAME para www, pero dado que ese registro ya existe en tus configuraciones de DNS, no puedes agregar uno adicional. Si necesitas agregar un CNAME para el subdominio www, primero debes configurar tu dominio como Solo DNS.

  • Configurar tu dominio como 'Solo DNS'

Si estás intentando agregar un CNAME para algo más, revisa tus registros DNS para ver si el registro que estás agregando ya existe.

Registro MX

Los registros MX son registros DNS necesarios para la entrega de correos. La compañía a la que apuntan tus registros MX es donde se envía tu correo. Hay dos opciones para agregar registros MX en el panel.

Opción 1 — Si tu dominio está alojado en DreamHost

Si has agregado alojamiento a tu dominio, ya tienes configurados registros MX en DreamHost.

Si necesitas alojar tu correo en otra compañía mientras continúas alojando tu sitio web en DreamHost, debes configurar registros MX personalizados para reemplazar los registros actuales en tu panel. Revisa este artículo para obtener más información.

Opción 2 — Si tu dominio no está alojado en DreamHost

Si tu dominio está registrado en DreamHost pero no está configurado en un plan de alojamiento, puedes agregar registros MX para dirigir tu correo a otra compañía.

Esto solo es necesario si tus nameservers aún están apuntando a DreamHost.

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro MX y haz clic en el enlace AGREGAR.
    • Host: Este campo generalmente se deja en blanco. Solo agrega algo aquí si tu nuevo proveedor de correo requiere un host, que a menudo es un subdominio, como www o mail. No necesitas agregar el nombre del dominio en el campo Host.
    • Apunta a: Ingresa el registro MX al que debe apuntar.
    • Prioridad: Ingresa el número de prioridad de este registro MX. Al agregar múltiples registros MX, este valor controla cuál registro MX se usa primero.
  3. Solo marca la casilla llamada Seguiré revisando mi correo electrónico en DreamHost si estás utilizando un Proveedor de Servicios de Correo.
  4. Haz clic en el botón Agregar Registro para guardar.

Registro TXT

Los registros TXT (Texto) te permiten asociar texto con un host específico. Un ejemplo común es agregar un registro de verificación de Google, o agregar un registro de DMARC de correos a tu sitio.

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro TXT y haz clic en el enlace AGREGAR.
  3. Ingresa lo siguiente:
    • Host: Este campo puede ser cualquier cosa. A menudo se deja en blanco. No necesitas agregar el nombre del dominio en el campo Host.
    • Valor de TXT *: Ingresa el valor de texto de este registro al que debe apuntar.
  4. Haz clic en el botón Agregar Registro para guardar.

Registro NAPTR

Los registros Name Authority Pointer (NAPTR) se utilizan con mayor frecuencia para la telefonía por Internet. Estos registros admiten el uso de expresiones regulares. Aquí tienes un ejemplo de un registro NAPTR:

$ORIGIN 4.3.2.1.5.5.5.0.0.8.1.e164.arpa.
IN NAPTR 100 10 "U" "E2U+sip" "!^.*$!sip:customer-service@example.com!" .
IN NAPTR 102 10 "U" "E2U+email" "!^.*$!mailto:information@example.com!" .

Revisa este enlace para obtener información detallada sobre este tipo de registro.

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro NAPTR y haz clic en el enlace AGREGAR.
    • Host: Ingresa el nombre del host para habilitar este registro, o déjalo en blanco. No necesitas agregar el nombre del dominio en el campo Host.
    • Orden: Un número (del 0 al 65535) que especifica el orden en que se procesan varios registros NAPTR. Los valores más bajos se procesan antes que los valores más altos.
    • Preferencia: Un número (del 0 al 65535) que especifica el orden en que se procesan varios registros NAPTR, con valores de Orden iguales. Los valores más bajos se procesan antes que los valores más altos.
    • Flags: Una cadena en minúsculas que contiene banderas para controlar aspectos de la reescritura e interpretación de los campos en el registro. Actualmente, solo se han definido cuatro banderas: s, a, u y p.
    • Servicio: Especifica el(los) servicio(s) disponible(s) en esta ruta de reescritura. También puede especificar el protocolo particular que se utiliza para comunicarse con un servicio.
    • Regex: Una cadena que contiene una expresión de sustitución que se aplica a la cadena original que tiene el cliente para construir el siguiente nombre de dominio a buscar.
    • Reemplazo: El siguiente nombre de dominio completamente calificado para consultar registros NAPTR, dependiendo de los valores encontrados en el campo de banderas. Este campo solo se usa si el campo de expresión regular está vacío. Si el campo Regex contiene un valor, este campo no se usa.
  3. Haz clic en el botón Agregar Registro para guardar.

Registro SPF

Los registros SPF Sender Policy Framework (SPF) especifican qué servidores de correo están autorizados para enviar correos desde tu dominio. Revisa este artículo para obtener detalles sobre cómo configurar los registros SPF.

Registro SRV

Los registros SRV (servicio) se utilizan para identificar computadoras que albergan un servicio específico y luego establecer una conexión entre ese servicio y el sitio web. Algunos programas y aplicaciones pueden requerir estos tipos de registros para activar sus servicios. Un ejemplo común son los siguientes dos registros SRV de Microsoft. Puedes encontrar más información en el siguiente enlace:

  • Dos registros SRV de Microsoft
Tipo Nombre Objetivo Protocolo Servicio Prioridad Peso Puerto TTL
SRV @ sipdir.online.lync.com _tls _sip 100 1 443 1 hour
SRV @ sipfed.online.lync.com _tcp _sipfederationtls 100 1 5061 1 hour

 

Formato

Tu registro SRV estándar tiene este tipo de formato:

_service._protocol.name. TTL class SRV priority weight port target.

Agragar

El ejemplo a continuación muestra cómo agregar el siguiente registro.

_sipfederationtls._tcp	SRV	100 1 5061 sipfed.online.lync.com.

Repite los siguientes pasos para agregar cualquier registro SRV adicional que necesites.

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección Registro SRV y haz clic en el enlace AGREGAR.
  3. Ingresa lo siguiente:

    Al ingresar un registro SRV en tu panel, ignora los campos TTL, class y SRV.

    • Nombre: Esto suele dejarse en blanco. No es necesario añadir el nombre de dominio en el Nombre field.
    • Servicio/Sip: sipfederationtls. No añadas el guion bajo antes de este valor, ni el punto al final.
    • Protocolo: tcp. No agregues el guion bajo antes de este valor.
    • Prioridad: 100
    • Peso: 1
    • Puerto: 5061
    • Objetico: sipfed.online.lync.com
  4. Haz clic en el botón Agregar Registro para guardar.

Registros CAA

Un registro de Autorización de Autoridad de Certificación (CAA) se utiliza para especificar qué autoridades de certificación (CA) están autorizadas a emitir un certificado para un nombre de dominio específico.

  • Si hay un registro CAA presente, solo las CAs listadas en el(los) registro(s) CAA están autorizadas a emitir certificados para el dominio.
  • Si no hay un registro CAA presente, cualquier CA está autorizada a emitir un certificado para el dominio.

Estos registros también permiten al propietario del dominio establecer reglas de notificación en caso de que alguien solicite un certificado a una autoridad certificadora no autorizada. Revisa el artículo sobre registros de políticas CAA para obtener más información.

Subdominioss

or lo general, un registro CAA se añade al dominio principal (example.com). Un registro CAA añadido al dominio principal (example.com) también se aplica a todos los subdominios, como blog.example.com. Sin embargo, un registro CAA añadido a cualquier subdominio tendrá prioridad sobre el registro CAA del dominio principal, lo que te permite tener más control sobre el registro del subdominio.

Agregar un registro CAA

  1. Haz clic en el botón Agregar Registro.
  2. Pasa el cursor sobre la sección de Registro CAA y haz clic en el enlace AGREGAR.

    Cada registro CAA creado solo se aplica a una única autoridad certificadora. Esto significa que necesitarías crear un registro CAA adicional para cada autoridad certificadora adicional que esté autorizada a emitir un certificado para este dominio.

    • flag: El valor de 1 se utiliza para representar la bandera crítica. Configurar este campo en 1 instruye a las autoridades certificadoras que deben comprender la etiqueta de propiedad siguiente antes de emitir un certificado.
    • tag: Escoge uno de los siguientes:
      • Entrada de Autorización por Dominio Esto configura la propiedad issue. El nombre de dominio de una única autoridad certificadora (por ejemplo, sectigo.com) autorizada a emitir certificados para el dominio.
      • Informe de incidente por informe IODEF Esto configura la propiedad iodef. Especifica una URL (https://example.com) o dirección de correo (mailto:admin@example.com) a la que una autoridad certificadora puede informar sobre violaciones de políticas. No todas las autoridades certificadoras admiten esta etiqueta, por lo que no se garantiza que todas las emisiones de certificados sean reportadas.
    • valor: El valor asociado con la etiqueta de propiedad elegida.
  3. Haz clic en el botón Agregar Registro para guardar.

Revisar el registro CAA a través de SSH

El comando dig se usa frecuentemente para consultar registros DNS. Dado que el registro CAA es un tipo nuevo de registro, también debes agregar type257 al ejecutar el comando. Por ejemplo:

[server]$ dig example.com type257

Registro DKIM

Los registros DKIM son una forma de 'firmar' los correos para demostrar que provienen de ti, utilizando claves DKIM listadas públicamente en el DNS de tu dominio. Se agregan como registros TXT con un formato específico. Revisa el artículo sobre registros DKIM para obtener más información.
