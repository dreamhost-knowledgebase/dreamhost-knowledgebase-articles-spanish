Generalidades

Para saber qué versión de Subversion tienes en tu servidor, inicia sesión en el servidor a través de SSH y corre el siguiente comando:

[server]$ svn --version

Si deseas instalar una versión más reciente de subversion (por ejemplo, para la sincronización del repositorio), deberás visitar esta página y descargar el archivo subversion-x.x.x.tar.bz2:

  • subversion.apache.org/download/

Actualizar a la última versión de Subversion

Advertencia: Las siguientes instrucciones no funcionan actualmente porque falla en el paso ./configure debido a la versión de servidor de APR. Los administradores de DreamHost están trabajando para resolver esto. Tan pronto como se resuelva este problema, este artículo se actualizará.

Sin embargo, aún puedes instalar tu propio paquete APR localmente y apuntar tu instalación de SVN para usar el paquete local. Pero ten en cuenta que el soporte de DreamHost no puede ayudar con tales instalaciones personalizadas.

Compilar subversion es bastante sencillo, pero puedes encontrar algunos errores de compilación dependiendo de la versión. En general, sigue estos pasos para compilar e instalar subversion localmente. Asegúrate de cambiar el username a tu usuario Shell.

[server]$ mkdir tmpwork
[server]$ cd tmpwork
[server]$ wget https://apache.spinellicreations.com/subversion/subversion-x.x.x.tar.bz2
[server]$ tar -xjf subversion-x.x.x.tar.bz2
[server]$ cd subversion-x.x.x
[server]$ ./configure --prefix=/home/username --with-ssl
[server]$ make
[server]$ make install

Esto instala la nueva subversión en tu directorio de usuario. Deberás cambiar la ruta para incluir tu directorio ~/bin para usar los nuevos binarios. Para hacer eso, corre el siguiente comando en tu terminal:

alias svn='/home/username/bin/svn'
