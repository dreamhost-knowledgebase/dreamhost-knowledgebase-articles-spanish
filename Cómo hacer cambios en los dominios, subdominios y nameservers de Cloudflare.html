Usar Cloudflare con WWW o un subdominio

Si estás usando Cloudflare sin un subdominio, debes seleccionar "Add WWW" al dominio. Esta es una limitación que Cloudflare impone al agregar tu servicio a través de DreamHost. Si deseas utilizar Cloudflare sin el subdominio www, debes comprar una cuenta directamente a través de tu sitio web.

Si tu dominio ya es un subdominio, como subdominio.example.com, entonces no es necesario cambiar las opciones de alojamiento del dominio y puedes configurar Cloudflare en subdominio.example.com.

Si tu dominio usa una aplicación de administración de contenido como WordPress, esto puede causar un problema ya que la URL utilizada por la aplicación ha cambiado. Si necesitas cambiar tu dominio para incluir WWW en la URL, también debes asegurarte de actualizar la URL utilizada en su aplicación o código. Es mejor hacer los cambios en tu programa o código CMS antes de habilitar Cloudflare en el dominio.

Mira el artículo Cambiar la URL del sitio de WordPress para obtener detalles sobre cómo ajustar esta configuración para un sitio de WordPress.

Uso de Cloudflare con subdominios dreamhosters.com

Cloudflare no se puede usar con los subdominios dreamhosters.com en este momento.

Usar Cloudflare con nameservers externos

Si tu dominio no utiliza los nameservers de DreamHost, debes actualizar manualmente los registros DNS de tu dominio. Puede ver todos los registros DNS que están configurados para cualquier sitio que aloje en DreamHost simplemente haciendo clic en el enlace DNS debajo del nombre de dominio en la página Administrar Dominios. Mira el artículo sobre DNS para más detalles.

Si el dominio utiliza WWW o la IP del subdominio cambió, deberás actualizar la dirección IP en la empresa que aloja tus nameservers. Al igual que con cualquier cambio de DNS, la propagación del DNS tarda varias horas en completarse.

Cambiar la cuenta de Cloudflare

Solo es posible cambiar la cuenta dentro del dominio principal (www.example.com). Los subdominios como blog.example.com no pueden cambiar la cuenta de Cloudflare asignada a ellos. Consulta el siguiente artículo para obtener más información:
