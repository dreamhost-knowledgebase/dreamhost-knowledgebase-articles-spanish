Generalidades

Es posible que se te indique que comprimas con GZIP para optimizar la velocidad de tu sitio web, especialmente si usas herramientas como GTMetrix, Pingdom, Page Speed ​​Insights o YSlow.

A continuación se describe cómo se habilita GZIP en DreamHost.

¿Qué es la compresión GZIP?

La compresión GZIP es un método para ayudar a acelerar la carga de sitios web al reducir el tamaño de ciertos archivos para transferirlos del servidor del sitio web el computador o dispositivo del visitante. Por ejemplo, una página web de 1MB de tamaño solo puede necesitar transferir 800 KB.

La carga de una página de 1MB en realidad puede ser un 25% más lenta que la carga de una página de 800 KB, por lo que asegurarte de aprovechar la compresión puede significar un sitio de carga más rápida.

¿DreamHost habilita la compresión GZIP?

DreamHost habilita la compresión GZIP de forma predeterminada en todos los planes de alojamiento web, por lo que no es necesario que hagas más pasos para activarla en tu sitio.

Sin embargo, ten en cuenta que SSLCompression está deshabilitado por defecto en los servidores de DreamHost debido a una vulnerabilidad de seguridad cuando la compresión es combinada con HTTPS (usando un Certificado TLS/SSL).

Si GZIP está habilitado de forma predeterminada, ¿por qué la herramienta de optimización de mi sitio me dice que necesito comprimir con GZIP?

Hay algunas razones por las que algunos servicios de análisis de sitios web pueden decirte que necesitas comprimir algunos archivos con GZIP a pesar de que ya está habilitado para tu sitio HTTP. Esto incluye:

  • Las URL que informan que necesitan compresión son para un sitio de terceros, no para tu dominio. En este caso, DreamHost no puede controlar la compresión en un sitio de terceros. Para compensar, puedes preguntar al sitio de terceros si pueden comprimir o crear alguna alternativa para tus URL, o puedes eliminar las URL del código de tu página y reemplazarlas con las URL de tu propio sitio o las URL de un sitio de terceros que pueda comprimir esos archivos.
  • Has creado tu sitio web solo HTTPS, donde todas las páginas y URL se cargan en https:// en lugar de http://. Debido a una vulnerabilidad de seguridad que se describe en la sección anterior, la compresión GZIP está desactivada para HTTPS.
  • Es posible que tengas un archivo .htaccess en tu sitio con un código que anula la compresión GZIP. Si tienes URLs http:// en tu dominio que parecen necesitar compresión, puedes revisar el código en tu archivo .htaccess y hacer las correcciones necesarias. Si no estás familiarizado con el código en .htaccess, también puedes intentar cambiar temporalmente el nombre de .htaccess a .htaccess_off e intentar cargar la página nuevamente. Para asegurarte de que otras funciones de tu sitio no se rompan permanentemente, recuerda cambiar el nombre del archivo .htaccess_off a .htaccess.

Alternativas a la compresión GZIP para HTTPS

Hay un par de opciones que puede usar que permiten la compresión de algunos archivos en tu sitio HTTPS incluso sin compresión GZIP, que al tiempo que mitigan la vulnerabilidad de seguridad mencionada anteriormente. Si utilizas los siguientes métodos y pruebas tu sitio utilizando un servicio de análisis de sitios web, no deberías ver ningún resultado que indique que los archivos en tu dominio deberían estar comprimidos.

Cloudflare

Cloudflare es un servicio que proporciona protección y aceleración de sitios web. La característica principal de Cloudflare es su red global de entrega de contenido (CDN), que proporciona tecnologías de optimización de contenido web que mejoran la rapidez con que se cargan los recursos en tu sitio. Todos los planes Cloudflare, incluidos sus planes gratuitos, ofrecen compresión Brotli de forma predeterminada, que es un método de compresión más nuevo y más rápido. Para obtener más información sobre cómo habilitar Cloudflare, consulta el siguiente artículo:

  • Cómo habilitar Cloudflare en tu dominio

DreamPress

Si tienes un sitio de WordPress, puedes cambiar tu sitio a DreamPress, donde la compresión Brotli también está habilitada de forma predeterminada sin ningún trabajo adicional. DreamPress es el alojamiento administrado de WordPress de DreamHost. Es un servicio escalable que permite a los usuarios administrar sus sitios de WordPress con precios simples, beneficios adicionales de robustez del servidor, almacenamiento en caché y memoria. Para obtener más información sobre cómo cambiar tu instalación de Un-clic de WordPress a DreamPress, consulta el siguiente artículo:
