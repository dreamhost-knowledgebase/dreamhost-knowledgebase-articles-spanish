Generalidades

Git es un sistema de control de versiones, que es un software que permite a los desarrolladores mantener un historial completo de cada archivo en un proyecto. Sin embargo, no necesitas ser un desarrollador para usar Git con tu sitio web. Por ejemplo, puedes usar Git para mantener un historial de todos los archivos de tu sitio web: cada vez que se cambias un archivo, se registra una versión del cambio en el software Git. Si es necesario, siempre puedes volver a cualquier versión y restaurarla.

Puedes sentirte tentado a pensar en Git como simplemente una herramienta de copia de seguridad, pero Git hace mucho más que eso. Git no solo almacena un historial completo de tus archivos, sino que también almacena la 'versión' de esos archivos a lo largo del historial de un proyecto. Esto te permite usar Git para recuperar información sobre cuándo se editó un archivo, así como la capacidad de ver los cambios exactos realizados en el archivo. Una copia de seguridad simple no incluye estos detalles adicionales.

¿Necesito usar Git?

No, pero se recomienda usar Git si continuamente realizas cambios en el código de tu sitio web. Por ejemplo, si estás actualizando el código de tus archivos HTML, CSS, JavaScript o cualquier otro archivo, se recomienda que mantengas copias de seguridad de esos archivos en caso de que un cambio que realice cause un error en tu sitio.

Con una simple copia de seguridad, tendrías los archivos antiguos que podrías restaurar. Pero con Git, puedes ver un historial completo no sólo de cuándo se editó un archivo, sino también los detalles exactos sobre cómo cambió el archivo. Esto puede ayudarte a solucionar problemas de codificación con tu sitio.
