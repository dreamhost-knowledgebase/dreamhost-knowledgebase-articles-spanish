Generalidades

Existen varios métodos manuales para hacer una copia de seguridad del contenido de tu cuenta. La siguiente es una descripción de las diversas opciones de respaldo y servicio:

Copia de seguridad a través de FTP

Si tu usuario tiene una gran cantidad de contenido, puedes hacer una copia de seguridad de ese usuario manualmente. Esto se hace a través de FTP, del que puedes leer más aquí:

  • Generalidades sobre FTP

Una vez conectado con un cliente FTP, puedes descargar tus datos. Visita el siguiente artículo que describe cómo seleccionar los datos del servidor en el panel derecho y descargarlos a tu computador:

  • Descargar archivos

Si tienes varios GB de datos, DreamHost recomienda comprimir primero los directorios y luego descargar los contenidos para acelerar las cosas. Esto se hace iniciando sesión en tu servidor web a través de tu usuario SHELL y luego comprimiendo manualmente las carpetas. Para obtener información sobre cómo iniciar sesión en tu cuenta a través de SSH, visita el siguiente artículo:

  • Generalidades sobre SSH

Copia de seguridad de tu base de datos

Existen algunos métodos para hacer una copia de seguridad de tus bases de datos. Una forma se realiza a través de la interfaz phpMyAdmin, que sería la forma más sencilla de completar este proceso. Las otras opciones para hacer una copia de seguridad de la base de datos MySQL se describen en el siguiente artículo:

  • Generalidades sobre copias de seguridad de bases de datos

Copia de seguridad de correo

Puedes hacer una copia de seguridad de tu correo manualmente a través de un cliente de correo. Esto se realiza a través de un cliente de correo configurado para descargar los mensajes de los servidores DreamHost a tu computador local. Luego, se hace una copia de seguridad de los mensajes en el computador que los descargó.

Puedes leer más sobre este proceso haciendo una copia de seguridad de los correos aquí:

  • Respaldar correo
