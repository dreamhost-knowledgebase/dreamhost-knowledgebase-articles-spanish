Generalidades

Este artículo describe varias opciones que puedes usar para acceder a una base de datos antes de apuntar tus nameservers a DreamHost.

Cuándo usar estas opciones

Lo siguiente se debe utilizar si los los registros DNS de tu dominio aún no apuntan a DreamHost. Esto se debe a que el hostname MySQL creado en el panel de tu dominio no funciona hasta que tus registro DNS se resuelva a DreamHost.

El hostname mysql es una URL que se utiliza para acceder a la base de datos de DreamHost. Este hostname suele ser un subdominio de tu dominio principal. Por ejemplo, si tu dominio es example.com, puedes crear un subdominio como mysql.example.com para conectarte al servidor de la base de datos.

Acceder a una base de datos de DreamHost

Al migrar un sitio web, es común que los nameservers sigan apuntando a la antigua compañía de alojamiento. Esto garantiza que el sitio web permanezca en línea durante la migración. Las siguientes opciones te permiten iniciar sesión en tu base de datos antes de apuntar tus nameservers a DreamHost.

Opción 1 — Apuntar tu registro mysql A a DreamHost (Recomendado)

La mejor opción es iniciar sesión en el panel de tu compañía de alojamiento actual y apuntar el registro mysql A a DreamHost.

  1. Ubica tu registro A de mysql en el panel de DreamHost, que es una dirección IP numérica. Revisa este artículo para obtener más información.
  2. Inicia sesión en el panel de tu compañía de alojamiento actual y apunta el registro A de mysql de tu dominio a la dirección IP de DreamHost.
  3. Espera unas horas para que el DNS se actualice en línea.

Una vez el DNS se actualice, podrás visitar tu subdominio mysql (mysql.example.com) e iniciar sesión.

No uses la dirección IP para conectarte a tu base de datos en el archivo de configuración de tu sitio web. Aunque es raro, esta IP está sujeta a cambios con el tiempo. Para evitar un posible tiempo de inactividad, asegúrate de usar solo el hostname para conectare.

Opción 2 — Usar la URL oficial de phpMyAdmin

También puedes utilizar el hostname oficial de phpMyAdmin de tu centro de datos. Visita la página Centro de Datos y desplázate hacia abajo para ubicar el centro de datos donde se encuentran tus servicios de MySQL. Puede ser Costa Oeste o Costa Este. A continuación, puedes utilizar los siguientes hostnames para conectarte según la ubicación de tu servidor de MySQL:

  • Costa Oeste:  https://west1-phpmyadmin.dreamhost.com
  • Costa Este:  https://east1-phpmyadmin.dreamhost.com

Opción 3 — Usar otro hostname del dominio

Si tienes otros dominios alojados en DreamHost, puedes usar su hostname mysql en vez. Esto es porque tu servidor MySQL (el cual aloja todas las bases datos) se accede desde cualquier subdominio asignado dentro de tu cuenta de DreamHost.

Por ejemplo, tu si tienes otro dominio como por ejemplo dreamhostexample.com y sus nameservers apuntan a DreamHost, puedes crear un hostname mysql llamado mysql.dreamhostexample.com. Luego puedes usar este hostname para conectarte a CUALQUIER base de datos en tu cuenta de DreamHost.
