Generalidades

Nginx (pronunciado engine-x) es un servidor HTTP gratuito, de código abierto, de alto rendimiento y proxy inverso, así como un servidor proxy IMAP / POP3. Es conocido por su alto rendimiento, estabilidad, rico conjunto de características, configuración simple y bajo consumo de recursos.

Nginx funciona con varios sitios de alta visibilidad, como Netflix, Hulu, Pinterest, Cloudflare, Airbnb, WordPress.com, GitHub, SoundCloud, Zynga, Eventbrite, Zappos, Heroku, RightScale, Engine Yard y MaxCDN

Revisa el siguiente artículo para más detalles:

  • https://www.nginx.com/resources/wiki/

Nginx está disponible en los planes de VPS de DreamHost como un servidor web opcional para ejecutar tus sitios.

Nginx en DreamHost

Todos los VPS de DreamHost usan Apache por defecto en lugar de Nginx. Esto se debe a que Apache admite una caja de herramientas más grande de cosas que puede hacer de inmediato y es probablemente el más compatible en todo el software web que existe en la actualidad.

Sin embargo, Nginx es una muy buena opción en los casos en que tienes un sitio que recibe grandes volúmenes de tráfico y se encuentra con problemas de rendimiento/memoria en un VPS. Un servidor que ejecuta Nginx sirve tu sitio más rápido mientras usa menos memoria que Apache. Esto te permite manejar una cantidad mucho mayor de tráfico con más gracia que Apache. Para una visión más detallada de esto, visita la página de comparación de rendimiento del servidor web:

  • Comparación de rendimiento de servidores web

Versión de Nginx

Para encontrar la versión actual que Nginx tiene instalada, corre lo siguiente:

[server]$ /usr/sbin/nginx -v

Compresión Gzip

Por defecto, la compresión gzip está habilitada en tu servidor Nginx. Sigue estos pasos para deshabilitar la compresión gzip:

  1. Visita el artículo archivos locales de configuración para crear tu directorio local  /nginx/example.com.
  2. Crea un archivo llamado settings.conf en este directorio /nginx/example.com.
  3. Agrega el siguiente contenido: 
    gzip off;
  4. Asegúrate de volver a cargar Nginx para que los cambios surtan efecto.

Hay opciones de compresión gzip adicionales que puedes configurar que se detallan en el siguiente enlace:

  • HttpGzipModule

Configuraciones de Nginx para aplicaciones comunes

Muchas aplicaciones web como WordPress y MediaWiki están configuradas originalmente para funcionar con Apache. Dado que los archivos .htaccess no son compatibles con Nginx, hay algunos ajustes que debes hacer para que tus aplicaciones se ejecuten correctamente.
