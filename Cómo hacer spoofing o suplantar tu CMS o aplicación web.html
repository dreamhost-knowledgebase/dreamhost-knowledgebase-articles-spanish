Generalidades

Lo siguiente describe cómo puedes suplantar la dirección del REMITENTE dentro varias aplicaciones de CMS.

Requisitos previos

Primero, asegúrate de que la dirección de correo del administrador esté alojada en DreamHost. Puedes verificar esto haciendo una búsqueda de registros MX usando cualquiera de los siguientes sitios:

  • whatsmydns.net — Un sitio popular que verifica los registros DNS en todo el mundo.
  • Ver tus registros DNS de DreamHost — El verificador de propagación de DNS de DreamHost en el panel.

Los registros MX de DreamHost aparecen como se muestra a continuación:

Host Valor
@ 0 mx1.mailchannels.net.
@ 0 mx2.mailchannels.net.
mail 0 mx1.mailchannels.net.
mail 0 mx2.mailchannels.net.

Una vez que hayas confirmado que tu correo está alojado en DreamHost, puedes proceder a falsificar la dirección del REMITENTE de una manera útil.

Suplantar dentro de un CMS o aplicación web

Las siguientes secciones le muestran cómo suplantar una dirección de correo dentro de varios CMS y aplicaciones web.

WordPress

Algunos formularios de contacto de WordPress creados por plugins o temas tienen configuraciones que te permiten controlar completamente cómo se envían los correos. Contact Form 7 es un ejemplo. Puedes usar la configuración del plugin en el panel de administración de WordPress para enviar correos con el nombre del visitante del sitio y una dirección de correo de un administrador del sitio, webmaster o cualquier persona asociada con el funcionamiento del sitio:

De: [your-name] <admin@example.com>

También puedes configurar la cabecera Reply-To para que las respuestas a estos correos vayan directamente al visitante del sitio:

Reply-To: [your-name] <[your-email]>

Hay un plugin diferente Configure SMTP que puedes usar para configurar la información de REMITENTE en todos los correos enviados desde WordPress, independientemente del plugin o tema que envía el correo.

Se puede ver información adicional de WordPress en la solución de problemas de los formularios de contacto de WordPress. Sin embargo, ten en cuenta que no todos los plugins y temas te permiten controlar cómo se envían los correos.

Drupal

El módulo Contact Reply-To cambia el formulario de contacto de Drupal para enviar desde la dirección de correo del sitio web. También establece un encabezado reply-to para el usuario real, evitando las restricciones discutidas aquí.

Joomla

Joomla tiene una "Respuesta personalizada" que envía correos con el REMITENTE del visitante del sitio. Puedes desactivar eso en la configuración del panel de Joomla. Dirígete a Components -> Contacts -> Options. En la pestaña Form, asegúrate de que "Custom Reply" esté puesto como NO.

Debes configurar el "Custom Reply" en NO para que Joomla genere el correo. De lo contrario, el formulario podría no funcionar.

Para las versiones de Joomla 3.1 o superior, puedes acceder a las configuraciones de autenticación SMTP en el panel de Joomla Site -> Global Configuration -> Server tab -> Mail Settings. 

WooCommerce

WooCommerce incluye la opción de cambiar el nombre del remitente de y dirección de correo en la pestaña Settings -> Emails. Esto debe configurarse en una cuenta de correo alojada en DreamHost.
