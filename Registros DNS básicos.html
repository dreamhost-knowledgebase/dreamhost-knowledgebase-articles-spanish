Generalidades

Este artículo describe los diversos tipos de registros que se configuran para un dominio y para qué sirven. Estos registros se pueden configurar a través de la página Manage Domains haciendo clic en el enlace 'DNS' debajo del nombre de dominio.

Ver también: Lista de tipos de registros DNS en Wikipedia

A
Este es el registro "Dirección" (registro A) para reenviar el nombre de host a una dirección IPv4.
  • Registro A
AAAA
Este es el registro de dirección IPv6. Puedes usar esto para reenviar la asignación del nombre de host a una dirección IPv6.
  • IPv6
CNAME
El "registro de nombre canónico" o registro CNAME apunta un nombre de dominio a otro nombre de dominio.
  • CNAME record
MX
También conocidos como registros de "intercambio de correo", muestran el registro de host al que apunta el dominio para el alojamiento de correo.
  • Registro MX
NS
Los "nameservers" delegan un dominio o subdominio a un servidor DNS.
  • Nameservers
PTR
También llamado "registro de puntero". Esto se utiliza para la asignación inversa de una dirección IP a un nombre de host. DreamHost actualmente solo admite registros DNS inversos cuando se agrega una IP única a un dominio.

Nota: Ten en cuenta que una dirección IP única no se puede utilizar para resolver problemas de bloqueo de correo en alojamiento de correo electrónico compartido.

Puedes leer más sobre sobre la dirección IP única aquí:
  • Dirección IP Única
SPF
Sender Policy Framework (SPF) Los registros del Marco de políticas del remitente especifican qué servidores de correo pueden enviar correos electrónicos desde tu dominio.
  • Registro SPF
SRV
Los registros del "localizador de servicios" anuncian un servicio específico que ofrece un servidor.
  • Registro SRV
TXT
Los registros de texto son cadenas de texto de forma libre. Se pueden usar para cosas como los registros de Verificación de Google y la firma de DKIM para servicios de correo. Para obtener más información sobre DKIM, consulta el siguiente artículo:
  • DKIM

Puedes encontrar más información sobre las direcciones IP y los dos tipos que se utilizan en el siguiente artículo:

  • Dirección IP

Detalles adicionales

A continuación se presentan algunos detalles adicionales sobre diferentes registros DNS o terminología:

Búsqueda de registros A de DNS

Al buscar un registro A, puedes obtener un registro CNAME en respuesta, por ejemplo:
[server]$ dig www.cloudflare.com
www.cloudflare.com. 260 IN CNAME
www.cloudflare.com.cdn.cloudflare.net.
www.cloudflare.com.cdn.cloudflare.net. 260 IN A 198.41.215.163
www.cloudflare.com.cdn.cloudflare.net. 260 IN A 198.41.214.163
  • Puedes buscar el registro A de ese registro CNAME. Se te presentará otro registro CNAME o un registro A.

Nombre de dominio completamente cualificado

Un FQDN es un dominio único que no se puede confundir con otro. Por ejemplo, si tu sitio era example.com, puedes tener varios FQDNs.

  • www.example.com

Revisa el siguiente artículo para obtener más detalles:

  • Fully qualified domain name

Archivo de zona

Un archivo de zona (Zone file) es un registro de texto que contiene la asignación de tu dominio y subdominios a sus correspondientes direcciones IP. Revisa el siguiente artículo para más detalles:

  • Zone file

DreamHost no ofrece un servicio para exportar el archivo de zona de tu dominio. Si necesitas una lista de los registros del dominio, debes copiar tus registros DNS de tu panel.

Registro SOA

El registro de Inicio de Autoridad es el primer registro de recursos en un archivo de Zona DNS. El registro SOA especifica la siguiente información autorizada sobre un dominio específico:

  • Nameserver primario de donde se creó el registro SOA.
  • Correo electrónico del administrador del archivo de zona. (Se utiliza un punto en lugar del símbolo @).
  • El número de serie del dominio. Este es el número de revisión del archivo de zona.
  • Temporizadores:
    • Tiempo de actualización
    • Tiempo de reintento
    • Tiempo de expiración
    • TTL Mínimo

Corre el siguiente comando vía SSH para ver un registro SOA. Este ejemplo muestra los resultados de dreamhost.com:

[server]$ nslookup -type=soa dreamhost.com

      Server:    64.90.63.202
      Address:   64.90.63.202#53
	  
	  Non-authoritative answer:
	  dreamhost.com
	  origin = ns1.dreamhost.com
	  mail addr = hostmaster.dreamhost.com
	  serial = 2015091000
	  refresh = 14534
	  retry = 1800
	  expire = 1814400
	  minimum = 14400
