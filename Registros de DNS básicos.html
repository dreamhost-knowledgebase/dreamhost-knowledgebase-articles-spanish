Generalidades

A continuación se describen los diversos tipos de registros que se configuran para un dominio y cómo funcionan.

Encontrar tus registros de DNS

Para encontrar tus registros DNS en el panel, navega a la página Administrar Sitios Web y abre la página Configuraciones de DNS:

  • Si estás en la Vista de Malla, da clic en el botón de los '3 puntos verticales' debajo de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
  • Si estás en la Vista de Lista, da clic en el botón de los '3 puntos verticales' a la derecha de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
    Los registros DNS del dominio aparecen.

Varios tipos de registros

Puedes encontrar más información sobre las direcciones IP y los dos tipos que se utilizan en el siguiente artículo:

  • Dirección IP
Registro Descripción Más información

A

 El registro de Dirección (A), que redirige el nombre de host a una dirección IPv4.
  • Registro A

AAAA

 El registro de dirección IPv6. Puedes usarlo para redirigir la asignación del nombre de host a una dirección IPv6.
  • Registro AAAA
  • IPv6

ALIAS

 An Registro ALIAS es un registro DNS virtual similar a un CNAME, pero con la capacidad de agregarlo al nombre de dominio raíz (lo cual no es posible con un CNAME). También puede agregarse a un registro de subdominio que ya existe, donde un CNAME debe ser único.
  • Registro ALIAS

CAA

 Un registro de Autorización de Autoridad de Certificación (CAA) se utiliza para especificar qué autoridades de certificación (CA) tienen permiso para emitir un certificado para un nombre de dominio específico.
  • Registro CAA

CNAME

 El registro de Nombre Canónico (CNAME) apunta un nombre de dominio a otro nombre de dominio.
  • Registro CNAME

MX

 También conocidos como registros de Intercambio de Correo (MX), que te muestran el registro de host al que el dominio está apuntando para el alojamiento de correo.
  • Registro MX

NAPTR

 Los registros de Puntero de Autoridad de Nombre (NAPTR) se utilizan con mayor frecuencia para la telefonía por Internet.
  • Nameservers

NS

 Los registros de Nameservers (NS) delegan un dominio o subdominio a un servidor DNS.
  • Registro NAPTR

PTR

También llamado "registro de puntero", se utiliza para la asignación inversa de una dirección IP a un nombre de host. Actualmente, DreamHost solo admite registros DNS inversos cuando se agrega una IP única a un dominio.

Nota: una dirección IP única no se puede utilizar para resolver problemas de bloqueo de correo en el alojamiento de correo compartido.

DreamCompute también admite registros PTR, pero primero debes configurar el DNS de tu(s) dominio(s) para que los registros se creen automáticamente. Si el registro PTR no parece ser correcto dentro de los 15 minutos posteriores a realizar este cambio, por favor contacta a soporte..
  • Dirección IP Única

SPF

 Sender Policy Framework Los registros (SPF) especifican qué servidores de correo tienen permiso para enviar correos desde tu dominio.
  • Registro SPF

SRV

 Los Registros de Localizador de Servicio (SRV) anuncian un servicio específico que ofrece un servidor.
  • Registro SRV

TXT

 Los registros de Texto (TXT) son cadenas de texto de formato libre. Por ejemplo, pueden usarse con los registros de verificación de Google y la firma DKIM para servicios de correo.
  • Registros TXT

Detalles adicionales

Las siguientes secciones describen detalles adicionales sobre los diferentes registros DNS y la terminología relacionada.

Búsqueda de registros A de DNS

Al buscar un registro A, puedes obtener un registro CNAME en respuesta. Por ejemplo:
[server]$ dig www.cloudflare.com
www.cloudflare.com. 260 IN CNAME
www.cloudflare.com.cdn.cloudflare.net.
www.cloudflare.com.cdn.cloudflare.net. 260 IN A 198.41.215.163
www.cloudflare.com.cdn.cloudflare.net. 260 IN A 198.41.214.163
  • Puedes buscar el registro A de ese registro CNAME. Se te presentará otro registro CNAME o un registro A.

Nombre de dominio completamente cualificado

Un FQDN es un dominio único que no se puede confundir con otro. Por ejemplo, si tu sitio era example.com, puedes tener varios FQDNs.

  • www.example.com

Revisa el siguiente artículo para obtener más detalles:

  • Nombres de dominio totalmente cualificados

Zone file

Un archivo de zona (Zone file) es un registro de texto que contiene la asignación de tu dominio y subdominios a sus correspondientes direcciones IP. Revisa el siguiente artículo para más detalles:

  • Zone file

DreamHost no ofrece un servicio para exportar el archivo de zona de tu dominio. Si necesitas una lista de los registros del dominio, debes copiar tus registros DNS de tu panel.

Registro SOA

El registro de Inicio de Autoridad es el primer registro de recursos en un archivo de Zona DNS. El registro SOA (Start Of Authority) especifica la siguiente información autorizada sobre un dominio específico:

  • Nameserver primario de donde se creó el registro SOA.
  • Correo del administrador del archivo de zona. (Se utiliza un punto en lugar del símbolo @).
  • El número de serie del dominio. Este es el número de revisión del archivo de zona.
  • Temporizadores:
    • Tiempo de actualización.
    • Tiempo de reintento.
    • Tiempo de expiración.
    • TTL Mínimo.

Corre el siguiente comando vía SSH para ver un registro SOA. Este ejemplo muestra los resultados de dreamhost.com:

[server]$ nslookup -type=soa dreamhost.com

      Server:    64.90.63.202
      Address:   64.90.63.202#53
	  
	  Non-authoritative answer:
	  dreamhost.com
	  origin = ns1.dreamhost.com
	  mail addr = hostmaster.dreamhost.com
	  serial = 2015091000
	  refresh = 14534
	  retry = 1800
	  expire = 1814400
	  minimum = 14400
