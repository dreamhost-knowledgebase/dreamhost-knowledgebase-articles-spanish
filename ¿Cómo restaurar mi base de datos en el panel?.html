Copias de seguridad diarias de DreamHost

DreamHost no puede garantizar que los datos se retendrán en cuentas inactivas (suspendidas/cerradas/deshabilitadas) durante más de dos semanas después del cambio de estado de la cuenta. Consulta los siguientes artículos del centro de ayuda para obtener más información sobre cómo puedes hacer una copia de seguridad de tu cuenta o realizar un pago:

  • Respaldar una cuenta en el panel
  • ¿Cómo puedo hacer un pago?

Todas las bases de datos DreamHost MySQL hacen copias de seguridad diariamente. Las copias de seguridad comienzan a medianoche y se completan después de que se haya hecho una copia de seguridad de cada base de datos. Se RECOMIENDA ALTAMENTE que crees tus propias copias de seguridad externas de todas tus bases de datos para complementar las copias de seguridad a corto plazo que DreamHost crea. DreamHost no garantiza que haya copias de seguridad disponibles.

Copia de seguridad de tu cuenta

Para hacer una copia de seguridad de toda tu cuenta (usuarios FTP, usuarios de correo y bases de datos), consulta la función de copia de seguridad de tu cuenta. Las copias de seguridad creadas se pueden descargar a otra ubicación externa para su custodia.

Restaurar una base de datos

Generalmente hay 5 días de copias de seguridad disponibles. Ten en cuenta que esto no está garantizado, por lo que siempre debes hacer tus propias copias de seguridad personales en caso de que no haya ninguna disponible para restaurar en tu panel.

  1. Navega a la página Bases de Datos MySQL.
  2. Haz clic en el botón Restaurar BD que se encuentra a la derecha de tu base de datos.
  3. Escoge lo que te gustaría restaurar:
    • ¿Restaurar qué copia de seguridad? — Elige cualquier copia de seguridad disponible del menú desplegable.
    • Tablas para restaurar — Si deseas restaurar toda la base de datos, deja este espacio con un asterisco, como *
    • Al restaurar — Tienes 4 opciones de cómo se restauran tus tablas.
    Opciones 1 y 2 — Las dos primeras opciones restauran tus tablas, pero los nombres cambian. De esta manera, los archivos restaurados NO toman el lugar de los archivos actuales. Deberás cambiar manualmente los nombres de las tablas o ajustar el archivo de configuración de tu sitio para ver la base de datos restaurada
    Opciones 3 y 4 — Las dos últimas opciones reemplazan inmediatamente las tablas de tu base de datos. Las tablas actuales se renombran. Con mayor frecuencia desearás una de las dos últimas opciones

    Sitios de WordPress

    Al restaurar una base de datos del sitio de WordPress, se recomienda encarecidamente que selecciones la Opción 4. Esto te permite revertir fácilmente el sitio a la versión anterior de tu base de datos si hay algún problema con los datos restaurados.

    Puedes volver rápidamente a la versión anterior actualizando el valor del prefijo en tu archivo wp-config.php. Inicia sesión en tu sitio a través de FTP o SSH y abre el archivo wp-config.php (está en el directorio de tu sitio). Verás la siguiente línea:

    $table_prefix = 'wp_';

    Reemplaza el valor con la nueva marca de tiempo de la copia de seguridad que creaste. Por ejemplo:

    $table_prefix = '20180924_141233_wp_';
  4. Haz clic en el botón ¡Restaurar mi copia de seguridad ahora!

Verás un mensaje de éxito que te notificará que tu base de datos se restaurará en 10 minutos.

Uso de disco de bases de datos MySQL

Los servidores MySQL tienen implementados discos muy rápidos, de alto RPM y discos más pequeños dentro de ellos, para proporcionar el aumento de velocidad requerido para hacer que MySQL sea rápido. Debido a esto, hay un espacio limitado disponible en los servidores MySQL y DreamHost alienta a sus clientes a desarrollar bases de datos tan pequeñas y eficientes como sea posible. Para obtener información específica sobre el tamaño de las bases de datos permitidas en un servidor MySQL DreamHost específico, revisa la Política Ilimitada y pónte en contacto con el servicio de soporte si tienes más preguntas.
