Visión general

Es posible que, al intentar visitar su página, se encuentre con un error que dice: "Página web no encontrada":

Este artículo le explicará las causas probables de que usted esté viendo este error, y cómo resolverlo.

La causa subyacente del error

El error de "página web no encontrada" significa que la dirección IP que su dominio está usando no está orientada hacia la IP correcta.

De manera más técnica, el error "página web no encontrada" significa que los registros de DNS para su dominio se están resolviendo hacia la dirección IP de un servidor web Apache que no está configurado para su dominio.

Cuando esto ocurre, el servidor web Apache está diseñado para simplemente mostrar el primer dominio listado en su archivo de configuración. Para garantizar que nunca ocurra que la página web de alguien muestre accidentalmente otra página cualquiera que esté alojada en el mismo servicio Apache, DreamHost deliberadamente hace que el primer dominio en todos sus archivos de configuración muestre la misma página de error personalizada.

Existen algunas razones, que se explican a continuación, por las cuales su IP pudiera no estar dirigida correctamente hacia la IP de su servidor web.

Ver la dirección IP de su página en su navegador

Debido a la forma en la que DreamHost configura Apache, a menos que usted haya comprado una dirección IP única para su dominio, el intentar ver su página web introduciendo la dirección IP en su navegador arrojará automáticamente el error de Página web no encontrada. Esto se debe a que usted estaría viendo la IP compartida de su servidor web, en vez de la IP de su página.

Si usted necesariamente debe ver su página a través de una dirección IP, deberá adquirir una IP única. Puede leer más acerca de las IP únicas aquí:

  • IP única

Cambios en el DNS

Es frecuente encontrar este error debido a un cambio en su dirección IP, que suele ser desencadenado por la eliminación del alojamiento (hosting) web para un dominio, seguido por la configuración de alguno de los siguientes: MIRROR, REDIRECT, PARK, o CLOAK. Si bien los cambios en el DNS de DreamHost son casi instantáneos, la nueva información no llega a los demás servidores en distintas partes del mundo hasta que los otros servidores hagan caducar los datos antiguos que aún contienen. A esto se le conoce como propagación de DNS.

El motivo por el cual usted recibe este error durante este período es que la información de DNS que usted está recibiendo en su computador local está desactualizada, y que la instancia de Apache a la cual se le está (incorrectamente) dirigiendo ya no cuenta con ningún registro de su página. Dependiendo de su ubicación, los cambios en el DNS pueden tardar algunas horas en propagarse, así que lo mejor que se puede hacer es dejar su dominio tal y como está durante algunas horas antes de intentar cualquier tipo de arreglo.

Adicionalmente, si usted se dirige a (Panel > 'Domains' > 'Manage Domains') y hace clic en el pequeño enlace "DNS" debajo del dominio, le aparecerá una página de DNS.

Al hacer clic en el botón azul Go! (¡Vamos!) en la parte superior, obligará al sistema de DreamHost a hacer una actualización de DNS. Pero, aun así, la propagación de DNS por internet puede tardar algunas horas en completarse.

Cómo consultar sus registros de DNS actuales

Para consultar el estado de su actualización de DNS, puede visitar ¿Cuál es mi DNS? para ver hacia cuál dirección IP se resuelve su dominio actualmente.

Si usa Linux o Apple Mac

Si usted usa un computador Linux o Mac, también puede ejecutar algunos comandos en su terminal para consultar la IP actual de su página. Visite el artículo de SSH acerca de cómo iniciar sesión en su terminal:

  • Resumen de SSH

Una vez que haya iniciado sesión en su terminal, puede ejecutar el comando "dig" para verificar el estado de la resolución de IP. Por ejemplo, este comando comprueba la dirección IP actual que se resuelve por internet en su ubicación. (Su IP será diferente dependiendo del servidor en el cual esté ubicada su página):

[server]$ dig +short example.com
64.90.47.165

Este comando consulta la dirección IP asignada actualmente a su servidor web de DreamHost. (Su IP será diferente dependiendo del servidor en el cual esté ubicada su página.)

[server]$ dig +short example.com @ns1.DreamHost.com
64.90.47.165

Si example.com está usando servidores de nombres de DreamHost, ambos comandos deberían arrojar la misma IP. Si no, intente purgar su DNS local. Puede leer más acerca de esto en el siguiente artículo:

  • Resumen de Purga de DNS

Si usa Windows

Los usuarios de Windows pueden usar el comando "nslookup" en el símbolo de sistema. Puede leer más acerca del uso del símbolo de sistema en Windows aquí:

  • Microsoft Windows 7 help

Ejemplo:

C:\Users\user> nslookup example.com
Nombre: example.com
Dirección: 64.90.47.165

Este comando comprueba la dirección IP actual que se resuelve por internet en su ubicación. (Su IP será diferente dependiendo del servidor en el cual esté ubicada su página.)

C:\Users\user> nslookup example.com ns1.DreamHost.com
Nombre: example.com
Dirección: 64.90.47.165

Este comando consulta la dirección IP asignada actualmente a su servidor de nombres de DreamHost. (Su IP será diferente dependiendo del servidor en el cual esté ubicada su página.)

Si example.com está usando servidores de nombres de DreamHost, ambos comandos deberían arrojar la misma IP. Si no, intente purgar su DNS local. Puede leer más acerca de esto en el siguiente artículo:

  • Resumen de Purga de DNS

Cómo resolver el problema

Existen varias maneras de resolver este problema.

Si el problema surge poco después de hacer un cambio de configuración para el dominio en el panel de control, la causa podría ser este cambio de dirección IP para el dominio. Si ese es el caso, este error debería desaparecer en unas 4 horas ─ una vez que el cambio de DNS se propague por la internet. Esto es normal. Si no desaparece después de unas 6 horas, entonces pruebe con lo siguiente.

Reconstrucción del httpd mediante el panel de control de DreamHost

Vaya a (Panel > 'Domains' > 'Manage Domains') en su panel, y haga clic en el botón Editar a la derecha del nombre de dominio, en la columna "Alojamiento web". No hará falca cambiar nada en esta página. Simplemente guarde los cambios presionando el botón azul Cambiar configuración. Después, espere hasta 10 minutos para ver si el problema se ha resuelto. Con esta solución, usted sencillamente habrá reconstruido la configuración httpd del servidor web.

Si usted no está usando el DNS de DreamHost

Si usted no está usando DreamHost para administrar el DNS de su dominio, lo más probable es que la dirección IP de su proveedor de DNS que indica hacia DreamHost sea incorrecta.

Usted puede averiguar la IP correcta haciendo una consulta al servidor de nombres de DreamHost con uno de los comandos en la sección anterior (dependiendo de su sistema operativo).

Aquí hay otra forma de averiguar la IP correcta:

  1. Inicie sesión en su panel en (Panel > 'Domains' > 'Manage Domains').
  2. Haga clic en el enlace "DNS" ubicado debajo su dominio.
  3. Cuando se abra la página de Administrar dominios, busque el registro A en la sección "no editable".
  4. Actualice su registro A con su proveedor de DNS para apuntar hacia esta IP de DreamHost.

Una vez que la DNS se haya propagado, no debería seguir apareciendo el robot con sueño de DreamHost.

Uso de cuentas suspendidas

Para cuentas que se hayan suspendido por falta de pago, DreamHost dejará de servir el DNS para la cuenta. Sus páginas mostrarán el mensaje de "página web no encontrada" hasta que se haya ejecutado el pago, y la cuenta se haya reactivado. Inicie sesión en su panel de control o consulte sus correos electrónicos de DreamHost para ver si este es el caso.

Contactar al soporte técnico

Si usted no puede resolver este problema con las sugerencias antes mencionadas, por favor contacte a nuestro departamento de soporte técnico para recibir asistencia.

Usted puede contactar a soporte técnico a través de su panel en (Panel > ‘Support’ > ‘Contact Support’). O consulte el siguiente artículo para otros métodos de contacto:

Usar Passenger WSGI activada

Este problema, o uno similar, se puede manifestar ocasionalmente en dominios que usan Passenger WSGI. En este caso, el error aparece de manera diferente, pues la página de error en el servidor no se muestra.

Los síntomas pueden incluir:

  • Otros dominios en la misma cuenta y servidor funcionan correctamente.
  • Se superan los límites de tiempo para acceder a la página.
  • No aparecen procesos de Passenger cuando se ejecuta ps -A desde el intérprete de comandos.

En esta instancia, la solución es la misma – editar el dominio y hacer clic en Cambiar Configuración. Y después esperar que se propague. También puede probar a desactivar y reactivar el Passenger en la misma página de editar dominios.

Esto cambiará las configuraciones del dominio sin alterar los registros de DNS de este.

Esto NO funciona para dominios encubiertos (con "cloaking"), porque el servicio de "cloaking" está en una máquina especial, y el DNS SIEMPRE tendrá que cambiar al activar/desactivar dominios encubiertos.
