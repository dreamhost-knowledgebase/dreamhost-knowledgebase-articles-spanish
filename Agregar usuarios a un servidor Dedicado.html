Generalidades

Este artículo explica cómo agregar un usuario administrador a tu servidor Dedicado. Este usuario es diferente de un usuario SFTP o Shell (SSH) normal ya que puede ejecutar comandos root/sudo en el servidor, lo que permite un mayor control sobre la personalización de las configuraciones del sistema o la instalación de software.

Una nota importante sobre los cambios personalizados de los usuarios Administradores

Con un usuario administrador, puedes hacer cambios de root/sudo en tu servidor, pero ten en cuenta que DreamHost aún administra tu servidor Dedicado. Esto significa que si haces cambios personalizados en Apache, se sobrescribirán cada vez que se ejecute una configuración de Apache. Esto puede suceder al agregar alojamiento a un dominio o al agregar un certificado SSL.

Si haces cambios personalizados, asegúrate de anotar claramente cuáles fueron para que pueda corregir rápidamente cualquier cosa que se haya sobrescrito.

Agregar un usuario administrador

Para agregar un usuario administrador:

  1. Ve a la página de Usuarios Administradores en tu panel.
  2. En la parte superior derecha, da clic en el botón Crear Usuario.
    Aparece un panel de 'Crear Usuario' en la parte derecha:
  3. Ingresa lo siguiente:
    • Nombre de Usuario — Un nombre de usuario para el nuevo usuario.
    • Servidor — Selecciona el servidor Dedicado del menú desplegable.
    • Contraseña — Puedes ingresar tu propia contraseña o dar clic en el botón Generar contraseña segura para permitir que el panel cree una por tí.
    • Acceso Shell Seguro (SSH) — Selecciona el tipo de Shell que deseas usar. Revisa el artículo tipos de Shell para más información.
    • Términos de uso del usuario administrador — Lee los términos de uso, luego revisa la casilla llamada He leído y acepto los términos de uso.
  4. Da clic en el botón Crear Usuario.
    Aparece entonces el usuario en la página Usuarios Administradores, señalado por un icono.

Asignar un usuario administrador a un dominio

Cuando se crea un usuario administrador, no se asigna a un dominio en específico. Esto es porque el propósito del usuario administrador es iniciar sesión directamente en el servidor para correr los comandos sudo/root. Si intentas agregar el usuario administrador a un dominio en un servidor Dedicado, aparece el siguiente error:

Cambios que no se pueden hacer.

Los usuarios Administradores no se pueden asignar a un sitio web.

Si necesitas asignar un usuario administrador específico a un dominio, primero debes revocar los privilegios de administrador como se describe a continuación.

¿Qué sucede si mi anterior usuario administrador se asigna a un dominio?

Los dominios que ya tienen asignado un usuario administrador seguirán funcionando normalmente con el mismo usuario.

¿Qué sucede si cambio mi antiguo usuario administrador asignado a un dominio?

Si se cambia el usuario administrador anterior asignado a un dominio, no se puede volver a asignar en el futuro, a menos que primero se revoque el acceso de administrador.

¿Puedo cambiar un usuario no administrador a un usuario administrador?

Un usuario ya asignado a un dominio no se puede cambiar a un usuario administrador Dedicado. Se debe crear un nuevo usuario administrador.

 

Revocar los privilegios de usuario administrador

Para revocar los privilegios de usuario administrador:

  1. Navega a la páginaUsuarios & Archivos SFTP en tu panel.
  2. Pasa el cursor sobr eel icono a la derecha de tu usuario administrador.
    Aparece una ventana emergente.
  3. Da clic en el interruptor para eliminar los privilegios de usuario.
