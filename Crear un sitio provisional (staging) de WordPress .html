Generalidades

Un sitio web provisional es un sitio de prueba que puedes usar para probar los cambios que te gustaría ver en tu sitio web en vivo. Por ejemplo, podrías probar nuevos temas, plugins y actualizaciones para ver si algo afecta negativamente la funcionalidad de tu sitio. Esto te permite trabajar en el sitio provisional sin afectar el sitio web en vivo.

Diferencias entre un sitio provisional de WordPress o DreamPress

¿Aún no tienes DreamPress?

Para actualizar tu sitio WordPress a DreamPress, por favor visita el siguiente artículo:

  • Actualizar un sitio de WordPress a DreamPress

DreamPress es el alojamiento administrado de WordPress de DreamHost. Es un servicio premium con varias características adicionales sobre el alojamiento estándar de WordPress.

DreamPress ya incluye la capacidad de provisionar tu sitio web, por lo que las instrucciones en este artículo no son necesarias. Consulta los siguientes artículos para obtener más información sobre el uso de sitio provisional con DreamPress:

  • Como hacer un sitio provisional (staging) en DreamPress
  • Copiar tu sitio DreamPress a tu sitio provisional para probar los cambios
  • Publicar tus cambios de tu sitio provisional a tu sitio DreamPress en vivo

Con tu instalación de WordPress de DreamHost estándar, deberás crear manualmente el sitio de provisional.

Crear el sitio de WordPress provisional

Estas instrucciones asumen que has instalado WordPress manualmente o usando el instalador de WordPress de DreamHost. Esto también supone que tu sitio de WordPress está en un alojamiento Shared, VPS o Dedicado.

Para el resto de esta guía, se supone que tu sitio web es example.com y el sitio provisional es example.dreamhosters.com.

Crear un subdominio dreamhosters.com

Para provisionar un sitio WordPress, necesitarás hacer un duplicado manualmente creando un nuevo subdomio dreamhosters.com.

Puedes llamar a este nuevo subdominio dreamhosters.com como quieras.

El DNS puede tardar unas horas en actualizarse, pero una vez que lo haga, deberías ver una página de 'Coming Soon'. Sigue las instrucciones en el siguiente artículo para copiar el contenido de example.com a example.dreamhosters.com mientras actualizas las URLs en la base de datos. Esto es muy importante ya que cada base de datos de WordPress tiene URLs codificadas que necesitan coincidir con el nombre de tu sitio web.

  • Cómo mover WordPress a un dominio diferente en DreamHost

Instalar y probar

En este punto, tu sitio example.dreamhosters.com debería ser una copia exacta de example.com. Ahora puedes instalar y probar todo lo que quieras.

Cuando hayas configurado tu sitio example.dreamhosters.com exactamente como desees, es momento de mover esos datos de vuelta a tu sitio en vivo en example.com

Mover el sitio provisional nuevamente al sitio en vivo

Para mover tu sitio provisional a tu sitio en vivo, puedes repetir el paso que realizaste anteriormente para copiar el sitio:

  • Cómo mover WordPress a un dominio diferente en DreamHost

El artículo de arriba te mostrará cómo volver a importar los datos a tu sitio web en vivo y cambiar las URL en la base de datos.

Una vez completado, tu sitio web en vivo se actualizará con el contenido de tu sitio provisional.
