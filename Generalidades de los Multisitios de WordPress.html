Generalidades

Los Multisitios de WordPress es una característica de WordPress que te permite crear múltiples sitios virtuales que comparten una sola instalación de WordPress. Esto permite a los usuarios crear sus propios sitios como subdominios de su sitio principal. Por ejemplo:

  • example.com/site1
  • example.com/site2
  • example.com/site3

También puedes configurar tu multisitio para permitir que solo el administrador cree nuevos sitios. Esto puede resultar útil para una empresa con diferentes departamentos. Por ejemplo:

  • example.com/sales
  • example.com/products
  • example.com/support
  • El soporte de Multisitios de WordPress NO ES PROVISIONADO POR DREAMHOST.
  • Si algo en tu instalación de varios sitios no funciona, debes solucionarlo tú mismo o contratar a un desarrollador.
  • Multisitio no es para principiantes de WordPress. Ya deberías saber cómo usar WordPress antes de considerar el uso de Multisitios.
  • Los usuarios deben tener conocimientos básicos de PHP, MySQL, cómo editar archivos en el servidor y cómo revisar los registros de errores en caso de que debas solucionarlos tú mismo.

¿Deberías usar Multisitios de WordPress?

Mira el artículo antes de crear una red de WordPress para más información.

Cuando NO usar Multisitios

Los Multisitios no son una buena opción si quieres:

  • Varios usuarios para colaborar en un solo proyecto.
  • Varios usuarios para bloguear juntos.
  • Un sitio donde se comparten datos o usuarios.
  • Diferentes usuarios para ver diferentes páginas (se puede usar un plugin en su lugar).
  • Diferentes páginas para que se vean diferentes (se puede usar un plugin en su lugar).

Cuando usar Multisitios

Los Multisistios puede ser una buena opción si tu:

  • Quieres crear sitios virtuales que sean independientes entre sí.
  • Quieres que cada usuario tenga control sobre cómo se presentan sus datos.
  • Quieres permitir a los usuarios crear su propio sitio.
  • Eres competente en PHP, MySQL y revisas los registros en busca de errores.
  • Te sientes cómodo en brindar soporte a (potencialmente) muchos usuarios.
  • Quieres dar a cada blog un subdominio o subdirectorio.
  • Deseas ser el único administrador de red sobre todos los sitios en una red.
  • Deseas actualizar todos los sitios en un solo paso.

Tipos de Multisitios

Hay dos formas en las que puedes configurar tu Multisitio:

  • Subdominios, o
  • Subdirectorios.

Subdirectorios en Multisitios

Esta opción crea subdirectorios en el siguiente formato:

  • example.com/site1
  • example.com/site2
  • example.com/site3

Subdominios en Multisite

Esta opción crea subdominios en el siguiente formato:

  • site1.example.com
  • site2.example.com
  • site3.example.com
  • La opción de subdominio solo está disponible en un Servidor VPS o Dedicado. Esto es por que el DNS de Wildcard es necesario para esta configuración. Si deseas utilizar Multisitios en un servidor compartido, debes usar la opción de subdirectorio.
  • NO uses el panel para ingresar un asterisco * como un registro CNAME. Esto no funciona y causará problemas en el futuro.

Tercera opción para usuarios avanzados

También es posible hacer un para separar un dominio de tu red. Por ejemplo, en vez de tener example.com y example.com/site1 tienes example.com y site1example.com.

Para hacer esto, necesitas un plugin llamado WordPress MU Domain Mapping.

Solución de Problemas

  • Si esto sale mal, trata de buscar tu problema en el Foro de Soporte para Multisitios de WordPress. Si no puedes encontrar una respuesta, pregúntala allí. Asegúrate de anotar cualquier información de depuración antes de publicar en los foros.
  • Asegúrate de haber creado el dominio para que funcione sin 'www', lo cual puedes hacer en la página de Administrar Dominios.
  • Asegúrate de instalar un software sin el https://www en el nombre de dominio si estás usando subdominios.
  • ¿Has creado accidentalmente una instalación de subdominio (en un VPS o Servidor Dedicado sin intención y sin haber contactado a soporte para crear un wildcard? Si es así, vuelca tu base de datos, e inicia de nuevo. O, contacta a soporte y pregunta por los wildcards para ponerlos en su lugar.
  • Si tiene problemas, el primer lugar para buscar es dentro del registro de errores del servidor.
  • Asegúrate de buscar los foros con palabras clave que cubren tu problema. Es probable que alguien ya se haya encontrado con el problema que tienes.
  • Si estás usando domain mapping, necesitas configurar una IP Única; si no, Apache no reconocerá el requerimiento..
