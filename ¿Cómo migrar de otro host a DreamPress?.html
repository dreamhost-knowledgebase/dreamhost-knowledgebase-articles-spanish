Oferta de migración gratuita

Si estás migrando un sitio de WordPress desde otra empresa de alojamiento, DreamHost transferirá tu sitio GRATIS a un plan de DreamPress. Si deseas que DreamHost avance con la migración de tu sitio, comunícate con el soporte con el nombre de dominio que deseas migrar.

  • Contactar a soporte — futuros clientes
  • Contactar a soporte — clientes actuales

Su boleto se pasará al equipo de Servicios Profesionales para que pueda comenzar.

Los siguientes pasos te guían a través de cómo migrar manualmente un sitio de WordPress existente.

Migrando

  1. Exporta las bases de datos MySQL de WordPress. Consulta el siguiente artículo para obtener más instrucciones:
    • Copias de seguridad de MySQL
  2. Descarga toda tu instalación de WordPress. Consulta el siguiente artículo para obtener más instrucciones:
    • FTP
  3. Ve el artículo Configuración de un nuevo dominio para agregar su sitio a DreamPress.
  4. Puedes importar tu base de datos a través de phpMyAdmin o mediante un usuario de SHELL.
    • Opción #1 — Importar mediante phpMyAdmin: ve el siguiente artículo que explica cómo importar utilizando phpMyAdmin:
    • Importar tu base de datos vía phpMyAdmin
    • Opción #2 — Importar a través de SSH: ve el siguiente artículo que explica cómo importar utilizando SSH:
    • Importar tu base de datos vía SSH
    • Opción #3 — Importar usando wp-cli
    • La herramienta wp-cli es útil para el volcado e importación de MySQL. Puedes leer sobre esta herramienta en el siguiente artículo:
      • WordPress wp-cli
      • Usar la herramienta wp-cli es el método más rápido.
      • Para usar la herramienta wp-cli, deberás asegurarte de que tu usuario de DreamPress tenga acceso a Shell.
      • También puedes visitar los siguientes artículos para obtener más instrucciones sobre cómo cambiar el tipo de usuario y cómo iniciar sesión a través de SSH:
        • Creación de una usuario con acceso Shell (SSH)
        • SSH

      espués de conectarte a través de SSH con tu usuario de DreamPress, ejecuta el siguiente comando (reemplazando "backup.sql" con el nombre de la copia de seguridad MySQL que cargaste anteriormente):

      [server]$ wp db import backup.sql
  5. Importa cualquier archivo de WordPress necesario (como tu tema personalizado, imágenes y archivo wp-config.php) a través de FTP.

La base de datos ahora se importa y tu archivo wp-config.php se vincula a la nueva instalación.

Solución de problemas de la nueva importación

Si tu sitio no se muestra como debería en este momento, es posible que debas vaciar tanto el caché de objetos como el caché de DreamPress. Ve los siguientes artículos para más detalles:

  • Limpiar el caché de DreamPress
  • Vaciar el caché de objetos usando 'wp cache flush'

A continuación hay ejemplos sobre cómo vaciar ambos vía SSH:

Vacía el caché DreamPress para todos tus archivos de tema

[server]$ curl -X PURGE "http://example.com/wp-content/themes/.*"

Vacía el caché de objetos utilizando WordPress wp-cli

[server]$ wp cache flush
