Generalidades

Las instancias de DreamCompute se crean con llaves par SSH especificas que se utilizan para obtener acceso inicial. Aquí hay un par de recomendaciones para evitar que se bloquee tu instancia de DreamCompute. 

El soporte de DreamHost no tiene acceso a tu instancia de DreamCompute. Se recomienda que sigas los pasos del artículo Cómo evitar quedarse fuera de las instancias de DreamCompute para evitar quedar bloqueado en tu instancia. Si no estableces una contraseña de usuario y tus claves ya no funcionan, no habrá forma de acceder a la instancia.

Si necesitas el soporte de DreamHost para solucionar problemas de tu instancia, debes proporcionar las credenciales de inicio de sesión para la instancia. Consulta el siguiente artículo para obtener más detalles.

  • Compartir contraseñas con el soporte de DreamHost

Establecer una contraseña para el usuario predeterminado

De forma predeterminada, el archivo de configuración ssh (/etc/ssh/sshd_config) no permite la autenticación de contraseña, por lo que se recomienda establecer una contraseña segura para tu usuario predeterminado. Si quedas bloqueado y no puedes iniciar sesión con las llaves par, aún puedes iniciar sesión utilizando el panel de DreamCompute. Una vez que haya iniciado sesión, puede ejecutar sudo para solucionar posibles problemas, establecer una nueva clave ssh, etc. Consulta el siguiente artículo para obtener instrucciones sobre cómo crear una contraseña para tu usuario predeterminado:

  • Establecer una contraseña para el usuario predeterminado

Crea tus propios usuarios

No es necesario utilizar el nombre de usuario predeterminado proporcionado por el sistema operativo. Una vez que hayas iniciado sesión con el usuario predeterminado, se pueden crear usuarios adicionales.


El usuario predeterminado y tu clave SSH pueden ser una opción de acceso de respaldo, en lugar de la única opción de inicio de sesión.

Hacer copias de seguridad

Al ser un servicio no administrado, el cliente es responsable de sus propios datos. Si hay datos importantes, archivos de configuración, etc. que desea mantener a salvo, por favor considera instalar un cliente CLI de S3 como s3cmd, awscli, u otra herramienta de la página de aplicaciones de DreamObjects.
