Generalidades

DreamCompute es el servicio de computación en la nube de DreamHost. Hay muchas cosas para las que puedes usar DreamCompute, desde la creación de servidores en la nube simples hasta configuraciones de red avanzadas.

Este artículo enumera muchos términos comunes que encontrarás al trabajar con DreamCompute.

Términos comunes de DreamCompute 

Término Definición
Bloque de Almacenamiento El espacio de almacenamiento compartido que existe junto con tus instancias de DreamCompute. Tu cuota de bloque de almacenamiento se usa al crear volúmenes, que luego se pueden adjuntar a tus instancias.
Panel Esta es la interfaz del navegador web que utilizas para interactuar con tu servicio DreamCompute. Se puede ver en la siguiente dirección:
  • Panel de DreamCompute
Usuario predeterminado El usuario creado automáticamente con tu nueva instancia. Puedes utilizar este usuario para iniciar sesión y administrar tu instancia.
Dirección IP flotante La dirección IP pública asignada a una instancia en una red privada. Esta dirección IP flotante pública se utiliza para comunicarse con Internet. Las direcciones IP flotantes también se usan solo con Redes Privadas.
Imagen Una plantilla para crear una nueva instancia. Puedes elegir varias imágenes pre configuradas al crear una instancia como 'Ubuntu', 'Debian' y 'CentOS'. También puedes cargar tu propia imagen para crear la instancia.
Instancia Las instancias son servidores virtuales en la Nube de DreamCompute.
Lanzamiento de una instancia El proceso de creación de un servidor virtual en la Nube de DreamCompute.
Archivo openrc Un pequeño script bash que establece variables de entorno para que las herramientas de línea de comandos de OpenStack sepan cómo comunicarse con DreamCompute.
CLI OpenStack  Una herramienta de línea de comandos para ayudarte a administrar tu instancia de DreamCompute.
Red Privada Tipo de red en la que varias instancias pueden comunicarse entre sí y acceder a Internet a través de un enrutador. Estas instancias no son accesibles para el resto de Internet, lo que lo convierte en un entorno seguro.
Proyecto Un proyecto es un grupo de cero o más usuarios. En DreamCompute, los clientes tienen acceso a un solo usuario y a un solo proyecto.
Cuota Cada proyecto de DreamCompute viene con un conjunto de límites operativos. Estos límites restringen aspectos como la cantidad de instancias que puedes tener y la cantidad de almacenamiento en bloque de volumen que puedes usar.
Grupos de seguridad Conjuntos de reglas de filtrado de IP que definen el acceso a la red. Por ejemplo, puedes crear un grupo de seguridad para permitir el acceso de puertos específicos a tu instancia.
Llaves SSH DreamCompute utiliza llaves SSH para acceder a tu instancia. Esta Llave de emparejamiento se inyectan en la instancia cuando se lanzan originalmente. Este es un método más seguro para acceder a tu instancia en lugar de usuarios y contraseñas SSH normales.
