Herramientas de red

Revisar los registros DNS de tu dominio

A continuación se explica cómo verificar los registros A de tu dominio para confirmar hacia dónde apuntan actualmente.

Usando Linux o Mac

Corre el comando dig para verificar la dirección IP actual de tu dominio resolviendo en tu ubicación.

[server]$ dig +short example.com

Corre el siguiente comando para verificar la dirección IP de tu dominio que se resuelve en el nameserver de DreamHost.

[server]$ dig +short example.com @ns1.dreamhost.com

Si example.com está usando los nameservers de DreamHost, ambos comandos deberían aparecer en la misma dirección IP. Si no, intenta Limpiar el DNS local.

Usando Windows

Corre el comando nslookup en tu prompt del sistema para verificar la dirección IP de tu dominio que se resuelve en tu ubicación.

C:\Users\user> nslookup example.com

Correo el siguiente comando para verificar la dirección IP de tu dominio que se resuelve en el nameserver de DreamHost.

C:\Users\user> nslookup example.com ns1.dreamhost.com

Si example.com está usando los nameservers de DreamHost, ambos comandos deberían aparecer en la misma dirección IP. Si no, intenta Limpiar el DNS local

Herramientas de línea de comando

También hay varias herramientas de línea de comandos que puedes usar para obtener información sobre un dominio o las condiciones de la red. Revisa el artículo sobre SSH para obtener detalles sobre cómo iniciar sesión en tu servidor.

A continuación se incluye una lista de ejemplos básicos. Asegúrese de leer los enlaces en cada sección para obtener más detalles.

host

El comando host convierte nombres en direcciones IP y viceversa.

Corre host en example.com para encontrar sus registros IP y MX:

[server]$ host example.com

Corre host en la IP para encontrar su nombre de dominio:

[server]$ host 173.236.241.100

nslookup

Buscador de nameserver interactivo.

  • nslookup en Windows
  • nslookup in Linux

WHOIS

Una consulta WHOIS que muestra detalles sobre el registro de un dominio.

[server]$ whois example.com

dig

El comando dig revisa varios registros de DNS.

Buscador de registros IPv4

[server]$ dig +short example.com

Buscador de registros IPv6

[server]$ dig +short AAAA example.com

Buscador de registros de Texto

[server]$ dig TXT example.com

Buscador de registros CNAME. Asegúrate de ingresar el registro CNAME correcto antes del dominio:

[server]$ dig CNAME abcdefg.example.com

Buscador de registros SRV. Asegúrate de ingresar el registro SRV correcto antes del dominio:

[server]$ dig SRV _sip.example.com

ping

ping & ping6 son usados para recolectar resultados de ping.

traceroute

traceroute & traceroute6 se utilizan para rastrear la ruta que toman los paquetes de un lugar a otro. Corre traceroute para ver todos los saltos entre tu ubicación y tu sitio web.

mtr

mtr combina la funcionalidad de los programas traceroute y ping en una sola herramienta de diagnóstico de red.

[server]$ mtr dreamhost.com

openssl

openssl es una herramienta de línea de comandos para utilizar varias funciones de la biblioteca de criptografía OpenSSL.

[server]$ openssl s_client -connect example.com:443
