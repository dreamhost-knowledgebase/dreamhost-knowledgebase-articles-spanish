Generalidades

Un Servidor Dedicado es un servidor que está completamente dedicado solo a tus datos. Esto significa que ningún otro cliente de DreamHost tiene acceso o comparte recursos con tus sitios en ese servidor.

DreamHost ofrece varios tipos de planes de alojamiento. Puedes ver los detalles de cada uno junto con una comparación en el artículo alojamiento.

Con todos los demás tipos de alojamiento, tus datos están en una máquina con otros usuarios.

Los Servidores Dedicados son máquinas premium, recomendadas para sitios web de alto rendimiento que requieren recursos dedicados para ejecutarlos de manera eficiente. Para más información, por favor consulta la página de Servidor Dedicado en el sitio web principal.

Características de un Servidor Dedicado

  • Hasta 2TB de almacenamiento en HDD.
  • Almacenamiento RAID: varios discos duros, lo cual significa que si uno falla, el otro se hace cargo, por lo que no se pierden datos.
  • Acceso root.
  • Servidor de base de datos MySQL local con acceso root.
  • Recursos dedicados de RAM y CPU.

Direcciones IP

Cada Servidor Dedicado viene con su propia dirección IP incluida. Luego puedes agregar una IP Única a cada dominio (IPv4 o IPv6) por una tarifa adicional. Puedes leer más sobre IP Únicas aquí:

  • Generalidades sobre IP Única

Ten en cuenta que DreamHost no puede ofrecer direcciones IP estáticas. Esto significa que la IP de tu sitio (incluidas las IP únicas) está sujeta a cambios. Si bien no sucede tan a menudo, DreamHost no garantiza que una dirección IP nunca cambie. Sin embargo, cuando se produce un cambio de IP, se te notificará por correo de manera oportuna.

Usuarios Administradores

Los Servidores Dedicados ofrecen la posibilidad de agregar un usuario administrador. Este usuario administrador tiene acceso root completo al Servidor Dedicado a través del comando sudo. Estos son usuarios especiales con acceso completo al servidor.

Ten cuidado cuando uses un usuario administrador, ya que tendrás la capacidad de alterar la configuración del sistema que podría deshabilitar tus sitios o el servidor. Asegúrate de saber lo que estás haciendo cuando usas un usuario administrador.
