Generalidades

WordPress tiene varios plugins de SMTP que puedes usar para forzar cualquier dirección FROM que desees. Esto es importante por dos motivos principales:

  • Te permite enviar correos directamente a través de tu servidor de correo, en lugar del servidor web. Esto ayuda a evitar problemas con la política de dominio de remitente (SPF) y la suplantación de identidad (Spoofing).
  • Te permite enviar correo electrónico desde otro servidor de correo. Por ejemplo, puedes usar tu dirección de GMAIL como el correo FROM en lugar de tu dirección de DreamHost.

Una opción de plugin popular es WP Mail SMTP. Este artículo te explica como configurar este plugin.

3 maneras diferences de configurar el plugin

Hay tres formas diferentes de configurar este complemento.

  • Opción 1: Usar el Asistente de Configuración para guiarte a través de cada paso
  • Opción 2: Configurarlo en la página de configuraciones
  • Opción 3: Agregar código en el archivo wp-config.php directamente

La tercera opción requiere que edites manualmente el archivo de configuración principal en tu instalación de WordPress. Esto es más avanzado, pero también más seguro. Si tu panel de administración se comparte con otros administradores, esto evitará que accedan a cualquier campo.

Instalar el plugin WP Mail SMTP

Si instalaste WordPress usando el instalador de Un-Clic en el panel

El plugin de WP Mail SMTP se instala automáticamente si instaló WordPress usando el instalador de Un-Clic en el panel a partir de junio de 2021.

  • Cómo instalar WordPress usando el Instalador de Un-Clic

Ahora debes activarlo en tu panel de control.

  1. Navega a la página Plugins > Installed Plugins.
  2. Debajo de WP Mail SMTP, da clic al enlace Activate.

Ahora serás llevado al Asistente de Configuración para completar la configuración.

Si tienes una instalación de Un-Clic antigua.

    1. En el panel de WordPress, haz clic en Plugins > Add New en el panel izquierdo.
    2. Búsca wp-mail-smtp. También se puede encontrar aquí.
    3. Haz clic en el botón Instalar, luego en Activar.
    4. Después de que esté instalado y activado, navega a la página Plugins en tu panel de WordPress.
    5. Haz clic en el enlace Settings debajo del plugin.

Opción 1 — Usar el Asistente de Configuración

El primer ítem en la página Settings es del Asistente de Configuración. Esta es una manera rápida de configurarlo.

      1. Navega a la página WP MAIL SMTP > Settings.
      2. Da clic en el botón Launch Setup Wizard.
      3. Da clic en el botón Let's Get Started.
      4. Selecciona Other SMTP. Da clic en Save and Continue.
      5. La siguiente página enumera todas sus opciones de configuración en diferentes secciones.
      6. Configuraciones SMTP.
        • SMTP Host - Ingresa smtp.dreamhost.com
        • Encryption - TLS
        • SMTP Port - 587
      7. Enable Authentication - prende esta opción
      8. SMTP Username - agrega tu correo de DreamHost
      9. SMTP Password - ingresa la contraseña de tu correo de DreamHost. También puedes restablecerlo si es necesario.
      10. From Name - Ingresa un nombre que te gustaría que aparezca como enviado de REMITENTE
      11. From Email - Ingresa el correo que deseas enviar como REMITENTE
      12. Da clic a Save and Continue.
      13. La siguiente página enumera algunas funciones adicionales que podría habilitar con la versión Pro. Haz clic en Save and Continue.
      14. La siguiente página le permite agregar tu correo para recibir recomendaciones sobre cómo optimizar la entrega de tu correo. Haz clic en Skip this Step o Save and Continue.
      15. La siguiente página te pide que ingreses tu clave de licencia si compraste la Versión Pro.
      16. La página final verifica tu configuración.
      17. Cuando se completa la verificación, se muestra un mensaje de éxito para confirmar que tu correo está configurado.

Opción 2 — Configurarlo en la página de configuraciones

  1. Navega a la página WP MAIL SMTP > Settings.
  2. Rellena los siguientes campos:
    • Correo electrónico remitente - Ingrese el correo electrónico que deseas enviar DESDE
    • Forzar remitente del mensaje - Marca esta casilla
    • Nombre remitente - Ingresa un nombre que te gustaría que aparezca como enviado DESDE
    • Forzar el nombre del remitente - Marque esto si deseas forzar el nombre de FROM
    • Servicio de correo electrónico - Selecciona 'Otro SMTP'
    • Return Path - Marca esta casilla
  3. Debajo de la sección Mailer selecciona Other SMTP.
  4. Debajo de la sección Other SMTP, asegúrate de prender el botón Authentication.
    • Servidor SMTP - smtp.dreamhost.com
    • Cifrado - Usar cifrado TLS
    • Puerto SMTP - 587

      En este ejemplo, se utiliza el puerto 587 con TLS. Si prefieres usar el puerto 465, asegúrate de usar la opción SSL en su lugar.

  • Nombre de usuario de SMTP - Ingresa tu dirección de correo electrónico de DreamHost
  • Contraseña SMTP - Ingresa la contraseña de tu dirección de correo electrónico
  • Haz clic en el botón Save Settings.

Tu formulario ahora se enviará DESDE el correo que ingresaste anteriormente, pero debe agregar el siguiente código para proteger aún más sus credenciales.

Opción 3 - Agregar código a tu archivo wp-config.php

Las instrucciones anteriores agregan tu contraseña al panel de WordPress. Para asegurar aún más tus credenciales, el plugin recomienda que agregues código a tu archivo wp-config.php en su lugar. Dirígete la página siguiente para más detalles.

      • Cómo proteger la configuración de SMTP mediante el uso de constantes

Si elijes esta opción, no necesitasingresar ninguna credencial en tu panel de WordPress. Se actualizarán automáticamente cuando los agregas a tu archivo wp-config.php.

Agregar las constantes a tu archivo wp-config.php

  1. Inicia sesión en tu servidor a través de FTP o SSH.
  2. Utiliza uno de los siguientes artículos para editar tu archivo wp-config.php.
    • Crear y editar archivos a través de FTP
    • Crear y editar un archivo a través de SSH
  3. Desplázate hacia abajo en este archivo hasta la siguiente línea:

    /* That's all, stop editing! Happy blogging. */

  4. Agrega las siguientes líneas justo encima de esta línea.
    // turn constants on        
define( 'WPMS_ON', true );

    // general configuration settings
define( 'WPMS_MAIL_FROM', 'email@example.com' );
define( 'WPMS_MAIL_FROM_FORCE', true ); // True turns it on, false turns it off.
define( 'WPMS_MAIL_FROM_NAME', 'Example Name' );
define( 'WPMS_MAIL_FROM_NAME_FORCE', true ); // True turns it on, false turns it off.
define( 'WPMS_SET_RETURN_PATH', true ); // Sets $phpmailer->Sender if true.

    // specific configuration settings
define( 'WPMS_SMTP_HOST', 'smtp.dreamhost.com' ); 
define( 'WPMS_SMTP_PORT', 587 ); 
define( 'WPMS_SSL', 'tls' ); 
define( 'WPMS_SMTP_AUTH', true ); 
define( 'WPMS_SMTP_USER', 'email@example.com' );
define( 'WPMS_SMTP_PASS', '********' );
define( 'WPMS_SMTP_AUTOTLS', true );
define( 'WPMS_MAILER', 'smtp' );
  5. La unica parque que necesitas cambiar es WPMS_MAIL_FROM, WPMS_MAIL_FROM_NAME, y WPMS_SMTP_PASS.
  6. Guarda el archivo
  7. Si ves la página del plugin ahora, verás que las credenciales no son accesibles.

Probar el plugin

Desplázate hasta la parte superior y haz clic en la pestaña Correo de prueba. Haz clic en el botón Enviar correo. Deberías ver que se envió con éxito.
