Generalidades

WordPress tiene varios plugins de SMTP que puedes usar para forzar cualquier dirección de correo que desees. Esto es importante por dos motivos principales:

  • Te permite enviar correos directamente a través de tu servidor de correo, en lugar del servidor web. Esto ayuda a evitar problemas con la política de dominio de remitente (SPF) y la suplantación de identidad (Spoofing).
  • Te permite enviar correo electrónico desde otro servidor de correo. Por ejemplo, puedes usar tu dirección de GMAIL como el correo FROM en lugar de tu dirección de DreamHost.

Una opción de plugin popular es WP Mail SMTP. Este artículo te explica como configurar este plugin.

Este artículo explica cómo configurar este plugin usando una dirección de correo de DreamHost creada en el panel usando el siguiente artículo:

Correo de Google

Si quieres usar una dirección Gmail para enviar desde tu sitio WordPress, revisa el siguiente artículo en vez.

Dos opciones para configurar el plugin

Existen dos opciones para configurar este plugin.

  • Opción 1: Usar el Asistente de Configuración para guiarte a través de cada paso
  • Opción 2: Configurarlo en la página de ajustes

Agregar seguridad extra

La sección final es muy recomendable ya que protege aún más tus credenciales de correo electrónico. Este paso adicional requiere que edites manualmente el archivo de configuración principal en tu instalación de WordPress.

Esto es más avanzado pero también más seguro. Si tu panel de administración se comparte con otros administradores, esto les impedirá acceder a cualquier campo.

 

Instalar el plugin WP Mail SMTP

Si instalaste WordPress usando el instalador de WordPress de DreamHost en el panel

El plugin de WordPress Mail SMTP se instala automáticamente cuando instalas WordPress usando el instalador de WordPress de DreamHost en el panel.

  • Instalar WordPress usando el instalador de WordPress de DreamHost

Ahora debes activarlo en tu panel de control. Navega a la página Plugins > Plugins instalados y da clic en el enlace Activar debajo de WP Mail SMTP.

Ahora serás llevado al Asistente de Configuración para completar la configuración.

Si tienes una instalación de WordPress de DreamHost antigua

Si instalaste WordPress usando el instalador de WordPress de DreamHost en el panel antes de Junio de 2021, debes instalar el plugin WP Mail SMTP manualmente.

  1. En el panel de WordPress, navega a Plugins > Añadir nuevo en el panel izquierdo.
  2. Búsca wp-mail-smtp. También se puede encontrar aquí.
  3. Haz clic en el botón Instalar, luego en Activar.

    Luego de dar clic en Activar, se abre el Asistente de Instalación. También puedes navegar a la página Plugins en el panel de WordPress y dar clic al enlace Ajustes debajo del plugin.

 

Opción 1 — Usar el Asistente de Configuración

El Asistente de Configuración es una manera rápida de configurar tus ajustes. Si el Asistente de Configuración no se abre durante la instalación, puedes abrirlo en la página de configuraciones del plugin.

  1. Navega a la página WP MAIL SMTP.
  2. Da clic en el botón Lanzar el asistente de configuración.
  3. Da clic en el botón Empecemos.
    Aparece una lista de SMTP Mailers.
  4. Selecciona Otro SMTP. Da clic en Guardar y Continuar.
    La siguiente página enumera todas sus opciones de configuración en diferentes secciones.
  5. Ingresa las siguientes configuraciones de SMTP.
    • Servidor SMTP - Ingresa smtp.dreamhost.com
    • Cifrado - TLS
    • Puerto SMTP - 587
    • Activar autenticación - activa esta opción
    • Nombre de usuario de SMTP - agrega tu correo de DreamHost
    • Contraseña SMTP - ingresa la contraseña de tu correo de DreamHost. También puedes restablecerlo si es necesario.
    • Nombre remitente - Ingresa un nombre que te gustaría que aparezca como REMITENTE
    • Correo electrónico remitente - Ingresa el correo que deseas enviar como REMITENTE
  6. Haz clic en Guardar y Continuar.
    La siguiente página muestra algunas herramientas adicionales que puedes habilitar con la versión Pro.
  7. Da clic en Saltar este Paso o Guardar y Continuar.
    Las siguiente página te permite agregar tu correo para recibir recomendaciones sobre cómo optimizar la entrega del mismo.
  8. Da clic en Saltar este Paso o Guardar y Continuar.
    La siguiente página te pide ingresar tu llave de licencia si compraste la versión Pro.
  9. Da clic en Saltar este Paso o Guardar y Continuar.
    La página final revisa la configuración. Cuando la revisión se complete, se muestra un mensaje exitoso confirmando que tu correo se ha configurado.

Es posible que veas una advertencia de DMARC después de la prueba. Tu correo seguirá funcionando sin él, pero se recomienda que configures DMARC para autenticarLO aún más. Revisa lo siguiente para obtener instrucciones sobre cómo configurar esto en tu dominio.

  • Crear una política de DMARC
 

Opción 2 — Configurar manualmente en la página de ajustes

  1. Navega a la página WP MAIL SMTP > General.
  2. Rellena los siguientes campos:
    • Correo electrónico remitente - Ingresa el correo del REMITENTE
    • Forzar remitente del mensaje - Marca esta casilla
    • Nombre remitente - Ingresa un nombre que te gustaría que aparezca como REMITENTE
    • Forzar el nombre del remitente - Marca esto si deseas forzar el nombre del REMITENTE
    • Ruta de retorno - Marca esta casilla
  3. Debajo de la sección Servicio de correo electrónico selecciona Otro SMTP.
  4. Debajo de la sección Otro SMTP, llena lo siguiente:
    • Servidor SMTP - smtp.dreamhost.com
    • Cifrado - Usar cifrado TLS
    • Puerto SMTP - 587

      En este ejemplo, se utiliza el puerto 587 con TLS. Si prefieres usar el puerto 465, asegúrate de usar la opción SSL en su lugar.

    • Enciende el botón Autenticación.
    • Nombre de usuario de SMTP - Ingresa tu dirección de correo de DreamHost
    • Contraseña SMTP - Ingresa la contraseña de tu correo
  5. Haz clic en el botón Guardar ajustes.

Tu formulario ahora se enviará DESDE el correo que ingresaste anteriormente.

 

Agregar seguridad extra

Las instrucciones anteriores agregan tu contraseña al panel de WordPress. Para asegurar aún más tus credenciales, el plugin recomienda que agregues código a tu archivo wp-config.php en su lugar. Dirígete la página siguiente para más detalles.

  • Cómo proteger la configuración de SMTP mediante el uso de constantes

Si elijes esta opción, no necesitasingresar ninguna credencial en tu panel de WordPress. Se actualizarán automáticamente cuando los agregas a tu archivo wp-config.php.

Agregar las constantes a tu archivo wp-config.php

  1. Inicia sesión en tu servidor a través de FTP o SSH.
  2. Utiliza uno de los siguientes artículos para editar tu archivo wp-config.php.
    • Crear y editar archivos a través de FTP
    • Crear y editar un archivo a través de SSH
  3. Desplázate hacia abajo en este archivo hasta la siguiente línea:

    /* That's all, stop editing! Happy blogging. */

  4. Agrega las siguientes líneas justo encima de esta línea.
    // turn constants on        
define( 'WPMS_ON', true );

    // general configuration settings
define( 'WPMS_MAIL_FROM', 'email@example.com' );
define( 'WPMS_MAIL_FROM_FORCE', true ); // True turns it on, false turns it off.
define( 'WPMS_MAIL_FROM_NAME', 'Example Name' );
define( 'WPMS_MAIL_FROM_NAME_FORCE', true ); // True turns it on, false turns it off.
define( 'WPMS_SET_RETURN_PATH', true ); // Sets $phpmailer->Sender if true.

    // specific configuration settings
define( 'WPMS_SMTP_HOST', 'smtp.dreamhost.com' ); 
define( 'WPMS_SMTP_PORT', 587 ); 
define( 'WPMS_SSL', 'tls' ); 
define( 'WPMS_SMTP_AUTH', true ); 
define( 'WPMS_SMTP_USER', 'email@example.com' );
define( 'WPMS_SMTP_PASS', '********' );
define( 'WPMS_SMTP_AUTOTLS', true );
define( 'WPMS_MAILER', 'smtp' );
    La unica parte que necesitas cambiar es WPMS_MAIL_FROM, WPMS_MAIL_FROM_NAME, y WPMS_SMTP_PASS. 
  5. Guarda el archivo

Si ves la página del plugin ahora, verás que las credenciales no son accesibles.

 

Probar el plugin

Navega a la página WP MAIL SMTP > Correo Prueba y da clic en el botón Enviar un correo electrónico de prueba para probar.

Deberías ver que fue enviado correctamente.

Es posible que veas una advertencia de DMARC después de la prueba. Tu correo seguirá funcionando sin él, pero se recomienda que configures DMARC para autenticarlo aún más. Revisa lo siguiente para obtener instrucciones sobre cómo configurar esto en tu dominio.

  • Crear una política de DMARC
