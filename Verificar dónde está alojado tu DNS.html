Generalidades

Este artículo explica cómo verificar dónde están alojados tus nameservers y registros A.

Revisa el artículo Visión general de los DNS de DreamHost para obtener más información sobre el DNS de tu sitio web y cómo se utilizan estos registros con tu sitio web.

Usar herramientas en línea para verificar el DNS

Hay muchos sitios web disponibles para que puedas verificar tu DNS. Por ejemplo.

Búsquedas generales de registros DNS

Estas herramientas verifican a dónde está apuntando actualmente tu DNS.

  • Centralops.net – Varias pruebas de DNS.
  • Network-tools.com – Varias pruebas de DNS.
  • Multiple record lookup – Especifica fácilmente de un rango de registros que deseas buscar.
  • Nameserver lookup – Se usa para verificar los detalles de los registros de nameservers.

Pruebas de propagación de DNS

Si acabas de actualizar tu DNS, puedes usar los siguiente enlaces para verificar qué ubicaciones se han actualizado en línea:

  • whatsmydns.net — Un sitio popular que verifica los registros DNS en todo el mundo.
  • Ver tus registros DNS de DreamHost — Usa el verificador de propagación de DNS de DreamHost en el panel.

Verificar el DNS a través de SSH

A continuación se enumeran varios comandos que puedes correr para verificar el DNS de tu sitio.

Para ejecutar estos comandos, asegúrate de haber creado un usuario de Shell y de poder iniciar sesión en tu servidor a través de SSH.

Verificar nameservers

Este comando verifica dónde están alojados actualmente los nameservers de tu sitio. La opción +short simplifica la salida para que solo veas los nameservers sin información adicional.

[server]$ dig ns example.com +short
ns2.dreamhost.com.
ns1.dreamhost.com.
ns3.dreamhost.com.

Comprobación de registros A

Esto verifica dónde están alojados los registros A (direcciones IP) de tu sitio.

[server]$ dig example.com +short
173.236.241.46
[server]$ dig www.example.com +short
173.236.241.46

Luego puedes correr el comando host para verificar desde dónde está alojada esa IP.

[server]$ host 173.236.241.46
46.214.236.173.in-addr.arpa domain name pointer apache2-argon.iad1-shared-b8-16.dreamhost.com.

La salida de esto es al revés. Por ejemplo, todo el camino a la derecha muestra que la IP está alojada en dreamhost.com.

A la izquierda está el nameserver en DreamHost. Esto muestra que está alojado en el servidor DreamHost llamado apache2-emu.

A la izquierda de eso está la instancia específica de apache en la que se llama el sitio iad1-shared-b8-16.
