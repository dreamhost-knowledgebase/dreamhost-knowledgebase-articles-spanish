Usar herramientas en línea para verificar tu DNS

Verificar el DNS de tu sitio a través de SSH

A continuación se enumeran varios comandos que puedes correr para verificar los valores DNS de tu sitio. Para correr estos comandos, debes usar un terminal SSH. Mira el artículo de Generalidades de SSH para más detalles.

Verificar nameservers

Este comando verifica dónde están alojados actualmente los nameservers de tu sitio. La opción +short simplifica la salida para que solo veas los nameservers sin información adicional.

[server]$ dig ns example.com +short
ns2.dreamhost.com.
ns1.dreamhost.com.
ns3.dreamhost.com.

Comprobación de registros A

Esto verifica dónde están alojados los registros A (direcciones IP) de tu sitio.

[server]$ dig example.com +short
173.236.185.214
[server]$ dig www.example.com +short
173.236.185.214

Luego puedes correr el comando host para verificar desde dónde está alojada esa IP.

[server]$ host 173.236.185.214
214.185.236.173.in-addr.arpa domain name pointer apache2-emu.tool.dreamhost.com.

La salida de esto es al revés. Por ejemplo, todo el camino a la derecha muestra que la IP está alojada en dreamhost.com.

A la izquierda está el nameserver en DreamHost. Esto muestra que está alojado en el servidor DreamHost llamado tool.

A la izquierda de eso está la instancia específica de apache en la que se llama el sitio apache2-emu.
