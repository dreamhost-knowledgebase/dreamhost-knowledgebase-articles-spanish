Generalidades

Si observas que los correos enviados o recibidos, especialmente dentro del mismo dominio, están llegando a una carpeta de correo no deseado, puede deberse a la dirección IP de tu proveedor de servicios de Internet (ISP) que está en la lista negra. Los filtros antispam de DreamHost están diseñados para marcar los correos como spam cuando se originan en una dirección IP en la lista negra.

A continuación se proporcionan los pasos que puedes seguir para determinar si tu dirección IP está en una lista negra.

Buscar tu dirección IP

  1. Visita whatismyipaddress.com para buscar la dirección IP actual de tu hogar.
  2. Usa los siguientes sitios para verificar si tu dirección IP está en una lista negra:
    • multirbl.valli.org
    • zy0.de
  3. Si tu dirección IP está en una lista negra, comunícate con tu ISP (la compañía que te proporciona acceso a Internet) para determinar los pasos que puedes seguir para eliminarla.

¿Qué sucede si mi dirección IP no está en una lista negra?

Si tu dirección IP no está en la lista negra, copia los encabezados del mensaje que terminó en tu carpeta de spam y pégalo en un Contactar a Soporte tiquete en tu panel DreamHost. Para obtener más información sobre cómo ver encabezados de correo, consulta el siguiente artículo:

  • Ver encabezados de correo electrónico

Asegúrate de explicar el problema que está experimentando para que el soporte pueda ayudarte más.

Solución de problemas adicionales

Proveedores de correo de terceros

Si tu dominio usa un proveedor de correo tercero pero usa el DNS de DreamHost, asegúrate de haber configurado los registros SPF personalizados.

Enviar correo desde tu sitio web

Si estás enviando un correo desde tu sitio web y esos mensajes se van a spam, asegúrate de que el correo esté autenticado vía SMTP:

  • WordPress: Puedes usar un plugin tal como 'WP Mail SMTP' para enviar correos de SMTP autenticados.
  • PHP: Si tu sitio usa una función de correo de PHP, puedes configurarlo para usar SMTP a través de PHPMailer.
  • Otros CMS: Si tu sitio tiene un CMS instalado que no sea WordPress, es posible que tengas soporte integrado para la autenticación de SMTP. Revisa la documentación de soporte de tu software para obtener más información.
