Generalidades

Un subdominio es un dominio o registro DNS que es agregado como dominio primario. Por ejemplo, si tu dominio primario es example.com, puedes agregar un subdominio para el llamado blog.example.com.

La parte 'blog' al frente del dominio primario es lo que le da sentido al subdominio. Esto es por que los dominios son leídos de derecha a izquierda. En el ejemplo de blog.example.com, las tres secciones se explican así:

  • .com — Este es nivel primario del dominio (TLD).
  • example. — Este es tu nombre de dominio único.
  • blog. — Este es tu subdominio.

Dos maneras de agregar un subdominio

Hay dos maneras de agregar un subdominio en DreamHost.

  • Como un subdominio completamente alojado — Esto quiere decir que va a tener su propio sitio web único.
  • Como un registro personalizado de DNS — Esto es solo un registro de DNS que apunta afuera de DreamHost.

Las siguientes secciones explican ambos métodos.

Opción 1 — Agregar un subdominio alojado completamente

Agregar un subdominio totalmente alojado es lo mismo que agregar un dominio primario. Por ejemplo, después de agregar tu dominio primario de example.com, puedes agregar un subdominio blog.example.com de la misma manera. Cuando agregas un subdominio, éste automáticamente es agregado a la página Administrar Dominios debajo de tu dominio primario. Puedes entonces alojar un nuevo sitio web en este subdominio.

  1. Navega hasta la página Administrar Dominios.
  2. Da clic en el botón Agregar alojamiento a un dominio / subdominio.
  3. La primera sección de esta página es títulada Completamente Alojado (completamente alojado). Aquí es donde agregarás el nuevo subdominio.

    No puedes ingresar un subdominio en la sección de Solo DNS.

  4. En el campo llamado 'Domain to host': ingresa el nombre del subdominio que deseas alojar. Por ejemplo, si quieres alojar completamente el subdominio, agrega el subdominio blog.example.com.
  5. Da clic en Alojar completamente este dominio para guardar.

    Si recibes el siguiente error:

    You can't add that domain: you must add the parent domain example.com to your account before adding a subdomain to it.

    Quiere decir que el dominio primario no está aún en el sistema. Un subdominio solamente puede ser creado si el dominio primario es creado primero (example.com). Esto es necesario inclusive si no planeas alojar el dominio de segundo nivel con DreamHost.

Una vez el subdominio esté completamente alojado, una dirección IP es asignada. Si quieres que tus nameservers  no estén alojados con DreamHost, debes apuntar los registros A a esta IP.

Revisa el artículo de DNS para ver instrucciones de como ver esta nueva dirección IP.

Opción 2 — Agregar un subdominio como un registro personalizado de DNS

Si estás alojando el dominio primario en DreamHost (example.com), pero quieres alojar el subdominio en otra compañía (blog.example.com), debes crear un registro personalizado de DNS en vez. Esto te permite configurar la dirección IP del subdominio para apuntar a otro proveedor de alojamiento.

Mira el siguiente artículo que describe cómo agregar un registro A como un registro personalizado de DNS. Este es el mismo método que verás para crear un subdominio.

Tiempo de propagación

Después de que el dominio es agregado, el DNS tiene que propagar. Esto puede tomar de 4-6 horas. Puedes usar cualquiera de los siguiente enlaces para confirmar si los DNS han actualizado en tu ubicación:

  • https://www.whatsmydns.net/

¿Puedo agregar correos para un subdominio?

Si. Una vez el subdominio ha sido agregado a la página de Administrar Dominios, dirígete a la sección de Administrar Correo en tu panel, procede con los pasos de configuración detallados en el siguiente artículo:

  • whatsmydns.net — Un sitio popular que verifica los registros DNS en todo el mundo.
  • Ver tus registros DNS de DreamHost — Usa el verificador de propagación de DNS de DreamHost en el panel.
