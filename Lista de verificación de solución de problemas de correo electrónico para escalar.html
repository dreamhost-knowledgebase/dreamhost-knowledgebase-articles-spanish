Generalidades

Si crees que tu correo electrónico no funciona como debería, el soporte de DreamHost necesita información específica para ayudarte. Este artículo te proporciona una lista de verificación de los elementos que debes tener el soporte para solucionar tu problema. Una vez que se reúne la información que se describe a continuación, ve a la página Contact Support para enviar tu solicitud de soporte.

Problemas de envío/recepción/conectividad

  • Error message – Describe el error exacto que ves en el cliente, incluidos los códigos de error. Pega el error que ves en tu solicitud de soporte.
  • Email address – La dirección que estás utilizando cuando se produce el problema.
  • Source IP (Es decir, la IP del cliente) – Puedes obtener esto en whatismyipaddress.com.
  • Provider – El ISP que está utilizando, como Verizon o Comcast.
  • Email client – Los ejemplos incluyen Webmail, Outlook, Thunderbird, and Mac Mail.
  • OS – ¿Qué sistema operativo o dispositivo estás usando?
  • Port # – ¿Cuáles son los puertos que estás usando? Visita el artículo de Configuración de cliente de correo para más detalles.
  • SSL – ¿Estás usando puertos seguros para conexiones entrantes y salientes?
  • Protocol – ¿Estás usando IMAP or POP3 para correo entrante? Visita el artículo POP3 vs IMAP para más detalles.
  • Authentication – ¿Estás utilizando la autenticación de contraseña para el correo saliente?
  • Time – Hora del error incluyendo tu zona horaria (u, hora actual).
  • Traceroute – Realiza un Traceroute a tu servidor de correo. Puede identificar en qué servidor de correo te encuentras visitando la página Data Center.
  • Server name – Consulta el siguiente artículo para obtener instrucciones sobre cómo localizar tu nombre de servidor de correo.
    • Cómo encontrar tu nombre de servidor de correo

Problemas de retraso

El soporte de DreamHost necesita lo siguiente:

  • Full email headers – Consulta el artículo Ver encabezados completos para obtener instrucciones sobre cómo obtenerlos.
  • Time – La hora en que se envió y recibió el correo electrónico, incluido zona horaria (u, hora actual).
  • Addresses – Dirección de envío y dirección de recepción
  • Where was the email sent from? – ¿Se envió el correo electrónico desde el servidor web utilizando phpmail/sendmail y así sucesivamente o mediante un cliente de correo electrónico?
  • Mail server – ¿En qué servidor de correo estás? Puedes identificar en qué servidor de correo te encuentras visitando la página Data Center.

Si no se recibió el correo

  • Addresses – La dirección de correo electrónico que envió el correo electrónico y la dirección a la que se envió el correo electrónico.
  • Time – La fecha y hora en que se envió la dirección de correo electrónico, incluidos zona horaria (u, hora actual).

Los registros de correo de DreamHost solo retroceden unos días, por lo que debe proporcionar ejemplos recientes.
