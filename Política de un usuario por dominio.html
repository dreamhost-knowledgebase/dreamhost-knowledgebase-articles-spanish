Generalidades

Cada dominio que alojes en DreamHost solo puede ser propiedad de un solo usuario FTP/SFTP/SHELL. Esta configuración ayuda a proteger un dominio ya que los datos de un usuario no pueden acceder a un dominio bajo un usuario diferente.

Este artículo explica esta configuración y cómo mover un dominio a un usuario diferente.

Cambiar el usuario del sitio web al que se asigna un dominio

Puedes cambiar el usuario al que se le asigna un dominio en cualquier momento.

  1. Navega a la página Administrar Sitios Web.
  2. Da clic en el botón Administrar para abrir la página Administrar Sitios Web, la cual te permite ajustar múltiples configuraciones de tu sitio.
    • Si estás en la Vista de Malla, da clic en el botón Administrar en la parte inferior de la caja del sitio.
    • Si estás en la Vista de Lista, da clic en el botón Administrar a la derecha de tu dominio.
  3. Da clic en el botón Datos de Ingreso debajo de Administrar Tu Sitio.
  4. Haz clic en el botón Cambiar usuario.
  5. Selecciona el usuario al que quieres que controle este dominio en el menú desplegable. Si deseas mover el sitio a un nuevo usuario, primero debes crearlo.

    Tienes la opción de mover todos los archivos al nuevo usuario en este momento. Si deseas mover todos los archivos del sitio web, marca la casilla junto a '¿Mover archivos al nuevo usuario?'.

  6. Desplázate hacia abajo y haz clic en el botón Cambiar Ajustes para guardar tus cambios.

La migración de los datos al usuario lleva más tiempo dependiendo de la cantidad de datos que el usuario original necesita copiar al nuevo usuario.

Si el nuevo nombre de usuario está en un servidor diferente, el DNS debe propagar antes de que el sitio se resuelva en línea con el nuevo usuario y nombre de servidor.

Verifica que el dominio se esté resolviendo a partir del nuevo nombre de usuario

Sigue estos pasos para confirmar que el sitio responde desde el nuevo nombre de usuario.

  1. Carga el sitio web.
  2. Inicia sesión en tu servidor a través de SFTP o SSH.
  3. Navega a la archivo. Cambia username a tu usuario Shell y example.com a tu sitio web. 
    /home/username/logs/example.com/http/access.log
    /home/<newuser>/logs/<domain>/http/access.log.
  4. Abre este archivo y verifique las entradas actuales.

Nombrar tu usuario de sitio web

Al crear un nuevo usuario del sitio web que utilizan varias personas, se recomienda darle un nombre general como ‘Domain_Admin’.

Ejemplo

Deseas crear un usuario para el sitio web de tu empresa, y habrá varias personas utilizando este inicio de sesión de usuario FTP.

  • No querrás nombrar la cuenta de usuario con tu nombre, como Bob_Smith
  • En su lugar, usa un nombre que esté más centrado en el dominio/empresa, como Nombrededominio_Admin

Permitir a un desarrollador aceder a tu sitio

En algún momento, puede ser necesario contratar a un desarrollador para que trabaje en tu sitio web. Hay diferentes formas en las que un desarrollador puede requerir acceso a tu sitio para trabajar en él. El acceso que le otorgas a un desarrollador depende exactamente de las herramientas que necesita para completar el trabajo para el que lo contrataste.

Mira el siguiente artículo para obtener información sobre cómo otorgar a tu desarrollador solo el acceso específico que requiere.
