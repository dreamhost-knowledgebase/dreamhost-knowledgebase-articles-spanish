Generalidades

La siguiente descripción detalla los registros DNS de sitios web más comunes y su uso.

Ver tus registros de DNS

Para encontrar tus registros DNS en el panel, navega a la página Administrar Sitios Web y abre la página Configuraciones de DNS:

  • Si estás en la Vista de Malla, da clic en el botón de los '3 puntos verticales' debajo de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
  • Si estás en la Vista de Lista, da clic en el botón de los '3 puntos verticales' a la derecha de tu dominio. Del menú que aparece da clic en el enlace Configuraciones DNS.
    Los registros DNS del dominio aparecen.

Registros de DNS del sitio web

Registro Descripción Más información

A

 El registro de Dirección (A) redirige el nombre de host (example.com) a una dirección IPv4.

Registro A

AAAA

 El registro de dirección IPv6 es una versión moderna del registro de dirección IPv4. Redirige un nombre de host (example.com) a una dirección IPv6.

Registro AAAA

IPv6

ALIAS

 An Registro ALIAS es un registro DNS virtual similar a un CNAME, pero con la capacidad de agregarlo al nombre de dominio raíz (lo cual no es posible con un CNAME). También puede agregarse a un registro de subdominio que ya existe, donde un CNAME debe ser único.

Registro ALIAS

CAA

 Un registro de Autorización de Autoridad de Certificación (CAA) especifica qué autoridades de certificación (CA) tienen permiso para emitir un certificado SSL para un nombre de dominio específico.

Registro CAA

CNAME

 El registro de Nombre Canónico (CNAME) apunta un nombre de dominio a otro nombre de dominio.

Registro CNAME

MX

 Los registros de Mail Exchange (MX) especifican la compañía donde tu correo está alojado.

Registro MX

NAPTR

 Los registros de Puntero de Autoridad de Nombre (NAPTR) se utilizan con mayor frecuencia para la telefonía por Internet.

Nameservers

NS

 Los registros de Nameservers (NS) controlan donde están alojados TODOS los registros DNS del dominio.

Registro NAPTR

PTR

Un "registro de puntero" se utiliza para la asignación inversa de una dirección IP a un nombre de host. DreamHost actualmente solo admite registros de DNS inverso cuando se agrega una IP única a un dominio.

DreamCompute también admite registros PTR, pero primero debes configurar el DNS de tu(s) dominio(s) para que los registros se creen automáticamente. Si el registro PTR no parece ser correcto dentro de los 15 minutos posteriores a realizar este cambio, por favor contacta a soporte.

Dirección IP Única

SPF

 Los registros (SPF) Sender Policy Framework especifica qué servidores de correo están autorizados para enviar correos desde tu dominio.

Registro SPF

SRV

 Los Registros de Localizador de Servicio (SRV) anuncian un servicio específico que ofrece un servidor.

Registro SRV

TXT

 Los registros de Texto (TXT) son cadenas de texto. Por ejemplo, pueden usarse con los registros de verificación de Google y la firma DKIM para servicios de correo.

Registros TXT

Detalles adicionales

Las siguientes secciones describen registros DNS adicionales y la terminología asociada.

Nombre de dominio completamente cualificado

Un FQDN es un dominio único que no se puede confundir con otro. Por ejemplo, si tu sitio era example.com, puedes tener varios FQDNs como:

  • www.example.com

Revisa esta página de Wikipedia para obtener más detalles.

Zone file

Un archivo de zona (Zone file) es un registro de texto que contiene la asignación de tu dominio y subdominios a sus correspondientes direcciones IP. Revisa esta página de Wikipedia para más detalles:

DreamHost no ofrece un servicio para exportar el archivo de zona de tu dominio. Si necesitas una lista de los registros del dominio, debes copiar tus registros DNS manualmente desde tu panel.

Registro SOA

El registro de Inicio de Autoridad es el primer registro de recursos en un archivo de Zona DNS. El registro SOA (Start Of Authority) especifica la siguiente información autorizada sobre un dominio específico:

  • Nameserver primario de donde se creó el registro SOA.
  • Correo del administrador del archivo de zona. (Se utiliza un punto en lugar del símbolo @).
  • El número de serie del dominio. Este es el número de revisión del archivo de zona.
  • Temporizadores:
    • Tiempo de actualización.
    • Tiempo de reintento.
    • Tiempo de expiración.
    • TTL Mínimo.

Corre el siguiente comando vía SSH (en tu computador o servidor) para ver un registro SOA. Este ejemplo muestra los resultados de dreamhost.com:

[server]$ nslookup -type=soa dreamhost.com

      Server:    64.90.63.202
      Address:   64.90.63.202#53
	  
	  Non-authoritative answer:
	  dreamhost.com
	  origin = ns1.dreamhost.com
	  mail addr = hostmaster.dreamhost.com
	  serial = 2015091000
	  refresh = 14534
	  retry = 1800
	  expire = 1814400
	  minimum = 14400
