Generalidades

Este artículo describe cómo preparar tu sitio web WordPress para el lanzamiento de una próxima versión.

Actualizar la versión PHP de tu sitio

Se recomienda encarecidamente que actualices a la última versión de PHP ya que proporciona un mejor rendimiento y compatibilidad con las últimas versiones de WordPress.

No todos los temas y plugins de terceros se han actualizado para que sean compatibles con PHP 8. Si tu sitio se basa en un plugin o tema específico, asegúrate de verificar que sea compatible con PHP 8 antes de actualizar.

Respalda tu sitio y su base de datos

Antes de hacer cualquier actualización en tu sitio WordPress, DreamHost recomienda hacer una copia de seguridad de tu instalación de WordPress. Los pasos son diferentes dependiendo de si tu sitio se instaló usando el instalador de WordPress de DreamHost o DreamPress.

Instalación de WordPress de DreamHost

Hay dos opciones para respaldar un sitio de WordPress creado por el instalador de WordPress de DreamHost en el panel.

  • Instalar un plugin de copias de seguridad de WordPress (el más fácil)
  • Hacer una copia de seguridad manual (avanazdo — requiere usar la línea de comando e iniciar sesión en tu sitio vía SSH)

Cualquiera de las opciones le proporciona un respaldo fuera del sitio que te permite restaurar rápidamente tu sitio en caso de que ocurra un error durante el proceso de actualización.

DreamPress

Los sitios web DreamPress tienen la capacidad de crear una copia de seguridad a pedido en el panel de DreamHost. Revisa el siguiente artículo para obtener instrucciones sobre cómo respaldar de tu sitio web DreamPress:

  • Crear y restaurar copias de seguridad para DreamPress

Actualizar tus plugins

Asegúrate de que los plugins estén actualizados antes de que se publique la nueva versión de WordPress. Puedes notar una o dos semanas antes de una nueva versión que varios plugins y temas de tu sitio tienen actualizaciones disponibles. Esto generalmente se debe a que los desarrolladores están preparando sus plugins para una nueva versión.

Revisa el siguiente artículo para obtener instrucciones sobre cómo actualizar tus plugins:

  • Administrar los plugins de WordPress

Actualizaciones de plugins manuales o automáticas

Si el plugin es parte de la funcionalidad central de tu sitio (por ejemplo, WooCommerce), DreamHost sugiere que crees una copia de seguridad antes de actualizar (ver arriba).

Sin embargo, si el plugin es más básico (por ejemplo, Akismet), puedes considerar habilitar las actualizaciones de los plugins automáticamente para asegurarte de que estén siempre en su última versión.

Usar un sitio de prueba o staging para probar la nueva versión

Si tu sitio depende de una funcionalidad esencial, como eCommerce o membresías, es mejor crear un sitio provisional para probar la nueva versión de WordPress antes de actualizar tu sitio en vivo.

No es necesario hacer pruebas con el sitio provisional, pero es una forma segura de asegurate de que tu sitio seguirá funcionando correctamente antes de actualizar.

Otra opción es usar un plugin como Migrate Guru para migrar los archivos y la base de datos de tu sitio a una instalación secundaria.

Usas el plugin WordPress Beta Tester para probar la nueva versión

Los siguientes pasos solo son necesarios si hay una próxima versión de WordPress disponible que aún no se haya lanzado oficialmente y que te gustaría probar.

Después de crear un sitio de prueba (como se mencionó anteriormente), usa las siguientes instrucciones para probar una versión preliminar de WordPress.

  1. Inicia sesión en su sitio provisional.
  2. Instala el plugin WordPress Beta Tester.
  3. Navega a Herramientas > Pruebas beta y selecciona la opción Desarrollo temprano.
  4. Da clic en Guardar cambios.
  5. Cuando la página cargue nuevamente, selecciona la opción Solo candidatas a definitivas.
  6. Da clic en Guardar cambios.
  7. Navega a Escritorio > Actualzaciones.
    Cuando esté disponible un Candidato de Lanzamiento (RC) de la próxima versión de WordPress, podrás actualizar tu sitio provisional en esta página:

    Los Candidato de Lanzamiento son efectivamente copias de "vista previa" de WordPress que están listas para lanzarse siempre que no haya errores importantes o que rompan el sitio.

  8. Una vez que tu sitio de prueba esté en el Candidato de Lanzamiento más reciente, asegúrate de que todo sea completamente funcional. Aquí hay algunas cosas para verificar:
    • ¿Todas las páginas se muestran como deberían?
    • ¿Sucede algo inesperado cuando navegas o intentas crear nuevo contenido?
    • Si usas un sitio eCommerce, ¿aún puedes agregar productos al carrito y pagar?

Si algo no funciona como se esperaba, anota lo que está fallando y comunícate con el desarrollador del plugin o tema específico, ya que es posible que ya tenga una actualización pendiente que solucione tu problema.
