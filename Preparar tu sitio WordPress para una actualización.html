Generalidades

Lo siguiente describe como preparar tu sitio para los lanzamientos venideros de WordPress.

Antecedentes

Desde la versión 5.6, que se lanzó a fines de 2020, las nuevas instalaciones de WordPress se configuraron para actualizar automáticamente el software principal a la última versión disponible, ya sea mayor o menor, poco después de que esté disponible para su descarga.

Esto significa que una gran cantidad de sitios de WordPress han recibido versiones importantes de su software principal de WordPress sin ningún problema durante un período de tiempo. Por lo tanto, puede estar seguro de que WordPress continuará funcionando como se espera cuando se realicen actualizaciones pendientes en tu sitio.

Sin embargo, dada la gran cantidad de temas, plugins y otras personalizaciones que puede haber agregado a tu instalación de WordPress, hay una serie de variables que se deben tenerse en cuenta para asegurarte de que tu proceso de actualización se realice sin problemas.

Respaldar tu sitio y base de datos

Antes de realizar cualquier actualización en tu sitio WordPress, Dreamhost recomienda hacer una copia de seguridad de tu instalación de WordPress. A continuación se describe cómo hacer una copia de seguridad de tu sitio para una Instalación de Un Clic o DreamPress.

Instalación de Un Clic

Si bien DreamHost no ofrece copias de seguridad a pedido en el panel de DreamHost para sitios de WordPress instalados a través del instalador de Un Clic, hay dos opciones para asegurarse de tener una copia de seguridad antes de que ocurra una actualización:

  • Instalar un plugin de respaldo de WordPress (lo más fácil)
  • Correr una copia de seguridad manual (avanzado — requiere usar la línea e iniciar sesión en tu sitio vía SSH)

Lo más importante aquí es asegurarse de que cualquier respaldo que realices se guarde en tu computador local o se cargue en un servicio Cloud como Google Drive o Dropbox. De esa manera, tendrás una copia de seguridad externa para restaurar en caso de que ocurra algo desafortunado en tu sitio durante el proceso de actualización.

Actualizar tus plugins

Una de las cosas que hacen que las actualizaciones automáticas de WordPress funcionen sin problemas es asegurarse de que los plugins desactualizados se hayan actualizado antes de que se publique la nueva versión de WordPress.

Antes de los principales lanzamientos de WordPress, se contacta a los desarrolladores de plugins y se les pide que se aseguren de que su código sea compatible con los cambios que se están realizando en el software principal. Puedes notar una o dos semanas antes de una nueva versión que varios plugins y temas de tu sitio tienen actualizaciones disponibles. Esto generalmente se debe a que los desarrolladores están preparando sus plugins para una nueva versión.

Afortunadamente, actualizar los plugins es tan simple como iniciar sesión en tu sitio y hacer algunas selecciones:

  • Administrar los plugins de WordPress

Si el plugin es parte de la funcionalidad principal de tu sitio (por ejemplo, WooCommerce, Easy Digital Downloads, Paid Memberships Pro, etc.), DreamHost sugiere que crees una copia de seguridad antes de actualizar (ver arriba). Sin embargo, si el plugin es un poco más básico (por ejemplo, Akismet, Smash Balloon, Site Kit, etc.), es posible que desees considerar habilitar actualizaciones automáticas de plugins, para que tengas una cosa menos de la que preocuparte.

Como probar una versión de WordPress lanzada previamente

Los siguientes pasos solo son necesarios si hay una próxima versión de WordPress disponible que aún no se ha lanzado oficialmente y te gustaría probar.

Si tu sitio aprovecha algún tipo de funcionalidad avanzada, como comercio electrónico o membresías, es mejor "probar" tu sitio con la nueva versión de WordPress antes de actualizar. Puedes crear un sitio provisional para lograr esto.

Un sitio provisional es básicamente un entorno secundario en el que hace una copia de tu sitio actual. Luego puedes usar un plugin de prueba para probar los cambios (como plugins actualizados, nuevos diseños y software central actualizado) en tu sitio provisional antes de actualizar tu sitio en vivo.

Usar los sitios provisionales o staging es totalmente opcional, pero es una práctica recomendada importante para garantizar que todo se vea, funcione y se sienta bien antes de que se produzca la actualización.

Usar el plugin WordPress Beta Tester para probar tu sitio

Después de crear un sitio provisional, usa las siguientes instrucciones para probarlo.

  1. Inicia sesión en tu sitio staging.
  2. Instala el plugin WordPress Beta Tester. Una vez activado, dirígete a Herramientas » Pruebas Beta en el panel de WordPress y luego selecciona Desarrollo temprano:
  3. Da clic en Guardar cambios y luego selecciona la opción Solo candidatas a definitivas cuando la página se recargue:
  4. Da clic en Guardar cambios nuevamente y luego dirígete a la página Escritorio » Actualizaciones.
    Cuando esté disponible un Candidatas a definitivas (RC) de la próxima versión de WordPress, podrás actualizar tu sitio provisional

    Candidatas a definitivas son copias de "vista previa" efectivas de WordPress que están listas para su lanzamiento siempre que no haya errores importantes o que rompan el sitio.

  5. Una vez que tu sitio staging esté en Candidatas a definitivas más reciente, asegúrate de que todo sea completamente funcional. Aquí hay algunas cosas para verificar:
    • ¿Se siguen mostrando todas las páginas como deberían?
    • ¿Ocurre algo inesperado cuando navegas o intentas crear contenido nuevo?
    • Si ejecutas un sitio de comercio electrónico, ¿aún puedes agregar productos al carrito y pagar?

Si algo no aparece o no funciona como esperabas, ten en cuenta lo que está fallando y comunícate con el desarrollador de cualquier plugin o tema que tengas problemas. Es posible que ya tengan una actualización pendiente que solucione lo que estás viendo. O, si no es así, es posible que haya encontrado algo que se puede arreglar rápidamente y evitará que muchos otros usuarios tengan una mala experiencia de actualización.

También puedes contactar al soporte de DreamHost si tienes preguntas sobre tu experiencia con WordPress.
