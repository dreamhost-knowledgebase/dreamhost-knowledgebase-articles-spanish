Generalidades

Este artículo es una descripción general de qué es phpMyAdmin y cómo usarlo en DreamHost.

Antecedentes

phpMyAdmin es un software que te permite utilizar un navegador web para administrar tu base de datos. Esto facilita la realización de operaciones comunes como la gestión de bases de datos, tablas, columnas, relaciones, índices, usuarios, permisos, etc. También puedes ejecutar cualquier instrucción de SQL dentro de esta interfaz.

Requisitos previos

Antes de usar phpMyAdmin en DreamHost, debes completar los siguientes pasos:

  1. Agrega un dominio como Completamente Alojado o Reflejado en la página Administrar Sitios Web.
  2. Agrega un hostname de MySQL en la página Bases de Datos MySQL.
  3. Dale suficiente tiempo para que los registros DNS propaguen después de agregar el hostname y el activar el alojamiento.

Una vez que completes estos pasos, phpMyAdmin estará disponible en tu dominio.

Usar phpMyAdmin

DreamHost proporciona phpMyAdmin para clientes en los planes de alojamiento compartidos, VPS, DreamPress, y Servidor Dedicado.

Acceder a phpMyAdmin

Tienes dos opciones para acceder:

  • Usa el hostname correcto para tu base de datos que aparece en el panel (mysql.example.com)
  • Usa el hostname oficial de phpMyAdmin de tu centro de datos. Este será uno de los siguientes:
    • Costa Oeste: https://west1-phpmyadmin.dreamhost.com
    • Costa Este: https://east1-phpmyadmin.dreamhost.com

    Puedes confirmar qué hostname utilizar verificando dónde se encuentran tus servicios de MySQL en la página Centro de Datos. Puede ser Costa Oeste o Costa Este.

Iniciar sesión en tu base de datos

Revisa este artículo para obtener más información sobre cómo ubicar las credenciales de tu base de datos e iniciar sesión en phpMyAdmin.

Administrar la base de datos

Borrar o eliminar una base de datos

Si necesitas borrar o eliminar una base de datos, debes hacerlo en la página Bases de Datos MySQL. phpMyAdmin no se puede usar para realizar esta acción.

Optimizar una base de datos

Con el tiempo, el tamaño de la base de datos reportado por phpMyAdmin comenzará a diferir del tamaño real de la base de datos en el disco de tu servidor MySQL. Esto ocurre porque phpMyAdmin muestra el tamaño de la base de datos usando los índices de la base de datos, que pueden ser más pequeños que el uso real del disco después de muchos ciclos de lectura y escritura.

Una forma posible de resolver este problema es optimizar tu base de datos, aunque esto no está garantizado. Los siguientes son dos métodos que puedes usar para optimizarla:

  • Optimizar una base de datos usando phpMyAdmin
  • Usando wp-cli de Wordpress
