Generalidades

Si encuentras picos de uso de la memoria de tu VPS en ciertos momentos del día, puedes usar una herramienta de evaluación comparativa para simular el entorno de alto tráfico. Una vez que se ejecuta, puedes verificar los procesos de memoria para determinar la causa del problema.

Este artículo explica cómo usar la línea de comando ab como herramienta de evaluación comparativa en tu VPS.

Para correr los comandos de este artículo, debes tener un usuario de Shell configurado en tu panel y conocimientos generales de UNIX Shell. Revisa los siguientes artículos para obtener más información:

  • Crear un usuario con acceso Shell (SSH)
  • Generalidades de SSH

Herramienta de referencia

Revisa el siguiente enlace para ver más detalles del comando ab.

  • ab [linux man page]

Para usar esta herramienta:

  1. Inicia sesión en tu servidor a través de SSH.
  2. Corre el siguiente comando: 
    [server]$ ab -n 1000 -c 20 https://www.example.com/index.html

    Este comando crea 1000 conexiones a https://www.example.com/index.html, limitándose a 20 conexiones concurrentes.

Ver tus procesos mientras prueba

Mientras el comando ab está corriendo, abre 2 pestañas más en tu terminal para observar el proceso del servidor.

  • En la primera ventana, corre top -c. Luego, clasifícalo por memoria (Shift + M).
  • En la segunda ventana, corre free -m (o watch free -m).

Estos comandos son efectivos para perfilar tus sitios y verificar si el almacenamiento en caché funciona. Puedes repetir este proceso para cada dominio para ver cuáles afectan más el uso de memoria y saber dónde optimizar.
