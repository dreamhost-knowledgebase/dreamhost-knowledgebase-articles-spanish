Generalidades

Al transferir el alojamiento a DreamHost, puede ser necesario migrar la base de datos de tu sitio web. Este artículo describe cómo transferir una base de datos MySQL a tu cuenta DreamHost.

Crear una base de datos MySQL en tu panel

Antes de poder importar una base de datos, primero debes crearla en tu panel. Como precaución de seguridad, una base de datos DreamHost solo se puede crear dentro del panel en la página MySQL Databases. No es posible crear la base de datos durante el proceso de importación.

Consulta el siguiente artículo para obtener detalles sobre cómo crear tu base de datos:

  • Crear una base de datos MySQL

Importar tu base de datos

Importar con phpMyAdmin

Si tu archivo de copia de seguridad SQL tiene un tamaño inferior a 64 MB, se puede importar a tu nueva base de datos a través de phpMyAdmin. evisa el siguiente artículo para importar una copia de seguridad de base de datos a través de phpMyAdmin:

  • Cómo importar / restaurar una base de datos

Importar por medio de SSH

Si tiene 64 MB o más, (o si encuentras un tiempo de espera al importar), la única otra opción es importarlo desde la línea de comandos a través de SSH.

  • Importar una base de datos a través de SSH

Pruebas

Al migrar MySQL de un host a otro, es posible probar la nueva instalación antes de modificar las entradas DNS de su dominio.

Para hacer esto, revisa las instrucciones en los siguientes artículos:

  • Ver tu sitio utilizando un subdominio dreamhosters.com antes de cambiar los DNS
  • Acceder a tu base de datos antes de cambiar los registros DNS

Apuntar tus registros DNS a DreamHost

Cuando tengas todo configurado en DreamHost, deberás actualizar tus DNS en tu host actual. Puedes apuntar los servidores de nombres a DreamHost, para apuntar TODOS tus registros DNS, o puedes simplemente apuntar registros A específicos. Revisa los siguientes artículos para más detalles:

  • Generalidades de los Nameservers
  • ¿A qué registros DNS apunto mi sitio?
