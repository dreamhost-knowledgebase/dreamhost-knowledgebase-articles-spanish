Generalidades

Este artículo describe cómo migrar una base de datos MySQL a tu cuenta de DreamHost.

Migrar una base de datos

Para migrar una base de datos:

 

Crear una base de datos MySQL en tu panel

Antes de poder importar una base de datos, primero debe crearse en el panel. Como medida de seguridad, solo se puede crear una base de datos DreamHost dentro del panel en la página Bases de Datos MySQL. No es posible crear la base de datos durante la importación. Revisa este artículo para obtener detalles sobre cómo crear tu base de datos.

Importar tu base de datos

Puedes importarla usando phpMyAdmin o SSH.

Importar con phpMyAdmin — Si tu archivo de respaldo de SQL tiene un tamaño inferior a 64 MB, puedes importarlo a tu nueva base de datos usando phpMyAdmin.

Importar usando SSH — Si el archivo de respaldo tiene 64 MB o más (o si se produce un tiempo de espera durante la importación), la única otra opción es importarlo desde la línea de comandos a través de SSH.

Probar tu base de datos

Antes de apuntar tus registros DNS a DreamHost, debes probar la nueva instalación para asegurarte de que esté configurada correctamente. Revisa los siguientes artículos para obtener más detalles:

  • Ver tu sitio utilizando un subdominio dreamhosters.com antes de cambiar los DNS
  • Acceder a una base de datos antes de apuntar el DNS a DreamHost

Apuntar tu DNS a DreamHost

Cuando todo esté configurado en DreamHost, es hora de actualizar tu DNS en el alojamiento actual. Puedes apuntar tus nameservers (que apuntan TODOS tus registros de DNS) o apuntar registros A específicos. Revisa este artículo para obtener más detalles.
