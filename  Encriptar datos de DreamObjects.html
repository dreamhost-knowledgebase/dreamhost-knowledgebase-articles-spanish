Generalidades

Cuando subes datos a DreamObjects, los datos no están cifrados de forma predeterminada. Los datos están seguros dentro de DreamObjects, sin embargo, hay algunas opciones disponibles para proteger aún más tus datos cifrándolos.

Este artículo explica diferentes formas en las que puedes asegurarte de que tus datos estén encriptados dentro de DreamObjects.

Encriptación del lado del servidor

El cifrado del lado del servidor es cuando tus datos se cifran mientras se escriben en el servidor. Cuando desees acceder a los datos, se descifrarán del servidor. DreamHost actualmente no admite el cifrado del lado del servidor con DreamObjects.

Cifrado del lado del cliente

El cifrado del lado del cliente es cuando usted cifra los datos ANTES de cargarlos en el servidor. Esto significa que los datos que cargas ya están encriptados mientras se escriben en el servidor. Las siguientes aplicaciones admiten el cifrado del lado del cliente.

  • Duplicati
  • CloudBerry
  • Encriptación de datos con S3cmd

SSE-C

SSE-C está disponible, sin embargo, esta es una opción solo para usuarios avanzados. Esto se debe a que requiere que crees tus propias llaves de cifrado para cargar/descargar contenido y cifrar datos. CloudBerry tiene un artículo que explica esto con más detalle:

  • Cómo configurar el cifrado del lado del servidor con claves proporcionadas por el cliente (SSE-C) utilizando CloudBerry S3 Explorer
