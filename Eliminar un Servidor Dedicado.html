Generalidades

Al eliminar tu Servidor Dedicado, tienes la opción de eliminar permanentemente todos los usuarios y datos, o migrar todo a un servidor diferente. Asegúrate de seguir los pasos a continuación cuidadosamente, de lo contrario podrías perder datos en tu servidor.

Si deseas migrar tus datos a un servidor diferente

Este suele ser el caso al eliminar un Servidor Dedicado. Esto también puede considerarse como una degradación de tu Servidor Dedicado a un plan de alojamiento diferente.

Si deseas mantener tu sitio web activo y migrarlo a un plan de alojamiento diferente, ya debes tener un plan de alojamiento compartido o VPS activo. Solo entonces tendrás la oportunidad de migrar los datos a este nuevo servidor cuando canceles.

Si no hay otro servicio de alojamiento, el sitio web se cerrará y tu cuenta solo estará disponible para registros de dominio.

Asegúrate de tener otro plan de alojamiento activo si desea mantener tu sitio en funcionamiento.

  1. Asegúrate de haber comprado un segundo plan de alojamiento. Este podría ser un plan Shared o un VPS.
  2. Navega a la página Dashboard.
  3. A la derecha de tu servidor, haz clic en el botón Eliminar.
  4. Haz clic en Mover todo en ds### a un <nuevoservidor> y luego destruir ds###!. Esto asegura que tu información se copia al nuevo servidor donde continuarás alojando tu sitio.

Si ya no deseas alojar tu sitio

Si ya no necesitas tu sitio web, puedes eliminar el Servidor Dedicado en la página Dashboard. Asegúrate de haber hecho una copia de seguridad de todos los datos necesarios primero, ya que no serán recuperables después de eliminar el servidor.

Si tus bases de datos contienen más de 3 GB de datos

Si deseas eliminar un Servidor Dedicado y las bases de datos asociadas contienen más de 3 GB de datos, el botón de Eliminar estará en gris y no podrás eliminar el servidor.

Hay varias cosas que puedes hacer para continuar con la eliminación del servidor:

  • Reduce el tamaño de las bases de datos existentes para que sea inferior a 3 GB (si planeas usar las bases de datos en un servidor MySQL compartido, DreamHost recomienda reducir significativamente el tamaño por debajo de 3 GB para permitir que las bases de datos tengan espacio para crecer).
  • Agrega un VPS MySQL y muevE las bases de datos del VPS dedicado al MySQL. Una vez que las bases de datos se hayan movido al VPS MySQL, se puede cancelar el servidor dedicado.
  • Eliminar cualquier base de datos de más de 3 GB que ya no se utilizan.
