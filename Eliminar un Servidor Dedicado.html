Generalidades

Lo siguiente describe cómo eliminar un Servidor Dedicado.

Asegúrate de tener otro plan de alojamiento activo si deseas mantener tu sitio en funcionamiento.

Si deseas mantener tu sitio web activo y moverlo a un plan de alojamiento diferente, debes haber añadido un plan de alojamiento compartido o VPS. Solo entonces tendrás la oportunidad de mover los datos a este nuevo servidor cuando estés cancelando.

  • Si no se añade primero otro plan de alojamiento, el sitio web se caerá y tu cuenta solo estará disponible para registros de dominios.

Cómo eliminar un Servidor Dedicado

Al eliminar tu Servidor Dedicado, hay dos opciones a considerar:

  • Migrar tu información a un servidor diferente
  • Eliminar permanentemente todos los usuarios y la información

Cada opción se describe a continuación. Asegúrate de seguir cuidadosamente estos pasos, de lo contrario podrías perder datos en tu servidor.

Opción 1 — Migrar tus datos a un servidor diferente

Este es el caso más común al eliminar un Servidor Dedicado. También se puede considerar como una degradación de tu Servidor Dedicado a un plan de alojamiento diferente.

  1. Asegúrate de haber comprado un segundo plan de alojamiento compartido o VPS.
  2. Navega a la página Panel.
  3. A la derecha de tu servidor, haz clic en el botón Eliminar.
  4. Haz clic en Mover todo en ds### a <nuevoservidor> y luego destruir ds###. Esto asegura que tus datos se copien al nuevo servidor donde continuarás alojando tu sitio.

Opción 2 — Eliminar permanentemente todos los usuarios y la información

Si ya no necesitas tu sitio web, puedes eliminar el Servidor Dedicado en la página del Panel. Asegúrate de haber respaldado cualquier dato necesario primero, ya que no será recuperable después de eliminar el servidor. Revisa los siguientes artículos para obtener más información:

  • Respaldar la información de un sitio web manualmente
  • Respaldar una cuenta en el panel

Si tus bases de datos contienen más de 3 GB de datos

Si deseas eliminar un Servidor Dedicado y las bases de datos asociadas contienen más de 3 GB de datos, el botón Eliminar estará desactivado y no podrás eliminar el servidor.

Hay varias opciones que puedes tomar para proceder con la eliminación del servidor:

  • Reduce el tamaño de las bases de datos existentes a menos de 3 GB (si planeas usar las bases de datos en un servidor MySQL compartido, DreamHost recomienda reducir el tamaño significativamente por debajo de 3 GB para permitir que las bases de datos crezcan).
  • Agrega un VPS MySQL y mueve las bases de datos del Servidor Dedicado a este. Una vez que las bases de datos se hayan trasladado al MySQL VPS, el Servidor Dedicado puede ser cancelado.
  • Elimina cualquier base de datos de más de 3 GB que ya no se esté utilizando.
