Generalidades

Este artículo describe cómo copiar tu sitio principal de DreamPress a un sitio web provisional. El sitio de preparación te permite probar los cambios que planeas realizar en gu sitio en vivo mientras gu sitio en vivo permanece en funcionamiento normalmente.

Cuando hayas terminado de trabajar en tu sitio de preparación y lo hayas configurado de la manera que desees, puedes publica esos cambios nuevamente en tu sitio web en vivo.

Notas sobre cómo copiar tu sitio DreamPress staging o provisional

Si tu sitio ha sido hackeado previamente y luego usaste el servicio de DreamHost DreamShield debes limpiar los archivos que fueron pirateados/deshabilitados antes de poder habilitar el sitio de prueba en tu sitio.

  • DreamPress Staging usa Git para copiar archivos. Si tienes tu propio repositorio Git activo en una de las carpetas de tu sitio en vivo, el contenido de la carpeta no se copiará en tu sitio de prueba. Para utilizar la puesta en escena con éxito, puedes copiar manualmente el contenido de la carpeta mediante SFTP o SSH. DreamPress recomienda como mejor práctica administrar tu repositorio Git en un entorno de desarrollo separado en lugar de tu sitio en vivo.
  • Cuando habilitas staging por primera vez, tu plan DreamPress debe tener al menos 5 GB de espacio libre.
  • La carpeta Cargas no se copia en tu sitio de prueba para utilizar el espacio en disco de manera eficiente. Reemplazar una imagen existente en tu Biblioteca de medios la modificará para el sitio en vivo.

Copiar tu sitio en vivo al sitio de preparación

Los siguientes pasos te muestran cómo copiar tu sitio web en vivo de DreamPress a tu sitio web provisional. Luego puedes probar y trabajar en el sitio de preparación sin afectar el sitio en vivo.

  1. Navega hasta la página WordPress Administrado.
  2. Da click en el botón Manage a la derecha de tu dominio principal.
    La página Manage DreamPress se abrirá:
  3. Da click en la pestaña Staging.
    Se abrirá la página del panel Manage Staging que te solicitará que cree un sitio de staging:
  4. Da click en el botón Create Staging Site.

    Un sitio provisional se crea automáticamente con un nuevo nombre. Ten en cuenta que no es posible darle un nombre personalizado a tu sitio de preparación.

    Un medidor de progreso muestra el estado:

Una vez que tu sitio web haya terminado de copiarse en tu sitio de preparación, se te enviará un correo de confirmación.

Administrar su sitio web provisional

El entorno de ensayo no está incluido en las copias de seguridad diarias del dominio DreamPress. Tú es responsable de mantener tus propias copias de seguridad de cualquier contenido que agregues en el entorno de ensayo.

Después de crear tu sitio web provisional, aparece la siguiente página en tu panel de control Managing Staging:

El nombre de dominio de tu sitio web provisional aparece en la parte superior. Este ejemplo muestra "exampledreamhostcom.stage.site".

  • Access Your Staging Website — Muestra el nombre de usuario y la contraseña que usas para acceder a tu sitio web protegido con inicio de sesión.
  • Staging WordPress Admin — Abre la página WP-Admin donde puedes realizar modificaciones en tu sitio web provisional.
  • File Upload — Muestra las credenciales FTP de tu sitio web provisional para que puedas transferir archivos a través del administrador de archivos de DreamHost o tu cliente FTP de terceros.
  • Database (MySQL) — Muestra las credenciales de tu base de datos (MySQL) para que puedas administrar tus bases de datos usando phpMyAdmin.

Iniciar sesión en tu sitio provisional

Si anteriormente actualizaste tu contraseña de administrador de DreamPress, puedes usar el mismo nombre de usuario y contraseña de tu sitio en vivo para iniciar sesión en tu sitio provisional.

Si no cambiaste manualmente tu contraseña, puedes iniciar sesión en tu sitio de WordPress desde el panel de DreamPress. Haz clic en el botón Managing Staging WordPress para iniciar sesión automáticamente en el panel de administración.
