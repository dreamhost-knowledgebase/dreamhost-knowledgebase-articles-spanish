Generalidades

Este artículo describe cómo copiar tu sitio principal de DreamPress a un sitio web provisional. El sitio de preparación te permite probar los cambios que planeas realizar en tu sitio en vivo mientras tu sitio en vivo permanece en funcionamiento normalmente.

Cuando hayas terminado de trabajar en tu sitio de preparación y lo hayas configurado de la manera que desees, puedes publica esos cambios nuevamente en tu sitio web en vivo.

Notas sobre cómo copiar tu sitio DreamPress Staging o provisional

Si tu sitio ha sido hackeado previamente y luego usaste el servicio de DreamHost DreamShield debes limpiar los archivos que fueron pirateados/deshabilitados antes de poder habilitar el sitio provisional en tu sitio.

  • DreamPress Staging usa Git para copiar archivos. Si tienes tu propio repositorio Git activo en una de las carpetas de tu sitio en vivo, el contenido de la carpeta no se copiará en tu sitio de prueba. Para utilizar la puesta en escena con éxito, puedes copiar manualmente el contenido de la carpeta mediante SFTP o SSH. DreamPress recomienda como mejor práctica administrar tu repositorio Git en un entorno de desarrollo separado en lugar de tu sitio en vivo.
  • Cuando habilitas Staging por primera vez, tu plan DreamPress debe tener al menos 5 GB de espacio libre.
  • La carpeta Cargas no se copia en tu sitio de prueba para usar el espacio en disco de manera eficiente. Reemplazar una imagen existente en tu Biblioteca de medios la modificará para el sitio en vivo.

Copiar tu sitio en vivo al sitio provisional

Los siguientes pasos te muestran cómo copiar tu sitio web en vivo de DreamPress a tu sitio web provisional. Luego puedes probar y trabajar en el sitio de preparación sin afectar el sitio en vivo.

  1. Navega hasta la página WordPress Administrado.
  2. Da clic en el botón Administrar a la derecha de tu dominio principal.
    La página Administrar DreamPress se abrirá:
  3. Da clic en la pestaña Vista de Prueba.
    Se abrirá la página del panel Administrar Staging que te solicitará que cree un sitio de staging:
  4. Da clic en el botón Crear Sitio de Vista Previa.

    Un sitio provisional se crea automáticamente con un nuevo nombre. Ten en cuenta que no es posible darle un nombre personalizado a tu sitio de vista previa

Un medidor de progreso muestra el estado:

Una vez que tu sitio web haya terminado de copiarse en tu sitio de preparación, se te enviará un correo de confirmación.

Administrar tu sitio web provisional

El entorno provisional no está incluido en las copias de seguridad diarias del dominio DreamPress. Tú es responsable de mantener tus propias copias de seguridad de cualquier contenido que agregues en el entorno provisional.

Después de crear tu sitio web provisional, aparece la siguiente página en tu panel de control Administrar Staging:

El nombre de dominio de tu sitio web provisional aparece en la parte superior. Este ejemplo muestra "exampledreamhostcom.stage.site".

  • Acceder al Sitio de Vista de Prueba — Muestra el nombre de usuario y la contraseña que usas para acceder a tu sitio web protegido con inicio de sesión.
  • Administrador de Vista de Prueba del Panel de WordPress — Abre la página WP-Admin donde puedes realizar modificaciones en tu sitio web provisional.
  • Subdir Archivos — Muestra las credenciales FTP de tu sitio web provisional para que puedas transferir archivos a través del administrador de archivos de DreamHost o tu cliente FTP de terceros.
  • Base de Datos (MySQL) — Muestra las credenciales de tu base de datos (MySQL) para que puedas administrar tus bases de datos usando phpMyAdmin.

Probar las nuevas versiones de PHP en Staging

La opción Configure PHP te permite probar versiones más nuevas de PHP en un sitio provisional, sin afectar tu sitio web DreamPress en vivo.

Si tu sitio provisional funciona en una versión más reciente de PHP, puedes publicar tu sitio web Staging, y la versión más nueva de PHP, en tu sitio web DreamPress en vivo. No es posible bajar PHP en tu sitio DreamPress en vivo usando este método.

Antes de continuar, asegúrate de haber publicado una copia de tu sitio web DreamPress en vivo en Staging, siguiendo los pasos anteriores.

Los siguientes pasos te muestran cómo cambiar la versión PHP de tu sitio web provisional desde el panel de Staging.

  1. Busca la opción Configurar PHP en la parte inferior del panel.
  2. Da clic en el botón Actualizar PHP a….
    Un medidor de progreso muestra el estado.
  3. Una vez que se completa la actualización, el panel de Staging mostrará la versión actual de PHP en uso en tu sitio provisional en Configurar PHP.
  4. Una vez que se complete la actualización, inicia sesión en tu sitio web Staging y examínalo para asegurarte de que esté funcionando como se esperaba. Revisa la sección Iniciar sesión en tu sitio Staging a continuación para obtener más detalles.
  5. Si tu sitio funciona en la nueva versión de PHP, puedes publicar tus cambios desde tu sitio web provisional en tu sitio DreamPress en vivo. Revisa el siguiente artículo para obtener instrucciones:
    • Publicar tus cambios de Staging en tu sitio de DreamPress en vivo

Iniciar sesión en tu sitio provisional

Es posible que veas el siguiente mensaje de error de estado de git, que aparece debajo Tools > Site Health (en la sección 'Es posible que las actualizaciones en segundo plano no funcionen correctamente' debajo de Security en el panel WP-Admin):

The folder /home/wp_xyz/example.com/ was detected as being under version control (.git).

Este mensaje siempre aparecerá para cada sitio provisional de DreamPress y puede ignorarlos con seguridad. Git no romperá tu sitio.

Si anteriormente actualizaste tu contraseña de administrador de DreamPress, puedes usar el mismo nombre de usuario y contraseña de tu sitio en vivo para iniciar sesión en tu sitio provisional.

Si no cambiaste manualmente tu contraseña, puedes iniciar sesión en tu sitio de WordPress desde el panel de DreamPress. Haz clic en el botón Administrar Vista de Prueba en WordPress para iniciar sesión automáticamente en el panel de administración.
