Generalidades

Cuando necesites configurar una cuenta de correo en cualquier cliente de correo, necesitarás cierta información específica. La información más básica que necesitarás es:

  • Nombre de usuario.
  • Contraseña.
  • Nombre del servidor de correo.

Vas a poder entonces implementar más configuraciones para tu correo, tales como establecer una dirección IMAP o POP y habilitar puertos seguros.

Este artículo te guiará paso a paso por estas opciones para configurar tu cuenta de correo tal como desees.

Este artículo te muestra cómo configurar una dirección de correo que está alojada en DreamHost. Si estás alojando tu correo en un proveedor de servicios de correo diferente, debes consultar la documentación de ellos para la configuración que se menciona a continuación.

 

Paso 1 — Agregar la dirección de correo al panel

Tu puedes crear tu dirección de correo a través de Administrar Correo.

Lee el siguiente artículo para recibir instrucciones detalladas acerca de cómo añadir una dirección de correo:

  • Agregar una dirección de correo electrónico

Paso 2 — Localiza tu nombre de usuario y contraseña

Una vez creada tu dirección de correo, tendrás tu nombre de usuario y contraseña.

Nombre de usuario

Esta es la parte que viene antes de tu dominio. Por ejemplo usuario@example.com.

Tu cliente de correo podría requerir solamente esta primera parte de la dirección de correo, pero también es posible que tu cliente de correo necesite la dirección completa para poder conectarse. Esto depende de tu cliente de correo.

Contraseña

La contraseña para el usuario será aquella que usted hayas elegido al crear la dirección.

La contraseña puede cambiarse en cualquier momento a través de tu página de Administrar Correo, o a través de la interfaz de Mailboxes.

La configuración de autentificación con nombre de usuario/contraseña debe estar habilitada y configurada a "Contraseña".

  • La configuración de autenticación de usuario/contraseña debe estar habilitada y configurada como ‘Contraseña’ en el cliente.
  • ¡Las contraseñas son sensibles a las mayúsculas y minúsculas!
  • Las contraseñas para tus usuarios de correo se pueden cambiar en cualquier momento.
  • Visita el artículo de Contraseñas para saber cómo elegir una contraseña segura.
 

Paso 3 — Ubicar tu nombre de servidor de correo

Para encontrar rápidamente tu configuración de correo en el panel, abre la página Administrar Correo y busca el enlace Información de configuración de correo en la esquina superior derecha. Haz clic en el texto para abrir las instrucciones de acceso rápido para tu dirección de correo:

Entrante
imap.dreamhost.com
pop.dreamhost.com
Saliente
smtp.dreamhost.com

Tu nombre de servidor (también conocido como nombre de equipo o nombre de host) controla desde y hacia dónde se envía tu correo. Los siguientes son los únicos servidores que debes usar para conectarte a tu correo. Visita el artículo de los protocolos de IMAP y POP y números de puerto para determinar si prefieres usar una conexión con IMAP o POP.

Servidor de Entrada

Servidor IMAP: imap.dreamhost.com

Servidor POP: pop.dreamhost.com

POP descarga mensajes de forma predeterminada desde el servidor a tu computador local.

Esto significa que después de conectarte, los correos se eliminan del servidor y ya no estarán disponibles cuando inicies sesión en tu dirección de correo en el futuro.

Si también deseas mantener tus correos en el servidor cuando utilizas una conexión POP, debes configurar tu cliente de correo para mantener una copia de ellos en el servidor. Los pasos para hacer esto difieren según tu cliente de correo.

Servidor de Salida

smtp.dreamhost.com

Si no usas uno de los servidores anteriores, tu cliente no podrá conectarse. Es posible que veas los siguientes errores si ese es el caso:

  • Error de coincidencia.
  • Esta identidad no puede ser validada.

Estos errores pueden evitarse utilizando los servidores mencionados arriba. Visita el siguiente artículo para más detalles:

  • Error de desfase en el dominio del certificado al conectarse a un servidor de correo DreamHost

Algunos proveedores de servicios de Internet (ISP) requieren que utilices tu servidor SMTP para enviar correo (por ejemplo, smtp.tuISP.com). Debes consultar con tu ISP para ver si requieren que uses ese servidor SMTP para el correo saliente, o si puedes usar el servidor de correo de tu dominio para el correo saliente.

Paso 4 — Configurar puertos seguros y elegir entre IMAP o POP

Muchos clientes de correo actuales habilitan automáticamente los puertos seguros durante su configuración. Esto quiere decir que es posible que ni siquiera se te solicite esta información. Sin embargo, algunos clientes de correo requieren que agregues específicamente estos detalles de forma manual.

Tu cliente te pedirá los números de puerto a los cuales conectarse. Algunos son seguros, mientras que otros son inseguros. Los números de puerto que elijas también determinarán si tu dirección usa una conexión IMAP o POP.

Al elegir un puerto, siempre recomendamos elegir la configuración segura. Al elegir entre POP e IMAP, vas a querer elegir siempre IMAP, a menos que haya una razón específica para usar POP.

En el siguiente artículo encontrarás más detalles acerca de la configuración de puertos seguros que debes utilizar:

  • Protocolos y números de puerto para clientes de correo.

Registros MX

Los registros MX son los registros de DNS que le indican a otros servidores de correo cómo encontrar tu dirección de correo en internet.

Los registros de MX para tu dominio se crearán una vez que se haya creado tu primera dirección de correo. Después de añadir la primera dirección, la configuración lleva entre 4 y 6 horas, de modo que deberás esperar esa cantidad de tiempo antes de poder usar la primera dirección de correo. Después de eso, ya no habrá más retardos al añadir una dirección de correo.

Si tu página web y tu correo están alojados en DreamHost, entonces no necesitarás preocuparte por estos registros.

Solamente haría falta dirigir tus registros de MX a DreamHost si tus nameservers están alojados en otra compañía.

Configuración de clientes de correo de terceros

Los clientes de correo más antiguos pueden tener problemas para conectarse a los servicios de correo de DreamHost y es posible que deban actualizarse a una versión que admita TLS 1.2 o superior. Esto se debe a que las versiones 1.0 y 1.1 de TLS ya no son compatibles con las conexiones de correo seguras.

Estos clientes de correo incluyen (pero no se limitan a):

  • Outlook 2007
  • Outlook 2010
  • MacMail en MacOS 10.11 o más antiguos
  • Cualquier cliente de correo de Microsoft que corra en Windows 7

DreamHost recomienda que actualices tu sistema operativo o uses un cliente de correo diferente como Mozilla Thunderbird, que viene con soporte TLS 1.2 listo para usar.

Bloqueos temporales

Multiples intentos de conexión de FTP o SFTP fallidos pueden resultar en que el servidor bloquee temporalmente tu IP. Este bloqueo se quitará automáticamente al pasar de unos momentos, pero si quieres que se remueva más pronto, por favor contacta a soporte.
