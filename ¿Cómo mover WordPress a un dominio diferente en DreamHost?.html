Generalidades

Este artículo describe cómo migrar una instalación de WordPress entre dos dominios alojados en DreamHost. Los dominios que se usan como ejemplo en estas instrucciones son:

  • oldsite.example.com
and
  • newsite.example.com

Paso 1 – Haz una copia de seguridad de tu información

Siempre debes hacer una copia de seguridad de tus datos antes de realizar cualquier cambio en tu sitio. Consulta el siguiente artículo para obtener instrucciones sobre cómo hacer una copia de seguridad de los archivos, la base de datos y el correo electrónico de tu sitio web:

  • Técnicas de respaldo manual

Paso 2 – Agrega alojamiento para el nuevo sitio

Agrega el nuevo dominio (newsite.example.com) a tu panel. Asegúrate de configurarlo como un dominio completamente alojado (Fully Hosted):

  • Agregar un Dominio

Paso 3 – Crea una nueva base de datos

Navega a la página MySQL Databases y crea un nuevo nombre de host MySQL y una base de datos en limpio.

  • Crear una nueva base de datos

Paso 4 – Transfiere tu instalación de WordPress

Transfiere los archivos de WordPress que respaldaste en el primer paso a tu nuevo dominio. Si tenías direcciones de correo electrónico en el sitio anterior (oldsite.example.com), asegúrate de crear las mismas direcciones en tu nuevo dominio. Luego puedes transferir esos correos electrónicos.

  1. Importa tus archivos web usando el siguiente artículo:
    • FTP
  2. Importa tu base de datos usando el artículo Migrar MySQL:
    • Migrar MySQL
  3. Transfiere correo electrónico de tu antiguo dominio:
    • Configuración de correo electrónico: Usa este artículo para configurar tus direcciones de correo electrónico en el nuevo dominio.
    • ¿Cómo importar correo electrónico?: Este artículo explica cómo importar tu correo electrónico al nuevo dominio.
    • Configuración del cliente de correo electrónico: Este artículo es una guía de las diferentes configuraciones de correo electrónico que puedes usar.

Paso 5 – Edita el nuevo archivo wp-config.php

Accede a tu nuevo sitio (newsite.example.com) vía FTP o  SSH y abre el archivo wp-config.php (dentro del directorio web de tu sitio). Verás el siguiente contenido:

define('DB_NAME', ‘exampledatabase’);
define('DB_USER', 'exampledbuser');
define('DB_PASSWORD', ‘database password here’);
define('DB_HOST', 'mysql.example.com');
define('DB_CHARSET', 'utf8');
define('DB_COLLATE', '');

$table_prefix = 'wp_2hdy121d_';

Actualiza lo siguiente:

  • Database name (Nombre de la base de datos)
  • Database user (Usuario de la base de datos)
  • Database user password (Contraseña del usuario)
  • Hostname (Nombre del host)
  • Reemplaza el valor existente de $table_prefix de WordPress con el valor importado de $table_prefix.

Paso 6 – Actualiza todas las URL en la base de datos

Ahora debes actualizar todas las URL en la base de datos. Consulta el siguiente artículo para ver varias opciones sobre cómo lograr esto:

  • Cambiar todas las URL en tu base de datos de WordPress

Cuando actualices todas las URL, también se actualizarán las URL de HOME (INICIO) y SITE URL (SITIO).

En este punto, tu sitio web debería cargar normalmente en el nuevo nombre de dominio.

Resumen

Una vez que se completan los pasos anteriores, el nuevo sitio se ve exactamente igual a como aparecía en su antiguo nombre de dominio.
