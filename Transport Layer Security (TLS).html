Generalidades

Transport Layer Security (TLS) y su predecesor, Secure Sockets Layer (SSL), son protocolos criptográficos que envían comunicaciones seguras en Internet para cosas como navegación web, correo electrónico, fax, mensajería instantánea y otras transferencias de datos.

Para obtener una descripción general de por qué deberías agregar un certificado SSL a tu sitio, revisa el siguiente artículo:

  • Generalidades de los certificados SSL

¿Qué versiones de TLS están disponibles en DreamHost?

TLS 1.3

Los servidores de DreamHost que ejecutan Ubuntu Bionic o Debian Buster son compatibles con TLS 1.2 y 1.3. Revisa el siguiente artículo para comprobar qué sistema operativo está ejecutando tu servidor.

  • Ver la versión del sistema operativo de tu servidor

TLS 1.2

Los servidores virtuales privados más antiguos (VPS) que ejecutan Debian Stretch solo admiten TLS 1.2. Revisa el siguiente artículo para comprobar qué sistema operativo está ejecutando tu servidor.

  • Ver la versión del sistema operativo de tu servidor

TLS 1.1

A partir del 29 de enero de 2018, DreamHost ya no es compatible con TLS 1.0 en tus servidores web. Si todavía usas un navegador que solo puede conectarse usando TLS 1.0, no podrás ver los sitios web alojados en DreamHost usando HTTPS. Esto significa que cualquier sitio que use un certificado SSL.

Revisa el sección Verificar la compatibilidad TLS del navegador de abajo par asegurarte que tu navegador es compatible con las versiones 1.2 en adelante.

Probar el soporte de tu sitio web

Si tu sitio web tiene un certificado SSL, puedes probar la versión TLS en los siguientes sitios:

  • SSL Server Test (SSL Labs)
  • WordPress TLS 1.2 compatibility test
