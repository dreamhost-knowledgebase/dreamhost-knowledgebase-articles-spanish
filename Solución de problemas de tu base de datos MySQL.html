Generalidades

A continuación se proporcionan soluciones a algunos problemas comunes que pueden ocurrir en una base de datos MySQL.

Acceso a SSH

No hay acceso SSH a un servidor MySQL Shared o Privado. Toda la resolución de problemas debe realizarse en el panel de DreamHost o dentro de tu software.

Si necesitas acceso a SSH, debes mejorar a un Servidor Dedicado o DreamCompute, donde la base de datos está alojada en la misma máquina.

Iniciar sesión

Si no puedes iniciar sesión con el usuario, el problema puede estar en la configuración de tu archivo Hosts permitidos.

Actualizar los 'Hosts Permitidos'

Puedes administrar esta configuración en la página Bases de Datos MySQL. A la derecha de tu base de datos, da clic en el nombre de usuario para ver la página de configuración.

En la imagen de arriba, reemplaza xxx.xxx.xxx.xxx por la dirección IP de tu computador. Puedes encontrarla en un sitio como whatismyipaddress.com. El panel también muestra tu dirección IP actual en esta sección.

Asegúrate de que tu dirección IP actual se agregue a este campo.

Inconvenientes del servidor

Planes Shared y VPS

Reiniciar el servidor puede resolver a menudo problemas comunes del servidor. Sin embargo, debido a la naturaleza del alojamiento MySQL compartido, no es posible reiniciar el servidor MySQL. Si notas salgún problema relacionado con el servidor, contacta a soporte para obtener más ayuda.

Planes Dedicados

El reinicio solo es posible si tienes un plan de Alojamiento Dedicado. En un Servidor Dedicado, el servidor MySQL está alojado en la misma máquina, lo que te permite tener acceso completo como root.

Erroress al importar a una base de datos con mysqlimport

DreamHost ha implementado algunas características más nuevas en MySQL 8, que pueden provocar que se produzca un error al utilizar el programa de línea de comandos mysqlimport. Para obtener más detalles, revisa la página Generalidades de MySQL.
