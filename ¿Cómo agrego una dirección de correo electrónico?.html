Generalidades

Este artículo explica cómo agregar una dirección de correo electrónico a tu panel DreamHost.

Prerrequisitos

Debes tener un plan de alojamiento activo, o un Plan adjunto de correo electrónico pago para usar correo.

Todos los planes de alojamiento, excepto Starter Shared, ofrecen correo electrónico de forma gratuita.

Las cuentas de planes de Starter Shared y registro de dominio solo no vienen originalmente con correo electrónico, sin embargo, puedes agregar un plan de correo electrónico pagado en su cuenta para hasta 5 direcciones de correo electrónico. Consulte el siguiente artículo para obtener más información.

  • Plan adjunto de correo pago

Agregar una nueva dirección de correo electrónico

Visita la página Manage Email en tu panel:

Haz clic en el botón Create New Email Address.

Correo totalmente alojado

En el pasado, era posible crear un correo electrónico totalmente alojado que también se reenviaba a otra bandeja de entrada. Esta característica ya no es compatible, y las nuevas direcciones de correo electrónico creadas no tendrán esta opción. Las direcciones de correo electrónico existentes que usan esta función no se verán afectadas hasta que se editen, momento en el que deberás seleccionar entre reenviar solo o totalmente alojado.

Una dirección de correo electrónico totalmente alojada es una dirección de correo electrónico que tiene un buzón activo para iniciar sesión para enviar y recibir mensajes. Estos usuarios de correo electrónico totalmente alojados tendrán acceso a su propio buzón, así como a la interfaz de Webmail y la interfaz de Mailboxes. También puede configurar estos usuarios en programas de correo de terceros si no desea acceder a su buzón a través de la interfaz de Webmail.

Descripción de los campos en la página de correo electrónico completamente alojada

Email settings

Email Address
A la izquierda del símbolo "@", ingrese tu alias deseado. A la derecha del símbolo "@", selecciona el dominio o subdominio en el que se creará el usuario.
Malibox Login
Como dice, "¡solo usa la dirección de correo electrónico completa!"
Mailbox Name
Este es el nombre que aparecerá en la página Manage Email del panel para este buzón.
Pick a new password for me
Si no tienes el sistema configurado para elegir la contraseña por ti, puedes establecer tu propia contraseña personalizada aquí.

La contraseña personalizada debe tener 8 caracteres y no puede contener el nombre de usuario del usuario para el que se configura. Ejemplo: El usuario "usuario@ejemplo.com" no puede tener "usuario" en la contraseña. Publicará el siguiente error cuando intente establecer la contraseña de esta manera:

“Bad Password: password contains username”

Change Password? / New Password Again
Este campo aparecerá si elige establecer su propia contraseña. Aquí, deberá ingresar nuevamente la contraseña personalizada ingresada en el campo ‘Password’ para verificar que la tiene correcta.
Limit total disk usage
Ver también: Limite el uso total del disco
En esta sección, puedes establecer el tamaño del buzón de tu usuario. El campo se completará con el tamaño predeterminado de 200 MB. Puedes editar el tamaño del buzón ingresando un valor de número diferente.

Si desmarcas la casilla 'MB', establecerás el tamaño del buzón del usuario en ilimitado. Los buzones ilimitados se mostrarán en la página Manage Email para el usuario como '10 MB/N/A MB (N/A%)' en la columna Mailbox.

Email me daily warnings
Puedes habilitar la opción de que DreamHost envíe un aviso por correo electrónico a esta dirección de correo electrónico cuando el espacio total en disco para el buzón esté dentro del 10% (mostrando el 90% en la página Manage Email para el usuario) o si está dentro de los 100 MB de espacio restante para ese usuario. Simplemente marca la casilla de esta opción para habilitarla. (Esta es una característica opcional, por lo que puede elegir no habilitarla si lo deseas).

Configuración del archivador de la bandeja de entrada

DreamHost purga todos los correos electrónicos no deseados y basura después de 15 días. Asegúrate de revisar tus mensajes de spam y basura dentro de este plazo.

La siguiente sección, qué está etiquetada como Tidy inboxes keep mail servers happy! En la sección Fully Hosted Email, es para la configuración del Archivador de Bandeja de entrada para su usuario. Esto archiva los mensajes antiguos de tu Bandeja de entrada en otra carpeta para que la Bandeja de entrada no se llene demasiado de mensajes, lo que puede provocar problemas de inicio de sesión con tu usuario.

Remove read messages when inbox reaches
Los mensajes que ha leído se eliminarán de la bandeja de entrada cuando el número total de correos electrónicos en la bandeja de entrada llegue a este número. Los correos electrónicos más antiguos se moverán primero.

El número que establezcas aquí debe estar entre 1 y 2000.

Remove read messages from inbox older than
Los mensajes que ha leído se eliminarán cuando sean mayores que la cantidad de días que configuraste en este campo.

El valor ingresado aquí debe estar entre 1 y 9999.

Even remove unread messages
Si marcas esta casilla, los mensajes que aún no se leyeron también se eliminarán de la Bandeja de entrada.
Save removed messages
En este campo, puedes establecer el nombre de la carpeta en la que deseas que se archiven los mensajes. De manera predeterminada, este campo se completará con "mensajes antiguos", por lo que todos los mensajes que archiva Inbox Archiver moverán los mensajes a esta carpeta siempre que tenga la casilla marcada.

Es vital que tengas marcada la casilla de verificación para esta configuración si deseas mantener los mensajes antiguos almacenados. Si no tienes marcada la casilla de verificación, los mensajes capturados por Inbox Archiver se eliminarán.

Email me when messages are removed
Marca esta casilla de verificación para que el sistema te envíe un correo electrónico (en esta dirección) cuando se hayan aplicado las reglas anteriores y se hayan archivado los mensajes.

Guardando tus cambios

Una vez que hayas configurado al usuario como un usuario de correo electrónico totalmente alojado, simplemente puedes hacer clic en el botón Create Address en la parte inferior de la página para guardar la nueva información.

Si el usuario se configuró correctamente, deberías ver el siguiente mensaje en la parte superior de la página Manage Email. Si eliges que el panel cree una contraseña por ti, se muestra ahora.

Si este es el primer usuario de correo electrónico creado para el dominio, el registro MX y los demás registros de correo se establecerán para el dominio una vez que se haya creado el usuario. Esto requerirá hasta 6 horas para que el DNS se resuelva para cada registro de correo. Debido a esto, el usuario no será accesible hasta que el DNS se haya actualizado.

Eliminar una dirección de correo electrónico

Para eliminar una dirección de correo electrónico en el panel:

  1. Navega hasta la página Manage Email.
  2. Haga clic en el botón Delete a la derecha de su dirección de correo electrónico.
  3. Cuando se le solicite, seleccione "OK" para confirmar la solicitud de eliminación.

¿Cuánto tiempo tarda una nueva dirección de correo electrónico en funcionar?

De forma predeterminada, tu dominio no está configurado con registros MX (alojamiento de correo) con DreamHost cuando el dominio está configurado para ser alojado. Los registros MX y todos los demás registros de alojamiento de correo se aplican al dominio una vez que se crea la primera dirección de correo electrónico.

Después de crear su dirección de correo electrónico (y si esta es la primera dirección de correo electrónico creada bajo el dominio), el dominio necesitará algo de tiempo para que se propague el DNS del registro de correo, lo que puede demorar hasta 6 horas para cada registro.

Si tu dominio ya está configurado con los registros para el alojamiento de correo, el usuario recién creado debe estar completamente activo dentro de los 15 minutos posteriores a su creación.

Enviar y recibir correos electrónicos utilizando tu nueva dirección de correo electrónico

A continuación se muestran los registros que se configuran una vez que se crea el primer usuario:

  • _domainkey (dkim)
  • _autodiscover (srv)
  • mail. (A)
  • mail. (MX)
  • mail. (MX)
  • mailboxes. (A)
  • www.mailboxes. (A)
  • webmail. (A)
  • www.webmail. (A)

Hay dos registros MX para el alojamiento de correo del dominio junto con los registros de los subdominios para buzones y correo web. Cada registro puede resolverse en diferentes momentos, por lo que un servicio puede estar activo mientras que otro aún no se ha actualizado.

Durante este tiempo, los mensajes enviados a tu usuario serán rechazados con una notificación de devolución que contiene el error a continuación:

“Recipient Address Rejected”

Hasta que se resuelva el DNS, los mensajes enviados a tu usuario continuarán siendo rechazados. Sin embargo, tu usuario debería poder enviar mensajes incluso cuando el DNS aún no se haya resuelto. Esto a menudo causa confusión sobre por qué un usuario puede enviar, pero no puede recibir correos electrónicos. Déle un poco más de tiempo para que el DNS se resuelva antes de que el nuevo usuario pueda recibir el correo entrante.

Para enviar y recibir correos electrónicos, DreamHost ofrece la Interfaz de Webmail or defecto. Se puede acceder a esto una vez que el DNS resuelve el registro del correo web. Puede acceder a la interfaz de Webmail con la siguiente URL: https://webmail.dreamhost.com.

Direcciones de correo electrónico estándar de Internet

Los estándares de Internet especifican algunas direcciones de correo electrónico que todos los dominios deberían tener y utilizar para fines estándar. No hay policía de Internet que te castigue por no tenerlos, pero tener estas direcciones de correo electrónico no es demasiado problema y puede ayudar a evitar problemas.

Dirección de correo electrónico Propósito
postmaster@example.com contacto para cualquier problema de correo electrónico, debe existir para cualquier dominio que tenga servicio de correo electrónico
webmaster@example.com contacto para cualquier problema del sitio web, debe existir para cualquier dominio que tenga un sitio web o servicio http
hostmaster@example.com contacto para cualquier problema de DNS, debe existir para todos los dominios ya que todos usan DNS para todo
abuse@example.com contacto para cualquier "comportamiento público inapropiado"
