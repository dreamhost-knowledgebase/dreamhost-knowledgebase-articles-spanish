Generalidades

Este artículo explica cómo habilitar el modo de depuración en WordPress para escribir errores en un archivo de registro sin alterar tu sitio web en vivo.

Antecedentes

WordPress ofrece una herramienta de depuración que te ayuda a encontrar la causa de los errores de tu sitio web. Esta herramienta puede mostrar información de dos formas:

  • En tu sitio en vivo — Esto no se recomienda si tu sitio web está activo para el público, pero puede ser una forma rápida de solucionar un problema.
  • Crear un archivo de registro de repuración — Esto crea un archivo de registro en el servidor para ayudarte a solucionar un problema.

Habilitar el modo DEBUG

Para habilitar el modo de depuración:

  1. Inicia sesión en tu servidor por medio de SSH o FTP.
  2. Edita el archivo wp-config.php usando SSH o tu cliente FTP.
    La siguiente línea aparece cerca de la parte inferior:
    define('WP_DEBUG', false);
  3. Reemplaza esa línea con estas tres líneas: 
    define( 'WP_DEBUG', true );
define( 'WP_DEBUG_DISPLAY', false );
define( 'WP_DEBUG_LOG', true );

    No se recomienda cambiar WP_DEBUG_DISPLAY a true (verdadero). Esto se debe a que cualquier error se mostrará en tu sitio web en vivo.

    Al habilitar WP_DEBUG_LOG, puedes ver los mismos errores en tu archivo /wp-content/debug.log en vez.

Cuando ocurre un error en WordPress, este escribe en un archivo titulado debug.log, ubicado en tu directorio /wp-content/. Abre este archivo usando SSH o tu cliente FTP para ver el informe de debug.

Usar un plugin para ver el registro de depuración

También puedes usar un plugin para ayudarte a depurar tu sitio y ver el registro de depuración. Revisa este artículo para obtener información sobre cómo administrar e instalar un plugin Cuando busques un plugin, intenta escribir debug como término de búsqueda, ya que hay muchas opciones diferentes entre las que puedes elegir.
