Generalidades

Si tienes problemas para migrar tu sitio web desde otro host, o simplemente no tienes tiempo, DreamHost actualmente ofrece un servicio que se encarga de esto por ti. Puedes encontrar más información en el artículo de DreamHost Servicios de migración para sitios web.

La migración de un sitio de WordPress es diferente de la migración de un sitio web HTML sencillo ya que los sitios de WordPress dependen tanto de los archivos web como de una base de datos MySQL para ejecutarse. Esta descripción general explica tus opciones según el tipo de migración que necesitas.

Migración de un sitio de WordPress desde un hosting diferente

Si estás migrando un sitio de WordPress desde una empresa de alojamiento diferente (host), hay muchas opciones disponibles para ti. Estos son algunos de los métodos más populares entre los que puedes elegir:

  • Migrar tu sitio usando el DreamHost Automated Migration WordPress plugin
  • Pedir el Servicio Profesional de DreamHost para migrar tu sitio por una tarifa (como se explica en el cuadro de notas en la parte superior de esta página)
  • Migrar tu sitio manualmente usando FTP y phpMyAdmin

Mantener el mismo nombre de dominio

Si mantienes el mismo nombre de dominio y deseas minimizar el tiempo de inactividad en el sitio actual mientras estás migrando, puedes beneficiarte de configurar un mirror de tu sitio DreamHost utilizando un subdominio temporal .dreamhosters.com:

  • Vista previa de tu sitio usando un subdominio gratuito dreamhosters.com

Cambiar el nombre de dominio

Si estás migrando su sitio de WordPress a DreamHost utilizando el plugin de migración automatizada DreamHost, o con la ayuda de los servicios profesionales de DreamHost, no es necesario actualizar manualmente WordPress con el nuevo nombre de dominio una vez que se complete el movimiento. El complemento de migración automatizada DreamHost cambia el nombre de dominio por tí durante el proceso de migración, y cualquier migración realizada por el equipo de servicios profesionales de DreamHost incluye cambiar el nombre de dominio si es necesario.

Si eliges no usar una de estas opciones y migrar manualmente usando FTP y phpMyAdmin, debe actualizar manualmente tu sitio migrado al nuevo nombre de dominio. Si tu sitio actual de WordPress no es administrado por el Instalador de un clic, puedes enviar un ticket para solicitar que tu sitio se convierta en una Instalación de un clic.

Solución de problmas al descargar archivos PHP después de una migración

En algunas circunstancias, un sitio migrado a DreamHost desde otro proveedor de alojamiento no carga correctamente. Cuando accedas al sitio, el buscador descargará un archivo PHP en vez de mostrar el sitio web. Esto es causado típicamente por código que sobre del anterior alojador, muy comunmente por la prescencia de una entrada AddHandler en el archivo .htaccess del sitio como el ejemplo de abajo:

AddHandler application/x-httpd-php70 .php

Este código no es necesario en los servidores de DreamHost y debería ser eliminado del archivo .htaccess de WordPress luego de que el sitio se haya migrado.
