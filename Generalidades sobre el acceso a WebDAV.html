Generalidades

WebDAV es esencialmente una forma colaborativa para que los usuarios editen y administren archivos en un servidor web remoto. Tomado de WebDAV en Wikipedia:

Web Distributed Authoring and Versioning (WebDAV) es una extensión del Protocolo de transferencia de hipertexto (HTTP) que permite a los clientes realizar operaciones remotas de creación de contenido web.

Esta guía describe los pasos necesarios para conectarte a un directorio WebDAV.

Credenciales de inicio de sesión

Debes tener los siguientes tres elementos básicos de información para conectarte:

  1. La URL WebDAV del sitio – Esta es la URL a la que te conectarás. Esta información se puede obtener en el panel web bajo Htaccess/WebDAV.
    • El panel de control de DreamHost requiere que WebDAV se configure en una sub carpeta real de la raíz de un dominio (no en la raíz de un dominio). Esto se puede solucionar mediante la configuración manual.
  2. El nombre de usuario y contraseña de la cuenta WebDAV.
  3. Un apodo para la conexión/atajo – Esto es para recordarte el sitio al que te estás conectando.

Programas de clientes alternativos de WebDAV

Puedes utilizar clientes alternativos que tienen características adicionales que incluyen:

  • caché local – para evitar viajes de ida y vuelta al servidor, que pueden ser lentos para usted o costosos para el servidor.
  • acceso fuera de línea – para trabajar mientras estás desconectado.
  • versioning – permisos personalizados u otras características que tu servidor WebDAV puede ofrecer solo a través de un cliente más avanzado.

Los clientes alternativos (y a veces no gratuitos) incluyen:

  • Map Drive
  • Web Drive
