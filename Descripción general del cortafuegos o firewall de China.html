Generalidades

El gobierno chino ha implementado un firewall nacional para bloquear el acceso de sus ciudadanos a dominios específicos (no solo dominios alojados con DreamHost sino también con otras empresas de alojamiento web). Si bien DreamHost no aprueba ni respalda su censura del contenido de Internet, no hay nada que se pueda hacer directamente para evitarlo. Sin embargo, hay algunas posibles soluciones que puedes probar.

Experiencia técnica

El tipo de servicio de alojamiento web que DreamHost proporciona se denomina "alojamiento virtual", lo que significa que cada una de las muchas instancias de servidor web Apache que se ejecutan en cualquier servidor tiene una dirección IP específica. Muchos dominios diferentes están alojados bajo esa misma dirección IP. En esta configuración, el servidor usa la parte del nombre de dominio de la URL solicitada para determinar qué sitio mostrar.

En pocas palabras, muchos dominios diferentes comparten la misma dirección IP.

Cuando el firewall chino bloquea un dominio, bloquea la dirección IP particular que está utilizando un dominio. Desafortunadamente, también bloquea todos los demás dominios que comparten esa misma dirección IP.

Mi dominio está bloqueado, ¿qué puedo hacer al respecto?

La única medida de prevención actual que puedes tomar es asignar una IP única a tu dominio. Al tener tu propia dirección IP única, tu dominio ya no comparte una dirección IP con otros dominios. Esto significa que si tu dominio se bloquea, es porque bloquearon TU dominio específicamente y no por el dominio de otra persona en la misma dirección IP del servidor compartido.

Lea la sección Notas y advertencias por completo antes de agregar una dirección IP única.

Notas y advertencias

  • DreamHost NO puede garantizar que al cambiar la dirección IP de tu dominio, el firewall chino te desbloqueara, ya que es posible que también bloqueen la nueva dirección IP única.
  • Ten en cuenta que la IP única que compras se asigna automáticamente y es posible que ya esté bloqueada.
  • Cuando compra una IP única, el DNS debe propagarse a través de Internet, lo que puede llevar varias horas.
  • Al agregar, eliminar o cambiar una dirección IP única, la dirección IP anterior continúa aceptando conexiones para tu dominio durante aproximadamente 5 días. Esto debería permitir suficiente tiempo para que la nueva información de DNS se propague completamente por todo Internet. La nueva dirección IP única debería comenzar a aceptar conexiones dentro de aproximadamente 4 horas, pero puede demorar hasta 72 horas.
  • Durante la transición de la configuración, no es raro ver la página "Site not found" al acceder a tu dominio durante las primeras horas. Esto generalmente se corrige solo.
  • Si no estás utilizando los Nameservers de DreamHost para manejar el DNS de tu dominio, debes actualizar sus servidores de nombres manualmente con la nueva dirección IP para tu dominio. Puedes ver tu información de DNS en tu panel. Por favor revisa el artículo DNS para más detalles. Asegúrate de duplicar tu configuración en la compañía que aloja tus servidores de nombres para que tu dominio continúe funcionando.
  • Hay un cargo adicional por cada dirección IP única que agregue.
  • Si tienes derecho a una dirección IP única gratuita, no se aplicará ningún cargo cuando agregue la primera y no se aplicará ningún reembolso cuando se elimine.
  • Al eliminar una dirección IP única de un dominio, recibirás un reembolso prorrateado a tu cuenta por la parte de tiempo no utilizada (para ese ciclo de facturación).
