Generalidades

Este artículo proporciona información sobre las fusiones de bases de datos y el proceso que se lleva a cabo cuando se programa una fusión de bases de datos en el panel DreamHost.

¿Por qué es esto un problema?

La mayoría de las veces, los servicios de tu base de datos están siempre en un solo servidor de base de datos. Sin embargo, es posible que tengas bases de datos divididas en dos servidores de bases de datos diferentes. Se recomienda encarecidamente utilizar un único servidor de base de datos, ya que esto hace que la resolución de problemas relacionados con la base de datos sea mucho más fácil y simplifica el acceso a todas tus bases de datos desde un único conjunto de usuarios y nombres de host.

Mover tus bases de datos si tienes múltiples servidores de bases de datos

  1. Navega hasta la página Bases de Datos MySQL.
  2. Desplázate hacia abajo.
  3. Haz clic en el botón ¡Mover mis bases de datos!.

Ten en cuenta que la migración no comienza inmediatamente. Generalmente, estas fusiones se hacen a media noche (PDT/PST). También se te da la oportunidad de cancelar el proceso si decides no continuar con la migración.

Después de hacer clic en el botón, un mensaje te informa que esta migración tomará varias horas en completarse y que tu sitio puede no funcionar correctamente durante este proceso.

Cuando termine la migración, se te enviará una confirmación por correo.

Que esperar

A la medianoche o después, se ejecuta un script automatizado y consolida tus bases de datos en un servidor. Una vez que la mudanza se realiza correctamente, se envía una notificación a tu dirección de correo electrónico.

Si toma más de unas pocas horas, es posible que el script haya tenido algún tipo de problema, y debes comunicarse con el soporte para obtener más ayuda:
