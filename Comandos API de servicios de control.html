Si ves que alguna de las instrucciones de este artículo no funciona como se esperaba, es posible que esté desactualizada. En caso de duda, por favor comunícate con el soporte para obtener más ayuda.

Cualquier cambio a tu cuenta que requiera tiempo para configuarse, se corre como un servicio, El API de services te permite ver el estado de esos servicios. Cuando un comando API necesita correr un servicio, retorna un token único para intentificar el servicio.

Comandos relacionados

Los siguientes comandos devuelven un "token" en caso de éxito, que se puede usar para verificar el estado:

  • jabber-add_user
  • jabber-remove_user
  • jabber-reactivate_user
  • jabber-deactivate_user
  • user-add_user
  • services-flvencoder

Comandos

services-progress

Usa este comando para obtener un informe de estado del trabajo que se está programando. Si el token es para un servicio que se completó hace más de varios días, puede devolver el error 'too_old'.

Comando services-progress
Valor requerido

token
Resultado

success
status scheduled executed finished
success 2010-01-15 14:47:05 2010-01-15 14:47:16 2010-01-15 14:47:17
Posibles errores

invalid_token
too_old
  • scheduled es el momento en el que este comando fue agregado a la cola del servicio.
  • executed es el momento en el que el comando empezó a correr
  • finished es el momento en el que el comando terminó.
  • status puede ser:
    • "waiting": este comando está programado para correrse y actualmente está esperando su turno para ejecutarse.
    • "running": este comando se está corriendo actualmente.
    • "success": este comando se completó con éxito.
    • "failure": este comando se completó sin éxito.

services-flvencoder

Inicia el conversor de Flash Video en un video especificado por una URL.

Comando services-flvencoder
Valor requerido

url (URL completa de un vídeo o directorio a convertir).
Valores opcionales

dim (tamaño del vídeo en formato "400x300").
snap (número de seguntos dentro del vídeo para generar una vista previa).
ab (tasa de bits de audio).
ar (frecuencia de muestreo de audio).
batch (establecer si pasa una ruta a un directorio para convertir todos los archivos).
quiet (solo envía un correo para una conversión por lotes).
noemail (no envía correos).

Devuelve un token de cola de servicio, se puede usar con services-progress.
