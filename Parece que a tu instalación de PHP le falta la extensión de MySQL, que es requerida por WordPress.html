Generalidades

Al visitar tu sitio de WordPress, es posible que veas este error:

Your PHP installation appears to be missing the MySQL extension which is required by WordPress.

Esto generalmente se debe a una personalización de la versión de PHP en tus archivos .htaccess o phprc.

Comprueba qué versión de PHP está usando tu sitio

Primero, verifica qué versión de PHP está usando tu sitio con un archivo phpinfo.php.

  • ¿Cómo veo la configuración de mi entorno PHP?

Asegúrate de que coincida con la versión de PHP que muestra tu panel para tu dominio.

  • ¿Qué versión de PHP está usando mi sitio?

Si hay una discrepancia, hay una personalización potencial en algún lugar que hace que las versiones sean diferentes.

Comprueba si hay personalizaciones

Revisa tu archivo .htaccess en búsqueda de personalizaciones de PHP

Primero, verifica el archivo .htaccess en el directorio principal de tu sitio web. Lo más probable es que se encuentre aquí:

  • /home/username/example.com/.htaccess

Abre ese archivo en un cliente FTP o SSH. Asegúrate de que no haya líneas que hagan referencia a una versión diferente de PHP.

Revisa tu archivo phprc en búsqueda de personalizaciones de PHP

Después de confirmar qué versión de PHP estás usando el sitio anterior, verifica tu archivo phprc file. Lo puedes encontrar aquí:

  • /home/username/.php/php#.#/phprc

Asegúrate de ajustar el php #. # a la versión de PHP que está usando tu sitio. Verifica tu archivo phprc utilizando un cliente FTP o SSH para asegurarte de que no haya personalizaciones que puedan causar un problema.

Para servidores Dedicated

Si estás en un servidor Dedicated, tu versión de MySQL puede ser 5.6. Si es así, y estás viendo este error, agrega la siguiente línea al archivo phprc de su sitio.

extension = mysql.so
