Generalidades

La vulnerabilidad de log4j no afecta a los servidores administrados de DreamHost. Esto incluye los siguientes planes de alojamiento:

  • Alojamientos compartidos
  • DreamPress
  • VPS
  • Servidores Dedicados (sin un usuario administrador)

Esto se debe a que los servidores administrados de DreamHost no proporcionan ni ofrecen ninguna herramienta o servicio Java que incluya la vulnerabilidad log4j.

Servidores no administrados

Hay dos tipos de planes de alojamiento de DreamHost que son administrados por la ONU. Éstos incluyen:

  • Servidores Dedicados con un usuario administrador
  • DreamCompute

Estos tipos de servidores permiten el acceso root/sudo, lo que permite al usuario administrador instalar manualmente software Java que podría incluir llamadas a la clase vulnerable.

DreamHost recomienda que los clientes que hayan instalado software basado en Java lo actualicen lo antes posible. Envía un ticket al soporte de DreamHost si tienes más preguntas.
