Generalidades

En caso de que haya problemas con los filtros de spam configurados para tus usuarios, existen algunos pasos básicos de solución de problemas que puedes seguir. A continuación, se muestran algunos problemas comunes que pueden ocurrir y lo que puede hacer para resolverlos.

Sigue apareciendo demasiado Spam en la bandeja de entrada

  • Configura listas de bloqueo para todo el dominio en lugar de solo el correo electrónico específico que envía el mensaje. De esta forma, bloquearás todos los correos electrónicos de ese dominio.
  • Mover manualmente un mensaje de la Bandeja de entrada a la carpeta Spam reportará automáticamente ese correo electrónico al sistema de filtrado. Puedes mover mensajes a la carpeta Spam para ayudar a entrenar el filtro.
  • Es posible que el filtro antispam no detecte todos los mensajes basura entrantes debido a cómo se envió o se creó el mensaje. DreamHost ofrece la opción de configurar filtros de mensajes personalizados, y puedes consultar el artículo Filtros de Mensaje (para usuarios avanzados) para obtener más información.

Los correos no se están enviando a la carpeta de Spam

Si el dominio no utiliza los nameservers de DreamHost, asegúrate de actualizar manualmente el DNS de tu dominio con los nuevos registros MX de filtrado antispam. Consulta el siguiente artículo para obtener instrucciones sobre cómo ubicar tus registros MX:

  • Ubicando tus registros DNS de correo de DreamHost

Se ven así:

  • 0 mx1.mailchannels.net
  • 0 mx2.mailchannels.net

Los correos de una dirección de lista de permitidos aún se envía a la carpeta Spam

  • Es posible que la dirección de remitente del correo no coincida con el lugar desde donde se envió el correo. Intenta ver los encabezados de correo y permite la dirección de envío que se muestra allí.
  • Si el dominio está en tu lista de Permitidos, verifica la dirección del remitente para ver si se envía desde un subdominio (ejemplo: @notices.domain.com en lugar de @domain.com). Las listas de dominios permitidos no se extienden a los subdominios. Es posible que también deba agregar el subdominio a tu lista de permitidos.
  • Es posible que el mensaje haya fallado en las políticas DMARC, DKIM o SPF de tu remitente. Cuando esto ocurre, las políticas de tu remitente pueden indicarle a DreamHost que anule tu lista de permitidos y prohíba la entrega de este mensaje en la Bandeja de entrada. Tu remitente debe asegurarse de que los correos electrónicos enviados cumplan con las políticas de tu dominio, o se ignorará la lista de permitidos.
  • Si todo está configurado correctamente, aún es posible que un RBL anule la entrada de la lista de permitidos. Esto sucede si una de las direcciones IP involucradas en el envío del mensaje está en una RBL y ha sido reportada globalmente por enviar spam.
