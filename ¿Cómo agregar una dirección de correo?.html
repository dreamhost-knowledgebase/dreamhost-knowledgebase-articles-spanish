Generalidades

Este artículo describe como agregar una dirección de correo a tu cuenta usando el panel de DreamHost.

Prerrequisitos

Debes tener un plan de alojamiento activo, o un Plan adjunto de correo pago para usar correo.

Todos los planes de alojamiento, excepto Shared Starter, ofrecen correo sin costo.

Mientras los planes de Shared Starter y registro de dominio solo no incluyen con correo, puedes agregar un plan de correo pagado en tu cuenta que incluye hasta 5 direcciones de correo. Consulta el siguiente artículo para más información.

  • Plan de correo pago

Agregar una nueva dirección de correo 

Visita la página Administrar Correo en tu panel:

Haz clic en el botón Create New Email Address.

Correo totalmente alojado

En el pasado, era posible crear un correo totalmente alojado que también se reenviaba a otra bandeja de entrada. Esta característica ya no es soportada, y las nuevas direcciones de correo creadas no tendrán esta opción. Las direcciones de correo existentes que usan esta función no se verán afectadas hasta que se editen, momento en el que deberás seleccionar entre reenviar solo o totalmente alojado.

Una dirección de correo totalmente alojada es una dirección de correo que tiene un buzón activo para que puedas iniciar sesión para enviar y recibir mensajes. Los usuarios de correo totalmente alojados pueden acceder a su propio buzón, así como usar la interfaz de Webmail y la interfaz de Mailboxes. También puedes configurar estos usuarios de coreo en programas de correo de terceros si no deseas acceder a tu buzón a través de la interfaz de Webmail.

Descripción de los campos en la página de correo completamente alojada

Email settings

Email Address
A la izquierda del símbolo "@", ingresa tu alias deseado. A la derecha del símbolo "@", selecciona el dominio o subdominio en el que se creará el usuario.
Malibox Login
Como dice, "¡solo usa la dirección de correo completa!"
Mailbox "Name"
El nombre que aparecerá en la página Administrar Correo del panel para este buzón.
Pick a new password for me / Password
Selecciona ésta casilla si quieres que el sistema genere una contraseña por ti. Si no quieres que el sistema escoja la contraseña, puedes establecer tu propia contraseña personalizada aquí.

La contraseña personalizada debe tener 8 caracteres y no puede contener el nombre de usuario del usuario para el que se configura. Por ejemplo: El usuario de correo usuario@example.com no puede tener "usuario" en la contraseña pues se publicará el siguiente error cuando intente establecer la contraseña:

“Contraseña incorrecta: la contraseña contiene el nombre de usuario"

Change Password? / New Password Again
Este campo aparecerá si eliges establecer tu propia contraseña. Ingresa nuevamente la contraseña personalizada que ingresaste en el campo ‘Password’ para verificar que está correcta.
Limit total disk usage
Ver también: Limite el uso total del disco
Puedes establecer el tamaño del buzón de tu usuario. El campo completa el tamaño predeterminado de 200 MB. Puedes editar el tamaño del buzón ingresando un valor de númerico diferente.

Si desmarcas la casilla 'MB', establecerás el tamaño del buzón del usuario en ilimitado. Los buzones ilimitados se mostrarán en la página Administrar Correo para el usuario como '10 MB/N/A MB (N/A%)' en la columna Mailbox.

Email me daily warnings
Cuando habilitas esta opción DreamHost enviará un aviso por correo a esta dirección de correo que indica cuando el espacio total en disco para el buzón esté dentro del 10% de la capacidad restante (mostrando el 90% en la página Administrar Correo para el usuario) o si está dentro de los 100 MB de espacio restante para ese usuario. Simplemente marca la casilla de esta opción para habilitarla. (Esta es una característica opcional, por lo que puede elegir no habilitarla si lo deseas).

Configuración de Inbox Archiver

DreamHost purga todos los correos no deseados y basura después de 15 días. Asegúrate de revisar tus mensajes de spam y basura dentro de este plazo.

La sección 'Tidy inboxes keep mail servers happy!' en Fully Hosted Email, contiene el Archivador de Bandeja de entrada para las configuraciones de tu usuario. El Archivador de Bandeja de Entrada archiva los mensajes antiguos de tu Bandeja de entrada a otra carpeta para que tu Bandeja de entrada no se llene, lo que puede provocar problemas de inicio de sesión con tu usuario.

Remove read messages when inbox reaches
Los mensajes que ha leído se eliminarán de la bandeja de entrada cuando el número total de correos en la bandeja de entrada llegue a este número. Los correos más antiguos son movidos primero.

El número que estableces aquí debe estar entre 1 y 2000.

Remove read messages from inbox older than
Los mensajes que ha leído se eliminarán cuando sean mayores que la cantidad de días que configuraste en este campo.

El valor ingresado aquí debe estar entre 1 y 9999.

Even remove unread messages
Si marcas esta casilla, los mensajes que aún no se leyeron también son eliminados de la Bandeja de Entrada.
Save removed messages
Puedes establecer el nombre de la carpeta en la que deseas que se archiven los mensajes. De manera predeterminada, este campo se completa con "old-messages", por lo que todos los mensajes que archiva Inbox Archiver moverán los mensajes a esta carpeta siempre que tengas la casilla marcada.

Es vital que tengas marcada la casilla de verificación para esta configuración si deseas mantener los mensajes antiguos almacenados. Si no tienes marcada la casilla de verificación, los mensajes capturados por Inbox Archiver se eliminarán.

Email me when messages are removed
Marca esta casilla de verificación para que el sistema te envíe un correo (en esta dirección) cuando se hayan aplicado las reglas anteriores y se hayan archivado los mensajes.

Guardar tus cambios

Una vez que hayas completado el ingreso de todos los detalles para el correo totalmente alojado, puedes solo hacer clic en el botón Create Address en la parte inferior de la página para guardar la nueva información.

Si el usuario se configuró correctamente, deberías ver el siguiente mensaje en la parte superior de la página Administrar Correo. Si eliges que el panel cree una contraseña por ti, se muestra en el mensaje.

Si este es el primer usuario de correo creado para el dominio, el registro MX y los demás registros de correo se establecen para el dominio una vez que se haya creado el usuario. Esto requiere hasta 6 horas para que el DNS se resuelva para cada registro de correo. Debido a esto, el usuario no será accesible hasta que el DNS se haya actualizado.

Eliminar una dirección de correo

Para eliminar una dirección de correo en el panel:

  1. Navega hasta la página Administrar Correo .
  2. Haz clic en el botón Delete a la derecha de tu dirección de correo.
  3. Cuando se te solicite, seleccione "OK" para confirmar la solicitud de eliminación.

¿Cuánto tiempo tarda una nueva dirección de correo en funcionar?

De forma predeterminada, tu dominio no está configurado con registros MX (alojamiento de correo) con DreamHost cuando el dominio está configurado para ser alojado. Los registros MX y todos los demás registros de alojamiento de correo se aplican al dominio una vez que se crea la primera dirección de correo.

Después de crear tu dirección de correo (y si esta es la primera dirección de correo creada bajo el dominio), el dominio necesitará algo de tiempo para que se propague el DNS del registro de correo, lo que puede demorar hasta 6 horas para cada registro.

Si tu dominio ya está configurado con los registros para el alojamiento de correo, el usuario recién creado debe estar completamente activo dentro de los 15 minutos posteriores a su creación.

Enviar y recibir correos usando tu nueva dirección de correo

Los siguientes registros se configuran cuando se crea el primer usuario:

  • _domainkey (dkim)
  • _autodiscover (srv)
  • mail. (A)
  • mail. (MX)
  • mail. (MX)
  • mailboxes. (A)
  • www.mailboxes. (A)
  • webmail. (A)
  • www.webmail. (A)

Hay dos registros MX para el alojamiento de correo del dominio junto con los registros de los subdominios para buzones y correo web. Cada registro puede resolverse en diferentes momentos, por lo que un servicio puede estar activo mientras que otro aún no se ha actualizado.

Durante este tiempo, los mensajes enviados a tu usuario serán rechazados con una notificación de devolución que contiene el siguiente error:

Recipient Address Rejected

Hasta que se resuelva el DNS, los mensajes enviados a tu usuario continuarán siendo rechazados. Sin embargo, tu usuario debería poder enviar mensajes incluso cuando el DNS aún no se haya resuelto. Esto a menudo causa confusión sobre por qué un usuario puede enviar, pero no puede recibir correos. Déle un poco más de tiempo para que el DNS se resuelva antes de que el nuevo usuario pueda recibir el correo entrante.

Para enviar y recibir correos, DreamHost ofrece la Interfaz de Webmail por defecto. Se puede acceder a esto una vez que el DNS resuelve el registro del correo web. Puedes acceder a la interfaz de Webmail con la siguiente URL: https://webmail.dreamhost.com.

Direcciones de correo estándar de Internet

Los estándares de Internet especifican que tu creas las siguientes direcciones de correo específicas, que todos los dominios deben tener y usar para los siguientes propósitos:

Dirección de correo electrónico Propósito
postmaster@example.com Contacto para cualquier problema de correo, debe existir para cualquier dominio que tenga servicio de correo
webmaster@example.com Contacto para cualquier problema del sitio web, debe existir para cualquier dominio que tenga un sitio web o servicio http
hostmaster@example.com Contacto para cualquier problema de DNS, debe existir para todos los dominios ya que todos usan DNS para todo
abuse@example.com Contacto para cualquier "comportamiento público inapropiado"
