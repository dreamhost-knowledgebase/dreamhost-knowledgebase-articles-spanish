Generalidades

Las siguientes instrucciones asumen que ya has creado una base de datos MySQL en el panel.

Inicia sesión en tu panel DreamHost. Cuando hayas ingresado, navega a la página Bases de Datos MySQL. En esta página verás una lista completa de tus nombres de host y bases de datos MySQL.

Administrar los hostnames

Los nombres de host se muestran en la parte superior, con enlaces útiles para el acceso a phpMyAdmin. Las bases de datos se enumeran debajo de los nombres de host.

Agregar un nuevo hostname

Haz clic aquí para mostrar la nueva página de hostname que permite ingresar un nombre y seleccionar un dominio para tu nuevo hostname.

Eliminar un hostname

Haz clic aquí para eliminar el hostname que figura a la izquierda del botón. Eliminar un hostname detiene el acceso a la base de datos para los sitios que aún lo utilizan. Siempre debe haber al menos un hostname, por lo que si solo existe uno, este botón se deshabilita hasta que se agregue otro hostname.

Administrar bases de datos MySQL existentes

Agregar un usuario

Haz clic aquí para mostrar una nueva página que permita crear un nuevo usuario de MySQL o seleccionar un usuario existente que debería tener privilegios para la base de datos.

Restaurar la base de datos

Haz clic aquí para mostrar la página de restauración de la base de datos que te permite seleccionar de una lista desplegable de copias de seguridad de la base de datos disponibles y cómo se deben restaurar las tablas. Las tablas de la copia de seguridad pueden restaurarse con una marca de tiempo en la base de datos, o restaurarse con nombres de tablas originales con tablas existentes renombradas con una marca de tiempo. No se recomienda restaurar una copia de seguridad de la base de datos a los nombres de la tabla original, ya que puede causar una interrupción en el servicio a tu sitio si no espera los datos más antiguos.

Modificar la base de datos

Esto te permite modificar el texto de descripción opcional de tu base de datos.

Eliminar la base de datos

Haz clic para eliminar la base de datos de tu cuenta. El acceso a la base de datos está deshabilitado y permanece en la Papelera de reciclaje de MySQL durante 30 días adicionales hasta que se elimine de forma permanente. Haz una copia de seguridad de la base de datos antes de eliminarla, ya que DreamHost no puede garantizar la disponibilidad de copias de seguridad para bases de datos eliminadas.
