Generalidades

Muchos sitios web usan PHP para enviar correos a través de SMTP. Por ejemplo, puedes crear un formulario de correo personalizado o usar un complemento SMTP de WordPress.

Un problema con el que te encontrarás es que la configuración de seguridad de Google no te permite enviar a través de SMTP a través de tus servidores a menos que lo permitas explícitamente. Hay dos formas de permitir que tu aplicación envíe correo SMTP a través de GMAIL.

Habilitar el protocolo OAuth 2.0 para autenticar conexiones

La primera opción y más segura es habilitar la autenticación OAuth 2.0 para tu aplicación.

  • https://developers.google.com/gmail/api/auth/about-auth
  • Habilitar el API OAuth 2 para usar con tu plugin Mail SMTP de WordPress

Ten en cuenta que el soporte no puede ayudarte con esta configuración personalizada.

Habilitar 'Less Secure Apps' dentro de GMAIL

Habilitar 'Less secure apps' requiere que primero desactives la verificación en dos pasos en tu cuenta de Google, por lo que no se recomienda.

La segunda opción es habilitar 'Less secure apps' dentro de GMAIL.

  1. Inicie sesión en tu cuenta de GMAIL.
  2. Navega hasta la página 'Less secure apps'.
  3. Alterna esto para activar esta función y dejarla en 'ON'.
  4. Espera al menos una hora para que Google actualice esta configuración.
  5. Prueba tu aplicación de correo nuevamente. Ahora deberías enviar correctamente a través de SMTP.
