Generalidades

Este artículo explica cómo actualizar todas las URL en la base de datos de WordPress, implementando un plugin.

Si recientemente has agregado un certificado SSL a tu sitio, algunos de los enlaces o “links” en tu base de datos están apuntando a la versión 'HTTP' de tu URL. Debes actualizar dichas URL para que apunten a la versión HTTPS de tu sitio web para que puedas usar el certificado SSL. Por ejemplo:

  • https://example.com

Better Search & Replace

  1. Primero, haz una copia de respaldo de tu base de datos. Sólo por si algo no sale como se esperaba, puedes remplazar fácilmente tu base de datos anterior.
  2. Descarga el Better Search Replace plugin. En tu panel de control de WordPress, dirígete a 'Plugins > Añadir nuevo'.
  3. Busca 'Better Search Replace'. 
  4. Haz click Instalar ahora.
  5. Haz click en Activar.
  6. En el Panel de WordPress, busca el plugin bajo la opción 'Herramientas > Better Search Replace'.
  7. Haz click en la pestaña 'Búsqueda/sustitución'.
  8. En el primer espacio en blanco, llamado 'Buscar por:', vas a ingresar tu URL actual en http. Ejemplo:
    • http://example.com
  9. En el segundo espacio llamado 'Sustituir con:', vas a ingresar tu URL en HTTPS. Ejemplo:
    • https://example.com
  10. Selecciona todas las tablas.
  11. Selecciona la casilla '¿Quieres ejecutar un simulacro?' (Run as dry run).
  12. Haz click en Ejecutar búsqueda/sustitución.
  13. Esto debería responder con un mensaje en la parte superior que te notifica lo que se habría actualizado si no hubieras marcado la casilla '¿Quieres ejecutar un simulacro?'.
  14. Si todo se ve bien, ejecuta nuevamente la acción, pero esta vez no selecciones la casilla '¿Quieres ejecutar un simulacro?' para que se puedan aplicar los cambios.
  15. A continuación, vas a ver un mensaje de confirmación que te mostrará cuantos cambios se han aplicado.

Pasos finales

En este punto, todas las URL en tu base de datos están apuntando a HTTPS, sin embargo, hay unos cuantos pasos más que debes tener en cuenta para asegurarte de que todos los links en tu sitio apunten a la version segura del dominio (es decir, la versión 'HTTPS').
