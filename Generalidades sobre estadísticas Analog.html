Generalidades

Analog es un programa de análisis de estadísticas de sitios web de código abierto y está instalado en todos los servidores de DreamHost de forma predeterminada, sin embargo, debes habilitarlo manualmente en tu panel para poder usarlo.

Puedes habilitar las estadísticas en la página Estadísticas de Sitio al crear un usuario de estadísticas.

Las estadísticas de sitio no están disponibles en los planes de DreamPress.

Habilitar las estadísticas

Visita el siguiente artículo para obtener detalles completos sobre cómo habilitar y ver las estadísticas:

  • Habilitar y acceder a las estadísticas

Algunas aplicaciones web (sistemas de administración de contenido, software de blogs, etc.) pueden implementar reglas de reescritura de URL de Apache que pueden impedirte acceder a esta página a menos que modifiques esas reglas. Las instrucciones generales para realizar dichas modificaciones (e instrucciones específicas para ciertas aplicaciones web que se encuentran comúnmente, como WordPress), se pueden encontrar en el artículo hacer las estadísticas accesibles con htaccess.

Resumen de funciones de estadísticas

Luego de iniciar sesión, la primera sección muestra las estadísticas del sitio.

Desde la sección Go To en la parte superior de la página, los usuarios pueden navegar a secciones y resúmenes separados. El área restante de la sección General Summary de la manera siguiente:

Successful requests
La cantidad de veces que alguien logró acceder a cualquier archivo en tu sitio.
Average successful requests per day
La cantidad promedio de solicitudes si el tráfico se mantuvo constante.
Successful requests for pages
La cantidad de veces que alguien logró acceder a un archivo de "page" en tu sitio (archivos que terminan en .html, .htm, .php, .cgi, y demás).
Average successful requests for pages per day
El número promedio si las visitas al sitio fueran constantes.
Failed requests
La cantidad de veces que alguien intentó acceder a un archivo desde tu sitio, pero no tuvo éxito por cualquier motivo.
Redirected requests
La cantidad de veces que un visitante intentó acceder a un archivo en particular, pero el servidor lo redirigió a otro diferente. Hay dos razones principales por las que esto sucede:
  • El visitante ha solicitado incorrectamente un nombre de directorio sin un slash seguido. Por ejemplo, una solicitud de www.example.com/tudirectorio (el nombre incorrecto) devuelve www.example.com/tudirectorio/ (el nombre correcto).
  • Los anuncios de banner "Click-thru" también pueden hacer que se redirija una solicitud.
Distinct files requested
Cuántos archivos (páginas, imágenes, programas, etc.) se solicitaron a tu sitio.
Distinct hosts served
Básicamente, cuántas personas diferentes visitaron tu sitio. Sin embargo, el número puede ser un poco engañoso, ya que los clientes de algunos ISP grandes pasan por un servidor proxy. Como resultado, una gran cantidad de visitantes puede representarse como una pequeña cantidad de alojamientos distintos.
Data transferred
La cantidad total de datos transferidos.
Average data transferred per day
Cuál sería la cantidad promedio de datos transferidos con tráfico constante.

Reportes basados en tiempo

Los tipos de informes se pueden dividir en resúmenes semanales, diarios, y por hora.

Aquí hay una descripción de las cuatro secciones:

Weekly Report
Un resumen de las últimas semanas por la fecha de inicio semanal. También enumera la semana más ocupada en la parte inferior.
Daily Report
Similar al Weekly Report, pero desglosa el sitio más ocupado por día.
Daily Summary
Resumen general de las estadísticas de un solo día.
Hourly Summary
Desglosa las solicitudes y las páginas por hora del día anterior.

Reportes basados en visitantes

Debido al tamaño de la página de estadísticas, la sección de informes basados en visitantes se divide ligeramente para revisar las diversas secciones individualmente.

Organization Report
Enumera las organizaciones, instituciones e ISP bajo los cuales se registraron los computadores que accedieron a la información.
Host Reports
Muestra todos los computadores individuales que accedieron a la información en tu sitio web.
User Reports
Listado de todos los usuarios que han visitado tu sitio durante el período de tiempo especificado. Si tu sitio requiere que los usuarios se autentiquen (iniciando sesión), aparecerá su nombre de usuario. De lo contrario, se muestra la cookie del usuario.
Browser Report/Summary
El informe/resumen de navegador desglosa las solicitudes del sitio y las visitas exitosas a páginas por diferentes navegadores.
Operating System Report
Una lista de los sistemas operativos utilizados por las personas que visitan tu sitio (por ejemplo, Windows, Linux, Apple, etc.). Estos están ordenados por número de solicitudes.
Status Code Report
Enumera cada código de estado HTTP que registra tu sitio.
File Type Report
La cantidad de veces que se solicitó a tu sitio cada tipo de archivo (por ejemplo, imagen, página, etc.).
Directory Report
Enumera donde se descargaron los archivos de directorios.
Failure Report
Enumera los nombres de archivo que provocaron un error.
Request Report
Enumera todos los archivos/páginas que se descargaron de tu sitio web durante el período de tiempo especificado.
