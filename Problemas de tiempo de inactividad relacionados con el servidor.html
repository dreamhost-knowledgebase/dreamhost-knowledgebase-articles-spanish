Generalidades

Los siguientes artículos describen problemas comunes relacionados con el servidor que pueden causar el tiempo de inactividad del sitio y los pasos para ayudarte a volver a conectarte.

Consulta el siguiente artículo para obtener una lista de otras causas por las que tu sitio puede estar inactivo.

  • Qué revisar si tu sitio está caido

Confirma si tu sitio está caído en todas partes

El primer paso a seguir es confirmar si el sitio está realmente inactivo en todas las ubicaciones, y no solo en tu computadora local. Hay un par de maneras de hacer esto.

¿Está abajo para todos, o solo para mí?

downforeveryoneorjustme.com comprueba si tu sitio está en línea desde varias ubicaciones. Luego te permite saber si el sitio está inactivo en todas partes, o si solo tu computadora no puede verlo.

Si solo tu computadora no puede verlo y ha actualizado recientemente tu DNS, consulta este sitio para ver si el nuevo DNS se ha actualizado en tu ubicación:

  • Verificador de propagación de DNS global

Si el DNS se ha actualizado, intenta forzar a tu navegador a volver a cargar la página. Esto borra el caché del navegador para que puedas ver la versión más reciente de la página.

  • Forzar una página para recargar

Sitios proxy

Los sitios proxy te permiten evitar problemas de red y ver el sitio desde una ubicación diferente. Ve el siguiente artículo para obtener una lista de sitios proxy que puedes usar:

  • Sitios Proxy

Si el sitio proxy carga tu sitio web, confirma que el sitio se está resolviendo en línea. Si aún no puedes verlo, intenta borrar tu caché y vaciar DNS:

  • Limpiar caché
  • Flush de DNS

¿Mi servidor está activo?

Si tu registro está actualizado y tu cuenta no ha sido suspendida, pero no puedes acceder a tu dominio a través de un sitio proxy, verifica que tu servidor esté activo. Puedes hacer esto ejecutando un traceroute en tu servidor y haciendo ping a tu dominio. Las instrucciones para realizar estos pasos se encuentran aquí:

  • Traceroute
  • Recopilar resultados de ping

Si no puedes hacer ping al servidor, o si el traceroute no se completa, por favor contacta el equipo de soporte. Si tu traceroute muestra varios saltos que exceden el tiempo de espera, deberás comunicarte con tu ISP para obtener ayuda.

¿Están mis archivos en el lugar correcto?

Asegúrate de que los archivos de tu sitio estén en el directorio correcto. Si están en el directorio incorrecto o asignados al usuario incorrecto, el sitio no se cargará.

Inicia sesión en tu sitio a través de FTP o SSH. Asegúrate de que los archivos de tu sitio estén en el directorio correcto. Ve el siguiente artículo para más detalles:

  • ¿Dónde subo mis archivos una vez conectado a tráves de FTP?
