Generalidades

Las versiones de PHP inferiores a 7.3 se han eliminado del panel DreamHost. Se recomienda encarecidamente que siempre actualices a una versión reciente, ya que las versiones anteriores pronto se eliminarán por completo de todos los servidores DreamHost.

PHP 8 ofrece varias mejoras con respecto a versiones anteriores. Puedes leer más sobre ellos aquí:

  • PHP 8.0: Que hay de nuevo y que ha cambiado

Un problema importante con las versiones anteriores es que ellos consumen más memoria de la necesaria y se ejecuta más lentamente que las versiones más recientes.

Las últimas versiones del software principal de WordPress han sido compatibles con PHP 7 durante algunos años. Actualizar a PHP 7 es una forma rápida y fácil de obtener más rendimiento para tu sitio WordPress y el servidor.

No todos los temas y plugins de terceros se han actualizado para que sean compatibles con PHP 8. Si tu sitio se basa en un plugin o tema específico, asegúrate de verificar que sea compatible con PHP 8 antes de actualizar.

¿Cómo sé si necesito actualizar?

WordPress.org recomienda es al menos PHP 7.4. Es posible que veas un mensaje de advertencia en tu panel de control de WordPress pidiéndote que actualices PHP si aún usas las versiones menores a 7.4. Puedes encontrar los pasos para actualizar tu sitio de WordPress continuación.

Planes de alojamiento de DreamPress

  1. Navega a la página WordPress Administrado del panel.
  2. Hacia la derecha de tu dominio, haz clic en el botón Manage.
  3. En la parte superior debajo de tu dominio, deberías ver la versión de PHP:
    • Si ves PHP-FPM, ya tienes una versión compatible de PHP 7 (o PHP 8.0, si solicitaste específicamente una actualización o la actualizaste manualmente usando un sitio provisional de DreamPress).
    • Si ves una versión anterior o si pediste específicamente cambiar en PHP-FPM, deberías actualizar a PHP 7.4 o 8.0.

Todos los demás planes de alojamiento

Esto incluye cualquier sitio de WordPress instalado en un plan que no sea DreamPress. Por ejemplo, Alojamiento Compartido (Shared), VPS o Dedicado.

  1. Navega a la página Administrar Sitios Web.
  2. Da clic en tl botón Administrar para abrir la página Administrar Sitios Web, la cual te permite ajustar varias configuraciones de tu sitio.
    • Si estás en la Vista de Malla, da clic al botón Administrar en la parte inferior de la casilla de tu sitio web.
    • Si estás en la Vista de Lista, da clic en el botón Administrar en la parte derecha de tu dominio.
  3. Revisa debajo de la sección PHP para ver la versión:
    • Debes actualizar a 7.4 o 8.0

¿A qué versión debo actualizar?

Se recomienda encarecidamente actualizar a la última versión de PHP si el software de tu sitio puede hacerlo sin problemas.

La última versión es la que será soportada por más tiempo y tiene la mayoría de las correcciones para mejorar el rendimiento. Varios estudios muestran que PHP 7.2 ofrece una mejora de velocidad de 20-30% sobre PHP 7.0 y una mejora de velocidad de 250-300% sobre PHP 5.6.

Si encuentras un problema con la última versión, puedes probar la siguiente versión más reciente sin mucha diferencia en rendimiento o soporte.

Cada versión de PHP ofrece dos opciones: FastCGI y CGI. Por lo general, debes elegir FastCGI, ya que ofrece un rendimiento más rápido para la mayoría de los sitios. Si la versión de PHP de tu sitio está configurada actualmente en CGI, es posible que en algún momento hayas encontrado problemas de rendimiento con FastCGI que provocaron el cambio a CGI. Puedes probar cualquiera de los dos para comparar el rendimiento.

Prepara tu sitio de WordPress para una actualización de PHP

El uso de la lista de verificación a continuación ayudará a minimizar el riesgo de problemas al actualizar la versión de PHP, sin embargo, siempre hay una pequeña posibilidad de un error o problema después de una actualización debido a los miles de temas, plugins y personalizaciones disponibles para WordPress.

Recuerda que siempre puedes probar la actualización y la degradación si hay un problema. Estos pasos no son obligatorios, pero son muy recomendables para una actualización de PHP, así como para el mantenimiento y la seguridad regulares de tu sitio.

Lista de Verificación

  • ¿Estás usando la última versión del software principal de WordPress?
  • ¿Tienes alguna actualización pendiente que se muestre en tu panel de administración de WordPress?
  • ¿Tienes algún tema o plugins más antiguo que el desarrollador no haya actualizado en más de un año, o algún tema/plugin pagado que haya expirado la licencia para las actualizaciones?
  • ¿Tienes alguna personalización escrita en PHP que necesita un desarrollador para revisar?

Las instalaciones de WordPress de DreamHost hechas a través del panel siempre están configuradas para actualizarse automáticamente (a menos de que lo desactives específicamente).

Probar la actualización utilizando un sitio de preparación

Una forma segura de probar la nueva versión de PHP es crear un sitio web provisional. Este sitio de preparación se convertirá en un duplicado de tu sitio web en vivo existente. Luego puedes proceder a ejecutar pruebas en este sitio de ensayo para ver cómo reacciona. Si no sucede nada inusual, puedes continuar con la actualización de tu sitio web en vivo. Actualizar PHP

DreamPress
