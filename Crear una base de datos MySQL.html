Generalidades

Podrás crear una base de datos en tu panel de DreamHost. Una vez hayas iniciado sesión, navega a la página Bases de Datos MySQL. Aquí podrás ingresar las nuevas credenciales para crear una base de datos nueva.

¿Es posible subir una base de datos existente?

No es posible cargar una base de datos existente en el panel de DreamHost.

Para subir una base de datos existente, necesitarás crear primero una base de datos en blanco como se explica a continuación. Una vez sea creada puedes usar phpMyAdmin para cargar el archivo de tu base de datos. Revisa el siguiente artículo para ver las instrucciones completas:

  • phpMyAdmin — Cómo importar o restaurar una base de datos o una tabla

Crear una base de datos MySQL

Para crear una base de datos, desplázate hasta la parte inferior de la página Bases de Datos MySQL y completa los siguientes campos.

Nombre de la base de datos

Ingresa un nombre único para la base de datos. Los nombres de las bases de datos deben ser únicos en todo el sistema DreamHost y solo pueden contener números, letras a-z y guiones bajos (_). Se recomienda elegir un nombre que sea más largo y descriptivo. Para mayor claridad, DreamHost sugiere poner tu dominio en el nombre de la base de datos como "blog_example_com" en lugar de un ejemplo menos claro de "blog12345".

Una vez que hayas creado la base de datos, no podrás cambiarle el nombre.

Nuevo hostname

Crea un nuevo hostname o seleccionar uno existente de un menú desplegable, si existe. En general, se recomienda crear un hostname para cada dominio para evitar el tiempo de inactividad en caso de que elimines un dominio de tu cuenta en el futuro y descubre que otros sitios usaron un hostname en él y ya no funcionan.

Primer usuario/nuevo nombre de usuario

Crea un nuevo usuario de MySQL Por razones de seguridad, se recomienda tener un usuario de MySQL para cada base de datos, de modo que cualquier sitio o credenciales comprometidos no puedan usarse para acceder a tus otras bases de datos.

Nueva contraseña

Si estás creando un nuevo usuario, ingresa una contraseña para ese usuario aquí. Si seleccionas un usuario existente, este campo está deshabilitado.

Nueva Contraseña de Nuevo

Si estás creando un nuevo usuario, vuelva a ingresar la misma contraseña que la anterior aquí. Si seleccionas un usuario existente, este campo está deshabilitado.

Comentario de base de datos

Ingresa una descripción de tu base de datos aquí para uso organizativo. Opcional.

Agregando

Cuando hayas ingresado todos los campos obligatorios, haz clic en ¡Añadir nueva base de datos ahora! para comenzar el proceso de creación de la base de datos.

  • Agregar nuevas bases de datos o usuarios, cambiar las contraseñas de los usuarios de MySQL y eliminar bases de datos o contraseñas generalmente tiene efecto en 5 minutos.
  • Los nombres de host recién agregados funcionarán para los sitios web en 5 minutos, debido a que todos los nombres de host se escriben en tu servidor web /etc/hosts.
  • El uso de un nuevo hostname para phpMyAdmin puede tardar hasta 4 horas en funcionar debido al tiempo de propagación de DNS.

Privilegios de Base de Datos

No todos los privilegios de la base de datos están disponibles en un entorno MySQL compartido. Mira el siguiente artículo para obtener una lista de privilegios y su disponibilidad:

  • Limitaciones de MySQL debido a un Shared Hosting
  • Manejar usuarios MySQL, hosts permitidos, y privilegios

¿En qué servidor MySQL se crea la base de datos si tengo un VPS MySQL?

Se recomienda tener un solo servidor de base de datos que usen todas tus bases de datos. Cuando compras un VPS MySQL, es posible que algunas de tus bases de datos se dividan entre el antiguo servidor MySQL compartido y el nuevo VPS MySQL. Para solucionar esto, debes fusionar las bases de datos en el nuevo servidor. Visita el siguiente artículo para obtener instrucciones:

  • Fusionar las bases de datos en un solo servidor
