Generalidades

Primero, inicia sesión en tu Panel de DreamHost. Una vez que hayas iniciado sesión, navega a la página MySQL Databases. En esta página verás una lista completa de tus nombres de host y bases de datos MySQL.

Crear una base de datos MySQL

Para crear una base de datos, desplázate hasta la parte inferior de la página  MySQL Databases y completa los siguientes campos.

Nombre de la base de datos

Ingresa un nombre único para la base de datos. Los nombres de las bases de datos deben ser únicos en todo el sistema DreamHost y solo pueden contener números, letras a-z y guiones bajos (_). Se recomienda elegir un nombre que sea más largo y descriptivo. Para mayor claridad, DreamHost sugiere poner tu dominio en el nombre de la base de datos como "blog_example_com" en lugar de un ejemplo menos claro de "blog12345".

Una vez que haya creado la base de datos, no podrá cambiarle el nombre.

Usar nombre de host/nuevo nombre de host

Aquí puedes crear un nuevo nombre de host o seleccionar uno existente de un menú desplegable, si existe. En general, se recomienda crear un nombre de host para cada dominio para evitar el tiempo de inactividad en caso de que elimines un dominio de tu cuenta en el futuro y descubre que otros sitios usaron un nombre de host en él y ya no funcionan.

Primer usuario/nuevo nombre de usuario

Aquí puedes crear un nuevo usuario de MySQL o seleccionar uno existente de un menú desplegable, si existe. Por razones de seguridad, se recomienda tener un usuario de MySQL para cada base de datos, de modo que cualquier sitio o credenciales comprometidos no puedan usarse para acceder a tus otras bases de datos.

Nueva contraseña

Si estás creando un nuevo usuario, ingresa una contraseña para ese usuario aquí. Si seleccionas un usuario existente, este campo está deshabilitado.

Nueva Contraseña de Nuevo

Si estás creando un nuevo usuario, vuelva a ingresar la misma contraseña que la anterior aquí. Si seleccionas un usuario existente, este campo está deshabilitado.

Comentario de base de datos

Ingresa una descripción de tu base de datos aquí para uso organizativo. Opcional.

Agregando

Cuando hayas ingresado todos los campos obligatorios, haz clic en Add new database now! para comenzar el proceso de creación de la base de datos.

  • Agregar nuevas bases de datos o usuarios, cambiar las contraseñas de los usuarios de MySQL y eliminar bases de datos o contraseñas generalmente tiene efecto en 5 minutos.
  • Los nombres de host recién agregados funcionarán para los sitios web en 5 minutos, debido a que todos los nombres de host se escriben en tu servidor web /etc/hosts.
  • El uso de un nuevo nombre de host para phpMyAdmin puede tardar hasta 4 horas en funcionar debido al tiempo de propagación de DNS.

Privilegios de Base de Datos

No todos los privilegios de la base de datos están disponibles en un entorno MySQL compartido. Consulte el siguiente artículo para obtener una lista de privilegios y su disponibilidad:

  • Limitaciones MySQL debido a alojamiento compartido
  • Manejar usuarios MySQL users, allowable hosts, y privilegios

¿En qué servidor MySQL se crea la base de datos si tengo un VPS MySQL?

Se recomienda tener un solo servidor de base de datos que utilicen todas sus bases de datos. Cuando compras un VPS MySQL, es posible que algunas de tus bases de datos se dividan entre el antiguo servidor MySQL compartido y el nuevo VPS MySQL. Para solucionar esto, debes fusionar las bases de datos en el nuevo servidor. Visita el siguiente artículo para obtener instrucciones:
