Generalidades

Hay muchas herramientas gratuitas que puedes usar para solucionar problemas de tu sitio web o cualquier problema de conexión que puedas estar viendo.

Este artículo enumera una variedad de servicios que puedes usar para solucionar problemas de tu sitio antes de contactar al soporte. Si aún necesitas ponerte en contacto con el soporte, asegúrate de incluir los resultados de cualquier prueba que hayas realizado.

General

  • archive.org – Ver cómo apareció un sitio web en una fecha específica en el pasado.
  • online-domain-tools.com – Una colección de herramientas en línea para ayudar a probar y solucionar problemas de tu sitio.
  • mxtoolbox.com – Una colección de herramientas (muchas relacionadas con correo electrónico) para solucionar tus servicios.

Registro de Dominio

  • whois.online-domain-tools.com – Búsqueda de WHOIS
  • hexillion.com – Búsqueda de WHOIS
  • whois.domaintools.com – Búsqueda de WHOIS
  • Nameserver lookup – úsqueda de registrador y servidor de nombres
  • whoisrequest.com - Muestra los cambios históricos del servidor de nombres

Herramientas de carga de sitios web

Pruebas de carga del sitio

Los siguientes sitios prueban cuánto tiempo tarda en cargar su página. También brindan detalles sobre lo que puede ser responsable del tiempo de carga:

  • Pingdom.com
  • gtmetrix.com
  • Webpagetest.org
  • Google's pagespeed insights
  • Bytecheck.com – Comprueba el tiempo hasta el primer byte
  • cloudmonitor.ca.com – Prueba mundial para verificar el tiempo de carga del sitio

Pruebas de propagación DNS

  • whatsmydns.net – Se usa para verificar si tus registros DNS se han propagado en diferentes ubicaciones
  • wheresitup.com – Comprueba si el sitio se está resolviendo en diferentes ubicaciones
  • viewdns.info - Comprueba si el DNS se ha resuelto en diferentes lugares del mundo

Pruebas basadas en localización

  • Speedtest.net – Prueba la velocidad de carga/descarga de su red local. Se usa para probar si tu ubicación específica es donde puede estar ocurriendo el problema de red.

Herramientas de línea de comando

También hay varias herramientas de línea de comandos que puedes usar para recuperar información sobre un dominio o condición(es) de red. Consulta el artículo de SSH para obtener detalles sobre cómo iniciar sesión en tu servidor.

La siguiente es una lista de ejemplos básicos. Asegúrese de leer los enlaces en cada sección para más detalles.

host

  • host – Convierte nombres a direcciones IP y viceversa.

Ejecuta host en example.com para encontrar tus registros IP y MX:

[server]$ host example.com
example.com has address 173.236.241.100
example.com mail is handled by 0 mx1.mailchannels.net.
example.com mail is handled by 0 mx2.mailchannels.net.

Ejecuta host en la IP para encontrar su nombre de dominio:

[server]$ host 173.236.241.100
100.241.236.173.in-addr.arpa domain name pointer example.com.

nslookup

  • nslookup en Windows
  • nslookup en Linux

WHOIS

  • WHOIS

Encuentra detalles sobre el registro de un dominio:

[server]$ whois example.com

dig

  • dig

Buscar el IPv4 de un dominio:

[server]$ dig +short example.com

Buscar el IPv6 de un dominio:

[server]$ dig +short AAAA example.com

dig TXT registro:

[server]$ dig TXT example.com

dig registro CNAME. Asegúrate de ingresar el registro CNAME correcto antes del dominio:

[server]$ dig CNAME abcdefg.example.com

dig registro SRV. Asegúrate de ingresar el registro SRV correcto antes del dominio:

[server]$ dig SRV _sip.example.com

ping

  • ping & ping6
  • Recopilar resultados de ping – haz ping a un host para ver si hay alguna pérdida de paquetes

traceroute

  • traceroute & traceroute6
  • Traceroute – Rastrea los paquetes de ruta que toman de una ubicación a otra. Ejecuta traceroute para ver todos los saltos entre tu ubicación y tu sitio web.

mtr

  • mtr

mtr combina la funcionalidad de los programas traceroute y ping en una única herramienta de diagnóstico de red:

[server]$ mtr dreamhost.com

openssl

  • openssl

Herramienta de línea de comando para usar varias funciones de la biblioteca criptográfica de OpenSSL:

[server]$ openssl s_client -connect example.com:443
