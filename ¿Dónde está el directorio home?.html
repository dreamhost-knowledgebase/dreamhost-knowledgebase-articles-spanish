Generalidades

Verás los términos directorio /home, directorio principal de usuario, y directorio de sitio web se utilizan con mucha frecuencia al leer artículos sobre dónde cargar y administrar los archivos de tu sitio web. Estos términos pueden ser confusos, ya que todos apuntan a una ubicación diferente en el servidor.

Este artículo explica qué son estos directorios y cómo confirmar que te encuentras en la ubicación correcta en el servidor al administrar tu sitio web.

Terminología

Los siguientes términos se explican en este artículo:

Términos Explicación
Usuario Un usuario/nombre de usuario es el nombre asignado a un dominio en el servidor.
El directorio /home del servidor Directorio donde se encuentran todos los usuarios
Directorios de usuario Varios directorios de usuarios en el directorio /home
El directorio home del usuario Directorio de un solo usuario. (También referenciado por el símbolo ~)
El directorio del sitio web Directorio de sitios web con un solo nombre de usuario

Usuario

Un usuario del sitio web es el nombre de usuario asignado a tu nombre de dominio en el servidor. Este es el nombre de usuario que se usa para iniciar sesión en tu servidor con un cliente SFTP o SSH para administrar tu sitio web. Se pueden crear varios sitios web con un solo nombre de usuario. Ve los siguientes artículos para obtener más información sobre los usuarios de DreamHost.

  • Añadir un usuario a un servidor
  • Cambiar el usuario asignado a un dominio

El directorio /home del servidor

El directorio /home es el directorio en el servidor donde TODOS los usuarios están ubicados. Este es referenciado como /home. En el servidor se verá simplemente así:

/home
  • Ningún usuario en DreamHost tiene acceso a este directorio.
  • Este directorio solo contiene varios directorios de nombre de usuario.
  • Estos directorios de usuarios contienen sitios web. 

Directorios de usuario

El directorio /home del servidor contiene muchos nombres de usuario diferentes. Cada nombre de usuario es su propio directorio independiente en el directorio /home.

Servidores Shared

Si estás en un servidor web compartido, los nombres de usuario que has creado junto con otros usuarios de ese servidor se encuentran en este directorio /home. Por ejemplo:

/home
    /youruser1
    /youruser2
    /youruser3
    /otheruser1
    /otheruser2
    /otheruser3

Servidores VPS o Dedicados

Si estás en un servidor VPS o Dedicados, solo tus nombres de usuario personales están ubicados en el directorio /home. Por ejemplo:

/home
    /youruser1
    /youruser2
    /youruser3

El directorio home del usuario

Este es el directorio al que se hace referencia en este centro de ayuda cuando se menciona el término directorio de home o inicio del usuario.

Este directorio a menudo es referenciado por el símbolo ~. Por ejemplo:

[server]$ cd ~

El directorio principal del usuario es específico para un solo usuario en el servidor. Es uno de los muchos usuarios que figuran en el directorio /home del servidor. Por ejemplo:

/home/youruser1

Solo tu usuario tiene acceso a este directorio de usuarios específico. Nadie más puede acceder a él. Aquí es donde residen todos los archivos de configuración de usuario y de tu sitio web.

Si has iniciado sesión en tu servidor a través de SSH, puedes correr el comando pwd para ver en qué directorio se encuentra actualmente. Así es como aparecería si ya estuvieras en el directorio principal de tu usuario.

[server]$ pwd
/home/username

¿Para qué se usa este directorio?

Este directorio se usa con mayor frecuencia para administrar los archivos de configuración de tu usuario (como .bash_profile) y revisa tus registros de error y acceso. Si necesitas administrar los archivos de tu sitio web, debes navegar al directorio de tu sitio web.

Así es como se vería en un cliente SFTP como Filezilla:

Archivos predeterminados en el directorio principal del usuario

Estos son los archivos predeterminados que verás en el directorio principal del usuario.

/home/youruser1/
               .alias
               .bash_profile
               .bashrc
               .cshrc
               /Maildir
               /logs
               /example.com

El directorio del sitio web

El directorio del sitio web es donde se encuentran los archivos reales del sitio web. Por ejemplo:

/home/youruser1/example.com

Si has iniciado sesión en tu servidor a través de SSH, puedes correr el comando pwd para ver en qué directorio te encuentras actualmente. Así es como aparecería si ya estuvieras en el directorio de tu sitio web.

[server]$ pwd
/home/username/example.com

Un dominio recién creado tendrá los siguientes archivos agregados al directorio del sitio web:

  • favicon.ico
  • favicon.gif

Estos archivos evitan la carga del servidor por exceso de páginas 404 cuando el sitio está experimentando un tráfico intenso de visitantes o rastreadores web.

  • quickstart.html

Este archivo actúa como la página de índice predeterminada cuando no hay otros archivos de índice válidos presentes. Para obtener más información, revias el siguiente artículo:

  • ¿Cómo puedo controlar mis índices de directorio con un archivo .htaccess?

Resumen

Cuando necesites administrar tu sitio web, estarás en el directorio principal de tu usuario o en el directorio del sitio web.

El directorio home del usuario

En este directorio, configurarías los ajustes para el usuario, ya que se aplica a todos los sitios web que se encuentran debajo. Por ejemplo, editar tú .bash_profile o personalizar tu configuración de PHP en un archivo phprc.

Directorio de sitio web

Este es el directorio donde editaría los archivos de un sitio web específico.
