Generalidades

En un esfuerzo por ayudar a mantener tu servidor seguro, Telnet se eliminará de la última versión de Ubuntu (Bionic). Si necesitas Telnet, DreamHost recomienda ejecutarlo en tu computador local.

Telnet es un protocolo obsoleto que se utiliza para iniciar sesión de forma remota en otra computador o equipo de comunicación a través de una red. El protocolo Telnet no ofrece seguridad. El uso de Telnet envía todas las comunicaciones, incluidas las contraseñas, a través del cable en texto sin formato. Por esta razón, solo se recomienda utilizar SSH en su lugar para correr cualquier comando con datos seguros. Telnet solo debe usarse para propósitos de prueba muy básicos.

Como se indicó anteriormente, Telnet nunca debe usarse con contraseñas o datos confidenciales, ya que no es seguro. Sin embargo, puedes usar Telnet para ejecutar algunas pruebas básicas en tus servicios DreamHost.

Probar una conexión a un servidor de correo

Consulta el siguiente artículo para obtener instrucciones sobre cómo ubicar tu nombre de servidor de correo correcto. 

  1. Consulte el siguiente artículo para obtener instrucciones sobre cómo ubicar tu nombre de servidor de correo correcto.

    • Cómo encontrar el nombre de tu servidor de correo
  2. Corre el siguiente comando a través de SSH o en el símbolo del sistema de Windows para probar tu conexión: 
    [server]$ telnet imap.dreamhost.com 143

    Si se conecta correctamente, responde con algo como esto:

    Trying 64.90.62.162...
Connected to imap.dreamhost.com.
Escape character is '^]'.
* OK [CAPABILITY IMAP4rev1 LITERAL+ SASL-IR LOGIN-REFERRALS ID ENABLE STARTTLS AUTH=PLAIN AUTH=LOGIN] Dovecot ready.
  3. Para salir de la sesión Telnet, escribe Ctrl + ] en tu teclado. Esto cambia el símbolo del sistema para que se muestre como telnet>.

    De vuelta en la terminal, escriba la palabra ‘cerrar’ para cerrar la sesiónn.

    telnet> close
