Este artículo contiene información/ejemplos usando usuarios root/sudo/admin. Si necesitas acceso sudo/admin, debes actualizar a un Servidor Dedicado.

Debes volver a cargar Nginx cada vez que hagas un cambio en un archivo de configuración. Puedes hacer esto en el panel o iniciando sesión en tu servidor a través de SSH con tu usuario administrador en un Servidor Dedicado.

Recargarlo en el panel (Servidores VPS y Dedicados)

  1. Navega a la página VPS o Dedicado.
  2. Da clic en el botón Volver a cargar HTTP.
  3. Espera 5 minutos para que reconstruya el archivo de configuración.

Recargar vía SSH (Solo para Servidores Dedicados)

Inicia sesión en tu servidor vía SSH con tu usuario admin. luego, correo el siguiente comando:

[server]$ sudo /etc/init.d/nginx reload
  • Recargar mantiene el servidor corriendo mientras relee cualquier actualización en el archivo de configuración.
  • Recargar es más seguro que reiniciar pues si un error de sintaxis es encontrado en el archivo de configuración no procederá con la recarga, y tu servidor seguirá corriendo.
  • Si hay un error de sintaxis y tu reinicias, es posible que el servidor no reinicie correctamente.

Reiniciar (Solo para Servidores Dedicados)

Puedes reiniciar Nginx con los siguientes comandos después de iniciar sesión en tu servidor vía SSH con tu usuario admin:

[server]$ sudo /etc/init.d/nginx stop
[server]$ sudo /etc/init.d/nginx start

También puedes reiniciarlo en tu panel de DreamHost. Navega hasta la página VPS, y da clic en el botón Reiniciar en la parte derecha de tu servidor.

Actualizar tu archivo phprc

Si has creado un archivo phprc y ves que el archivo phpinfo.php no ha sido actualizado, tienes tres opciones:

Actualizar la configuración de tu sitio en tu panel

Puedes guardar nuevamente la configuración de tu sitio en el panel para actualizar tu archivo phprc.

Recargar Nginx (Solo para Servidores Dedicados)

Si estás en un Servidor Dedicado con un usuario admin, puedes recargar Nginx usando los pasos que están más arriba. Si recargando no actualiza los cambios, trata reiniciando y matando todos los procesos de PHP como muestra abajo.

Reiniciar Nginx y matar los procesos de PHP (Solo para Servidores Dedicados)

Si estás en un Servidor Dedicado con un usuario admin, puedes reiniciar Nginx y matar todos los procesos de PHP que se están corriendo. Corre el siguiente comando en tu usuario admin Dedicado:

[server]$ sudo /etc/init.d/nginx stop
[server]$ pkill -9 php
[server]$ sudo /etc/init.d/nginx start
