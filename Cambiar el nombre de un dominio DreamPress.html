Generalidades

Este artículo explica cómo cambiar el nombre de tu sitio DreamPress a un nombre de dominio diferente. Estas instrucciones se pueden usar para un nombre de dominio regular como 'example.com' o un dominio temporal de DreamPress como 'example.dream.press'.

Asegúrate de que el dominio al que está cambiando el nombre ya esté configurado como DNS Only. Luego puedes usar el cambiador de nombres de dominio en el panel.

Puedes usar el cambiador de nombres de dominio en el panel de DreamPress para cambiar cualquier dominio temporal, no solo los subdominios dream.press.

Preparando tu dominio para la migración

Confirma lo siguiente para el dominio al que estás migrando. Si alguno de los siguientes elementos no está configurado, tu sitio no se conectará después de migrarlo.

Eliminar alojamiento

El dominio al que migras no se puede alojar. Si es así, debes eliminar el alojamiento del dominio antes de continuar. Puede hacer esto configurando el dominio como DNS Only. Esto es para que el dominio se pueda migrar de forma segura sin romper un sitio en vivo.

Elimina cualquier nombre de host mysql 'previamente existente'

Este paso es extremadamente importante.

Si ya existe un nombre de host mysql en tu panel para el nuevo dominio, primero debes eliminarlo. Por ejemplo, navega hasta la página MySQL Databases. Si tienes bases de datos, verá los nombres de host enumerados bajo el nombre del servidor MySQL. Este servidor MySQL tendrá un nombre de servidor compartido o un nombre de VPS. Este ejemplo muestra un servidor MySQL compartido llamado 'bowl' en el que el nombre de host ya existe:

Si estás moviendo DreamPress a un nuevo dominio llamado example.com, busca mysql.example.com. Si lo ves en la lista, elimínalo.

Si existe y no lo eliminas, tu panel no podrá actualizar tu base de datos con el nombre de host correcto. Como no se conectará al nombre de host correcto, verás un error 'Error al establecer una conexión de base de datos' cuando migre.

Eliminar DNS personalizado

Asegúrate de que no haya Registros DNS personalizados registros A o CNAME en el dominio al que estás migrando (si existe alguno que apunte a un host web diferente). Si los hay, elimina los registros personalizados. Otros registros DNS personalizados están bien para dejarlos en su lugar.

Paso 1 — Cambiar tu dominio

JetPack Premium no se conectará automáticamente cuando cambies el nombre de tu dominio. Para volver a conectar, simplemente vuelve a iniciar sesión en tu página de administración de WordPress. JetPack debe volver a conectarse después de iniciar sesión.

Para cambiar el dominio de DreamPress:

  1. Navega hasta la página Managed WordPress.
  2. Da click en el botón Manage al lado de tu dominio.
  3. Da click en la pestaña Domain en la parte superior, y luego da click 'Choose existing domains'.
  4. Tienes dos opciones para cambiar el nombre.

    Usa un dominio de tu propiedad

    Esta opción te permite ingresar un nombre de dominio que ya posees. El panel lo agregará por ti.

    Elige dominios existentes

    Esta opción te permite elegir un dominio que ya has agregado a tu panel.
  5. Haz clic en el botón Change Domain.
    Aparece la siguiente advertencia que Te notifica que se eliminarán las copias de seguridad anteriores del antiguo nombre de dominio.
    El DNS tarda entre 4 y 6 horas en actualizarse para tu nuevo dominio en DreamHost.

Paso 2 — Esperando a que se actualice tu DNS DreamHost

Como se indicó anteriormente, el DNS puede tardar entre 4 y 6 horas en actualizarse en DreamHost. Si apuntas tu DNS en su host actual a DreamHost antes de que el DNS se haya actualizado en los servidores de DreamHost, el sitio no se conectará.

Se recomienda que espere de 4 a 6 horas antes de actualizar su DNS en su host actual.

Paso 3 — Actualización de tu DNS en su host actual

Finalmente, inicia sesión en su registrador actual y apunta tu DNS a DreamHost. Después de que el DNS propague, tu sitio estará alojado con DreamPress.

Error al establecer una conexión de base de datos

Lo más probable es que este error se haya producido porque no eliminaste el antiguo nombre de host como se mencionó anteriormente en la sección titulada Remove any mysql hostnames.

La razón es porque el panel todavía tiene el antiguo nombre de host para el sitio 'DNS Only' todavía configurado en la página MySQL Databases. Como esto no se eliminó, el panel no pudo mover el antiguo nombre de host a la base de datos DreamPress.

Para solucionarlo, debes actualizar su archivo wp-config.php con el antiguo nombre de host. Navega a la página de bases de datos MySQL. Busca su base de datos DreamPress. Anote el nombre de host.

Inicie sesión en tu servidor a través de SSH y edita tu archivo wp-config.php. En ese archivo, actualiza la línea de nombre de host con este antiguo nombre de host (el que aún se muestra en el panel junto a tu base de datos DreamPress).
