Generalidades

Este artículo describe el copiar tu sitio principal de DreamPress a un sitio web provisional. El sitio de preparación te permite probar los cambios que planeas realizar en tu sitio en vivo mientras tu sitio en vivo permanece en funcionamiento normalmente.

Cuando hayas terminado de trabajar en tu sitio de preparación y lo hayas configurado como desees, puedes publica esos cambios nuevamente en tu sitio web en vivo.

Notas sobre cómo copiar tu sitio DreamPress Staging o provisional

Si tu sitio ha sido hackeado previamente y luego usaste el servicio de DreamHost DreamShield debes limpiar los archivos que fueron pirateados/deshabilitados antes de poder habilitar el sitio provisional en tu sitio.

  • DreamPress Staging usa Git para copiar archivos. Si tienes tu propio repositorio Git activo en una de las carpetas de tu sitio en vivo, el contenido de la carpeta no se copiará en tu sitio provisional. Para utilizar la puesta en escena con éxito, puedes copiar manualmente los contenidos de la carpeta mediante SFTP o SSH. DreamPress recomienda como mejor práctica administrar tu repositorio Git en un entorno de desarrollo separado en lugar de tu sitio en vivo.
  • Cuando habilitas Staging por primera vez, tu plan DreamPress debe tener al menos 5 GB de espacio libre.
  • La carpeta Cargas no se copia en tu sitio provisional para usar el espacio en disco de manera eficiente. Reemplazar una imagen existente en tu Biblioteca de medios la modificará para el sitio en vivo.

Copiar tu sitio en vivo al sitio provisional

Los siguientes pasos te muestran cómo copiar tu sitio web en vivo de DreamPress a tu sitio web provisional. Luego puedes probar y trabajar en el sitio de preparación sin afectar el sitio en vivo.

  1. Navega a la página Administrar Sitios Web.
  2. Da clic en el botón Administrar para abrir la página de Configuraciones de Dominios, la cual te permite ajustar múltiples configuraciones de tu sitio.
    • Si estás en la Vista de Malla, da clic en el botón Administrar en la parte inferior de la caja del sitio.
    • Si estás en la Vista de Lista, da clic en el botón Administrar a la derecha de tu dominio.
  3. Da clic en la pestaña Sitio Provisional para abrir el panel de Sitio Provisional.
  4. Haz clic en el botón Crear Sitio Provisional.

    Un sitio provisional se crea automáticamente con un nuevo nombre. Ten en cuenta que no es posible darle un nombre personalizado a tu sitio de vista previa

Una vez que tu sitio web haya terminado de copiarse en tu sitio de preparación, se te enviará un correo de confirmación.

Administrar tu sitio web provisional

El entorno provisional no está incluido en las copias de seguridad diarias del dominio DreamPress. Tú es responsable de mantener tus propias copias de seguridad de cualquier contenido que agregues en el entorno provisional.

Luego de crear el ambiente provisional, aparece la página Sitio Web Provisional donde se ve el nombre de tu dominio en la parte superior. También aparecen las siguientes configuraciones:

  • Acceder al Sitio Provisional — Muestra el nombre de usuario y la contraseña que usas para acceder a tu sitio web protegido con inicio de sesión.
  • Administrador de Vista Provisional del Panel de WordPress — Abre la página WP-Admin donde puedes editar en tu sitio web provisional.
  • Subdir Archivos — Muestra las credenciales FTP de tu sitio web provisional para que puedas transferir archivos a través del administrador de archivos de DreamHost o tu cliente FTP de terceros.
  • Base de Datos (MySQL) — Muestra las credenciales de tu base de datos (MySQL) para que puedas administrar tus bases de datos usando phpMyAdmin.

Probar las nuevas versiones de PHP en Staging

La opción Configurar PHP te permite probar versiones más nuevas de PHP en un sitio provisional sin afectar tu sitio web DreamPress en vivo.

Si tu sitio provisional funciona en una versión más reciente de PHP, puedes publicar tu sitio web Staging, y la versión más nueva de PHP, en tu sitio web DreamPress en vivo. No es posible bajar PHP en tu sitio DreamPress en vivo usando este método.

Antes de continuar, asegúrate de haber publicado una copia de tu sitio web DreamPress en vivo en Staging, siguiendo los pasos anteriores.

Los siguientes pasos te muestran cómo cambiar la versión PHP de tu sitio web provisional desde el panel de Staging.

  1. Busca la sección Configurar PHP en la parte inferior del panel de Sitio Provisional.
  2. Haz clic en el botón Cambiar PHP a…. Una vez se complete la actualización, el panel mostrará la versión de PHP actual en uso en tu sitio provisional debajo de Configurar PHP.
  3. Inicia sesión en tu sitio web provisional y examínalo para asegurarte de que funciona como se espera. Revisa la sección Inicias sesión en tu sitio web Provisional de debajo para ver más detalles.

Si tu sitio funciona con la nueva versión de PHP, puedes publicar cambios desde tu sitio web provisional en tu sitio DreamPress en vivo. Revisa el siguiente artículo para obtener instrucciones:

  • Publicar tus cambios de Staging en tu sitio de DreamPress en vivo

Iniciar sesión en tu sitio provisional

Es posible que veas el siguiente mensaje de error de estado de git, que aparece debajo Tools > Site Health (en la sección 'Es posible que las actualizaciones en segundo plano no funcionen correctamente' debajo de Security en el panel WP-Admin):

The folder /home/wp_xyz/example.com/ was detected as being under version control (.git).

Este mensaje siempre aparecerá para cada sitio provisional de DreamPress, y puede ignorarlos con seguridad. Git no romperá tu sitio.

Si anteriormente actualizaste tu contraseña de administrador de DreamPress, puedes usar el mismo nombre de usuario y contraseña de tu sitio en vivo para iniciar sesión en tu sitio provisional.

Si no cambiaste la contraseña manualmente, puedes iniciar sesión en tu Sitio Provisional de WordPress desde el panel de Sitio Web Provisional. Haz clic en el botón Administrar Sitio Web Provisional de WordPress para iniciar sesión automáticamente en el panel administrador.

Publicar tus cambios

Cuando estés listo, puedes usar la página de Sitio Web Provisional del panel para publicar cambios en tu sitio web en vivo. Revisa el siguiente artículo para obtener más información:

  • Publicar tus cambios de staging en tu sitio de DreamPress en vivo

Solucionar problemas de inconvenientes de copiado

Si recibes un mensaje de 'Publicación Fallida' o el contenido copiado no coincide con el sitio de prueba, no hagas ningún cambio adicional en el entorno. Debes contactar a soporte para más asistencia.
