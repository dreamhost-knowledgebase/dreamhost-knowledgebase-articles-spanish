Este artículo explica cómo eliminar WordPress instalado a través del instalador de WordPress de DreamHost en el panel.

¿Qué se elimina en el servidor?

Cuando eliminas una instalación de WordPress de DreamHost, se pueden eliminar varios archivos dependiendo de dónde se encuentre la instalación. Asegúrate de leer los ejemplos a continuación antes de eliminar la instalación de WordPress de DreamHost.

Si tu instalación está en el directorio web principal

El directorio web principal de tu sitio es el directorio que el servidor busca para servir archivos. Revisa el artículo ¿Cómo agregar un dominio?. El paso 4 muestra dónde se encuentra el directorio web. También puedes visitar la página Instalar WordPress

Haz clic en el botón Administrar mis aplicaciones.

Luego haz clic en el nombre de tu sitio web. Mostrará una barra diagonal al lado del ícono instalar WordPress de DreamHost.

Cuando se le da clic, el botón Quitar de la Lista desactiva una Instalación de WordPress de DreamHost y lo elimina de la lista. NO elimina ningún archivo.

En general, este directorio es el nombre de tu sitio web. Por ejemplo, si tu sitio se llama example.com, tu directorio web también sería /home/username/example.com. Esto significa que cuando visitas tu sitio en example.com, estás viendo la instalación de WordPress de DreamHost.

Si aquí es donde instalaste tu aplicación de WordPress de DreamHost, ten mucho cuidado al eliminarlo en el panel. Si eliminas la aplicación de WordPress de DreamHost almacenada en tu directorio web principal, TODOS LOS ARCHIVOS en el directorio web y todos los subdirectorios se eliminan. Esto significa que si tienes subdirectorios como example.com/images o example.com/blog, también se eliminan.

Asegúrate de usar esta opción solo si deseas que todo se elimine de tu sitio web y servidor, incluidos todos los demás archivos de este directorio web. Si necesitas alguno de esos archivos o subdirectorios, asegúrate de hacer una copia de seguridad antes de eliminar la instalación de WordPress de DreamHost.

Si tu instalación está en un subdirectorio

Si tu instalación está en un subdirectorio como example.com/blog, solo se eliminan los archivos dentro de ese subdirectorio.

Cuando se le da clic, el botón Quitar de la Lista desactiva una Instalación de WordPress de DreamHost y lo elimina de la lista. NO elimina ningún archivo.

Por ejemplo, si tu sitio era example.com/blog, solo se eliminarán los archivos dentro de ese subdirectorio /blog. Sin embargo, el directorio vacío /blog permanecerá. No se tocan archivos encima de este directorio, por lo que queda todo dentro del directorio /example.com.

Eliminar una instalación de WordPress de DreamHost

  1. Navega a la página Instalar WordPress.
    Se abre la página Instalar WordPress:
  2. Haz clic en el botón Administrar mis aplicaciones.
  3. Haz clic en el dominio que está usando la aplicación para expandirlo.
    Se muestran bajo el dominio todas las aplicaciones instaladas de WordPress de DreamHost.
  4. Haz clic en el botón Borrar todos los archivos que se encuentra a la derecha.
    Aparece un mensaje de advertencia después de hacer clic en Borrar todos los archivos:
  5. Haz clic en Ok para confirmar y espera que la desinstalación se complete. Un mensaje que dice 'Éxito' se mostrará cuando la aplicación sea eliminada:
    • Después de desinstalar la aplicación, ya no se muestra en la página Administrar Aplicaciones.
    • Si aún aparece en la lista, haz clic en el botón Quitar de la lista.

¿Cómo eliminar la base de datos?

Cuando haces clic en el botón Borrar todos los archivos, solo los archivos en el servidor son eliminados. Tu base de datos de WordPress sigue en tu panel. Para eliminar tu base de datos, navega a la página Bases de Datos MySQL. Hacia la derecha de la respectiva base de datos, haz clic en el botón Eliminar BD.

¿Qué sucede si no se muestra el botón Eliminar?

Si encuentras que el botón Eliminar no se muestra en tu panel junto a la instalación de WordPress de DreamHost, contacta a soporte para obtener más ayuda.
