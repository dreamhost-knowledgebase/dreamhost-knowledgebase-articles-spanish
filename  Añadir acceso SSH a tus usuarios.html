Generalidades

Si has creado un nuevo usuario para administrar tu sitio web, puede ser una buena idea habilitar el acceso SSH para ese usuario. Esto suele ser necesario para sitios como WordPress que requieren que el usuario realice configuraciones adicionales.

Para habilitarlo, deberás editar tu archivo sshd_config. Para obtener información sobre cómo editar un archivo a través de SSH, consulta el siguiente artículo:

  • Crear y editar un archivo a través de SSH

Habilitar SSH

  1. Inicia sesión en tu instancia usando la Llave par asignada y el usuario por defecto asginado.
  2. Una vez que hayas iniciado sesión, ejecuta lo siguiente para editar tu archivo sshd_config: 
    [user@instance]$ sudo vim /etc/ssh/sshd_config
  3. En el archivo, busca la configuración llamada PasswordAuthentication. A la derecha, cambia no a yes.
  4. Guarda el archivo, y luego reinicia el SSHD con el siguiente comando: 
    [user@instance]$ sudo service sshd restart

Ahora puedes iniciar sesión en la instancia a través de SSH con este nuevo usuario.

¿Esto hará que tu instancia sea menos segura? No significativamente. Dado que este nuevo usuario solo tiene acceso a sus propios archivos, el daño que podría causar si estuviera comprometido es mínimo.
