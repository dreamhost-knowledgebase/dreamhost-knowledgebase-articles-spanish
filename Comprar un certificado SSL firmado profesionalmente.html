Generalidades

Este artículo sólo explica cómo comprar un certificado SSL firmado profesionalmente en tu panel DreamHost. Este tipo de certificado cuesta $15 dólares por año y está firmado por Sectigo, que muestra un sello de confianza de certificado en tu sitio.

Cualquier certificado de Comodo emitido después de enero de 2019 hace referencia a Sectigo como la autoridad de certificación, mientras que cualquier certificado emitido antes de esta fecha hace referencia a Comodo. Ambos nombres se refieren a la misma compañía.

Este tipo de certificado se recomienda para sitios de comercio electrónico, áreas de inicio de sesión de clientes o cualquier otro sitio que transmita información confidencial.

Antes de empezar

Correo de confirmación

Antes de comenzar el proceso de compra de un certificado SSL, asegúrate de haber creado una dirección de correo de confirmación en tu dominio. Este correo puede ser necesario para finalizar la compra.

Este correo debe ser una dirección en tu dominio. Por ejemplo, si tu sitio web es example.com, asegúrate de haber creado una de las siguientes direcciones:

  • admin@example.com
  • administrator@example.com
  • hostmaster@example.com
  • postmaster@example.com
  • webmaster@example.com

Consulta el siguiente artículo para obtener instrucciones sobre cómo crear una dirección de correo :

  • Agregar una dirección de correo electrónico

Tu correo puede ser alojado en otra compañía, solo debes asegurarte de que una de las direcciones anteriores esté activa y pueda recibir correo.

Registros CAA

El propietario de un sitio web puede crear registros de CAA para especificar qué autoridades de certificación (CA) pueden emitir certificados SSL para un dominio.

Mira el siguiente artículo si deseas agregar registros CAA a tu dominio antes de comprar un certificado SSL.

  • Políticas de registros CAA

Comprar un certificado firmado en el panel DreamHost

  1. Navega a la página Certificados Seguros.
  2. A la derecha de tu dominio, haz clic en el botón Add.
  3. Elige la opción para agregar un certificado Sectigo por $15 dólares (Sectigo) y haz clic  'Select this Certificate'.
  4. Ingresa tu información personal y del sitio web.
    • Debes completar TODOS los campos (incluido el 'Nombre de la organización') para poder avanzar con éxito al siguiente paso. Si no tienes el nombre de una organización, ingresa tu nombre y apellido. 'Calle 2' y 'Calle 3' son campos opcionales; puedes dejarlos en blanco si tu dirección no incluye nada de esta información (como un # de apt.).
    • Esta información se utiliza para generar la Solicitud de Firma de Certificado (CSR) que se utiliza para generar su nuevo certificado SSL firmado profesionalmente.
    Haz clic en Proceed to Next Step cuando hayas terminado.
  5. En la página siguiente, selecciona uno de los correos de tu dominio en el menú desplegable.

    Dirección de confirmación

    Este debe ser un correo en tu dominio. Asegúrate de que uno de los correos que figuran en ese menú desplegable se haya creado en tu dominio y de que pueda recibir correo con él. Si no existe, primero debes crear esta dirección de correo antes de continuar.

    Ten en cuenta que el proceso para verificar tu dominio mediante correo es el último recurso. Esto significa que es posible que nunca recibas un correo de confirmación y el SSL se procesará correctamente. Sin embargo, es posible que recibas un correo si otros pasos del DCV no se verifican. Por lo tanto, sigue siendo importante configurar esta dirección de correo. Consulta el siguiente artículo para más detalles:

    • ¿Qué es el proceso DCV?
  6. En la última página, elige cómo deseas pagar. Tus opciones son tarjeta de crédito o cheque electrónico.
  7. Cuando termines, ingresa tu información, haz clic en el botón Place Certificate Order.

¿Qué sucede después de comprarlo?

Correo de DreamHost

Pronto recibirás un correo de support@dreamhost.com enviado a la dirección de Validación de Control de Dominio (DCV) que seleccionaste para este pedido. Esto es solo una confirmación de tu compra, pero también puede pedirte que hagas clic en un enlace para verificar tu información de contacto.

Correo de Sectigo

También puedes recibir un segundo correo de Sectigo. Esto es parte del proceso DCV para verificar tu dominio. Esta es la última opción en el proceso DCV, por lo que es posible que nunca recibas este correo. Si recibes este correo de Sectigo, sigue los pasos para completar la verificación de tu dominio.

La mayoría de los pedidos se completan e instalan dentro de las 2 horas posteriores a su aprobación, sin embargo puede tomar hasta 24 horas procesarlos por completo.

Instalar el certificado Sectigo en tu sitio web

En este punto, solo has comprado el certificado Sectigo. NO está instalado todavía. Puedes ver en la página Certificados Seguros que aparece como 'pendiente'.

Cuando das clic al botón Settings a la derecha, verás instalado actualmente un certificado 'Temporal', que puede ser visto debajo de la pestaña Current Certificate:

Este certificado temporal solo permanecerá activo mientras tu certificado Sectigo esté en proceso de instalación. Una vez instalado, tu certificado Sectigo tomará su lugar y se convertirá en el único certificado activo en tu dominio.

Una vez que el certificado Sectigo esté activo, se mostrará lo siguiente en la página Certificados Seguros hacia el lado derecho del dominio.

Prueba si el certificado SSL funciona

El certificado ahora está instalado en tu dominio. Visita la URL segura de tu sitio web en:

    • https://example.com

Puedes ver que la barra del navegador muestra un icono de bloqueo que confirma que la conexión es segura.

Si acabas de instalar el certificado, es posible que debas esperar unas horas para que se actualice en línea.

Forzar a tu sitio a cargar HTTPS por defecto

Incluso después de haber agregado el certificado SSL a tu sitio, los visitantes no podrán usarlo a menos que escriban manualmente https antes del nombre de tu dominio. Esto anula el propósito de agregarlo en primer lugar ya que el certificado está destinado a proteger todo el tráfico de tu sitio web.

Hay varias formas de solucionar esto. Revisa el siguiente artículo para obtener instrucciones sobre cómo crear un archivo de configuración para redirigir todo el tráfico de visitantes a la versión segura de tu URL (https).

  • Cómo forzar tu sitio para redirigir a https (SSL)

Solución de problemas

Después de instalar el certificado, es posible que tu navegador aún muestre una advertencia insegura cuando visitas la versión HTTPS de tu URL. Hay algunas razones por las que esto podría ocurrir. Consulta el siguiente artículo para obtener instrucciones sobre cómo resolver este problema.
