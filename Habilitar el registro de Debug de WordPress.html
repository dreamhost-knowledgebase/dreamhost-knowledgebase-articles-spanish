Generalidades

WordPress proporciona una herramienta de Debug para ayudar a descubrir cuál puede ser la causa de un error en tu sitio web. Esta herramienta puede mostrar información en tu sitio web en vivo, sin embargo, esto no se recomienda si tu sitio web es en vivo para el público. En cambio, se recomienda crear un archivo de registro de Debug. Luego puedes verificar ese archivo de registro para determinar la causa del problema que está intentando resolver.

Este artículo le guía sobre cómo habilitar el modo Debug y escribir en un archivo de registro sin alterar tu sitio web en vivo.

Habilitar el modo DEBUG

  1. Inicia sesión en tu servidor por medio de SSH o FTP.
  2. Edita el archivo wp-config.php usnado SSH o tu cliente FTP.
  3. Cerca del final del archivo verás lo siguiente: 
    define('WP_DEBUG', false);
    Ajusta esa linea a estas tres lineas: 
    define( 'WP_DEBUG', true );
define( 'WP_DEBUG_DISPLAY', false );
define( 'WP_DEBUG_LOG', true );

    No se recomienda cambiar WP_DEBUG_DISPLAY a true (verdadero). Esto se debe a que cualquier error se mostrará en tu sitio web en vivo. Al habilitar WP_DEBUG_LOG, puedes ver los mismos errores en tu archivo /wp-content/debug.log.

  4. Cuando se produce un error en WordPress, se escribirá en un archivo titulado 'debug.log'. Este archivo se encuentra en tu directorio /wp-content/. Abre este archivo usando SSH o tu cliente FTP para ver el informe de debug.

Ver tu registro Debug usando un plugin

También hay varios plugins que ayudan a depurar tu sitio y ver tu registro de debug. Consulta el siguiente artículo para obtener información sobre cómo administrar e instalar un plugin.
