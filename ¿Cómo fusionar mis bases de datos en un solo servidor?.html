Generalidades

Este artículo proporciona información sobre las fusiones de bases de datos y el proceso que se lleva a cabo cuando se programa una fusión de bases de datos en el panel DreamHost.

¿Por qué es esto un problema?

La mayoría de las veces, los servicios de tu base de datos están siempre en un solo servidor de base de datos. Sin embargo, es posible que tengas bases de datos divididas en dos servidores de bases de datos diferentes. Se recomienda encarecidamente utilizar un único servidor de base de datos, ya que esto hace que la resolución de problemas relacionados con la base de datos sea mucho más fácil y facilita el acceso a todas tus bases de datos desde un único conjunto de usuarios y nombres de host.

Comprobar si tienes varios servidores de base de datos

  1. Navega hasta la página Bases de Datos MySQL.
  2. Desplázate hacia abajo.

Mover tus bases de datos

Al hacer clic en ¡Mover mis bases de datos!, la migración no comienza inmediatamente. Generalmente, estas fusiones se realizan en medio de la noche (PDT/PST).

También tienes la oportunidad de cancelar el proceso si eliges no continuar con la migración.

Después de hacer clic en el botón, aparece un mensaje que le informa que esta migración tardará varias horas en completarse y que es posible que tu sitio no funcione correctamente durante este proceso.

Cuando finalice la migración, se te enviará una confirmación por correo.

Que esperar

A la medianoche o después, se ejecuta un script automatizado y consolida tus bases de datos en un servidor. Una vez que la mudanza se realiza correctamente, se envía una notificación a tu dirección de correo electrónico.

Si toma más de unas pocas horas, es posible que el script haya tenido algún tipo de problema y debes comunicarse con el soporte para obtener más ayuda:
