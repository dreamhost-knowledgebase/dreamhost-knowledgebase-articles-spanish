Generalidades

Lo siguiente describe cómo funciona el bloqueo del puerto 25 y algunos pasos para ayudarte a resolverlo.

¿Qué es un bloquedo de puertos?

El correo se transmite desde un número de puerto específico. El correo enviado a través de SMTP tradicionalmente usa el puerto 25 de manera predeterminada. El problema es que el puerto 25 es bloqueado con frecuencia por los Proveedores de Servicios de Internet (ISP) como una técnica antispam. Para enviar correo a través de dichos ISP, uno debe:

  • usar SMTP para enviar — la opción recomendada para enviar correos desde DreamHost es usar el puerto 465 o 587.
  • usar el servidor SMTP propio del ISP — esto puede requerir que utilices el dominio del Proovedor de Servicio de Internet para enviar correos.

Puerto 25 en DreamHost

DreamHost bloquea el puerto 25 para llamadas salientes de SMTP en servidores compartidos (esto no es una restricción en los Servidores VPS/Dedicados).

Esto sucede a menudo cuando se envía desde un formulario de correo o un script de correo personalizado mientras se intenta conectarse a un servidor de correo externo (no a un servidor de correo DreamHost).

Resolver el inconveniente

Cambiar el puerto de salida

La solución más sencilla es configurar tu cliente de correo para utilizar uno de los siguientes puertos de salida:

  • 465 con SSL (recomendado)
  • 587 con STARTTLS

Usar el servidor SMTP del ISP

Si ves el error "smtp.example.com no responde o no se encuentra", comunícate con tu ISP y averigüe si están bloqueando el puerto 25. Si es así, probablemente puedas enviar correo saliente a través de tus servidores de correo de salida.
