Visión general

Si está teniendo problemas para migrar un sitio de WordPress de otro servidor o simplemente no tiene tiempo, DreamHost actualmente ofrece un servicio que se encarga de esto por usted: por solo $99 y en menos de 48 horas, DreamHost moverá su sitio de WordPress sin ningún tiempo de inactividad. Por favor, consulte los Términos de servicio (TOS) para obtener más información sobre el proceso. Si usted desea que DreamHost se encargue de la migración de su página, por favor envíenos una solicitud con el nombre del dominio que desea migrar, incluyendo la siguiente afirmación:

I agree to the WordPress migrations TOS and the $99 charge.

DreamHost le transmitirá su solicitud al altamente capacitado equipo de Servicios Profesionales. 

Si aún no tienes una cuenta DreamHost

Si actualmente no tiene una cuenta de DreamHost, puede registrarse en WordPress Hosting.

Durante el proceso de registro, solicite que se instale un sitio de WordPress, luego se mostrará la opción para el servicio de migración que le permitirá solicitarlo durante el registro.

Este artículo detalla el proceso de cómo mover su sitio de WordPress desde su compañía de hospedaje actual a DreamHost.

Si se está mudando de WordPress.com, deberá seguir sus instrucciones oficiales: Migrar un sitio en wordpress.com a otros hospedaje. (Moving to WordPress.org)

Creando una copia de seguridad de sus datos

Antes de realizar cambios en su sitio web, DreamHost recomienda realizar una copia de seguridad de su base de datos. Las instrucciones se pueden encontrar en el siguiente artículo:

  • Copia de seguridad de base de datos en MySQL

Es posible que deba ponerse en contacto con su compañía de hospedaje actual para obtener instrucciones sobre cómo exportar desde el sistema de ellos.

Migrando todos tus archivos manualmente

Para poder migrar manualmente sus datos, deberá hacer los siguiente:

  1. Realice una copia de seguridad de su base de datos con formato .sql desde su compañía de hospedaje anterior.
  2. Realice una copia de seguridad de los archivos web vía FTP desde su compañía de hospedaje anterior.
  3. Cree un nombre de host SQL (hostname) y una base de datos en DreamHost para la nueva instalación.
  4. Suba estos archivos a través de FTP/SFTP/SSH a su servidor en DreamHost.
  5. Importe su base de datos al servidor de DreamHost.
  6. Actualice su archivo wp-config.php para su sitio a la base de datos en DreamHost.

Paso 1 – Crear una copia de seguridad de su base de datos

Antes de hacer cualquier otra cosa, cree una copia de seguridad de su base de datos MySQL con su compañía de hospedaje actual. La manera en que se crea una copia de seguridad varía de un host a otro. El método más sencillo es usar phpMyAdmin, una herramienta web que está disponible con la mayoría de compañías de hospedaje.

Para obtener más información, consulte con su compañía de hospedaje actual sobre cómo hacer una copia de seguridad de su base de datos con formato o tipo de archivo .sql.

Paso 2 – Crear una copia de seguridad de tus archivos web a través de FTP

Con su compañía de hospedaje actual, haga una copia de seguridad de todos sus archivos web a través de FTP/SFTP/SHELL. Visite el artículo de FTP que explica cómo hacer esto con DreamHost:

  • FTP

Es posible que deba ponerse en contacto con su compañía de hospedaje actual para confirmar qué configuración usar para conectarse a su servidor.

  • Con la compañía de hospedaje actual cree un archivo de su antigua instalación de WordPress y comprímalo antes de continuar para así obtener resultados más rápidos. Póngase en contacto con compañía de hospedaje actual para obtener instrucciones sobre cómo realizar esto.
  • Luego de eso puede subir el archivo a su servidor en DreamHost e iniciar sesión a vía SSH para descomprimirlo allí. Este proceso no es necesario, pero puede ayudar a acelerar el proceso si tiene una gran cantidad de datos para transferir.

Paso 3 – crear un nombre de host (hostname) y una base de datos en su panel de DreamHost

Una vez tenga los archivos de respaldo guardados en su computadora, cree un nuevo nombre de host (hostname) de MySQL y una nueva base de datos vacía para su instalación de WordPress en el panel de DreamHost. Puede crear ambos (el nombre de host y la base de datos) en la página de bases de datos MySQL.

Si aún no ha apuntado su DNS a DreamHost, debe crear un nombre de host (hostname) temporal para conectarse a su base de datos. Visite el siguiente artículo para más detalles:

  • How do I connect to my database with a temporary MySQL hostname?

Si usted crea el nombre de host (hostname), usa el nombre de la base de datos, el usuario y la contraseña tal como la tenía con su antiguo nombre de host y la base de datos, puede evitar editar su archivo wp_config.php en el paso#5 a continuación.

Paso cuatro - Transfiriendo su instalación de WordPress

Transfiera sus archivos de WordPress de los que ha hecho una copia de seguridad a su nueva cuenta de DreamHost.

  1. Importe sus archivos web usando el siguiente artículo:
    • FTP
  2. Importe su base de datos utilizando el artículo de Migrar MySQL:
    • Migrate MySQL

Suponiendo que haya creado la base de datos en su panel de DreamHost con el mismo nombre de host (hostname), nombre de usuario, contraseña y nombre de base de datos que se configuró en su compañía de hospedaje anterior, es posible que su sitio ya aparezca en línea. Si no es así, vaya al paso # 5 a continuación.

Paso cinco - Editar el archivo wp-config.php

Es probable que el nombre de host, la base de datos, el nombre de usuario y la contraseña que creó en su panel de DreamHost sean diferentes a los que tenía con su compañía de hospedaje anterior. Si este es el caso, deberá actualizar su archivo wp-config.php para asegurarse de que la información coincida con las credenciales de inicio de sesión de su nueva base de datos en el panel de DreamHost.

Abra el archivo wp-config.php y va a ver la siguiente información:

define('DB_NAME', ‘exampledatabase’);
define('DB_USER', 'exampledbuser');
define('DB_PASSWORD', ‘database password here’);
define('DB_HOST', 'mysql.example.com');
define('DB_CHARSET', 'utf8');
define('DB_COLLATE', '');

$table_prefix = 'wp_2hdy121d_';

Asegúrese de que la configuración en table_prefix del wp-config.php coincida con el prefijo de la tabla de su copia de seguridad, de lo contrario, WordPress le pedirá que instale desde cero.

Para confirmar el prefijo de la tabla, use phpMyAdmin para iniciar sesión en la base de datos en DreamHost a la cual le acaba de importar la información en el Paso#4.

  • Una vez que haya iniciado sesión, haga clic en el nombre de su base de datos a la izquierda. Van a aparecer las tablas debajo del nombre.
  • La primera parte de la tabla es el prefijo. En el ejemplo anterior, el prefijo es wp_q2e97u_.
  • Actualice el campo de prefijo del archivo wp-config.php para que coincida con el prefijo que tiene la tabla.
