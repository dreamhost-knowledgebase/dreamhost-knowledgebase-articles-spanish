Generalidades

Para que un sitio web sea visible en internet, necesitas dos cosas:

  • Comprar el registro de un dominio
  • Agregar alojamiento a un dominio

Este artículo asume que ya has comprado el registro del nombre de dominio que deseas usar.

Qué es un dominio 'Completamente Alojado' (Fully Hosted)?

Cuando un dominio está completamente alojado quiere decir que está configurado en un servidor y tiene los Registros DNS asignados. Una vez los registros DNS han sido asignados, puedes entonces visitar y ver el sitio en internet. Las instrucciones para agregar un dominio a tu cuenta como 'completamente alojado' se muestran debajo. Esto requiere algunos pasos:

  1. Comprar el registro del dominio.
  2. Comprar el plan de alojamiento.
  3. Seguir las instrucciones en este artículo para agregar tu dominio al plan de alojamiento.

Cómo agregar un dominio 'Completamente Alojado'

  1. Navega a la página Manage Domains.
    La página Administrar Dominios se abre:
  2. Add Hosting to a Domain / Sub-Domain.
    En la siguiente página verás varias opciones que debes habilitar:
  3. Nombre del dominio

    La primera sección es donde ingresas tu dominio y cómo aparece:

    • Domain to host: ingresa el nombre del dominio o subdominio que deseas agregar.

      NO agregues www frente al nombre del dominio o al inicio de éste. Podrás agregar www usando las opciones debajo.

    • Do you want the www in your URL?: Tienes tres opcione
    Sin preferencia: Ambas versiones de tu sitio (con y sin www) van a funcionar.
    Agregar WWW: Esto obliga a que la 'www' sea agregada a tu dominio una vez lo visites, así siempre se muestra como www.example.com
    Quitar WWW: Esto obliga a que la 'www' sea removida para que tu dominio siempre se muestre como example.com
  4. Users, Files, and Paths
    Esta sección te permite escoger tu nombre de usuario y el directorio donde tu sitio estará instalado.
    • Run this domain under the user: Selecciona un usuario para que los archivos estén bajo este.

      También puedes crear un nuevo usuario aquí. Cuando creas un nuevo usuario, también lo asignas a un servidor específico.

    • Web directory: Este es el directorio en el servidor donde tus archivos estarán ubicados. En general, no es necesario ajustarlo porque se crea con la misma ruta del nombre del dominio que deseas alojar.
    • Logs directory: Esto solamente es para que sepas dónde encontrar los registros de acceso y errores de tu sitio en el servidor.
  5. Web Options
    Esta sección te permite escoger la versión de PHP en la que correrá tu sitio, así como las opciones de seguridad y optimización.
    • PHP mode: Haz click en el menú desplegable para seleccionar la versión de PHP que deseas para tu sitio.
    • Automatically upgrade PHP: Marca esta casilla para que DreamHost actualice automáticamente la versión de PHP cuando esté disponible. Esto previene que versiones de PHP obsoletas e inseguras se sigan ejecutando.
    • Extra Web Security?: Se recomienda mantener esto habilitado ya que provee mayor seguridad a tu sitio.
    • PHP OPcache Support: Solamente está disponible en el plan VPS y ayuda a optimizar los procesos PHP.
    • Passenger (Ruby/Python apps only): Habilita esta opción si vas a correr Python o Ruby on Rails en tu sitio web.
  6. Cloudflare Services
    Esta sección permite habilitar diferentes servicios de Cloudflare a tu dominio.
  7. Haz click en el botón que dice Fully host this domain para guardar tus cambios.
  8. Verás un mensaje verde que dice 'Exitoso' confirmando que el dominio ha sido agregado. Este dominio es inmediatamente añadido a la página 'Manage Domains'.

Apuntar los DNS a DreamHost

Tu sitio puede ya encontrarse alojado en los servidores de DreamHost, pero si los DNS de tu sitio no apuntan aún a DreamHost, el sitio no resolverá en línea. Revisa las siguientes secciones dependiendo de dónde compraste el registro del dominio.

Si compraste el registro con DreamHost

Si compraste tu registro del dominio en DreamHost, los DNS ya se encuentran configurados correctamente. (A menos de que tu los personalices).

Si compraste el registro con otra compañía

Mensajes de error al agregar un dominio

En algunos casos, aparece un mensaje de error cuando intentas agregar alojamiento para un dominio en tu cuenta. Los mensajes de error más comunes se describen a continuación.

Debes agregar el dominio principal

Si ves:

You can't add that domain: you must add the parent domain example.com to your account before adding a subdomain to it.

Esto significa que el dominio de segundo nivel aún no está en el sistema. Un subdominio solo se puede crear si su dominio de segundo nivel se crea primero (example.com)— incluso si no planeas alojar el dominio de segundo nivel aquí en DreamHost.

No tienes permiso para agregar ese dominio

Si ves:

You can't add that domain: you don't have permission to add.

significa que el dominio que estás intentando agregar a su cuenta ya está alojado en otra cuenta DreamHost. Si estás moviendo el dominio de una cuenta existente a la que tiene acceso, recuerda eliminar primero el dominio de la cuenta anterior antes de agregarlo a una cuenta nueva. Si no sabes en qué cuenta está alojado actualmente el dominio, pónte en contacto con el equipo de soporte.

El nombre de dominio ya existe en el sistema.

El nombre de dominio ya existe en el sistema, el error puede ocurrir en situaciones similares a las que no tienes permiso para agregar. Si ves este mensaje de error, confirma que el dominio no está alojado en otra cuenta DreamHost y luego pónte en contacto con el equipo de soporte.
