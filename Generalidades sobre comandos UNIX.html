Uso del Disco

Para mostrar el uso total del disco de tu VPS o servidor Dedicado, usa df. Esto te mostrará el espacio total, usado y disponible en el disco. Agrega el indicador -h ara mostrar los números en un formato más legible (KB, MB, GB, etc.).

[server]$ df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        24G  4.4G   19G  20% /
/dev/sda5       2.3G  1.1G  1.2G  49% /tmp
/dev/sda6        51G  2.0G   48G   4% /var
/dev/sdb1       3.6T  3.2T  448G  88% /home
/dev/ram0       3.9M   92K  3.6M   3% /var/dovecot

Ten en cuenta que el resultado anterior se truncó. El resultado real en los servidores de DreamHost puede ser bastante más largo.

Tu directorio de inicio y todos tus archivos están en /home (no mencionados anteriormente).

En el ejemplo a continuación, se enumera el directorio de inicio, y puedes ver a partir de esto que se está utilizando el 26% por ciento de una partición de 30 GB.

[server]$ df -h /home
Filesystem      Size  Used Avail Use% Mounted on
/dev/sdb1       3.6T  3.2T  448G  88% /home

Verifica si tus archivos están cargados correctamente (verifica la integridad del archivo)

Si necesitas cargar algunos archivos grandes en el servidor, es posible que desees verificar que se cargan correctamente sin errores. Para hacer eso, simplemente crea un archivo MD5 y cárgalo en el servidor, en la misma carpeta que los archivos que deseas verificar.

Un archivo MD5 es un archivo de texto sin formato que contiene sumas de verificación de tus archivos para que luego se pueda verificar su integridad; Para crearlo, puedes usar cualquier herramienta MD5 para tu sistema operativo, o la utilidad de línea de comandos en un sistema similar a UNIX con openssl md5.

Una vez que cargues el archivo MD5, ve al directorio que contiene los archivos que deseas verificar (y dónde cargaste el archivo MD5) e ingresa el siguiente comando:

[server]$ md5sum -c yourfile.md5
list: OK
of: OK
uploaded: OK
files: OK
being: OK
verified: OK

Si todos los archivos informan ON, la carga fue completamente exitosa. Si alguno de los informes de archivos muestran FAILED (FALLÓ), la carga no fue exitosa.
