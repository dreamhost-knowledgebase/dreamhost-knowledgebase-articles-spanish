Generalidades

LDAP son las siglas de Lightweight Directory Access Protocol y es un protocolo de red para buscar y modificar servicios de un directorio.

LDAP es un protocolo común para acceder a datos que se almacenan en un servidor de servicios de directorio central (base de datos). Hay muchos servidores de servicios de directorio diferentes, como (slapd, IBM Lotus Domino Directory, Microsoft Active Directory, Mozilla Open Directory Project, Novell eDirectory, y otros)

Muchas libretas de direcciones de correo (como la "Libreta de direcciones" .app incluida en Apple) tienen soporte LDAP incorporado.

Usos

LDAP se puede utilizar para acceder a la información del usuario que está almacenada en un servidor de servicios de directorio (una base de datos de detalles del usuario, como nombre, permisos, dirección de correo). Esa información puede incluir datos de usuario y permisos para un sitio web complejo o datos de la libreta de direcciones colaborativa.

  • Los usuarios de correo utilizarían LDAP para acceder a un servicio de directorio para buscar la dirección de correo de un usuario.
  • Los desarrolladores de aplicaciones web utilizarían LDAP en sus diseños de aplicaciones para almacenar y recuperar de forma centralizada información de usuarios autorizados almacenada en un servicio de directorio a través de LDAP.

¿Por qué? El beneficio de un servicio de directorio central (base de datos de usuarios) es que varias aplicaciones web podrían ahorrar tiempo y recursos al compartir los mismos datos de usuario.

LDAP en DreamHost

Los paquetes del sistema LDAP se instalan en todos los servidores DreamHost, pero las bibliotecas PHP no. Para utilizar LDAP, deberás instalar una versión personalizada de PHP. Durante la instalación, ajusta el archivo 'make' para usar la ubicación correcta de libldap.so.

LDAP no es adecuado para el alojamiento compartido ya que utiliza muchos recursos de CPU. Alojamientos VPS o Dedicados es la mejor opción. Si se requiere sudo durante la instalación, debes actualizar a un Servidor Dedicado y crear un usuario admin.
